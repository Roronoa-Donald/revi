Chapitre 3

Securite reseau et filtrage

Comprendre pare-feu, filtrage, segmentation et protection applicative.

Objectifs

  • Comprendre le role des pare-feu et leurs types.
  • Connaitre les criteres de filtrage et les regles.
  • Utiliser segmentation, DMZ, WAF.

1) Pare-feu (types)

Un pare-feu filtre le trafic entrant et sortant selon des regles.

  • Stateless: filtre paquet par paquet.
  • Stateful: suit l'etat des connexions.
  • NGFW: ajoute inspection applicative.
Memo technique: "deny by default" + ouvrir seulement le necessaire.

2) Criteres de filtrage

  • IP source/destination
  • Port source/destination
  • Protocole (TCP/UDP/ICMP)
  • Etat de connexion (stateful)
Piege: une regle "any any allow" annule toute securite.

3) Segmentation et DMZ

  • Segmentation: separer LAN, serveurs, admin.
  • DMZ: zone exposee pour les services publics.
  • Moindre privilege: acces minimal.
Memo technique: segmentation limite la propagation d'un incident.

4) WAF (Web Application Firewall)

Le WAF analyse le trafic HTTP/HTTPS pour bloquer des attaques web (SQLi, XSS).

  • Controle applicatif (couche 7).
  • Protection des API et sites web.
Piege: un WAF ne remplace pas les correctifs applicatifs.

Recap / Cheat Sheet

Pare-feu:
  • Stateless / stateful / NGFW
  • IP / port / protocole
Segmentation:
  • LAN / DMZ / Admin
  • Moindre privilege
WAF:
  • Protection web
  • Bloque SQLi / XSS
