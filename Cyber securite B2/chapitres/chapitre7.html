Chapitre 7

Snort: IDS et IPS

Comprendre l'architecture Snort, les regles et la detection.

Objectifs

  • Comprendre Snort et ses composants.
  • Differencier IDS et IPS.
  • Lire et expliquer une regle Snort.

1) Qu'est-ce que Snort ?

Snort est un outil open source de detection d'intrusions. Il analyse les paquets et declenche des alertes selon des regles.

  • Mode sniffer: affiche le trafic.
  • Mode packet logger: enregistre les paquets.
  • Mode IDS/IPS: detection et prevention.
Memo technique: Snort = capture + preprocess + detection + alertes.

2) IDS vs IPS

  • IDS: detecte et alerte (ne bloque pas).
  • IPS: detecte et bloque automatiquement.
En mode IPS, Snort est place inline pour pouvoir bloquer le trafic.

3) Regle Snort - structure

Exemple minimal:

  • alert tcp any any -> any 80 (msg:"HTTP detecte"; sid:1000001;)
  • action: alert
  • protocole: tcp
  • source/destination: any any -> any 80
  • options: msg, sid, content, etc.
Piege: un sid doit etre unique, sinon conflit de regles.

4) Pipeline Snort (simplifie)

  • Capture des paquets.
  • Preprocessors (normalisation, reassemblage).
  • Detection engine (regles).
  • Output (alertes, logs).
Memo technique: preprocess = nettoyer, detection = comparer, output = alerter.

5) Exemples d'usage

  • Detecter des scans Nmap.
  • Surveiller le trafic HTTP ou DNS.
  • Bloquer des signatures connues en mode IPS.

Recap / Cheat Sheet

Modes:
  • Sniffer
  • Logger
  • IDS / IPS
IDS vs IPS:
  • IDS = alerte
  • IPS = bloque
Regle:
  • action protocole src -> dst
  • options (msg, sid, content)
