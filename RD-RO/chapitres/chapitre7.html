Module 07

Le GPS dans le Brouillard

Quand on a 50 variables, on ne peut plus dessiner. On est dans le noir. L'algorithme du Simplexe est votre canne d'aveugle : il tâte le terrain et trouve le sommet le plus haut, pas après pas.

1. La Logique du Marcheur

Imaginez André perdu sur la montagne du Profit. Il est dans un brouillard total.

  • 1. Il part du camp de base (0 tables, 0 chaises). Profit = 0.
  • 2. Il se demande : "Quelle direction monte le plus vite ?" (Tables ou Chaises ?).
  • 3. Il marche dans cette direction jusqu'à cogner une barrière (manque de bois).
  • 4. Il recommence.

2. Le Tableau de Bord (Simplex Tableau)

Voici à quoi ressemble le cerveau d'André sous forme mathématique. C'est le tableau de départ.

Base Tables ($x_1$) Chaises ($x_2$) Stock Bois ($e_1$) Stock Temps ($e_2$) RHS (Dispo)
$e_1$ 2 1 1 0 20
$e_2$ 1 1 0 1 40
Z -50 -30 0 0 0

Test d'Optimalité (AVANT toute itération)

On examine la ligne Z. Y a-t-il des coefficients négatifs ? Ici : -50 et -30. Oui → la solution n'est pas optimale, on peut encore améliorer $Z$. On continue.

Critère de Dantzig — Gain Marginal (Variable Entrante)

On choisit la variable avec le coefficient le plus négatif dans la ligne Z. C'est la règle du plus grand gain marginal (critère de Dantzig).
Ici : -50 (Tables) est plus négatif que -30 (Chaises). Le gain marginal de $x_1$ est de 50 par unité → c'est la direction de montée la plus rapide.
André décide : "Je commence par fabriquer des Tables !". La colonne $x_1$ entre dans la base.

Le Ratio (Variable Sortante)

Combien de tables peut-il faire ?
• Bois : 20 / 2 = 10 Tables.
• Temps : 40 / 1 = 40 Tables.
Le chiffre le plus bas gagne (c'est le goulot d'étranglement). Il fera 10 Tables. Le stock de Bois ($e_1$) va tomber à 0 (sortir).

Résolvons entièrement le problème d'André avec l'algorithme du Simplexe.

$$x_1 + x_2 \leq 40 \quad \text{(Temps)}$$

$$x_1, x_2 \geq 0$$

$$2x_1 + x_2 + e_1 = 20$$

$$x_1 + x_2 + e_2 = 40$$

$$Z - 50x_1 - 30x_2 = 0$$

Résumé des Règles du Simplexe (Maximisation)

  • • Test d'optimalité : vérifier si tous les coefficients de la ligne Z sont $\geq 0$. Si oui → STOP (solution optimale). Si non → continuer.
  • • Variable entrante (Critère de Dantzig) : colonne du coefficient le plus négatif dans la ligne Z. C'est la variable avec le plus grand gain marginal.
  • • Variable sortante (Ratio minimum) : ligne du ratio minimum positif : $\theta = \min\left(\frac{b_i}{a_{ij}}\right)$ pour $a_{ij} > 0$.
  • • Pivot : élément à l'intersection de la colonne entrante et de la ligne sortante. On effectue Gauss-Jordan sur le pivot.

4. Quand le Simplexe de base ne suffit plus

Le Simplexe standard démarre facilement quand toutes les contraintes sont du type $\leq$ : les variables d'écart forment une base initiale naturelle (matrice identité). Mais avec des contraintes $\geq$ ou $=$, il n'y a pas de base de départ évidente. Deux techniques résolvent ce problème.

M 4.1 La Méthode du Big-M (ou Méthode de la Pénalité)

Principe

  1. Pour chaque contrainte $\geq$ : on soustrait une variable de surplus $s_i$ et on ajoute une variable artificielle $a_i$ :

    $\sum a_{ij} x_j - s_i + a_i = b_i, \quad s_i, a_i \geq 0$

  2. Pour chaque contrainte $=$ : on ajoute directement une variable artificielle $a_i$.
  3. Dans la fonction objectif, on ajoute une pénalité géante $-M$ (maximisation) ou $+M$ (minimisation) par variable artificielle, avec $M \to +\infty$ :

    $\text{Max } Z = \sum c_j x_j - M \cdot a_1 - M \cdot a_2 - \ldots$

  4. On résout par le Simplexe classique. La pénalité force les $a_i$ à quitter la base.
  5. Si une $a_i > 0$ à l'optimum → le problème est infaisable.

Exemple Big-M

Problème :

$$\text{Min } Z = 2x_1 + 3x_2$$

$$x_1 + x_2 \geq 4 \quad \text{(Contrainte 1)}$$

$$x_1 + 2x_2 \geq 6 \quad \text{(Contrainte 2)}$$

$$x_1, x_2 \geq 0$$

Mise en forme standard :

On convertit en maximisation : $\text{Max } Z' = -2x_1 - 3x_2$

On soustrait les surplus et ajoute les artificielles :

$x_1 + x_2 - s_1 + a_1 = 4$
$x_1 + 2x_2 - s_2 + a_2 = 6$

Objectif pénalisé :

$\text{Max } Z' = -2x_1 - 3x_2 - M \cdot a_1 - M \cdot a_2$

La base initiale est $(a_1, a_2)$. Le Simplexe va progressivement remplacer $a_1$ et $a_2$ par $x_1, x_2$ grâce à la pénalité $M$. À l'optimum : $x_1 = 2$, $x_2 = 2$, $Z = 10$.

φ 4.2 La Méthode des Deux Phases

Principe

On évite le choix arbitraire de la valeur $M$ en séparant le travail en deux étapes distinctes :

Phase 1 — Trouver une base réalisable
  • Ajouter les variables artificielles comme dans Big-M.
  • Résoudre un problème auxiliaire : $\text{Min } W = a_1 + a_2 + \ldots$
  • Si $W^* = 0$ → toutes les artificielles sont sorties. On a une base réalisable !
  • Si $W^* > 0$ → le problème original est infaisable.
Phase 2 — Optimiser l'objectif original
  • Supprimer les colonnes des variables artificielles du tableau.
  • Réintroduire la fonction objectif originale $Z$.
  • Poursuivre le Simplexe normalement avec la base trouvée en Phase 1.

Exemple Deux Phases

Même problème : Min $Z = 2x_1 + 3x_2$ s.c. $x_1 + x_2 \geq 4$, $x_1 + 2x_2 \geq 6$.

Phase 1 : On résout Min $W = a_1 + a_2$ avec les mêmes contraintes. Le Simplexe donne $W^* = 0$ avec la base $(x_1, x_2)$ → le problème est réalisable.

Phase 2 : On supprime les colonnes $a_1, a_2$ du tableau. On remet l'objectif $Z = 2x_1 + 3x_2$ et on continue le Simplexe → $x_1 = 2$, $x_2 = 2$, $Z = 10$.

4.3 Comparaison : Big-M vs Deux Phases

Critère Big-M Deux Phases
Nombre de problèmes résolus 1 seul (avec pénalité) 2 problèmes distincts
Avantage Plus simple à mettre en œuvre Pas de paramètre $M$ arbitraire, plus stable numériquement
Inconvénient Choix de $M$ délicat (erreurs d'arrondi en grande dimension) Deux optimisations successives
Détection infaisabilité $a_i > 0$ à l'optimum $W^* > 0$ en fin de Phase 1
Usage typique Résolution manuelle (petits problèmes) Solveurs informatiques

Quand utiliser quoi ?

  • • Toutes les contraintes sont $\leq$ → Simplexe standard (pas besoin d'artificielles).
  • • Au moins une contrainte $\geq$ ou $=$ → Big-M (à la main) ou Deux Phases (solveur).
  • • Les deux méthodes donnent toujours la même solution optimale.

5. Dégénérescence et Cyclage

Qu'est-ce que la dégénérescence ?

Une solution de base est dégénérée lorsqu'une ou plusieurs variables basiques sont égales à zéro. Cela arrive quand, lors du test du ratio minimum, deux lignes ou plus donnent le même ratio.

// Exemple de dégénérescence

Ratios : 12/3 = 4, 8/2 = 4

→ Égalité ! La variable basique

  non choisie restera à 0.

Le risque : le Cyclage

Quand la base est dégénérée, le Simplexe peut effectuer des itérations sans améliorer Z (le pivot change la base mais pas la solution). Dans de rares cas, l'algorithme peut boucler indéfiniment entre les mêmes bases.

En pratique : le cyclage est extrêmement rare mais théoriquement possible. Il faut le prévenir.

Règle de Bland (anti-cyclage)

La règle de Bland garantit la terminaison du Simplexe même en cas de dégénérescence :

  1. Variable entrante : parmi toutes les colonnes avec un coefficient négatif dans Z, choisir celle de plus petit indice (pas la plus négative).
  2. Variable sortante : en cas d'égalité de ratios, choisir la ligne de plus petit indice.

Cette règle simple suffit à empêcher tout cyclage, au prix éventuel de quelques itérations supplémentaires.
