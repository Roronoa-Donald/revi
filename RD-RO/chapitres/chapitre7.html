Module 07

Le GPS dans le Brouillard

Quand on a 50 variables, on ne peut plus dessiner. On est dans le noir. L'algorithme du Simplexe est votre canne d'aveugle : il tâte le terrain et trouve le sommet le plus haut, pas après pas.

1. La Logique du Marcheur

Imaginez André perdu sur la montagne du Profit. Il est dans un brouillard total.

  • 1. Il part du camp de base (0 tables, 0 chaises). Profit = 0.
  • 2. Il se demande : "Quelle direction monte le plus vite ?" (Tables ou Chaises ?).
  • 3. Il marche dans cette direction jusqu'à cogner une barrière (manque de bois).
  • 4. Il recommence.
🏔️ 🦯

Algorithme itératif =
Un pas après l'autre

2. Le Tableau de Bord (Simplex Tableau)

Voici à quoi ressemble le cerveau d'André sous forme mathématique. C'est le tableau de départ.

Base Tables ($x_1$) Chaises ($x_2$) Stock Bois ($e_1$) Stock Temps ($e_2$) RHS (Dispo)
$e_1$ 2 1 1 0 20
$e_2$ 1 1 0 1 40
Z -50 -30 0 0 0

Pourquoi -50 ? (Variable Entrante)

Dans la ligne Z, les nombres négatifs sont des opportunités de profit. Ici, -50 (pour les Tables) est plus fort que -30 (Chaises).
Donc André décide : "Je commence par fabriquer des Tables !". La colonne $x_1$ entre dans la base.

Le Ratio (Variable Sortante)

Combien de tables peut-il faire ?
• Bois : 20 / 2 = 10 Tables.
• Temps : 40 / 1 = 40 Tables.
Le chiffre le plus bas gagne (c'est le goulot d'étranglement). Il fera 10 Tables. Le stock de Bois ($e_1$) va tomber à 0 (sortir).
