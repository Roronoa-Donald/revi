Introduction a la Virtualisation

Comprendre les definitions, les concepts cles et les liens entre les briques de la virtualisation.

La virtualisation, c'est quoi ?

La virtualisation permet d'executer plusieurs machines virtuelles (VM) sur un seul serveur physique. Chaque VM fonctionne comme un ordinateur complet, avec son propre systeme, ses logiciels et ses ressources (CPU, RAM, disque).

Imagine un immeuble : le serveur physique est le batiment, chaque VM est un appartement independant avec ses propres regles.

Cadre de l'examen

L'evaluation est 100% theorique. On attend des definitions claires, des comparaisons justes et la capacite a relier les notions.

  • Definir hyperviseur, VM, snapshot, template.
  • Expliquer les relations entre compute, stockage et reseau.
  • Comparer type 1 vs type 2, et justifier un choix.

Pourquoi virtualiser ?

  • Consolidation : un serveur physique peut heberger 5 a 20 VMs selon la charge.
  • Economies : moins d'achats materiels, moins d'energie, moins de maintenance.
  • Disponibilite : snapshots, sauvegardes et restauration rapide.
  • Agilite : creation d'une VM en quelques minutes pour un projet ou un cours.
Exemple terrain : une PME qui avait 4 serveurs physiques (fichiers, appli, base de donnees, sauvegarde) passe a 1 serveur + un NAS. Elle reduit ses couts d'energie et son temps d'admin.

Hyperviseurs : type 1 vs type 2

TypeFonctionnementPour qui ?Exemples
Type 1 (bare-metal)Installe directement sur le serveurProduction, entreprisesProxmox VE, ESXi
Type 2 (hosted)Installe sur un OS existantTests, formationVMware Workstation, VirtualBox
Regle simple : Type 1 pour la production, Type 2 pour apprendre et tester.

Panorama des solutions

Proxmox VE

Solution open-source basee sur KVM + LXC. Interface web simple, snapshots, clustering et sauvegarde native.

VMware ESXi

Reference entreprise pour la fiabilite. Ideale pour les environnements critiques.

VMware Workstation / VirtualBox

Solutions de poste pour apprendre, tester, simuler des reseaux ou lancer un laboratoire.

Choisir un hyperviseur : regarde le budget, le niveau technique, et le besoin de haute disponibilite.

Vocabulaire essentiel

  • Host : le serveur physique.
  • Guest : l'OS installe dans la VM.
  • vCPU : coeur processeur virtuel attribue a une VM.
  • Snapshot : photo instantanee d'une VM a un instant T.
  • Template : modele reutilisable pour creer plusieurs VMs.
  • Cluster : groupe de serveurs pour HA et migration.
