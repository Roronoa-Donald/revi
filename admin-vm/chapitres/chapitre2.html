Chapitre 2

Prerequis & Virtualisation Imbriquee

Tout preparer pour lancer un lab solide : materiel, BIOS/UEFI, systeme et verification.

Pourquoi la virtualisation imbriquee ?

La virtualisation imbriquee (nested) permet d'executer un hyperviseur dans une VM. C'est ideal pour :

  • les laboratoires (ex. installer Proxmox dans une VM),
  • la formation sans multiplier les serveurs,
  • les tests d'architecture ou de migration.
Limite : la performance est plus faible. Utilise-la pour apprendre et tester, pas pour la production.

Prerequis materiels

  • CPU avec Intel VT-x ou AMD-V active.
  • RAM : minimum 16 Go (32 Go recommande pour un lab complet).
  • Disque : SSD avec 100 Go libres (plus si plusieurs VMs).
  • BIOS/UEFI a jour (met a jour si possible).
Astuce : un CPU 6 a 8 coeurs + 32 Go RAM permet de simuler un mini datacenter.

Activer la virtualisation dans le BIOS/UEFI

  1. Redemarre le PC et entre dans le BIOS/UEFI (F2, Del, Esc selon la marque).
  2. Active Intel VT-x ou AMD-V/SVM.
  3. Active aussi VT-d (IOMMU) si disponible.
  4. Sauvegarde et redemarre.
Apres activation, Windows doit afficher "Virtualisation : activee" dans le Gestionnaire des taches.

Configurer Windows (Hyper-V vs VMware/VirtualBox)

Sur Windows, Hyper-V peut bloquer VMware/VirtualBox. Choisis une seule pile :

  • Option A : utiliser Hyper-V (bon pour WSL2, mais limite VMware).
  • Option B : desactiver Hyper-V pour VMware/VirtualBox.
# Desactiver Hyper-V (option B)
bcdedit /set hypervisorlaunchtype off
# Redemarrer ensuite

Verifie avec msinfo32 : la ligne "Hyper-V - Extensions de virtualisation" doit etre OK.

Sous Windows 11, desactive aussi "Integrite de la memoire" si VirtualBox refuse de demarrer.

Configurer Linux (KVM)

Sur Linux, verifie le support CPU :

egrep -c '(vmx|svm)' /proc/cpuinfo

Installe KVM et verifie :

sudo apt install qemu-kvm libvirt-daemon-system
kvm-ok
Si "KVM acceleration can be used", ton systeme est pret.

Configurer macOS

  • Mac Intel : VMware Fusion ou Parallels supportent bien la virtualisation.
  • Apple Silicon (M1/M2) : prefere UTM ou Parallels, mais la nested virtualisation est limitee.
Sur Apple Silicon, l'objectif est surtout d'apprendre les concepts, pas de monter un cluster complet.

Verifier avant de continuer

  • Le BIOS/UEFI affiche VT-x/AMD-V actif.
  • Windows : "Virtualisation activee" dans le Gestionnaire des taches.
  • Linux : kvm-ok renvoie OK.
  • Une VM peut demarrer sans erreur.
Si tout est OK, tu peux installer Proxmox VE dans le chapitre suivant.
