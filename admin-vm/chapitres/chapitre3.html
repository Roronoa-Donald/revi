Chapitre 3

Installation & Configuration des Hyperviseurs

Installer Proxmox VE proprement, configurer le reseau, les depots et la securite de base.

Avant de commencer

  • ISO Proxmox VE telechargee et cle USB bootable.
  • Plan IP simple (ex. 192.168.10.0/24) et un nom de host clair.
  • Mot de passe root solide.
Proxmox se gere via une interface web : tu auras besoin d'un navigateur et d'une IP fixe pour l'hote.

Installation de Proxmox VE (pas a pas)

  1. Boote sur la cle USB Proxmox.
  2. Accepte la licence et choisis le disque (LVM ou ZFS).
  3. Definis la langue, le fuseau horaire et le clavier.
  4. Configure le reseau : IP fixe, passerelle, DNS.
  5. Fixe le mot de passe root et un email d'admin.
Choisis un disque fiable. Les VMs dependent directement de la qualite du stockage.

Premier acces a l'interface web

Apres l'installation :

  • Accede a https://IP_DU_HOST:8006
  • Connecte-toi avec root et ton mot de passe.
  • Verifie l'heure, la zone et l'etat du reseau.
Si l'interface web s'affiche, Proxmox est operationnel.

Depots et mises a jour

Proxmox propose deux depots :

  • Enterprise : payant, tres stable.
  • No-subscription : gratuit, ideal pour les labs.
# Activer le depot gratuit
nano /etc/apt/sources.list.d/pve-enterprise.list
# Commente la ligne, puis ajoute :
echo "deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription" | tee /etc/apt/sources.list.d/pve-no-subscription.list
apt update && apt full-upgrade

Comptes et securite de base

  • Cree un compte admin non-root pour la gestion quotidienne.
  • Active le pare-feu Proxmox (Datacenter > Firewall).
  • Limite l'acces a l'interface web par IP si possible.
Ne laisse pas l'interface exposee sur Internet sans VPN ou filtre IP.

Stockage et reseau initial

  • Verifie la presence de local et local-lvm.
  • Configure un bridge vmbr0 pour les VMs.
  • Ajoute un stockage NFS/iSCSI si besoin (chapitre 5).
Un bridge = un switch virtuel. Il relie les VMs au reseau physique.

Et VMware ESXi ?

ESXi suit une logique similaire : installation sur serveur nu, interface web, configuration reseau et datastore. Il reste une reference, mais Proxmox est plus accessible en open-source.

Objectif du cours : maitriser Proxmox, comprendre ESXi pour pouvoir s'adapter sur le terrain.
