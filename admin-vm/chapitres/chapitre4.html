Chapitre 4

Creation & Gestion des VMs

Creer des VMs propres, gerer templates, clones et snapshots, puis connecter les VMs entre elles.

Concepts essentiels

  • VM : machine virtuelle isolee.
  • Template : modele pour cloner rapidement.
  • Clone : copie d'une VM (full ou linked).
  • Snapshot : etat fige pour retour arriere rapide.
Astuce : utilise un template Windows/Ubuntu propre, puis clone pour gagner du temps.

Creer une VM (Proxmox)

  1. Upload de l'ISO dans local (Datacenter > Storage).
  2. Bouton Create VM : nom clair (ex. SRV-AD).
  3. Choisir l'ISO, la taille disque, le type de stockage.
  4. Allouer CPU, RAM et reseau (bridge vmbr0).
  5. Demarrer la VM et installer l'OS.
Pour Windows, active le disque VirtIO et installe les drivers VirtIO.

Allocation de ressources

  • vCPU : 1 a 2 vCPU pour un serveur leger, 4+ pour base de donnees.
  • RAM : ne donne pas tout, laisse 20-30% libre pour l'hote.
  • Disque : thin si stockage limite, thick si performance.
Regle simple : commence petit, puis augmente selon la charge.

Reseau entre VMs

Un bridge relie les VMs au reseau physique. Pour que les VMs se voient :

  • Attribue des IP fixes (ex. 192.168.10.10 / 11 / 12).
  • Verifie le ping entre les VMs.
  • Utilise un VLAN si tu veux isoler les environnements.
Exemple : une VM AD + une VM Web + une VM DB sur le meme bridge.

Snapshots et restauration

Un snapshot permet de revenir en arriere apres un test risque.

  • Prends un snapshot avant une mise a jour.
  • Supprime les snapshots anciens pour economiser le stockage.
Un snapshot n'est pas un backup complet. Il reste sur le meme stockage.

Templates et clones

  1. Installe un OS propre (updates + agents).
  2. Nettoie et generalise (sysprep pour Windows).
  3. Convertis en template.
  4. Clone a partir du template pour tes nouvelles VMs.
Resultat : creation d'une VM en moins de 2 minutes.

Bonnes pratiques

  • Nommer clairement : SRV-AD, WEB-01, DB-01.
  • Tagger les VMs (prod, test, dev).
  • Activer QEMU Guest Agent pour une meilleure gestion.
  • Surveiller l'usage CPU/RAM via le dashboard.
