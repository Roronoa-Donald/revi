Chapitre 4

Cycle de vie des VMs

Comprendre le cycle de vie des VMs, les ressources et les liens entre identite, stockage et reseau.

Concepts essentiels

  • VM : unite d'execution isolee avec ses ressources.
  • Template : modele standardise pour uniformiser.
  • Clone : duplication d'un etat existant.
  • Snapshot : point de retour temporel.
Ces notions se combinent pour accelerer la creation sans perdre le controle.

Cycle de vie d'une VM

  • Conception : objectifs, charges, criticite.
  • Provisioning : ressources, stockage, reseau.
  • Exploitation : supervision, mises a jour, sauvegarde.
  • Retrait : archivage, nettoyage, tracabilite.
Une VM sans cycle de vie clair devient vite un risque.

Allocation de ressources

  • vCPU : partage temporel du CPU physique.
  • RAM : ressource non elastique, a dimensionner finement.
  • Disque : thin pour la souplesse, thick pour la performance.
Principe : dimensionner selon la charge et la priorite.

Identite et reseau des VMs

Le reseau donne une identite et un perimetre a la VM.

  • Bridge : lien entre virtuel et physique.
  • VLAN : isolation logique des flux.
  • Adressage : lisibilite et gouvernance.
Une topologie claire facilite la securite et le diagnostic.

Snapshots et restauration

Un snapshot fige l'etat d'une VM a un instant T.

  • Utile pour revenir en arriere apres un changement.
  • Ne remplace pas une sauvegarde externe.
Snapshot = retour rapide, backup = resilence.

Templates et clones

  • Template : standard de reference pour coherence.
  • Clone full : independant, plus lourd.
  • Clone linked : rapide, depend du template.
La strategie template garantit la qualite et la rapidite.

Bonnes pratiques

  • Nommer et classer pour la lisibilite.
  • Tagger les VMs selon leur criticite.
  • Surveiller les ressources pour eviter la contention.
  • Documenter les dependances entre VMs.
