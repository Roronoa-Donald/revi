Chapitre 6

Reseau Virtualise & Segmentation

Organiser les flux, isoler les zones et securiser les communications entre VMs.

Les briques reseau

  • Bridge (vmbr) : switch virtuel relie au physique.
  • NAT : permet aux VMs d'acceder a Internet sans IP publique.
  • VLAN : segmentation logique dans un meme switch.
  • Firewall virtuel : filtre le trafic entre zones.
Un bridge, c'est comme une multiprise : plusieurs VMs branchent dessus.

Configuration d'un bridge Proxmox

Par defaut, Proxmox cree vmbr0 lie a la carte reseau physique.

  • vmbr0 : reseau principal (LAN).
  • vmbr1 : reseau isole (test ou DMZ).
# Exemple de bridge (Linux)
auto vmbr0
iface vmbr0 inet static
  address 192.168.10.2/24
  gateway 192.168.10.1
  bridge-ports eno1

VLAN et segmentation

Les VLAN isolent les flux sans multiplier les switchs physiques.

  • VLAN 10 : utilisateurs
  • VLAN 20 : serveurs
  • VLAN 30 : DMZ
Avec Proxmox, active le VLAN tagging dans la configuration de la VM.

DMZ et pare-feu

Une DMZ contient les services exposes (web, mail). Elle doit etre isolee du LAN.

  • Place les VMs web en DMZ.
  • Autorise uniquement les ports necessaires.
  • Utilise un pare-feu (pfSense, OPNsense).
Ne mets jamais la base de donnees en DMZ.

Diagnostiquer le reseau

  • ping : test de connectivite basique.
  • traceroute : chemin des paquets.
  • tcpdump : inspection du trafic.
  • Verifie les regles firewall et les VLAN tags.

Mini cas pratique

Une agence veut heberger un site web et un ERP interne.

  • Web en DMZ (VLAN 30).
  • ERP en LAN (VLAN 20).
  • Utilisateurs en VLAN 10.
  • Firewall : LAN <-> DMZ uniquement sur port 443.
Resultat : web accessible, ERP protege.
