Chapitre 7

Administration Avancee

Haute disponibilite, migration, sauvegarde et securite renforcee - vision theorique.

Gestion avancee des ressources

  • Reservations : minimum garanti pour une VM critique.
  • Limits : plafonner CPU/RAM.
  • Shares : priorite relative entre VMs.
Exemple : un serveur de bases peut avoir une reservation RAM plus elevee.

Cluster & haute disponibilite (HA)

Un cluster relie plusieurs hyperviseurs pour assurer la continuite de service.

  • Quorum : garante la prise de decision.
  • Fencing : isole un noeud instable.
  • Failover : redemarre les VMs ailleurs.
Concept cle : la redondance reduit le risque d'arret.

Migration

La migration deplace une VM d'un noeud a un autre. Elle illustre le lien entre reseau, stockage et RAM.

  • Stockage partage : facilite la mobilite.
  • Reseau : conditionne la duree.
  • Charge : impacte la stabilite.
Plus les dependances sont fortes, plus la migration est delicate.

Sauvegarde avancee

  • Deduplication : reduction d'espace.
  • Chiffrement : protection des sauvegardes.
  • Restauration granulaire : fichier ou VM.
Le principe compte : resilence et rapidite de retour.

Monitoring & alertes

  • Metriques : CPU, RAM, I/O, latence.
  • Seuils : definir ce qui est critique.
  • Gouvernance : suivre l'evolution des usages.

Securite renforcee

  • Desactive l'acces root en SSH si possible.
  • Active le pare-feu au niveau Datacenter.
  • Separation des comptes et logs d'audit.
  • Mises a jour planifiees avec snapshots prealables.
Un hyperviseur non patche expose toutes les VMs.
