Chapitre 7

Administration Avancee

Haute disponibilite, migration a chaud, sauvegardes deduplees et securite renforcee.

Gestion avancee des ressources

  • Reservations : minimum garanti pour une VM critique.
  • Limits : plafonner CPU/RAM.
  • Shares : priorite relative entre VMs.
Exemple : un serveur de bases peut avoir une reservation RAM plus elevee.

Cluster & haute disponibilite (HA)

Un cluster permet de relier plusieurs hyperviseurs pour assurer la continuitie de service.

  • Quorum : garantit la decision dans le cluster.
  • Fencing : isole un noeud instable.
  • Failover : relance auto des VMs sur un autre noeud.
Avec 3 noeuds, tu peux perdre 1 serveur sans arreter le service.

Live migration

La migration a chaud deplace une VM sans interruption visible.

  • Stockage partage (NFS, Ceph, SAN).
  • Reseau rapide entre noeuds (1G minimum, 10G ideal).
  • Temps de migration selon RAM et charge.
Si le reseau est instable, la migration peut echouer. Planifie hors heures critiques.

Proxmox Backup Server (PBS)

  • Deduplication : stockage optimise.
  • Chiffrement : sauvegardes protegees.
  • Restauration granulaire : fichier ou VM.
PBS est ideal pour une strategie GFS automatisee.

Monitoring & alertes

  • Surveille CPU, RAM, I/O disque et latence reseau.
  • Configure alertes mail/Telegram.
  • Planifie des rapports mensuels pour l'equipe.

Securite renforcee

  • Desactive l'acces root en SSH si possible.
  • Active le pare-feu au niveau Datacenter.
  • Separation des comptes et logs d'audit.
  • Mises a jour planifiees avec snapshots prealables.
Un hyperviseur non patche expose toutes les VMs.
