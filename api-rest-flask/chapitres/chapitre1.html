Chapitre 1

API Web et HTTP

Comprendre le dialogue client-serveur.

API

API signifie Application Programming Interface: une interface pour faire communiquer des programmes.

Dans le web, une API permet a un frontend ou a une autre application d echanger des donnees avec un backend.

Elle utilise souvent HTTP pour recevoir une requete et renvoyer une reponse.

Memo rapide : API = contrat de communication entre logiciels.

Requete et reponse HTTP

Une requete contient une methode, une URL, des headers et parfois un body.

Une reponse contient un code d etat, des headers et un body.

ElementRole
MethodeAction souhaitee: GET, POST, PUT, DELETE
RouteRessource visee: /students/42
HeaderInformations: Content-Type, Authorization
BodyDonnees envoyees, souvent JSON
Tip : Methode = verbe. Route = nom de la ressource.

Pourquoi JSON

JSON est un format texte lisible par beaucoup de langages. Il represente objets, tableaux, chaines, nombres, booleens et null.

Une API REST renvoie souvent du JSON au lieu du HTML.

{
  "id": 42,
  "firstName": "Lisa",
  "semesterPaid": true
}
Piege : En JSON strict, les cles doivent etre entre guillemets.

Questions de fin de chapitre

Reponds avant de lire la correction. Elle se debloque apres une bonne reponse, ou apres deux erreurs.

1. Que signifie l'acronyme API ?
Correction : Application Programming Interface (API) désigne une interface permettant à différents programmes de communiquer entre eux.
2. Quel est le rôle principal d'une API Web ?
Correction : Une API Web expose des points d'accès (endpoints) sur le réseau pour permettre à un client de lire ou modifier des données sur un serveur.
3. Dans le modèle client-serveur d'une API, que représente le 'client' ?
Correction : Le client est l'émetteur de la requête. Il demande des informations ou des actions au serveur.
4. Dans le modèle client-serveur, quel est le rôle du serveur ?
Correction : Le serveur est à l'écoute des requêtes. Il traite les demandes (ex: lire en base de données) et répond au client.
5. Quel protocole réseau est le fondement du transfert de données pour les API REST ?
Correction : HTTP (Hypertext Transfer Protocol) est le protocole utilisé par REST pour transporter les requêtes et réponses.
