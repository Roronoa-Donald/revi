Chapitre 4

Status codes

Dire clairement ce qui s est passe.

Codes frequents

Le code HTTP fait partie de la reponse. Il explique le resultat technique de la requete.

Une API claire utilise les bons codes au lieu de tout renvoyer en 200.

CodeSens
200OK
201Created
400Bad Request
401Non authentifie
403Interdit
404Ressource introuvable
500Erreur serveur
Memo rapide : 401 = qui es-tu ? 403 = je te connais mais tu n as pas le droit.

Erreurs JSON

Meme en erreur, une API peut renvoyer un JSON comprehensible.

Le frontend peut alors afficher le message proprement.

return jsonify({"error": "student_not_found"}), 404
Tip : Une erreur utile a un code HTTP et un message machine lisible.

Validation

Si un champ obligatoire manque, on renvoie 400.

Si une ressource n existe pas, on renvoie 404.

Piege : Ne renvoie pas 500 pour une erreur utilisateur. 500 signifie probleme serveur.
