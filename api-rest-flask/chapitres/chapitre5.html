Chapitre 5

Tester une API

Verifier autrement qu au hasard.

Navigateur

Le navigateur teste surtout GET. Si tu ouvres /students, il fait une requete GET.

Pour POST, PUT ou DELETE, il faut un outil adapte.

Memo rapide : Navigateur = GET facile, pas suffisant pour toute l API.

Postman, Bruno, Insomnia, cURL

Ces outils permettent de choisir la methode, les headers et le body.

cURL est pratique pour garder une commande reproductible.

curl -X POST http://localhost:5000/students \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"Lisa\"}"
Tip : Un test API precise methode + URL + headers + body attendu.

Checklist

Teste les cas normaux et les erreurs: id introuvable, champ manquant, JSON invalide.

Une API n est pas fiable tant que ses erreurs ne sont pas testees.

  • GET liste.
  • GET detail existant et absent.
  • POST valide et invalide.
  • PUT ressource absente.
  • DELETE puis GET pour confirmer.

Questions de fin de chapitre

Reponds avant de lire la correction. Elle se debloque apres une bonne reponse, ou apres deux erreurs.

1. Pourquoi le navigateur web standard n'est-il pas suffisant pour tester une API complète ?
Correction : Pour tester les requêtes POST, PUT ou DELETE avec des en-têtes complexes, il est nécessaire d'utiliser des outils spécialisés comme Postman ou cURL.
2. Qu'est-ce que l'outil 'cURL' dans le développement web ?
Correction : cURL permet d'effectuer des appels HTTP directement depuis un terminal, ce qui est très pratique pour tester rapidement ou scripter des requêtes.
3. Quelle option cURL permet de définir la méthode HTTP à employer (ex: POST ou DELETE) ?
Correction : L'option `-X` ou `--request` est utilisée pour spécifier le verbe HTTP (GET, POST, PUT, DELETE, etc.).
4. Comment ajoute-t-on un en-tête (Header) personnalisé dans une commande cURL ?
Correction : L'option `-H` permet de configurer les en-têtes HTTP de la requête de test.
5. Quelle option cURL est utilisée pour envoyer des données dans le corps (Body) d'une requête ?
Correction : L'option `-d` ou `--data` permet de passer la chaîne de données ou le JSON (ex: `-d '{"name":"Lisa"}'`).
