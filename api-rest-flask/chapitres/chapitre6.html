Chapitre 6

Mini API complete

Students et classes comme dans le support.

Modele de donnees

Le support donne deux ressources: students et classes.

On peut commencer avec une liste Python en memoire avant de brancher une vraie base.

students = [
  {"id": "1842421", "firstName": "Lisa", "classes": ["420-123-SH"]}
]
classes = [
  {"id": "420-123-SH", "title": "Creation de pages Web"}
]
Memo rapide : Avant la base de donnees, maitrise deja routes + JSON + codes.

CRUD minimal

CRUD signifie Create, Read, Update, Delete.

Dans une API REST, CRUD se traduit souvent par POST, GET, PUT, DELETE.

CRUDHTTP
CreatePOST /students
Read allGET /students
Read oneGET /students/id
UpdatePUT /students/id
DeleteDELETE /students/id

Securite de base

Une vraie API doit proteger certaines routes: authentification, permissions, validation, logs.

Le cours API REST ici reste centre sur HTTP, JSON et Flask, mais le reflexe securite doit deja exister.

Piege : Ne jamais exposer de secrets dans le code frontend. Une cle API reste cote serveur.
