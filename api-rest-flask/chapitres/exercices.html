Exercices progressifs

Applique les notions avec des corrections qui se debloquent apres reussite ou deux echecs.

Exercice 1 - Identifier une requete

Decoupe une requete HTTP en methode, route, headers et body.

Correction :
POST /students HTTP/1.1
Content-Type: application/json

{"name":"Lisa"}

Methode: POST
Route: /students
Header: Content-Type
Body: JSON envoye

Cette lecture evite de confondre l action, la ressource et les donnees.

Exercice 2 - Choisir les routes REST

Propose les routes pour lister, lire, creer, modifier et supprimer des cours.

Correction :
GET /classes
GET /classes/<id>
POST /classes
PUT /classes/<id>
DELETE /classes/<id>

La ressource est classes. La methode porte l action.

Exercice 3 - Route Flask GET

Ecris une route GET /classes qui renvoie une liste JSON.

Correction :
@app.get("/classes")
def list_classes():
    return jsonify(classes)

Une route GET lit une collection sans modifier les donnees.

Exercice 4 - Route Flask POST

Ecris une route POST /students avec request.get_json.

Correction :
@app.post("/students")
def create_student():
    data = request.get_json() or {}
    if not data.get("firstName"):
        return jsonify({"error": "firstName_required"}), 400
    students.append(data)
    return jsonify(data), 201

201 indique qu une ressource a ete creee.

Exercice 5 - Erreur 404

Gere le cas ou GET /students/<id> ne trouve rien.

Correction :
return jsonify({"error": "student_not_found"}), 404

Une ressource absente doit produire une erreur claire.

Exercice 6 - Test cURL

Ecris une commande cURL pour creer un etudiant.

Correction :
curl -X POST http://localhost:5000/students -H "Content-Type: application/json" -d "{\"firstName\":\"Lisa\"}"

La commande precise methode, URL, header JSON et body.
