Chapitre 1

Numeration, representation et logique

Objectif: comprendre les bases binaires, les entiers signes et les masques, avec methodes de conversion et reflexes pratiques.

Objectifs

  • Expliquer la logique des bases et convertir sans se tromper.
  • Comprendre la representation des entiers signes et non signes.
  • Utiliser les masques (AND/OR/XOR/NOT) pour manipuler des bits.
  • Appliquer les decalages pour multiplier/diviser par des puissances de 2.

1) Systemes de numeration (idee cle)

Un systeme de numeration est defini par une base. Chaque chiffre vaut une puissance de cette base.

  • Base 2 (binaire): chiffres 0 et 1.
  • Base 8 (octal): chiffres 0 a 7.
  • Base 10 (decimal): chiffres 0 a 9.
  • Base 16 (hexadecimal): 0-9 puis A-F.

Exemple de lecture positionnelle: 1011b = 1*8 + 0*4 + 1*2 + 1*1 = 11.

Memo technique: les puissances de 2 a retenir: 1, 2, 4, 8, 16, 32, 64, 128, 256.

2) Conversion pas a pas (decimal -> binaire)

La methode la plus fiable est la division successive par 2 et la lecture des restes a l'envers.

Pas a pas: 25 / 2 = 12 reste 1, 12 / 2 = 6 reste 0, 6 / 2 = 3 reste 0, 3 / 2 = 1 reste 1, 1 / 2 = 0 reste 1. Lecture inverse: 11001.

Pour revenir en decimal, on additionne les poids: 11001b = 16 + 8 + 1 = 25.

Memo technique: binaire -> hexa: grouper par 4 bits (nibbles). binaire -> octal: grouper par 3 bits.

3) Entiers non signes vs signes

Un entier non signe utilise tous les bits pour la valeur. Un entier signe reserve le bit de poids fort pour le signe.

  • Plage non signee sur n bits: 0 a (2^n - 1)
  • Plage signee sur n bits: -(2^(n-1)) a (2^(n-1) - 1)

Exemple sur 8 bits: non signe 0..255, signe -128..127.

Piege: confondre signe et non signe change totalement l'interpretation des bits.

4) Complement a 2 (signe)

Le complement a 2 est la methode standard pour coder les entiers negatifs.

  • Ecrire la valeur positive en binaire.
  • Inverser tous les bits.
  • Ajouter 1.
Pas a pas: -13 sur 8 bits: 13 = 00001101, inversion = 11110010, +1 = 11110011.

Verification: si vous refaites le complement a 2 de 11110011, vous retombez sur 13.

5) Logique booleenne et masques

Les operations AND, OR, XOR, NOT servent a manipuler des bits precis.

  • AND: garde uniquement les bits a 1 du masque.
  • OR: force des bits a 1.
  • XOR: inverse (toggle) les bits a 1 dans le masque.
  • NOT: inverse tous les bits.
Memo technique: AND = garder, OR = forcer, XOR = basculer.

Exemple: pour effacer le bit 3, on fait x AND 11110111b (masque avec 0 au bit 3).

6) Decalages (shifts)

Un decalage gauche (<<) multiplie par 2, un decalage droite (>>) divise par 2.

  • Decalage logique: remplit avec 0.
  • Decalage arithmetique: preserve le bit de signe.
Piege: un decalage gauche peut provoquer un depassement si un bit sort du registre.

7) Exemple complet (masques)

Soit x = 01011010b.

  • Mettre le bit 1 a 1: x OR 00000010b = 01011010b (deja a 1).
  • Effacer le bit 4: x AND 11101111b = 01001010b.
  • Basculer le bit 0: x XOR 00000001b = 01011011b.

Recap / Cheat Sheet

Bases:
  • Binaire: 2
  • Octal: 8
  • Decimal: 10
  • Hexa: 16
  • Hexa = groupes de 4 bits
Complement a 2:
  • Inverse bits + 1
  • Plage 8 bits: -128..127
Masques:
  • AND: garder
  • OR: forcer
  • XOR: basculer
  • SHIFT: x2 / /2
