Chapitre 3

Microcontroleurs AVR et ATmega328P

Comprendre l'architecture interne, la memoire, les ports et les interruptions avant d'ecrire du code AVR.

Objectifs

  • Expliquer microcontroleur vs microprocesseur.
  • Connaitre Flash, SRAM, EEPROM et registres internes.
  • Manipuler DDR/PORT/PIN pour les E/S.
  • Comprendre horloge, timers et interruptions.

1) Microcontroleur vs microprocesseur

Un microcontroleur integre le CPU, les memoires et les peripheriques sur une seule puce. Un microprocesseur est surtout un CPU qui depend d'element externes.

  • Microcontroleur: CPU + Flash + SRAM + timers + UART + ADC.
  • Microprocesseur: CPU + composants externes (RAM, chipset).
Memo technique: microcontroleur = tout-en-un, microprocesseur = CPU nu.

2) Architecture ATmega328P (interieur)

L'AVR est de type Harvard: instructions en Flash, donnees en SRAM.

  • Flash: memoire programme (code).
  • SRAM: variables et pile.
  • EEPROM: donnees persistantes.
  • Registres: R0 a R31, directement branches sur l'ALU.

Les registres E/S sont mappes en memoire. On pilote le materiel en ecrivant dans ces adresses.

Memo technique: Flash = code, SRAM = variables, EEPROM = persistant.

3) Ports E/S (DDR, PORT, PIN)

Chaque port (B, C, D) a trois registres:

  • DDRx: direction (1 = sortie, 0 = entree).
  • PORTx: ecriture (sortie) ou pull-up (entree).
  • PINx: lecture de l'etat des broches.
Exemple: PB5 en sortie -> mettre le bit 5 de DDRB a 1, puis ecrire 1 dans PORTB5.
Piege: sur AVR, ecrire 1 dans PINx peut toggler la sortie.

4) Horloge, timers, PWM

  • Horloge: cadence le CPU (MHz), base de tous les timings.
  • Prescaler: divise la frequence pour ralentir un timer.
  • Timers: comptent les cycles, generent des interruptions.
  • PWM: sortie periodique pour controler une LED ou un moteur.

Un timer declenche une interruption quand il deborde ou atteint une valeur comparee.

Memo technique: timer = compteur + interruption + prescaler.

5) Interruptions

Une interruption est un evenement qui force le CPU a executer une routine (ISR).

  • Table des vecteurs: adresse de chaque ISR.
  • Activation globale via l'instruction SEI.
  • Retour via RETI.
Memo technique: SEI = enable interrupts, RETI = retour ISR.

6) Peripheriques courants

  • UART: communication serie (debug, console).
  • SPI / I2C: bus serie pour capteurs et memoires.
  • ADC: convertir une tension en valeur numerique.

Chaque peripherique se configure par registres, puis se pilote par des bits.

Recap / Cheat Sheet

Memoire:
  • Flash: code
  • SRAM: variables/stack
  • EEPROM: persistant
Ports:
  • DDR: direction
  • PORT: sortie/pull-up
  • PIN: lecture
Interruptions:
  • Vecteurs + ISR
  • SEI / RETI
  • Timers, UART, etc.
