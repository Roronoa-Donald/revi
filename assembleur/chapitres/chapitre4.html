Chapitre 4

Toolchain AVR et premier programme

Mettre en place l'environnement et compiler un code embarque.

Objectifs

  • Connaitre AVR-GCC, AVR-ASM et AVRDude.
  • Comprendre la compilation croisee.
  • Produire un premier programme type LED blink.

1) Compilation croisee

Le code est compile sur un PC, mais execute sur un microcontroleur. On parle de compilation croisee.

  • Code source (.c, .S, .asm)
  • Compilation: creation d'un .elf
  • Conversion: .hex pour le flash

2) Outils clefs

  • AVR-GCC: compilateur C/ASM pour AVR.
  • AVR-AS: assembleur (ou utiliser AVR-GCC pour assembler).
  • AVRDude: chargement du firmware sur la carte.
En pratique: compiler, produire le .hex, puis flasher avec AVRDude.

3) Premier programme: LED blink (idee)

Exemple conceptuel:

  • Configurer une broche en sortie (DDR).
  • Mettre la broche a 1 (PORT) pour allumer la LED.
  • Attendre un delai simple (boucle).
  • Mettre la broche a 0 pour eteindre.

4) Etapes type de build

  • Assembler: avr-gcc -mmcu=atmega328p -c main.S
  • Lier: avr-gcc -mmcu=atmega328p -o main.elf main.o
  • Hex: avr-objcopy -O ihex main.elf main.hex
  • Flash: avrdude -p m328p -c arduino -P COMx -U flash:w:main.hex

Recap / Cheat Sheet

Pipeline:
  • source -> elf
  • elf -> hex
  • flash sur carte
Outils:
  • avr-gcc
  • avr-objcopy
  • avrdude
LED Blink:
  • DDR = sortie
  • PORT = on/off
  • Delai par boucle
