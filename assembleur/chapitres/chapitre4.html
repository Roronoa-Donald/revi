Chapitre 4

Toolchain AVR et premier programme

Comprendre tout le flux: du code source au microcontroleur, avec les fichiers et outils intermediaires.

Objectifs

  • Expliquer la compilation croisee et la chaine d'outils.
  • Identifier les fichiers .o, .elf, .hex.
  • Flasher un programme et verifier la cible.
  • Repeter un LED blink simple.

1) Compilation croisee (idee cle)

On compile sur un PC pour executer sur une autre architecture (ATmega). C'est une compilation croisee.

  • Source (.c, .S, .asm)
  • Assemblage/Compilation -> .o
  • Linkage -> .elf
  • Conversion -> .hex
Memo technique: .o = objet, .elf = binaire lie, .hex = format pour flasher.

2) Outils clefs

  • avr-gcc: compilateur et driver principal.
  • avr-as: assembleur (souvent pilote par avr-gcc).
  • avr-objcopy: conversion ELF -> HEX.
  • avrdude: flash du microcontroleur.
Memo technique: compiler -> objcopy -> avrdude.

3) Premier programme: LED blink (pas a pas)

Objectif: faire clignoter une LED connectee sur une broche (ex: PB5).

  • Configurer DDRB5 = 1 (sortie).
  • Ecrire PORTB5 = 1 (allumer).
  • Attendre via une boucle de delai.
  • Ecrire PORTB5 = 0 (eteindre).
Piege: oublie de DDR -> la broche reste en entree, la LED ne s'allume pas.

4) Etapes type de build

  • Assembler: avr-gcc -mmcu=atmega328p -c main.S
  • Lier: avr-gcc -mmcu=atmega328p -o main.elf main.o
  • HEX: avr-objcopy -O ihex main.elf main.hex
  • Flash: avrdude -p m328p -c arduino -P COMx -U flash:w:main.hex
Piege: mauvais modele MCU (-mmcu) ou mauvais port COM = flash impossible.

5) Verification rapide

  • Le bon MCU est selectionne (m328p).
  • Le programmateur est correct (arduino, usbasp...).
  • La frequence d'horloge correspond au materiel.
Memo technique: MCU + programmeur + port = trio gagnant.

Recap / Cheat Sheet

Pipeline:
  • source -> .o
  • .o -> .elf
  • .elf -> .hex
  • flash sur carte
Outils:
  • avr-gcc
  • avr-objcopy
  • avrdude
LED Blink:
  • DDR = sortie
  • PORT = on/off
  • Delai par boucle
