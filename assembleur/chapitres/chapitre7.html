Chapitre 7

NASM avance: pile, syscall et compilation

Couvrir les points frequents des controles et de l'examen final.

Objectifs

  • Comprendre la pile (stack) et les appels.
  • Utiliser les syscalls basiques.
  • Compiler sous Windows avec NASM + GCC (MinGW).

1) Pile et conventions

La pile est geree par RSP. Les instructions PUSH/POP stockent et restaurent des valeurs. RBP sert souvent de base pour les variables locales.

  • PUSH X: RSP diminue, X est stocke.
  • POP X: X est restaure, RSP augmente.

2) Syscalls (mode Linux)

Un syscall passe par RAX (numero) et des registres d'arguments (RDI, RSI, RDX...). Exemple de sortie:

  • RAX = 60 (exit)
  • RDI = code retour
  • SYSCALL
En environnement Windows, on utilise les appels WinAPI et une autre convention.

3) Compilation NASM sous Windows

  • Assembler: nasm -f win64 prog.asm -o prog.obj
  • Lier: gcc prog.obj -o prog.exe

En controle continu, on vous demande souvent les etapes et les outils.

4) Debug et erreurs frequentes

  • Oublier une virgule dans MOV.
  • Confondre adresse et valeur (MOV vs LEA).
  • Mauvais format NASM (win64 vs elf64).

Recap / Cheat Sheet

Pile:
  • RSP = haut de pile
  • PUSH / POP
Syscall (Linux):
  • RAX = numero
  • RDI/RSI/RDX = args
Build Windows:
  • nasm -f win64
  • gcc prog.obj
