Chapitre 7

NASM avance: pile, syscall et compilation

Maitriser la pile, les appels, les syscalls et les differences Linux/Windows.

Objectifs

  • Comprendre la pile, CALL/RET et l'alignement.
  • Utiliser les syscalls Linux et leurs registres d'arguments.
  • Compiler sous Windows et reconnaitre les differences d'ABI.

1) Pile et appels (stack frame)

La pile est geree par RSP. CALL empile l'adresse retour, RET la depile.

  • PUSH X: RSP diminue (8 octets en x64), X est stocke.
  • POP X: X est restaure, RSP augmente.
  • Frame classique: push rbp, mov rbp, rsp, sub rsp, N.
Memo technique: avant un CALL, la pile doit etre alignee a 16 octets (ABI).

2) Syscalls Linux (x86_64)

Un syscall utilise RAX pour le numero et des registres d'arguments.

  • RDI, RSI, RDX, R10, R8, R9 = arguments 1..6
  • RAX = numero du syscall
  • SYSCALL pour entrer noyau
Exemple exit: RAX=60, RDI=0, SYSCALL.

3) Linux vs Windows (ABI)

  • Linux (SysV): args dans RDI, RSI, RDX, RCX, R8, R9.
  • Windows (Win64): args dans RCX, RDX, R8, R9.
Piege: une routine Linux ne fonctionne pas tel quel sous Windows.

4) Compilation NASM sous Windows

  • Assembler: nasm -f win64 prog.asm -o prog.obj
  • Lier: gcc prog.obj -o prog.exe

Sur Linux: nasm -f elf64 puis ld ou gcc.

5) Debug et erreurs frequentes

  • Oublier une virgule dans MOV.
  • Confondre adresse et valeur (MOV vs LEA).
  • Mauvais format NASM (win64 vs elf64).
  • Oublier l'alignement 16 octets avant CALL.
Memo technique: verify format, verify virgules, verify alignement.

Recap / Cheat Sheet

Pile:
  • RSP = haut de pile
  • PUSH / POP
  • Alignement 16
Syscall (Linux):
  • RAX = numero
  • RDI/RSI/RDX = args
Build Windows:
  • nasm -f win64
  • gcc prog.obj
