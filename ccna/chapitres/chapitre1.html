Module 1

Fondamentaux Réseau

Comprendre les modèles de communication, les couches, l'encapsulation et les bases de la transmission de données.

Pourquoi les réseaux informatiques ?

Définition

Un réseau informatique est un ensemble d'équipements (ordinateurs, serveurs, switches, routeurs…) interconnectés qui communiquent entre eux pour partager des ressources (fichiers, imprimantes, connexion Internet).

Types de réseaux

TypePortéeExemple
PAN (Personal Area Network)Quelques mètresBluetooth entre téléphone et écouteurs
LAN (Local Area Network)Un bâtiment / campusRéseau d'entreprise, réseau Wi-Fi maison
MAN (Metropolitan Area Network)Une villeRéseau intersite d'une université
WAN (Wide Area Network)Pays / mondialeInternet, liaisons MPLS entre succursales
Retiens : LAN = local (un site), WAN = étendu (entre sites distants). Internet est le plus grand WAN du monde.

Le Modèle OSI (Open Systems Interconnection)

Le modèle OSI est un modèle de référence à 7 couches créé par l'ISO en 1984. Il décrit comment les données transitent d'une application sur une machine vers une application sur une autre machine, à travers un réseau.

Astuce mémorisation (couche 7 → 1) : « All People Seem To Need Data Processing » → Application, Presentation, Session, Transport, Network, Data Link, Physical.

Les 7 couches en détail

Couche 7 — Application
Interface entre l'utilisateur et le réseau. C'est la couche que « voit » l'utilisateur final.
Protocoles : HTTP, HTTPS, FTP, SMTP, DNS, DHCP, SSH, Telnet
PDU : Données
Couche 6 — Présentation
Traduction, chiffrement et compression des données. Assure que le format est lisible par le récepteur.
Exemples : SSL/TLS (chiffrement), JPEG, ASCII, UTF-8, compression GZIP
Couche 5 — Session
Gère l'ouverture, le maintien et la fermeture des sessions de communication entre applications.
Exemples : NetBIOS, RPC, sessions TCP (SYN/ACK)
Couche 4 — Transport
Assure la livraison fiable (TCP) ou rapide (UDP) des données de bout en bout. Segmente les données et gère le contrôle de flux.
Protocoles : TCP (fiable, connexion, 3-way handshake), UDP (rapide, sans connexion)
PDU : Segment (TCP) / Datagramme (UDP)
Couche 3 — Réseau (Network)
Adressage logique (adresses IP) et routage des paquets d'un réseau à un autre. Le routeur opère à cette couche.
Protocoles : IPv4, IPv6, ICMP, ARP*, OSPF, EIGRP
PDU : Paquet
Couche 2 — Liaison de données (Data Link)
Adressage physique (adresses MAC) et transmission fiable sur un lien local. Le switch opère à cette couche.
Protocoles : Ethernet (802.3), Wi-Fi (802.11), STP, VLANs (802.1Q)
PDU : Trame (Frame)
Couche 1 — Physique
Transmission des bits bruts (0 et 1) sur le support physique (câble cuivre, fibre optique, radio Wi-Fi).
Éléments : Câbles RJ-45/UTP, fibre, hubs, répéteurs, connecteurs
PDU : Bits
*ARP fonctionne entre les couches 2 et 3 : il résout une adresse IP (couche 3) en adresse MAC (couche 2). On le classe souvent en couche 3 par convention.

Le Modèle TCP/IP

Le modèle TCP/IP (ou modèle Internet) est le modèle utilisé en pratique sur Internet et dans la quasi-totalité des réseaux modernes. Il comporte 4 couches :

OSI vs TCP/IP — quand utiliser lequel ?

OSI est un modèle théorique (pour apprendre et dépanner couche par couche). TCP/IP est le modèle pratique implémenté dans tous les systèmes. En examen CCNA, tu dois maîtriser les deux et savoir les mapper l'un à l'autre.

L'Encapsulation et la Décapsulation

Quand des données sont envoyées sur le réseau, chaque couche ajoute un en-tête (header) avec des informations de contrôle. Ce processus s'appelle l'encapsulation. À la réception, c'est l'inverse : la décapsulation.

Processus d'encapsulation (émetteur, couche 7 → 1)

  1. Couche Application → génère les données
  2. Couche Transport → ajoute l'en-tête TCP/UDP → Segment (ports source/destination)
  3. Couche Réseau → ajoute l'en-tête IP → Paquet (IP source/destination)
  4. Couche Liaison → ajoute en-tête Ethernet + trailer FCS → Trame (MAC source/destination)
  5. Couche Physique → convertit en bits (signaux électriques/optiques/radio)

Schéma visuel d'encapsulation

┌─────────────────────────────────────────────────────────┐ │ DONNÉES │ Couche Application ├──────────┬──────────────────────────────────────────────┤ │ TCP/UDP │ DONNÉES │ Couche Transport → Segment ├──────────┼──────────┬───────────────────────────────────┤ │ IP │ TCP/UDP │ DONNÉES │ Couche Réseau → Paquet ├──────────┼──────────┼──────────┬────────────────┬───────┤ │ ETH hdr │ IP │ TCP/UDP │ DONNÉES │ FCS │ Couche Liaison → Trame ├──────────┴──────────┴──────────┴────────────────┴───────┤ │ 01101001 10110100 11001010 01110011... │ Couche Physique → Bits └─────────────────────────────────────────────────────────┘
PDU (Protocol Data Unit) = l'unité de données à chaque couche. Retiens : Données → Segment → Paquet → Trame → Bits.

Le 3-Way Handshake TCP

Établissement d'une connexion TCP

Client Serveur │ │ │─── SYN ──────────────────────────────────────→│ 1. Client envoie SYN (seq=100) │ │ │←── SYN-ACK ────────────────────────────────│ 2. Serveur répond SYN-ACK (seq=300, ack=101) │ │ │─── ACK ──────────────────────────────────────→│ 3. Client confirme ACK (ack=301) │ │ │←══════════ Connexion établie ═══════════════→│

SYN = « Je veux communiquer » | SYN-ACK = « OK, moi aussi » | ACK = « C'est parti ! »

ÉquipementCouche OSIRôleAdressage

Hub vs Switch : Un hub envoie les données à tout le monde (collision domain unique). Un switch envoie les données uniquement au destinataire (une collision domain par port). Le hub est obsolète.
