Adressage IPv4

Classes d'adresses, masques de sous-réseau, CIDR, VLSM et calcul de plages — la compétence numéro 1 du CCNA.

Structure d'une adresse IPv4

Format

Une adresse IPv4 est codée sur 32 bits (4 octets), notée en décimal pointé :

Exemple : 192.168.1.10 En binaire : 11000000.10101000.00000001.00001010 Octet : 192 . 168 . 1 . 10 Chaque octet = 8 bits → valeur de 0 à 255

Partie réseau vs Partie hôte

Adresse = Réseau + Hôte

Adresse : 192.168.1.10 Masque : 255.255.255.0 (/24) 192.168.1 . 10 ───────── ── │ └─ Partie HÔTE (identifie la machine dans le réseau) └─ Partie RÉSEAU (identifie le réseau)

Le masque de sous-réseau détermine où se situe la frontière entre la partie réseau et la partie hôte. Un bit à 1 dans le masque = bit réseau, bit à 0 = bit hôte.

Le Subnetting (sous-réseaux)

Pourquoi sous-réseauter ?

  • Réduire le broadcast — chaque sous-réseau a son propre domaine de broadcast
  • Sécurité — isoler les départements (RH, IT, Direction)
  • Optimiser l'adressage — ne pas gâcher d'adresses IP

Formules essentielles

// Nombre de sous-réseaux 2^n où n = nombre de bits empruntés à la partie hôte // Nombre d'hôtes par sous-réseau 2^h - 2 où h = nombre de bits hôte restants (-2 car on enlève : adresse réseau + adresse broadcast) // Taille de saut (block size) 256 - valeur du masque dans l'octet concerné

Exemple complet : sous-réseauter 192.168.10.0/24 en 4 sous-réseaux

Calcul pas à pas

Réseau de départ : 192.168.10.0/24 (masque 255.255.255.0) Objectif : 4 sous-réseaux Étape 1 : Combien de bits emprunter ? 2^n ≥ 4 → n = 2 bits empruntés Étape 2 : Nouveau masque /24 + 2 = /26 → 255.255.255.192 Étape 3 : Taille de saut 256 - 192 = 64 (chaque sous-réseau contient 64 adresses) Étape 4 : Hôtes par sous-réseau 2^6 - 2 = 62 hôtes utilisables Étape 5 : Tableau des sous-réseaux

CIDR et VLSM

CIDR — Classless Inter-Domain Routing

Le CIDR permet d'utiliser des masques de n'importe quelle longueur (pas seulement /8, /16, /24). Notation : adresse/préfixe (ex : 10.0.0.0/12).

VLSM — Variable Length Subnet Masking

Le VLSM permet de créer des sous-réseaux de tailles différentes pour optimiser l'utilisation des adresses.

Exemple : Réseau 192.168.1.0/24 à diviser pour : - Département A : 100 machines → besoin de /25 (126 hôtes) - Département B : 50 machines → besoin de /26 (62 hôtes) - Département C : 20 machines → besoin de /27 (30 hôtes) - Lien WAN : 2 machines → besoin de /30 (2 hôtes) Règle VLSM : Toujours commencer par le plus grand sous-réseau ! Dept A : 192.168.1.0/25 (1 à 126) Dept B : 192.168.1.128/26 (129 à 190) Dept C : 192.168.1.192/27 (193 à 222) WAN : 192.168.1.224/30 (225 à 226)
