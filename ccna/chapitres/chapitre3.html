ARP — Address Resolution Protocol

Problème résolu

Un PC connaît l'IP de destination mais pas son adresse MAC. Or, pour envoyer une trame Ethernet (couche 2), il faut la MAC. ARP fait le pont entre couche 3 (IP) et couche 2 (MAC).

Fonctionnement pas à pas

PC1 (192.168.1.10) PC2 (192.168.1.20) MAC: AA:AA:AA:AA:AA:AA MAC: BB:BB:BB:BB:BB:BB │ │ │── ARP Request (Broadcast) ──────────→│ "Qui a 192.168.1.20 ?" │ Dest MAC = FF:FF:FF:FF:FF:FF │ (envoyé à TOUS les appareils) │ │ │←─ ARP Reply (Unicast) ────────────│ "C'est moi ! Ma MAC = BB:BB:..." │ Dest MAC = AA:AA:AA:AA:AA:AA │ (réponse directe à PC1) │ │ │ → PC1 met à jour sa table ARP │
# Commandes utiles PC> arp -a # Afficher la table ARP (Windows/Linux) Switch# show arp # Afficher la table ARP (Cisco) PC> arp -d # Vider la table ARP
Sécurité : ARP n'a aucune authentification. Un attaquant peut envoyer de faux ARP Reply → ARP Spoofing (attaque man-in-the-middle). Protection : DAI (Dynamic ARP Inspection) sur le switch.

ICMP — Internet Control Message Protocol

Rôle

ICMP est le protocole de diagnostic du réseau. Il ne transporte pas de données utilisateur, mais des messages de contrôle et d'erreur. Il opère à la couche 3, encapsulé directement dans IP (pas de port TCP/UDP).

Type ICMPMessageUtilisation
Type 0Echo ReplyRéponse à un ping
Type 3Destination UnreachableRéseau/hôte/port inaccessible
Type 8Echo RequestEnvoi d'un ping
Type 11Time ExceededTTL expiré (utilisé par traceroute)
# Outils basés sur ICMP PC> ping 192.168.1.1 # Echo Request (type 8) → attend Echo Reply (type 0) PC> tracert 8.8.8.8 # Trace la route (TTL incrémental, attend Time Exceeded) PC> traceroute 8.8.8.8 # Version Linux

DHCP — Dynamic Host Configuration Protocol

Rôle et Transport

DHCP attribue automatiquement une configuration IP aux clients : adresse IP, masque, passerelle, serveurs DNS.

Transport : UDP — Port 67 (serveur) / Port 68 (client)

Le processus DORA (4 étapes)

Client (pas encore d'IP) Serveur DHCP (192.168.1.1) │ │ │── Discover ──────────────────────────────→│ Broadcast : "Qui est serveur DHCP ?" │ (UDP src:68, dst:67, dest:255.255.255.255) │ │ │ │←─ Offer ────────────────────────────────│ "Je te propose 192.168.1.50" │ (UDP src:67, dst:68) │ │ │ │── Request ─────────────────────────────→│ "OK, j'accepte 192.168.1.50" │ (encore broadcast) │ │ │ │←─ Acknowledge ──────────────────────────│ "Confirmé ! Durée du bail : 24h" │ │ │ Client configuré : 192.168.1.50/24 │
DORA = Discover, Offer, Request, Acknowledge. Retiens cet acronyme !
# Commandes de diagnostic PC> ipconfig /release # Libérer l'IP DHCP (Windows) PC> ipconfig /renew # Renouveler le bail DHCP Router# show ip dhcp binding # Voir les baux DHCP sur un routeur Cisco

DNS — Domain Name System

Rôle et Transport

DNS traduit les noms de domaine (google.com) en adresses IP (142.250.74.46). C'est l'annuaire d'Internet.

Transport : UDP/TCP — Port 53

UDP 53 pour les requêtes courantes (< 512 octets). TCP 53 pour les transferts de zone et les réponses volumineuses.

Fonctionnement de la résolution DNS

Client tape "www.example.com" │ ├→ 1. Vérifie le cache DNS local │ (ipconfig /displaydns sur Windows) │ ├→ 2. Interroge le serveur DNS récursif (ex: 8.8.8.8) │ Le serveur fait la recherche à ta place : │ │ DNS récursif → Root Server (.) → "Va voir les serveurs .com" │ DNS récursif → TLD Server (.com) → "Va voir ns.example.com" │ DNS récursif → Authoritative Server → "Voici l'IP: 93.184.216.34" │ └→ 3. Le client reçoit 93.184.216.34 et met en cache

NTP, Syslog & SNMP — Services d'infrastructure

NTP — Network Time Protocol

UDP 123 — Synchronise l'heure sur tous les équipements du réseau. Essentiel pour corréler les logs, les certificats SSL et l'authentification.

Router(config)# ntp server 0.pool.ntp.org Router# show clock Router# show ntp associations

Syslog — Centralisation des logs

UDP 514 — Envoie les journaux d'événements (logs) des routeurs/switches vers un serveur central pour analyse et archivage.

SNMP — Simple Network Management Protocol

UDP 161 (requêtes) / UDP 162 (traps) — Supervise et gère les équipements réseau à distance. Le serveur SNMP (NMS) interroge les agents sur les équipements.

# Concepts clés MIB : Management Information Base (base de données des objets supervisés) OID : Identifiant unique d'un objet dans la MIB Agent : Software sur le switch/routeur qui répond aux requêtes SNMP NMS : Network Management System (serveur de supervision) Trap : Alerte envoyée spontanément par l'agent vers le NMS

STP — Spanning Tree Protocol

Problème résolu

Quand des switches sont interconnectés en boucle (redondance), les trames tournent indéfiniment = tempête de broadcast → réseau saturé, tables MAC instables, CPU à 100%.

STP (IEEE 802.1D) détecte et bloque les liens redondants pour éliminer les boucles, tout en gardant un chemin de secours en cas de panne.

Fonctionnement simplifié

┌────────────────┐ ┌────────────────┐ │ Switch A │──────────│ Switch B │ │ BID: 32768.AA │ actif │ BID: 32768.BB │ └───────┬────────┘ └────────┬───────┘ │ │ │ actif │ bloqué ✖ │ │ ┌───────┴────────────────────────────┴───────┐ │ Switch C (Root Bridge) │ │ BID: 4096.CC (le plus bas) │ └────────────────────────────────────────────┘ STP élit le Root Bridge (BID le plus bas) et bloque un port pour casser la boucle

Étapes de STP

  1. Élection du Root Bridge — switch avec le BID le plus bas (priorité + MAC)
  2. Root Port — sur chaque switch non-root, le port le plus proche du root bridge
  3. Designated Port — sur chaque segment, le port qui mène le mieux au root
  4. Blocked Port — tous les autres ports sont bloqués (pas de trafic)
Switch# show spanning-tree # État STP complet Switch# show spanning-tree vlan 10 # STP pour un VLAN spécifique Switch(config)# spanning-tree vlan 10 priority 4096 # Forcer ce switch comme Root

Protocoles de messagerie

Expéditeur Serveur Mail Destinataire │ │ │ │── SMTP (TCP 25) ──────────→│ │ │ "Envoie cet email" │ │ │ │── stocke le mail ──→ │ │ │ │ │ │←─ POP3 (TCP 110) ─────────│ "Télécharge et supprime" │ │ OU │ │ │←─ IMAP (TCP 143) ─────────│ "Synchronise (reste sur serveur)"
POP3 vs IMAP : POP3 télécharge les mails et les supprime du serveur (1 seul appareil). IMAP synchronise et garde tout sur le serveur (multi-appareil). IMAP est le standard moderne.
🧠 Mémo — Protocoles essentiels : « Des Héros Très Forts »
• DNS (53) → nom → IP (l'annuaire d'internet)
• HTTP/S (80/443) → pages web
• TFTP (69) → transfert simple (configs Cisco)
• FTP (20/21) → transfert de fichiers
TCP vs UDP : TCP = fiable (accusé de réception), UDP = rapide (pas de vérification).
« TCP = Téléphone (conversation), UDP = Uberisation (on envoie sans attendre) »
