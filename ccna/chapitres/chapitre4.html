Ethernet & Switching

Trames Ethernet, adresses MAC, tables CAM, fonctionnement d'un switch L2, domaines de collision et de broadcast.

La Trame Ethernet (IEEE 802.3)

Ethernet est le standard couche 2 dominant dans les réseaux LAN. Les données sont encapsulées dans des trames :

┌──────────┬──────────┬──────────┬──────────┬─────────────────┬─────────┐ │ Préambule │ MAC Dest │ MAC Src │ Type/Len │ Données │ FCS │ │ 8 oct. │ 6 oct. │ 6 oct. │ 2 oct. │ 46-1500 oct. │ 4 oct. │ └──────────┴──────────┴──────────┴──────────┴─────────────────┴─────────┘ Préambule : Synchronisation (10101010... + SFD) MAC Dest : Adresse MAC du destinataire MAC Src : Adresse MAC de l'émetteur Type : 0x0800 = IPv4, 0x0806 = ARP, 0x86DD = IPv6 Données : Payload (paquet IP encapsulé) FCS : Frame Check Sequence (CRC pour détecter les erreurs)
MTU (Maximum Transmission Unit) = 1500 octets pour Ethernet. Si le paquet IP est plus grand, il sera fragmenté.

Fonctionnement du Switch L2

Les 3 actions d'un switch

  1. Learn (Apprendre) — Le switch lit la MAC source de chaque trame reçue et l'associe au port d'entrée dans sa table MAC (CAM)
  2. Forward (Transmettre) — Si la MAC destination est dans la table → envoie sur le port correspondant (unicast)
  3. Flood (Inonder) — Si la MAC destination est inconnue → envoie sur tous les ports sauf le port source (unknown unicast flood)

Exemple visuel

Domaines de Collision & Broadcast

Domaine de collision

Zone où deux appareils qui transmettent en même temps provoquent une collision.

  • Hub : tous les ports = 1 seul domaine de collision
  • Switch : chaque port = son propre domaine de collision (micro-segmentation)

Domaine de broadcast

Zone où un broadcast (MAC: FF:FF:FF:FF:FF:FF) est propagé à tous.

  • Switch : tous les ports du même VLAN = 1 domaine de broadcast
  • Routeur : sépare les domaines de broadcast (chaque interface = un domaine)
Retiens : Un switch segmente les domaines de collision. Un routeur (ou un VLAN) segmente les domaines de broadcast.

Duplex et Vitesse

ModeDescriptionExemple
Half-duplexEnvoi OU réception (pas les deux en même temps)Hub, Wi-Fi
Full-duplexEnvoi ET réception simultanésSwitch moderne
Switch(config-if)# duplex full # Forcer full-duplex Switch(config-if)# speed 100 # Forcer 100 Mbps Switch(config-if)# duplex auto # Auto-négociation (recommandé) Switch# show interfaces fa0/1 # Vérifier duplex et speed
Duplex mismatch : Si un côté est en full et l'autre en half → performances terribles, erreurs CRC, late collisions. Toujours vérifier avec show interfaces !

🧠 Mémo — Le Switch en 3 actions : « A.I.F = Apprendre, Inonder, Filtrer »
• Apprendre → le switch lit la MAC source et la mémorise dans sa table
• Inonder → si la MAC destination est inconnue, il envoie à TOUS les ports
• Filtrer → si la MAC est connue, il envoie UNIQUEMENT au bon port
La table MAC est comme un carnet d'adresses : au début vide, elle se remplit à chaque trame reçue.
