CLI Cisco IOS

Modes d'exécution, commandes show, configuration de base, sécurisation de l'accès, sauvegarde et restauration.

Les Modes de l'IOS

Cisco IOS utilise une hiérarchie de modes. Chaque mode a son propre prompt et ses commandes disponibles :

┌─────────────────────────────────────────────────────────────────┐ │ HIÉRARCHIE DES MODES IOS │ │ │ │ User EXEC Switch> │ │ │ enable │ │ ▼ │ │ Privileged EXEC Switch# │ │ │ configure terminal │ │ ▼ │ │ Global Config Switch(config)# │ │ │ │ │ ├──► Interface Config Switch(config-if)# │ │ ├──► Line Config Switch(config-line)# │ │ ├──► Router Config Switch(config-router)# │ │ └──► VLAN Config Switch(config-vlan)# │ └─────────────────────────────────────────────────────────────────┘
ModePromptAccèsUsage
User EXECSwitch>Connexion initialeCommandes basiques (ping, show version)
Privileged EXECSwitch#enableshow complets, copy, debug, reload
Global ConfigSwitch(config)#configure terminalConfiguration générale (hostname, etc.)
Interface Config(config-if)#interface fa0/1IP, shutdown, duplex, speed
Line Config(config-line)#line console 0Console, VTY (SSH/Telnet)
# Navigation entre les modes Switch> enable # User → Privileged Switch# configure terminal # Privileged → Global Config Switch(config)# interface fa0/1 # Global → Interface Config Switch(config-if)# exit # Retour au mode précédent Switch(config)# end # Retour direct au Privileged Switch# disable # Privileged → User

Commandes Show Essentielles

CommandeModeDescription
show running-config#Configuration active en RAM
show startup-config#Configuration sauvegardée en NVRAM
show version> / #Version IOS, uptime, mémoire, modèle
show ip interface brief#Résumé de toutes les interfaces (IP, status)
show interfaces fa0/1#Stats détaillées d'une interface
show mac address-table#Table MAC du switch
show vlan brief#VLANs configurés et ports
show ip route#Table de routage
show arp#Table ARP (IP ↔ MAC)
show protocols#Protocoles de routage actifs
Astuce : Tape le début d'une commande + Tab pour l'auto-compléter. Utilise ? à tout moment pour voir les commandes disponibles.

Configuration de Base

# ========================================== # CONFIGURATION INITIALE D'UN SWITCH/ROUTEUR # ========================================== # 1. Hostname Switch(config)# hostname SW1 # 2. Banner (message d'avertissement) SW1(config)# banner motd # Accès réservé - Unauthorized access prohibited # # 3. Mot de passe console SW1(config)# line console 0 SW1(config-line)# password cisco123 SW1(config-line)# login SW1(config-line)# exit # 4. Mot de passe enable (chiffré) SW1(config)# enable secret class123 # 5. Chiffrer tous les mots de passe en clair SW1(config)# service password-encryption # 6. Désactiver la résolution DNS (évite les délais) SW1(config)# no ip domain-lookup # 7. Configurer SSH (accès distant sécurisé) SW1(config)# ip domain-name lab.local SW1(config)# crypto key generate rsa modulus 2048 SW1(config)# username admin secret admin123 SW1(config)# line vty 0 15 SW1(config-line)# transport input ssh SW1(config-line)# login local SW1(config-line)# exit # 8. Interface VLAN 1 (management) — Switch SW1(config)# interface vlan 1 SW1(config-if)# ip address 192.168.1.2 255.255.255.0 SW1(config-if)# no shutdown SW1(config-if)# exit SW1(config)# ip default-gateway 192.168.1.1

Sauvegarde & Restauration

┌──────────────────────────────────────────────────────────┐ │ MÉMOIRES CISCO │ │ │ │ RAM → running-config (actif, perdu au reboot) │ │ NVRAM → startup-config (persistant) │ │ Flash → Image IOS (système d'exploitation) │ │ ROM → Bootstrap, POST, ROMMON │ └──────────────────────────────────────────────────────────┘
# Sauvegarder la config active SW1# copy running-config startup-config # ou : write memory (wr) # Restaurer la config de démarrage SW1# copy startup-config running-config # Sauvegarder sur TFTP SW1# copy running-config tftp: # → Entrer l'IP du serveur TFTP + nom fichier # Effacer la config et redémarrer à zéro SW1# erase startup-config SW1# reload
Attention : Si tu ne fais pas copy run start, ta configuration est perdue au prochain reboot ! La running-config est en RAM (volatile).

🧠 Mémo — Les 3 modes Cisco : « U.P.C = User, Privileged, Config »
• Router> → Mode User (lecture seule)
• Router# → Mode Privileged (enable pour y accéder)
• Router(config)# → Mode Config (configure terminal)
Règle : enable pour monter, exit pour descendre, end pour retourner tout en haut.
Comme un ascenseur : enable = monter d'un étage, exit = descendre, end = rez-de-chaussée.
