Routage Statique

Table de routage, routes statiques, passerelle par défaut, route de dernier recours, dépannage réseau multi-routeur.

Rôle du Routeur

Le routeur opère à la couche 3 (Réseau) et prend des décisions de transfert basées sur l'adresse IP destination.

Caractéristiques clés

  • Chaque interface du routeur = un réseau différent
  • Le routeur sépare les domaines de broadcast
  • Le routeur consulte sa table de routage pour transférer les paquets
  • Si aucune route ne correspond → le paquet est dropped

La Table de Routage

La table de routage contient les réseaux connus et comment les atteindre :

R1# show ip route Codes: C - connected, S - static, R - RIP, O - OSPF, D - EIGRP Gateway of last resort is 10.0.0.2 to network 0.0.0.0 C 192.168.1.0/24 is directly connected, GigabitEthernet0/0 L 192.168.1.1/32 is directly connected, GigabitEthernet0/0 C 10.0.0.0/30 is directly connected, Serial0/0/0 L 10.0.0.1/32 is directly connected, Serial0/0/0 S 192.168.2.0/24 [1/0] via 10.0.0.2 S* 0.0.0.0/0 [1/0] via 10.0.0.2
CodeTypeDescription
CConnectedRéseau directement connecté à une interface active
LLocalAdresse IP exacte de l'interface locale (/32)
SStaticRoute configurée manuellement par l'admin
S*Default StaticRoute par défaut (0.0.0.0/0)
[1/0] = [Distance Administrative / Métrique]. La route statique a une AD de 1. Plus l'AD est basse, plus la route est préférée.

Configuration des Interfaces Routeur

# Configurer une interface GigabitEthernet R1(config)# interface GigabitEthernet0/0 R1(config-if)# ip address 192.168.1.1 255.255.255.0 R1(config-if)# description LAN Réseau 1 R1(config-if)# no shutdown R1(config-if)# exit # Configurer une interface Série (WAN) R1(config)# interface Serial0/0/0 R1(config-if)# ip address 10.0.0.1 255.255.255.252 R1(config-if)# clock rate 128000 # Côté DCE seulement R1(config-if)# no shutdown R1(config-if)# exit # Vérification R1# show ip interface brief Interface IP-Address OK? Method Status Protocol Gi0/0 192.168.1.1 YES manual up up Se0/0/0 10.0.0.1 YES manual up up

Routes Statiques

Une route statique indique au routeur comment atteindre un réseau distant :

# Syntaxe générale ip route {réseau_dest} {masque} {next-hop | exit-interface} # Via next-hop (IP du routeur voisin) — RECOMMANDÉ R1(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2 # Via interface de sortie R1(config)# ip route 192.168.2.0 255.255.255.0 Serial0/0/0 # Route par défaut (passerelle de dernier recours) R1(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.2 # Supprimer une route R1(config)# no ip route 192.168.2.0 255.255.255.0 10.0.0.2
Route par défaut 0.0.0.0/0 = "Si tu ne sais pas où envoyer le paquet, envoie-le vers cette adresse." C'est la Gateway of Last Resort.

Dépannage Routage

Checklist de dépannage

  1. ✅ Les interfaces sont-elles up/up ? → show ip interface brief
  2. ✅ Les adresses IP sont-elles correctes ? (même réseau pour le lien WAN)
  3. ✅ La route statique pointe vers le bon next-hop ?
  4. ✅ La route de retour existe sur le routeur distant ?
  5. ✅ Le PC a la bonne passerelle par défaut ?
  6. ✅ Pas de shutdown oublié sur une interface ?
Erreur n°1 : Oublier la route de retour. Si R1 sait atteindre LAN2 mais R2 ne sait pas atteindre LAN1, le ping partira mais la réponse n'aura aucun chemin de retour.
🧠 Mémo — Route statique : « R.M.N = Réseau, Masque, Next-hop »
ip route [réseau] [masque] [next-hop]
Exemple : ip route 192.168.2.0 255.255.255.0 10.0.0.2
→ « Pour aller au réseau .2.0, passe par 10.0.0.2 »
Route par défaut : ip route 0.0.0.0 0.0.0.0 [next-hop] → « tout ce que je ne connais pas, envoie-le là-bas »
0.0.0.0 = « n'importe quelle destination ». C'est le GPS par défaut du routeur.
