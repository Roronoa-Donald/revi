Module 7 — Chapitre Phare

VLANs sans Trunking

Comprendre, créer, assigner et vérifier des VLANs sur un switch. Segmentation logique du réseau sans liaison trunk.

Qu'est-ce qu'un VLAN ?

Un VLAN (Virtual LAN) est un réseau local logique créé au sein d'un switch. Il permet de segmenter un réseau physique en plusieurs domaines de broadcast indépendants, sans avoir besoin de routeurs ou de switches supplémentaires.

Pourquoi utiliser des VLANs ?

  • Sécurité — Isoler les départements (Comptabilité ne voit pas la R&D)
  • Performance — Réduire la taille des domaines de broadcast → moins de trafic inutile
  • Organisation — Grouper les utilisateurs par fonction, pas par emplacement physique
  • Flexibilité — Déplacer un utilisateur dans un autre VLAN sans changer le câblage
  • Coût — Un seul switch physique peut remplacer plusieurs switches dédiés
AVANT les VLANs : 1 switch = 1 seul domaine de broadcast ┌─────────────────────────────────────────────────┐ │ SWITCH │ │ Tous les ports = même broadcast │ │ PC1 ←→ PC2 ←→ PC3 ←→ PC4 ←→ PC5 ←→ PC6 │ └─────────────────────────────────────────────────┘ APRÈS les VLANs : 1 switch = PLUSIEURS domaines de broadcast ┌─────────────────────────────────────────────────┐ │ SWITCH │ │ │ │ ┌─VLAN 10──┐ ┌─VLAN 20──┐ ┌─VLAN 30──┐ │ │ │ PC1 PC2 │ │ PC3 PC4 │ │ PC5 PC6 │ │ │ │ Compta │ │ R&D │ │ Direction │ │ │ └──────────┘ └──────────┘ └──────────┘ │ │ Broadcast Broadcast Broadcast │ │ isolé ✓ isolé ✓ isolé ✓ │ └─────────────────────────────────────────────────┘
Important : Les PCs de VLANs différents ne peuvent PAS communiquer entre eux sans un routeur (ou du routage inter-VLAN, cf. Module 8). C'est voulu : c'est la segmentation !

Types de Ports Switch

TypeDescriptionUsage
AccessAppartient à un seul VLAN. La trame n'est PAS taggée (802.1Q).Connexion de PCs, imprimantes, serveurs
TrunkTransporte plusieurs VLANs. Les trames sont taggées avec 802.1Q.Liaison switch↔switch ou switch↔routeur
Dans ce chapitre, on ne configure que des ports Access. Le trunking est abordé au Module 8 (ROAS). Un port Access = 1 VLAN = 1 broadcast domain.

Créer et Assigner des VLANs

Étape 1 — Créer les VLANs

SW1(config)# vlan 10 SW1(config-vlan)# name COMPTABILITE SW1(config-vlan)# exit SW1(config)# vlan 20 SW1(config-vlan)# name INFORMATIQUE SW1(config-vlan)# exit SW1(config)# vlan 30 SW1(config-vlan)# name DIRECTION SW1(config-vlan)# exit

Étape 2 — Assigner les ports aux VLANs

# Assigner UN port au VLAN 10 SW1(config)# interface FastEthernet0/1 SW1(config-if)# switchport mode access SW1(config-if)# switchport access vlan 10 SW1(config-if)# exit # Assigner une PLAGE de ports au VLAN 20 SW1(config)# interface range FastEthernet0/5 - 10 SW1(config-if-range)# switchport mode access SW1(config-if-range)# switchport access vlan 20 SW1(config-if-range)# exit # Assigner les ports 11-15 au VLAN 30 SW1(config)# interface range Fa0/11 - 15 SW1(config-if-range)# switchport mode access SW1(config-if-range)# switchport access vlan 30 SW1(config-if-range)# exit
switchport mode access force le port en mode Access (pas de négociation trunk). switchport access vlan X assigne le port au VLAN X.

Commandes de Vérification

SW1# show vlan brief VLAN Name Status Ports ---- -------------------- --------- ------------------------- 1 default active Fa0/16-24, Gi0/1-2 10 COMPTABILITE active Fa0/1-4 20 INFORMATIQUE active Fa0/5-10 30 DIRECTION active Fa0/11-15 SW1# show vlan id 10 # Détails du VLAN 10 uniquement SW1# show interfaces fa0/1 switchport Name: Fa0/1 Switchport: Enabled Administrative Mode: static access Operational Mode: static access Access Mode VLAN: 10 (COMPTABILITE)

Supprimer / Modifier un VLAN

# Retirer un port de son VLAN (retourne au VLAN 1) SW1(config)# interface fa0/1 SW1(config-if)# no switchport access vlan SW1(config-if)# exit # Supprimer un VLAN SW1(config)# no vlan 30 # ⚠️ Les ports qui étaient dans le VLAN 30 deviennent INACTIFS # Ils apparaîtront dans un VLAN qui n'existe plus → pas de communication # Il faut les réassigner manuellement !
Piège classique : Supprimer un VLAN ne déplace pas automatiquement les ports vers le VLAN 1. Les ports restent assignés au VLAN supprimé et ne peuvent plus communiquer. Toujours réassigner les ports avant la suppression.

C'est normal ! Les VLANs sont conçus pour isoler. Pour que VLAN 10 communique avec VLAN 20, il faut du routage inter-VLAN → c'est exactement ce qu'on fait au Module 8 (ROAS) !

Bonnes Pratiques VLANs

Règles de sécurité

  1. ❌ Ne jamais utiliser le VLAN 1 pour le trafic utilisateur (VLAN par défaut = cible d'attaque)
  2. ✅ Créer un VLAN de management dédié (ex: VLAN 99) pour l'accès SSH au switch
  3. ✅ Désactiver les ports non utilisés : shutdown
  4. ✅ Assigner les ports non utilisés à un VLAN « parking » (ex: VLAN 999)
  5. ✅ Documenter le plan d'adressage VLAN (tableau VLAN-ID / Nom / Sous-réseau)
# Sécuriser les ports non utilisés SW1(config)# vlan 999 SW1(config-vlan)# name PARKING SW1(config-vlan)# exit SW1(config)# interface range fa0/16 - 24 SW1(config-if-range)# switchport mode access SW1(config-if-range)# switchport access vlan 999 SW1(config-if-range)# shutdown SW1(config-if-range)# exit
🧠 Mémo — VLANs : « 1 VLAN = 1 Réseau = 1 Broadcast »
• Un VLAN isole le trafic comme un mur virtuel
• VLAN 1 = défaut (ne JAMAIS l'utiliser en production)
• Les PCs de VLANs différents ne peuvent PAS communiquer sans routeur
Commandes clés : vlan 10 → créer, switchport access vlan 10 → assigner un port
Imagine un immeuble : chaque VLAN est un étage. Sans ascenseur (routeur), impossible de changer d'étage.
