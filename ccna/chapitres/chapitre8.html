Module 8 — Chapitre Phare

ROAS : Router on a Stick

Trunking 802.1Q, sous-interfaces, encapsulation dot1Q, routage inter-VLAN complet avec un seul lien physique.

Le Problème à Résoudre

Au Module 7, on a créé des VLANs. Les PCs du même VLAN communiquent, mais ceux de VLANs différents sont totalement isolés. Comment les faire communiquer ?

3 méthodes de routage inter-VLAN

MéthodePrincipeInconvénient
1 interface par VLANChaque VLAN a son propre câble vers le routeur❌ Coûteux, gaspille des ports
ROAS (Router on a Stick)1 seul câble Trunk + sous-interfaces virtuelles✅ Économique, solution classique
Switch L3Le switch fait lui-même le routage (SVI)Switch plus cher, config différente

Le Standard 802.1Q (Trunking)

Un trunk est un lien qui transporte le trafic de plusieurs VLANs. Le standard IEEE 802.1Q ajoute un tag de 4 octets dans la trame Ethernet pour identifier le VLAN :

Trame Ethernet NORMALE (port Access) : ┌───────────┬───────────┬──────────┬───────────────┬─────┐ │ MAC Dest │ MAC Src │ Type │ Données │ FCS │ └───────────┴───────────┴──────────┴───────────────┴─────┘ Trame Ethernet TAGGÉE (port Trunk — 802.1Q) : ┌───────────┬───────────┬────────────┬──────────┬──────────┬─────┐ │ MAC Dest │ MAC Src │ 802.1Q Tag │ Type │ Données │ FCS │ └───────────┴───────────┴────────────┴──────────┴──────────┴─────┘ │ ┌───────────┴───────────────┐ │ Tag 802.1Q (4 octets) │ │ ┌──────┬───┬──────────┐ │ │ │ TPID │PRI│ VLAN ID │ │ │ │0x8100│3b │ 12 bits │ │ │ └──────┴───┴──────────┘ │ │ VLAN ID: 0-4095 │ └───────────────────────────┘

Concepts clés du trunk

  • Native VLAN — Le VLAN dont les trames ne sont PAS taggées sur le trunk (par défaut : VLAN 1)
  • Allowed VLANs — Liste des VLANs autorisés à traverser le trunk
  • DTP (Dynamic Trunking Protocol) — Négociation automatique trunk/access (désactiver en prod !)

Architecture ROAS

Le principe : une seule interface physique du routeur reçoit un lien trunk, puis on crée des sous-interfaces virtuelles, chacune associée à un VLAN :

PC1 (VLAN 10) PC2 (VLAN 20) PC3 (VLAN 30) 192.168.10.10 192.168.20.10 192.168.30.10 GW: .1 GW: .1 GW: .1 │ Fa0/1 │ Fa0/5 │ Fa0/9 └───────┐ ┌──┘ ┌──┘ │ │ │ ┌───┴─────┴───────────────────┴───┐ │ SW1 │ │ VLAN 10: Fa0/1-4 │ │ VLAN 20: Fa0/5-8 │ │ VLAN 30: Fa0/9-12 │ │ │ │ Fa0/24 ← TRUNK (802.1Q) │ └──────┬───────────────────────────┘ │ │ Lien Trunk │ (transporte VLAN 10, 20, 30) │ ┌──────┴───────────────────────────┐ │ ROUTEUR R1 │ │ Gi0/0 (interface physique) │ │ │ │ Gi0/0.10 → VLAN 10 → 192.168.10.1/24 │ │ Gi0/0.20 → VLAN 20 → 192.168.20.1/24 │ │ Gi0/0.30 → VLAN 30 → 192.168.30.1/24 │ └───────────────────────────────────┘ Flux : PC1 (VLAN10) → SW1 (tag VLAN10) → Trunk → R1 (Gi0/0.10) R1 route → Gi0/0.20 → Trunk (tag VLAN20) → SW1 → PC2

Configuration du Switch (Trunk)

# ========================================== # CONFIGURATION DU SWITCH — TRUNK + VLANs # ========================================== # 1. Créer les VLANs SW1(config)# vlan 10 SW1(config-vlan)# name COMPTABILITE SW1(config-vlan)# exit SW1(config)# vlan 20 SW1(config-vlan)# name INFORMATIQUE SW1(config-vlan)# exit SW1(config)# vlan 30 SW1(config-vlan)# name DIRECTION SW1(config-vlan)# exit # 2. Assigner les ports Access SW1(config)# interface range fa0/1 - 4 SW1(config-if-range)# switchport mode access SW1(config-if-range)# switchport access vlan 10 SW1(config-if-range)# exit SW1(config)# interface range fa0/5 - 8 SW1(config-if-range)# switchport mode access SW1(config-if-range)# switchport access vlan 20 SW1(config-if-range)# exit SW1(config)# interface range fa0/9 - 12 SW1(config-if-range)# switchport mode access SW1(config-if-range)# switchport access vlan 30 SW1(config-if-range)# exit # 3. Configurer le port Trunk vers le routeur SW1(config)# interface FastEthernet0/24 SW1(config-if)# switchport mode trunk SW1(config-if)# switchport trunk allowed vlan 10,20,30 SW1(config-if)# switchport trunk native vlan 99 # Sécurité : native ≠ VLAN 1 SW1(config-if)# exit # Vérification SW1# show interfaces trunk Port Mode Encapsulation Status Native vlan Fa0/24 on 802.1q trunking 99 Vlans allowed: 10,20,30

Configuration du Routeur (Sous-interfaces)

# ========================================== # CONFIGURATION DU ROUTEUR — SOUS-INTERFACES # ========================================== # 1. Activer l'interface physique (PAS d'IP dessus !) R1(config)# interface GigabitEthernet0/0 R1(config-if)# no shutdown R1(config-if)# exit # 2. Sous-interface pour VLAN 10 R1(config)# interface GigabitEthernet0/0.10 R1(config-subif)# description Gateway VLAN 10 - COMPTABILITE R1(config-subif)# encapsulation dot1Q 10 R1(config-subif)# ip address 192.168.10.1 255.255.255.0 R1(config-subif)# exit # 3. Sous-interface pour VLAN 20 R1(config)# interface GigabitEthernet0/0.20 R1(config-subif)# description Gateway VLAN 20 - INFORMATIQUE R1(config-subif)# encapsulation dot1Q 20 R1(config-subif)# ip address 192.168.20.1 255.255.255.0 R1(config-subif)# exit # 4. Sous-interface pour VLAN 30 R1(config)# interface GigabitEthernet0/0.30 R1(config-subif)# description Gateway VLAN 30 - DIRECTION R1(config-subif)# encapsulation dot1Q 30 R1(config-subif)# ip address 192.168.30.1 255.255.255.0 R1(config-subif)# exit # 5. (Optionnel) Sous-interface Native VLAN 99 R1(config)# interface GigabitEthernet0/0.99 R1(config-subif)# encapsulation dot1Q 99 native R1(config-subif)# ip address 192.168.99.1 255.255.255.0 R1(config-subif)# exit
encapsulation dot1Q <VLAN_ID> dit au routeur : "Les trames qui arrivent avec le tag 802.1Q de ce VLAN-là, traite-les sur cette sous-interface." Le numéro de sous-interface (.10, .20, .30) est une convention — il ne doit pas forcément correspondre au VLAN ID, mais c'est une bonne pratique.

Checklist en cas d'échec

#VérificationCommandeErreur courante

3Trunk actif sur le switch ?show int trunkPort en mode access au lieu de trunk
4VLANs autorisés sur trunk ?show int trunkVLAN pas dans la allowed list
5Native VLAN identique ?show int trunkNative VLAN différent côté switch/routeur
6VLAN ID correct sur sous-if ?show run int gi0/0.10dot1Q 20 au lieu de dot1Q 10
7Gateway correcte sur les PCs ?ipconfigMauvaise gateway ou pas de gateway
8Ports access dans le bon VLAN ?show vlan briefPort dans VLAN 1 au lieu de VLAN 10
Erreur n°1 : Oublier no shutdown sur l'interface physique Gi0/0. Les sous-interfaces héritent de l'état de l'interface physique. Si Gi0/0 est down, toutes les sous-interfaces sont down.

Récapitulatif ROAS

RÉSUMÉ DE LA CONFIG ROAS SWITCH : ├── Créer les VLANs (vlan X, name Y) ├── Assigner les ports Access (switchport mode access + switchport access vlan X) └── Configurer le Trunk (switchport mode trunk + allowed vlan X,Y,Z) ROUTEUR : ├── Activer l'interface physique (no shutdown) ├── Créer les sous-interfaces (int gi0/0.X) ├── Associer chaque sous-interface à un VLAN (encapsulation dot1Q X) └── Attribuer l'IP gateway de chaque VLAN (ip address ...) PCs : └── Configurer l'IP dans le bon sous-réseau + gateway = sous-interface du routeur ✓ Résultat : Tous les VLANs communiquent via le routeur !
🧠 Mémo — ROAS : « Une patte, plusieurs saveurs »
• Router On A Stick = 1 seul câble physique, plusieurs sous-interfaces
• Chaque sous-interface = 1 VLAN = encapsulation dot1Q [vlan-id]
• Exemple : interface g0/0.10 → sous-interface pour VLAN 10
3 étapes : créer la sous-interface → encapsulation dot1Q → assigner l'IP
C'est comme un cornet de glace : une seule main (un câble) qui tient plusieurs parfums (VLANs).
