Module 01

Installation & Premiers Pas

Sharpy vient de décrocher son premier stage. Son chef lui dit : « Bienvenue ! Ici on code en C#. » Installons tout et écrivons notre première ligne.

C#, c'est quoi exactement ?

C# (prononcé « C sharp ») est un langage de programmation créé par Microsoft en 2000. Il fait partie de la famille .NET et il est utilisé partout : applications de bureau (Windows Forms, WPF), sites web (ASP.NET), jeux vidéo (Unity), applications mobiles (Xamarin/MAUI) et même de l'intelligence artificielle.

Pourquoi commencer par C# ? Parce qu'il est fortement typé (il te force à écrire du code propre), orienté objet (tu vas structurer ton code comme un pro), et la documentation Microsoft est gigantesque. Si tu apprends C#, tu peux ensuite passer à Java, TypeScript ou Kotlin sans trop de difficultés — les concepts sont les mêmes.

Définition — .NET

.NET est la plateforme (le framework) qui permet d'exécuter du code C#. Pense à .NET comme le moteur d'une voiture : C# est le volant, .NET est ce qui fait tourner les roues. Depuis .NET 6+, tout est unifié et cross-platform (Windows, Mac, Linux).

Bonne nouvelle

Tu n'as pas besoin de connaître un autre langage pour suivre ce cours. On part vraiment de zéro, et chaque concept est expliqué avec des analogies concrètes.

Installer l'environnement

Pour coder en C#, tu as besoin de deux choses : un éditeur de code et le .NET SDK (le kit de développement). Voici les deux options principales :

Option A — Visual Studio Community (recommandé pour débutants)

C'est l'IDE complet de Microsoft. Il intègre tout : éditeur, compilateur, débogueur, IntelliSense (auto-complétion intelligente). C'est comme avoir une cuisine équipée plutôt que juste un couteau.

étapes1. Va sur visualstudio.microsoft.com/fr/
2. Télécharge "Community" (gratuit)
3. Coche la charge de travail ".NET Desktop Development"
4. Laisse l'installation se terminer (~5-10 Go)

Option B — VS Code + .NET SDK (léger et rapide)

Si tu préfères un éditeur plus léger, installe VS Code puis le .NET SDK séparément. C'est comme un couteau suisse : plus compact, mais tu dois installer les lames toi-même.

terminal# Vérifier que .NET est installé
dotnet --version
8.0.100

# Créer un projet console
dotnet new console -n MonPremierProjet
cd MonPremierProjet
dotnet run
Attention

Assure-toi d'installer le .NET SDK et pas seulement le Runtime. Le SDK inclut le compilateur (csc) et l'outil dotnet. Sans lui, tu ne peux pas créer ni compiler de projets.

Ton premier programme C#

Le moment tant attendu ! Sharpy ouvre Visual Studio, crée un projet « Console App », et voici ce qu'elle voit dans le fichier Program.cs :

C#// Mon tout premier programme C#
Console.WriteLine("Bonjour, je suis Sharpy !");
Console.WriteLine("Bienvenue dans le monde de C# 🚀");

Quand tu lances ce programme (F5 ou dotnet run), tu vois dans la console :

sortieBonjour, je suis Sharpy !
Bienvenue dans le monde de C# 🚀
Console.WriteLine vs Console.Write

Console.WriteLine() affiche du texte puis saute une ligne. Console.Write() affiche du texte sans saut de ligne. C'est la différence entre appuyer sur Entrée ou non.

Le fichier .csproj

Chaque projet C# a un fichier .csproj — c'est la carte d'identité du projet. Il contient la version de .NET cible, les dépendances, les paramètres de compilation. Pour l'instant, tu n'as pas besoin de le modifier, mais sache qu'il existe.

XML<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net8.0</TargetFramework>
  </PropertyGroup>
</Project>
Bravo !

Si tu vois « Bonjour » dans ta console, tu as réussi ton premier programme C#. Sharpy est fière de toi ! Maintenant, direction les variables et les types pour stocker des données.

Les concepts à retenir

Avant de passer au module suivant, voici les 5 points essentiels à graver dans ta mémoire :

  • 1. C# est un langage compilé — ton code est transformé en instructions machine avant d'être exécuté. C'est pas comme Python qui interprète ligne par ligne.
  • 2. .NET est le framework, C# est le langage. Tu peux utiliser .NET avec F# ou VB.NET aussi, mais C# est le plus populaire.
  • 3. Console.WriteLine() est ta première fonction — elle affiche du texte dans le terminal.
  • 4. Chaque instruction se termine par un point-virgule ; — oublie-le et le compilateur te le rappellera.
  • 5. C# est case-sensitive : Console ≠ console. Les majuscules comptent !
Erreur classique

Oublier le ; en fin de ligne. Le compilateur te dira : CS1002: ; expected. C'est l'erreur n°1 des débutants, et même les pros la font parfois !

Mnémotechnique — Console.WriteLine — ta première commande

🧠 « C.W.L — Console, Write, Line ! »

  • Console → l'écran noir (ton terminal)
  • Write → écrire (afficher du texte)
  • Line → retour à la ligne automatique

Astuce : Write = sans retour. WriteLine = avec retour. Le ; est obligatoire à chaque fin de ligne !
