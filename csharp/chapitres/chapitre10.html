Module 10

Switch & Enums

Sharpy a une chaîne de if/else if/else if qui fait 50 lignes. SharpBot secoue la tête : « Utilise un switch, c'est comme un distributeur automatique — tu appuies sur un bouton et tu obtiens exactement ce que tu veux. Et avec les enums, tu as un menu fixe de choix possibles ! »

Le switch statement — le distributeur automatique

Le switch compare une valeur à plusieurs cas possibles et exécute le bloc correspondant. C'est plus lisible qu'une longue chaîne de if/else if quand tu compares une seule variable à des valeurs fixes. Imagine un distributeur : tu tapes A1, il te donne un café. Tu tapes B2, il te donne un jus. Le switch fait pareil avec ton code.

C#// Switch sur un string
string jour = "mercredi";

switch (jour)
{
    case "lundi":
        Console.WriteLine("😩 C'est le début de la semaine...");
        break;

    case "mardi":
    case "mercredi":
    case "jeudi":
        Console.WriteLine("💼 Milieu de semaine, on tient bon !");
        break;

    case "vendredi":
        Console.WriteLine("🎉 VENDREDI ! Presque le week-end !");
        break;

    case "samedi":
    case "dimanche":
        Console.WriteLine("😎 Week-end ! Repos et code perso.");
        break;

    default:
        Console.WriteLine($"❌ '{jour}' n'est pas un jour valide.");
        break;
}

// Switch sur un int
int note = 15;

switch (note / 4)  // Astuce : diviser pour regrouper
{
    case 5:
        Console.WriteLine("🌟 Excellent (16-20)");
        break;
    case 4:
        Console.WriteLine("🌟 Excellent (16-20)");
        break;
    case 3:
        Console.WriteLine("👍 Bien (12-15)");
        break;
    case 2:
        Console.WriteLine("👌 Passable (8-11)");
        break;
    default:
        Console.WriteLine("📚 À retravailler (0-7)");
        break;
}
sortie💼 Milieu de semaine, on tient bon !
👍 Bien (12-15)
N'oublie JAMAIS le break !

En C#, chaque case doit se terminer par break, return, throw ou goto case. Contrairement à C/C++ ou Java, le « fall-through » accidentel est interdit — le compilateur te donnera l'erreur CS0163. La seule exception : les case vides empilés (comme case "samedi": case "dimanche":) qui partagent le même bloc de code.

Switch vs if/else — quand choisir quoi ?

switch → quand tu compares une seule variable à des valeurs constantes (string, int, enum…). Plus de 3 cas ? Utilise switch.
if/else → quand tu testes des conditions complexes (plages, comparaisons, combinaisons avec && ou ||). Le switch classique ne gère pas case > 10 (mais les switch expressions oui !).

Switch expressions (C# 8+) — la version moderne

Depuis C# 8, il existe une syntaxe beaucoup plus concise : les switch expressions. Au lieu de switch { case: break; }, tu écris une expression avec => qui retourne directement une valeur. C'est plus compact, plus élégant, et parfait quand le switch doit produire un résultat.

C#// Switch expression avec string
string saison = "été";

string activite = saison switch
{
    "printemps" => "🌸 Jardinage et promenades",
    "été"       => "☀️ Plage et glaces",
    "automne"   => "🍂 Cueillette et randonnée",
    "hiver"     => "❄️ Ski et chocolat chaud",
    _           => "❓ Saison inconnue"  // _ = default
};

Console.WriteLine($"{saison} → {activite}");

// Switch expression avec int et pattern matching
int note = 17;

string mention = note switch
{
    >= 16 => "Très Bien 🌟",
    >= 14 => "Bien 👍",
    >= 12 => "Assez Bien 👌",
    >= 10 => "Passable 😅",
    _     => "Recalé 😢"
};

Console.WriteLine($"Note {note}/20 → {mention}");

// Switch expression dans une méthode
static double Calculer(double a, string op, double b) => op switch
{
    "+" => a + b,
    "-" => a - b,
    "*" => a * b,
    "/" when b != 0 => a / b,
    "/" => throw new DivideByZeroException("Division par zéro !"),
    _ => throw new ArgumentException($"Opérateur inconnu : {op}")
};

Console.WriteLine($"10 + 5 = {Calculer(10, "+", 5)}");
Console.WriteLine($"10 * 3 = {Calculer(10, "*", 3)}");
sortieété → ☀️ Plage et glaces
Note 17/20 → Très Bien 🌟
10 + 5 = 15
10 * 3 = 30
Switch expressions vs switch statement

Switch statement (classique) → exécute du code (peut contenir plusieurs instructions)
Switch expression (moderne) → retourne une valeur (chaque branche est une expression)

Utilise les switch expressions quand tu veux assigner une valeur ou retourner un résultat. Le _ remplace le default, et chaque branche utilise => au lieu de case/break.

Le mot-clé when — conditions supplémentaires

Tu peux ajouter des conditions avec when dans un switch expression : "/" when b != 0 => a / b. C'est ce qu'on appelle un guard clause. Le cas ne match que si la valeur ET la condition sont vraies. C'est extrêmement puissant pour du pattern matching avancé.

Les Enums — un menu fixe de choix

Un enum (énumération) est un type qui définit un ensemble fixe de constantes nommées. C'est comme le menu d'un restaurant : les options sont définies à l'avance, tu ne peux pas inventer un plat qui n'existe pas. Les enums rendent ton code plus lisible, plus sûr, et tu évites les strings magiques.

C#// Déclarer un enum
enum Saison
{
    Printemps,   // = 0 par défaut
    Ete,         // = 1
    Automne,     // = 2
    Hiver        // = 3
}

// Enum avec valeurs personnalisées
enum NiveauAcces
{
    Visiteur = 0,
    Utilisateur = 10,
    Moderateur = 50,
    Admin = 100
}

// Enum pour les jours
enum Jour
{
    Lundi, Mardi, Mercredi, Jeudi, Vendredi, Samedi, Dimanche
}

// Utiliser un enum
Saison maSaison = Saison.Ete;
NiveauAcces monNiveau = NiveauAcces.Admin;

Console.WriteLine($"Saison : {maSaison}");           // Affiche "Ete"
Console.WriteLine($"Valeur : {(int)maSaison}");       // Affiche 1
Console.WriteLine($"Niveau : {monNiveau}");          // Affiche "Admin"
Console.WriteLine($"Code   : {(int)monNiveau}");     // Affiche 100

// Conversion int → enum
Saison fromInt = (Saison)2;
Console.WriteLine($"Saison index 2 : {fromInt}");  // Affiche "Automne"

// Conversion string → enum
Saison fromStr = Enum.Parse<Saison>("Hiver");
Console.WriteLine($"Depuis string : {fromStr}"); // Affiche "Hiver"
sortieSaison : Ete
Valeur : 1
Niveau : Admin
Code   : 100
Saison index 2 : Automne
Depuis string : Hiver
Pourquoi utiliser des enums plutôt que des strings ?

Sécurité → Le compilateur vérifie les valeurs. Saison.Xyz = erreur à la compilation, contrairement à "Xyz" qui passe mais plante au runtime.
IntelliSense → VS Code te montre toutes les valeurs possibles quand tu tapes Saison.
Lisibilité → NiveauAcces.Admin est plus clair que le nombre magique 100
Performance → Les enums sont des int en mémoire, donc très rapides à comparer.

Enum.Parse est sensible à la casse

Enum.Parse<Saison>("hiver") lève une ArgumentException car "hiver" ≠ "Hiver". Pour ignorer la casse : Enum.Parse<Saison>("hiver", ignoreCase: true). Et comme pour les autres Parse, utilise Enum.TryParse() pour une conversion sécurisée sans exception.

Enum + Switch — le combo parfait

Les enums et les switch sont faits pour travailler ensemble. Un enum définit les options possibles, et le switch traite chaque option. C'est comme un distributeur avec un menu fixe : chaque bouton (enum) déclenche une action (switch). SharpBot dit : « C'est la combinaison la plus propre en C#. »

C#enum StatutCommande
{
    EnAttente,
    Confirmee,
    EnPreparation,
    Expediee,
    Livree,
    Annulee
}

// Switch expression avec enum
static string ObtenirMessage(StatutCommande statut) => statut switch
{
    StatutCommande.EnAttente      => "⏳ Votre commande est en attente de validation.",
    StatutCommande.Confirmee     => "✅ Commande confirmée ! Préparation imminente.",
    StatutCommande.EnPreparation => "📦 Votre commande est en cours de préparation.",
    StatutCommande.Expediee      => "🚚 C'est parti ! Votre colis est en route.",
    StatutCommande.Livree        => "🎉 Commande livrée ! Bon déballage !",
    StatutCommande.Annulee       => "❌ Commande annulée. Remboursement en cours.",
    _ => "❓ Statut inconnu."
};

// Utilisation
StatutCommande maCommande = StatutCommande.Expediee;
Console.WriteLine(ObtenirMessage(maCommande));

// Parcourir toutes les valeurs d'un enum
Console.WriteLine("\n=== Tous les statuts possibles ===");
foreach (StatutCommande s in Enum.GetValues<StatutCommande>())
{
    Console.WriteLine($"  {(int)s}. {s} → {ObtenirMessage(s)}");
}

// Exemple avec Jour et switch pour les horaires
enum JourSemaine { Lundi, Mardi, Mercredi, Jeudi, Vendredi, Samedi, Dimanche }

static string Horaires(JourSemaine jour) => jour switch
{
    JourSemaine.Lundi or JourSemaine.Mardi or JourSemaine.Mercredi
        or JourSemaine.Jeudi or JourSemaine.Vendredi
        => "🕘 Ouvert 9h-18h",
    JourSemaine.Samedi => "🕙 Ouvert 10h-16h",
    JourSemaine.Dimanche => "🔒 Fermé",
    _ => "❓ Jour inconnu"
};

Console.WriteLine($"\nLundi : {Horaires(JourSemaine.Lundi)}");
Console.WriteLine($"Samedi : {Horaires(JourSemaine.Samedi)}");
Console.WriteLine($"Dimanche : {Horaires(JourSemaine.Dimanche)}");
sortie🚚 C'est parti ! Votre colis est en route.

=== Tous les statuts possibles ===
  0. EnAttente → ⏳ Votre commande est en attente de validation.
  1. Confirmee → ✅ Commande confirmée ! Préparation imminente.
  2. EnPreparation → 📦 Votre commande est en cours de préparation.
  3. Expediee → 🚚 C'est parti ! Votre colis est en route.
  4. Livree → 🎉 Commande livrée ! Bon déballage !
  5. Annulee → ❌ Commande annulée. Remboursement en cours.

L'attribut [Flags] — combiner des enums

L'attribut [Flags] permet de combiner plusieurs valeurs d'un enum avec l'opérateur | (OR binaire). Chaque valeur doit être une puissance de 2 :
[Flags] enum Permissions { Lire = 1, Ecrire = 2, Supprimer = 4, Admin = 8 }
Ensuite : var droits = Permissions.Lire | Permissions.Ecrire;
Et pour vérifier : droits.HasFlag(Permissions.Lire) → true. C'est comme ça que fonctionnent les permissions dans les vrais systèmes !

Résumé — Switch & Enums

switch (val) { case X: break; } → switch classique
val switch { X => result, _ => default } → switch expression (C# 8+)
when → condition supplémentaire dans un case
enum → ensemble fixe de constantes nommées
(int)enumVal → enum vers int
(MonEnum)intVal → int vers enum
Enum.Parse<T>() → string vers enum
Enum.GetValues<T>() → toutes les valeurs
[Flags] → combiner des valeurs d'enum

Mnémotechnique — Switch = Aiguillage

🧠 « Switch = Aiguillage ferroviaire, Enum = Panneau de direction »

  • switch → « J'ai UNE variable, et PLUSIEURS cas possibles » — plus propre que 10 if/else
  • case → chaque voie possible. break; OBLIGATOIRE à la fin de chaque case
  • default → le filet de sécurité (comme else)
  • enum → un type personnalisé avec des valeurs nommées. Plus lisible que des nombres magiques !

Oubli classique : sans break;, le code « tombe » dans le case suivant. C'est le piège n°1 du switch.
