Module 12

Introduction à la POO

Sharpy a appris les bases — variables, boucles, méthodes. Mais comment organiser un vrai projet avec des centaines de fichiers ? La Programmation Orientée Objet (POO) est la clé. C# est un langage 100% orienté objet.

Classe et Objet : le plan et la maison

Imagine que tu construis des maisons. La classe est le plan d'architecte — elle décrit les caractéristiques (nombre de pièces, couleur) et les actions (ouvrir la porte, allumer la lumière). L'objet est une maison réelle construite à partir de ce plan. Un plan → plusieurs maisons.

Classe = plan, Objet = instance
// Le plan (la classe)
public class Personnage
{
    public string Nom;
    public int PointsDeVie;
    public int Attaque;
}

// Construction de 2 maisons (2 objets)
Personnage heros = new Personnage();
heros.Nom = "Sharpy";
heros.PointsDeVie = 100;
heros.Attaque = 25;

Personnage ennemi = new Personnage();
ennemi.Nom = "Bug";
ennemi.PointsDeVie = 50;
ennemi.Attaque = 10;

Console.WriteLine($"{heros.Nom} a {heros.PointsDeVie} PV");
// Sharpy a 100 PV
Vocabulaire essentiel
  • Classe = le plan, le modèle, le type
  • Objet = l'instance, la réalisation concrète
  • Instancier = créer un objet avec new
  • Membre = une variable ou méthode appartenant à la classe
Convention importante

En C#, les noms de classes utilisent PascalCase (Personnage, pas personnage). Les champs publics aussi. C'est une convention stricte dans l'écosystème .NET.

Le Constructeur : initialiser proprement

Assignar chaque champ un par un, c'est fastidieux. Le constructeur est une méthode spéciale qui s'exécute automatiquement quand tu créés un new objet. Il a le même nom que la classe et pas de type de retour.

Constructeur avec paramètres
public class Personnage
{
    public string Nom;
    public int PointsDeVie;
    public int Attaque;

    // Constructeur
    public Personnage(string nom, int pv, int atk)
    {
        Nom = nom;
        PointsDeVie = pv;
        Attaque = atk;
    }
}

// Création simplifiée !
Personnage heros = new Personnage("Sharpy", 100, 25);
Personnage ennemi = new Personnage("Bug", 50, 10);
Constructeur par défaut

Si tu ne définis aucun constructeur, C# en crée un implicite sans paramètre. Mais dès que tu définis un constructeur avec paramètres, le constructeur par défaut disparaît. Si tu veux les deux, il faut les écrire explicitement.

Plusieurs constructeurs (surcharge)
public class Personnage
{
    public string Nom;
    public int PointsDeVie;

    // Constructeur complet
    public Personnage(string nom, int pv)
    {
        Nom = nom;
        PointsDeVie = pv;
    }

    // Constructeur simplifié (PV par défaut)
    public Personnage(string nom) : this(nom, 100) { }
}

var p1 = new Personnage("Sharpy", 150);
var p2 = new Personnage("Noob"); // PV = 100 par défaut

Propriétés & Encapsulation

Les champs publics sont dangereux : n'importe qui peut modifier heros.PointsDeVie = -999. L'encapsulation consiste à protéger les données internes et à contrôler l'accès via des propriétés.

Propriétés avec get/set
public class Personnage
{
    public string Nom { get; set; }

    private int _pointsDeVie;  // champ privé

    public int PointsDeVie
    {
        get { return _pointsDeVie; }
        set
        {
            // Empêche les PV négatifs
            if (value < 0)
                _pointsDeVie = 0;
            else if (value > 200)
                _pointsDeVie = 200;
            else
                _pointsDeVie = value;
        }
    }

    public bool EstVivant => PointsDeVie > 0;  // propriété calculée
}
Niveaux d'accès en C#
Mot-cléAccessible depuis
publicPartout
privateLa classe elle-même uniquement
protectedLa classe + ses classes enfants
internalLe même assemblage (projet)
Auto-propriétés : la syntaxe express

En C#, si tu n'as pas de logique spéciale, utilise les auto-propriétés : public string Nom { get; set; }. Le compilateur crée le champ privé en arrière-plan. C'est la forme la plus courante dans les projets professionnels.

Méthodes d'instance : faire agir les objets

Une classe ne contient pas que des données — elle définit aussi des comportements. Les méthodes d'instance sont des actions qu'un objet peut effectuer.

Classe complète avec méthodes
public class Personnage
{
    public string Nom { get; set; }
    public int PointsDeVie { get; private set; }
    public int Attaque { get; set; }

    public Personnage(string nom, int pv, int atk)
    {
        Nom = nom; PointsDeVie = pv; Attaque = atk;
    }

    // Méthode d'instance : attaquer un autre personnage
    public void Attaquer(Personnage cible)
    {
        Console.WriteLine($"{Nom} attaque {cible.Nom} !");
        cible.SubirDegats(Attaque);
    }

    public void SubirDegats(int degats)
    {
        PointsDeVie -= degats;
        if (PointsDeVie < 0) PointsDeVie = 0;
        Console.WriteLine($"  → {Nom} perd {degats} PV ({PointsDeVie} restants)");
    }

    public void Soigner(int soin)
    {
        PointsDeVie += soin;
        Console.WriteLine($"  💚 {Nom} récupère {soin} PV ({PointsDeVie} PV)");
    }

    // Override ToString pour un affichage élégant
    public override string ToString()
        => $"[{Nom}] PV:{PointsDeVie} ATK:{Attaque}";
}

// Utilisation :
var sharpy = new Personnage("Sharpy", 100, 25);
var bug = new Personnage("Bug", 60, 15);

sharpy.Attaquer(bug);
// Sharpy attaque Bug !
//   → Bug perd 25 PV (35 restants)

Console.WriteLine(sharpy); // [Sharpy] PV:100 ATK:25

Héritage & Polymorphisme : aperçu

L'héritage permet de créer une classe enfant qui hérite des propriétés et méthodes d'une classe parent. Un Guerrier est un Personnage avec des capacités supplémentaires.

Héritage en C#
public class Guerrier : Personnage
{
    public int Armure { get; set; }

    public Guerrier(string nom, int pv, int atk, int armure)
        : base(nom, pv, atk)   // Appel du constructeur parent
    {
        Armure = armure;
    }

    // Override : le guerrier réduit les dégâts avec son armure
    public new void SubirDegats(int degats)
    {
        int degatsReduits = degats - Armure;
        if (degatsReduits < 0) degatsReduits = 0;
        base.SubirDegats(degatsReduits);
        Console.WriteLine($"  🛡️ (armure a absorbé {degats - degatsReduits} dégâts)");
    }
}

public class Mage : Personnage
{
    public int Mana { get; set; }

    public Mage(string nom, int pv, int atk, int mana)
        : base(nom, pv, atk) { Mana = mana; }

    public void LancerSort(Personnage cible)
    {
        if (Mana >= 20)
        {
            Mana -= 20;
            Console.WriteLine($"🔥 {Nom} lance une boule de feu !");
            cible.SubirDegats(Attaque * 2);
        }
        else
            Console.WriteLine("Pas assez de mana !");
    }
}

// Utilisation :
var guerrier = new Guerrier("Tank", 150, 20, 10);
var mage = new Mage("Merlin", 60, 35, 100);

mage.LancerSort(guerrier);
guerrier.Attaquer(mage);
Polymorphisme en un mot

Le polymorphisme signifie qu'un objet peut prendre « plusieurs formes ». Un Guerrier est aussi un Personnage. Tu peux mettre des Guerriers, des Mages et des Personnages dans une même List<Personnage> et appeler leurs méthodes — chaque type se comportera selon sa propre implémentation.

Ce n'est que le début

Ce module est une introduction. La POO en C# comprend aussi les interfaces, les classes abstraites, les génériques, les records, et bien plus. Mais avec ces bases, tu peux déjà structurer des projets solides.
