Module 02

Variables & Types

Sharpy découvre les boîtes magiques qui stockent des données. Chaque boîte a une étiquette (le type) et un contenu (la valeur).

Qu'est-ce qu'une variable ?

Imagine un entrepôt rempli de boîtes. Chaque boîte a une étiquette (le nom de la variable), un type (ce qu'elle peut contenir — des chiffres, du texte, vrai/faux) et un contenu (la valeur actuelle). En C#, tu dois toujours dire au compilateur quel type de boîte tu veux avant de l'utiliser.

C'est ce qu'on appelle le typage statique : une boîte étiquetée « entier » ne pourra jamais contenir du texte. Ça peut paraître contraignant, mais c'est ce qui te protège des bugs idiots. Le compilateur est ton allié, pas ton ennemi.

C#// Déclaration + affectation
int age = 25;                  // Entier
string prenom = "Sharpy";       // Texte
double taille = 1.72;           // Décimal
bool estEtudiante = true;       // Vrai/Faux
char initiale = 'S';            // Un seul caractère
Définition — Typage statique

En C#, le type d'une variable est fixé à la compilation. Un int restera un int pour toujours. C'est l'opposé de Python ou JavaScript où une variable peut changer de type à volonté.

Les types fondamentaux

C# propose une gamme de types pour couvrir tous les besoins. Voici les plus utilisés au quotidien :

Piège classique — double vs decimal

Pour de l'argent, utilise toujours decimal et pas double. Le double a des erreurs d'arrondi (0.1 + 0.2 ≠ 0.3 exactement). Le decimal est précis mais plus lent — parfait pour les calculs financiers.

Le mot-clé var et l'interpolation

C# offre un raccourci : le mot-clé var. Il demande au compilateur de deviner le type automatiquement à partir de la valeur assignée. C'est du sucre syntaxique — le type est toujours fixé à la compilation.

C#var nom = "Sharpy";      // Le compilateur déduit : string
var age = 22;              // Le compilateur déduit : int
var prix = 9.99;           // Le compilateur déduit : double

// ❌ Interdit : var sans valeur initiale
// var mystere; → Erreur, le compilateur ne peut pas deviner

L'interpolation de chaînes ($"")

L'interpolation est la manière la plus élégante d'insérer des variables dans du texte. Mets un $ devant la chaîne, puis utilise {} pour insérer des expressions :

C#string prenom = "Sharpy";
int age = 22;
double moyenne = 15.75;

// ✅ Interpolation (moderne, lisible)
Console.WriteLine($"Je suis {prenom}, j'ai {age} ans");
Console.WriteLine($"Ma moyenne : {moyenne:F1}/20");

// ❌ Concaténation (ancien style, moins lisible)
Console.WriteLine("Je suis " + prenom + ", j'ai " + age + " ans");
Astuce pro

Utilise {variable:F2} pour formater un nombre avec 2 décimales, {variable:C} pour un format monétaire, ou {variable:N0} pour un nombre avec séparateurs de milliers.

Conversions de types (Casting)

Parfois tu as un string et tu veux un int. Ou un int et tu veux un double. C# propose plusieurs manières de convertir :

C#// Conversion implicite (sans risque : petit → grand)
int entier = 42;
double decimal_ = entier;    // OK : int → double automatique

// Conversion explicite (cast : grand → petit, risque de perte)
double pi = 3.14159;
int piEntier = (int)pi;      // 3 — la partie décimale est perdue !

// Conversion string → nombre
string texte = "123";
int nombre = int.Parse(texte);        // 123
int nombre2 = Convert.ToInt32(texte);  // 123 aussi

// Conversion sûre avec TryParse
if (int.TryParse("abc", out int resultat))
    Console.WriteLine(resultat);
else
    Console.WriteLine("Pas un nombre !");
Ne jamais faire

N'utilise pas int.Parse() sur une entrée utilisateur sans vérification. Si l'utilisateur tape « abc », ton programme crash avec une FormatException. Utilise toujours int.TryParse() pour les entrées non fiables.
