Module 03

Conditions IF / ELSE

Sharpy arrive à un carrefour. À gauche, le café. À droite, le bureau. Comment choisir ? En programmation, on utilise des conditions pour prendre des décisions automatiquement.

C'est quoi une condition ?

Imagine un feu tricolore. Quand c'est vert, tu avances. Quand c'est rouge, tu t'arrêtes. Quand c'est orange, tu ralentis (enfin… normalement). C'est exactement ce que fait un if en C# : il évalue une expression qui est soit true (vrai), soit false (faux), et il exécute un bloc de code en conséquence.

Sharpy, notre dev junior, doit écrire un programme qui vérifie si un utilisateur a le droit de conduire. La règle est simple : si l'âge est supérieur ou égal à 18, c'est bon. Sinon, c'est non. Voyons comment ça se traduit en C# :

C#int age = 20;

if (age >= 18)
{
    Console.WriteLine("Tu peux conduire 🚗");
}
else
{
    Console.WriteLine("Encore un peu de patience ⏳");
}
sortieTu peux conduire 🚗
Définition — Le type bool

Une condition retourne toujours un bool : soit true, soit false. C'est le type le plus simple de C# — il n'a que deux valeurs possibles. L'expression age >= 18 est évaluée comme un bool avant que le if ne décide quoi faire.

Anatomie d'un if

Le if se compose de trois parties : 1) le mot-clé if, 2) la condition entre parenthèses (), 3) le bloc de code entre accolades {}. Les accolades sont optionnelles si tu n'as qu'une seule instruction, mais mets-les toujours par bonne habitude — ça t'évitera des bugs vicieux.

Les opérateurs de comparaison

Pour construire des conditions, tu as besoin d'opérateurs de comparaison. Ce sont les outils qui comparent deux valeurs et renvoient true ou false. Sharpy les utilise tous les jours, comme un détective qui compare des indices :

Opérateur Signification Exemple Résultat
==Égal à5 == 5true
!=Différent de5 != 3true
<Strictement inférieur3 < 5true
>Strictement supérieur5 > 3true
<=Inférieur ou égal5 <= 5true
>=Supérieur ou égal3 >= 5false

La chaîne if / else if / else

Parfois, le choix n'est pas binaire. Sharpy veut attribuer une mention selon la note. C'est comme un aiguillage de train avec plusieurs voies :

C#int note = 15;

if (note >= 16)
{
    Console.WriteLine("Mention Très Bien 🌟");
}
else if (note >= 14)
{
    Console.WriteLine("Mention Bien 👍");
}
else if (note >= 12)
{
    Console.WriteLine("Mention Assez Bien 👌");
}
else if (note >= 10)
{
    Console.WriteLine("Passable 😅");
}
else
{
    Console.WriteLine("Recalé 😢");
}
sortieMention Bien 👍
Piège mortel — == vs =

= c'est l'affectation (donner une valeur). == c'est la comparaison (vérifier l'égalité). Écrire if (x = 5) au lieu de if (x == 5) est une erreur classique. Heureusement, le compilateur C# refuse ce code car un int n'est pas un bool. Mais en d'autres langages, ce bug passerait inaperçu !

Astuce SharpBot

L'ordre des else if compte ! C# évalue les conditions de haut en bas et s'arrête à la première qui est vraie. Si tu mets note >= 10 en premier, tout le monde aurait « Passable » même avec un 18. Classe toujours du plus restrictif au moins restrictif.

Opérateurs logiques : combiner les conditions

Parfois, une seule condition ne suffit pas. Sharpy veut vérifier si quelqu'un peut voter : il faut avoir au moins 18 ans ET être citoyen français. Ou bien, pour une réduction cinéma : avoir moins de 12 ans OU plus de 65 ans. C'est là qu'interviennent les opérateurs logiques :

Opérateur Nom Signification Exemple
&&ET logiqueLes DEUX doivent être vraiesa && b
||OU logiqueAu moins UNE doit être vraiea || b
!NON logiqueInverse la valeur!a
 
C#int age = 25;
bool estCitoyen = true;
bool aCarteEtudiant = false;

// ET logique : les DEUX conditions doivent être vraies
if (age >= 18 && estCitoyen)
{
    Console.WriteLine("Tu peux voter ! 🗳️");
}

// OU logique : au moins UNE condition doit être vraie
if (age < 12 || age > 65)
{
    Console.WriteLine("Tarif réduit au cinéma 🎬");
}

// NON logique : inverse la condition
if (!aCarteEtudiant)
{
    Console.WriteLine("Pas de réduction étudiante.");
}
sortieTu peux voter ! 🗳️
Pas de réduction étudiante.

Les conditions imbriquées (nested if)

Tu peux mettre un if à l'intérieur d'un autre if. C'est comme des poupées russes, mais attention : si tu imbriquer trop, ton code devient illisible. Sharpy a appris ça à ses dépens :

C#int age = 20;
bool aPermis = true;
bool aVoiture = false;

if (age >= 18)
{
    if (aPermis)
    {
        if (aVoiture)
        {
            Console.WriteLine("Tu peux conduire !");
        }
        else
        {
            Console.WriteLine("Il te faut une voiture 🚙");
        }
    }
    else
    {
        Console.WriteLine("Passe ton permis d'abord ! 📝");
    }
}

// ✅ Version plus propre avec && :
if (age >= 18 && aPermis && aVoiture)
{
    Console.WriteLine("Tu peux conduire !");
}
Short-circuit evaluation

Avec &&, si la première condition est false, C# ne vérifie même pas la deuxième (inutile, le résultat sera forcément false). Avec ||, si la première est true, il ne vérifie pas la suite. C'est le court-circuit — ça optimise les performances et ça évite des erreurs (division par zéro, accès null, etc.).

L'opérateur ternaire : le if en une ligne

Quand ton if/else ne fait qu'assigner une valeur, tu peux utiliser l'opérateur ternaire. C'est un raccourci élégant, comme commander un espresso au lieu d'un café long — même résultat, plus rapide :

C#// Syntaxe : condition ? valeurSiVrai : valeurSiFaux

int age = 20;

// Version classique (if/else)
string statut;
if (age >= 18)
    statut = "Majeur";
else
    statut = "Mineur";

// ✅ Version ternaire (1 ligne !)
string statut2 = age >= 18 ? "Majeur" : "Mineur";

Console.WriteLine($"Statut : {statut2}");

// Directement dans un WriteLine
Console.WriteLine(age >= 18 ? "Accès autorisé ✅" : "Accès refusé ❌");

// Ternaire avec calcul
int prix = 50;
bool estMembre = true;
double prixFinal = estMembre ? prix * 0.8 : prix;
Console.WriteLine($"Prix à payer : {prixFinal}€");
sortieStatut : Majeur
Accès autorisé ✅
Prix à payer : 40€
Attention — N'abuse pas du ternaire

Le ternaire est génial pour des conditions simples. Mais ne l'imbrique jamais : a ? b ? c : d : e est illisible et un cauchemar à déboguer. Si ta condition tient sur plus d'une ligne, utilise un if/else classique. La lisibilité prime toujours sur la concision.

Résumé express

if → exécute un bloc si la condition est vraie
else → exécute un bloc si la condition est fausse
else if → ajoute des conditions intermédiaires
? : → ternaire, raccourci pour assigner une valeur conditionnellement
&& / || / ! → combiner, alterner, inverser des conditions
