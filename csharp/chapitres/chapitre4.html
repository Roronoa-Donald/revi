Module 04

Boucles FOR / WHILE / FOREACH

Sharpy doit afficher les nombres de 1 à 100. Écrire 100 fois Console.WriteLine ? Hors de question. Les boucles sont là pour automatiser les tâches répétitives.

La boucle for — quand tu sais combien de fois

La boucle for est comme faire des pompes au sport : tu décides à l'avance combien tu vas en faire. « Je fais 10 pompes, point final. » La boucle for est parfaite quand tu connais le nombre exact de répétitions.

Elle se compose de trois parties séparées par des ; : l'initialisation (point de départ), la condition (quand s'arrêter), et l'incrémentation (comment avancer).

C#// Syntaxe : for (initialisation; condition; incrémentation)

// Afficher les nombres de 1 à 5
for (int i = 1; i <= 5; i++)
{
    Console.WriteLine($"Pompe numéro {i} 💪");
}

// Compter de 0 à 9 (classique en programmation)
for (int i = 0; i < 10; i++)
{
    Console.Write($"{i} ");
}
Console.WriteLine();  // Saut de ligne

// Compter à l'envers
for (int i = 5; i >= 1; i--)
{
    Console.WriteLine($"Décompte : {i}");
}
Console.WriteLine("Décollage ! 🚀");

// Compter de 2 en 2 (nombres pairs)
for (int i = 0; i <= 10; i += 2)
{
    Console.Write($"{i} ");
}
// Sortie : 0 2 4 6 8 10

Pourquoi commencer à 0 ?

En informatique, on commence souvent à compter à partir de 0. C'est une convention héritée du langage C où l'index représente un décalage mémoire. Tu t'y habitueras vite — les tableaux et les listes commencent tous à l'index 0 en C#.

i++ expliqué

i++ est un raccourci pour i = i + 1. De même, i-- signifie i = i - 1, et i += 2 signifie i = i + 2. Ce sont les opérateurs d'incrémentation/décrémentation — tu les verras partout.

while & do-while — quand tu ne sais pas combien de fois

La boucle while est comme manger à un buffet : tu manges tant que tu as faim. Tu ne sais pas combien d'assiettes tu vas prendre, ça dépend de ton appétit. Le while répète le bloc tant que la condition est vraie.

C#// while : vérifier AVANT d'exécuter
int compteur = 1;

while (compteur <= 5)
{
    Console.WriteLine($"Tour {compteur}");
    compteur++;  // ⚠️ Ne jamais oublier l'incrémentation !
}

// Exemple réaliste : deviner un nombre
int secret = 7;
int tentative = 0;
int essais = 0;

while (tentative != secret)
{
    essais++;
    tentative = essais;  // Simulation : on essaye 1, 2, 3...
}
Console.WriteLine($"Trouvé en {essais} essais ! 🎯");

do-while : au moins une fois

Le do-while est comme lire un livre : tu lis au moins la première page avant de décider si tu continues. La différence avec while ? Le bloc est exécuté au moins une fois, même si la condition est fausse dès le départ.

C#// do-while : exécuter PUIS vérifier
int choix;

do
{
    Console.WriteLine("=== MENU ===");
    Console.WriteLine("1. Jouer");
    Console.WriteLine("2. Options");
    Console.WriteLine("3. Quitter");
    choix = 3;  // Simulation de l'entrée utilisateur
} while (choix != 3);

Console.WriteLine("Au revoir ! 👋");
Danger — La boucle infinie

Si tu oublies de modifier la variable de la condition, ta boucle ne s'arrêtera jamais. Le programme tourne en boucle, consomme 100% du CPU, et tu dois le forcer à s'arrêter (Ctrl+C). C'est l'erreur la plus dangereuse avec les boucles. Vérifie toujours que ta condition finira par devenir false.

C#// ❌ BOUCLE INFINIE — NE FAIS JAMAIS ÇA
// int x = 1;
// while (x > 0)  ← x sera toujours > 0 !
// {
//     Console.WriteLine("Je suis infini...");
//     x++;  ← x augmente, la condition reste vraie !
// }

// ✅ Version corrigée
int x = 1;
while (x <= 5)
{
    Console.WriteLine($"x = {x}");
    x++;
}

foreach — lire un livre page par page

La boucle foreach est la manière la plus élégante de parcourir une collection (tableau, liste, etc.). C'est comme lire un livre page par page : tu prends chaque élément un par un, dans l'ordre, sans te soucier des index. Sharpy adore foreach parce que c'est impossible de faire une erreur d'index avec :

C#// Parcourir un tableau de prénoms
string[] prenoms = { "Alice", "Bob", "Sharpy", "Diana" };

foreach (string prenom in prenoms)
{
    Console.WriteLine($"Bonjour {prenom} ! 👋");
}

// Calculer la somme d'un tableau de notes
int[] notes = { 15, 12, 18, 9, 14 };
int somme = 0;

foreach (int note in notes)
{
    somme += note;
}
Console.WriteLine($"Moyenne : {(double)somme / notes.Length:F1}");

// Parcourir une chaîne caractère par caractère
string mot = "C#";
foreach (char lettre in mot)
{
    Console.WriteLine($"Lettre : {lettre}");
}

break et continue : contrôler le flux

Parfois tu veux sortir de la boucle prématurément (break) ou sauter une itération (continue). C'est comme au sport : break = « j'arrête l'exercice », continue = « je saute cette répétition mais je continue ».

C#// break : quitter la boucle immédiatement
for (int i = 1; i <= 100; i++)
{
    if (i == 5)
    {
        Console.WriteLine("Stop ! J'ai trouvé le 5 🛑");
        break;  // On sort de la boucle
    }
    Console.WriteLine($"Recherche... {i}");
}

Console.WriteLine();

// continue : sauter une itération
for (int i = 1; i <= 10; i++)
{
    if (i % 2 == 0)
        continue;  // Sauter les nombres pairs

    Console.Write($"{i} ");
}
// Sortie : 1 3 5 7 9
Piège classique — Off-by-one

L'erreur « off-by-one » (décalage d'un) est le cauchemar des développeurs. for (int i = 0; i <= 10; ...) fait 11 itérations, pas 10 ! Et for (int i = 1; i < 10; ...) en fait 9. Astuce : si tu pars de 0, utilise <. Si tu pars de 1, utilise <=.

Quelle boucle choisir ?

for → tu connais le nombre d'itérations (afficher de 1 à N)
while → tu ne sais pas combien de fois, tu vérifies avant
do-while → tu veux exécuter au moins une fois (menus, saisies)
foreach → tu parcours une collection élément par élément

Boucles imbriquées — les boucles dans les boucles

Tu peux mettre une boucle à l'intérieur d'une autre. C'est comme une horloge : l'aiguille des minutes tourne 60 fois pour chaque tour de l'aiguille des heures. Sharpy utilise ça pour dessiner des motifs ou parcourir des grilles :

C#// Table de multiplication
for (int i = 1; i <= 5; i++)
{
    for (int j = 1; j <= 5; j++)
    {
        Console.Write($"{i * j,4}");
    }
    Console.WriteLine();
}

// Dessiner un triangle d'étoiles
for (int ligne = 1; ligne <= 5; ligne++)
{
    for (int etoile = 0; etoile < ligne; etoile++)
    {
        Console.Write("⭐");
    }
    Console.WriteLine();
}

Performance

Attention aux boucles imbriquées ! Deux boucles de 1000 itérations chacune = 1 000 000 d'opérations. Trois boucles imbriquées de 100 = 1 000 000 aussi. La complexité augmente exponentiellement. Si ton programme rame, c'est souvent à cause de boucles imbriquées trop profondes.
