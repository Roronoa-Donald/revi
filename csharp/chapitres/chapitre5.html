Module 05

Tableaux & Listes

Sharpy doit gérer une liste de tâches. Stocker chaque tâche dans une variable séparée ? Avec 100 tâches, c'est impossible. Les tableaux et les listes sont la solution.

Les tableaux (arrays) — les casiers numérotés

Imagine une rangée de casiers numérotés dans un vestiaire. Chaque casier a un numéro fixe (l'index), et tu peux y mettre exactement un objet. Le nombre de casiers est défini à la construction et ne change jamais. C'est exactement ce qu'est un tableau en C# : une collection de taille fixe, avec des éléments accessibles par leur index.

C#// Déclaration et initialisation d'un tableau
int[] notes = { 15, 12, 18, 9, 14 };

// Déclaration avec taille (vide au départ, valeurs par défaut)
string[] jours = new string[7];

// Accéder à un élément par son index (commence à 0 !)
Console.WriteLine($"Première note : {notes[0]}");  // 15
Console.WriteLine($"Dernière note : {notes[4]}");   // 14

// Modifier un élément
notes[1] = 16;  // Remplace 12 par 16

// Longueur du tableau
Console.WriteLine($"Nombre de notes : {notes.Length}");

// Parcourir avec for
for (int i = 0; i < notes.Length; i++)
{
    Console.WriteLine($"Note {i + 1} : {notes[i]}");
}

// Parcourir avec foreach (plus élégant)
foreach (int note in notes)
{
    Console.Write($"{note} ");
}

IndexOutOfRangeException

Si tu accèdes à un index qui n'existe pas (par exemple notes[10] sur un tableau de 5 éléments), C# lance une IndexOutOfRangeException et ton programme crash. Souviens-toi : un tableau de taille N a des index de 0 à N-1. Le dernier élément est toujours à l'index Length - 1.

Tableau = taille fixe

Un tableau ne peut pas grandir ni rétrécir une fois créé. Si tu crées new int[5], tu auras toujours exactement 5 cases. Pour une collection dynamique qui peut s'agrandir, utilise une List<T> (section suivante).

List<T> — le sac à dos extensible

Si le tableau est une rangée de casiers fixes, la List<T> est un sac à dos extensible. Tu peux y ajouter, enlever, chercher des objets à volonté. C'est la collection la plus utilisée en C# au quotidien. Sharpy ne jure que par elle !

C#using System.Collections.Generic;

// Créer une liste vide
List<string> courses = new List<string>();

// Ajouter des éléments
courses.Add("Lait");
courses.Add("Pain");
courses.Add("Beurre");
courses.Add("Tomates");

// Nombre d'éléments
Console.WriteLine($"Ma liste a {courses.Count} articles");

// Vérifier si un élément existe
if (courses.Contains("Pain"))
    Console.WriteLine("Le pain est dans la liste ✅");

// Supprimer un élément
courses.Remove("Beurre");

// Accéder par index (comme un tableau)
Console.WriteLine($"Premier article : {courses[0]}");

// Parcourir
foreach (string article in courses)
{
    Console.WriteLine($"🛒 {article}");
}

// Initialisation directe
List<int> scores = new List<int> { 100, 85, 92, 78, 95 };
Console.WriteLine($"Meilleur score : {scores.Max()}");
Console.WriteLine($"Moyenne : {scores.Average():F1}");

C'est quoi le <T> dans List<T> ?

Le <T> est un générique (generic). Le T est remplacé par le type que tu veux stocker : List<string> pour du texte, List<int> pour des entiers, List<double> pour des décimaux. C'est comme dire « une liste de quelque chose ». Le compilateur s'assure que tu ne mets que des éléments du bon type dedans.

Tableau vs Liste — quand utiliser quoi ?

Tableau → taille connue et fixe, performance maximale (jours de la semaine, coordonnées XYZ)
List<T> → taille variable, ajout/suppression fréquents (panier d'achat, résultats de recherche, logs)

Introduction à LINQ — interroger tes données

LINQ (Language Integrated Query) est la baguette magique de C#. C'est comme avoir un moteur de recherche intégré directement dans le langage. Au lieu d'écrire des boucles pour filtrer, trier ou transformer tes données, tu écris des requêtes élégantes en une ligne. Sharpy a découvert LINQ et elle ne peut plus s'en passer :

C#using System.Linq;

List<int> notes = new List<int> { 15, 8, 12, 18, 6, 14, 9, 16 };

// Where — filtrer les éléments
var bonnesNotes = notes.Where(n => n >= 12);
Console.WriteLine("Notes >= 12 :");
foreach (int n in bonnesNotes)
    Console.Write($"{n} ");
Console.WriteLine();

// OrderBy — trier par ordre croissant
var triees = notes.OrderBy(n => n);
Console.WriteLine($"Triées : {string.Join(", ", triees)}");

// OrderByDescending — trier par ordre décroissant
var descTriees = notes.OrderByDescending(n => n);
Console.WriteLine($"Décroissant : {string.Join(", ", descTriees)}");

// Select — transformer chaque élément
var surVingt = notes.Select(n => $"{n}/20");
Console.WriteLine($"Format : {string.Join(", ", surVingt)}");

// Chaîner les méthodes LINQ
var topNotes = notes
    .Where(n => n >= 12)
    .OrderByDescending(n => n)
    .ToList();
Console.WriteLine($"Top notes : {string.Join(", ", topNotes)}");

// Agrégations rapides
Console.WriteLine($"Min : {notes.Min()}");
Console.WriteLine($"Max : {notes.Max()}");
Console.WriteLine($"Somme : {notes.Sum()}");
Console.WriteLine($"Moyenne : {notes.Average():F1}");
Console.WriteLine($"Combien >= 10 ? {notes.Count(n => n >= 10)}");

C'est quoi le => (lambda) ?

L'expression n => n >= 12 est une expression lambda — une mini-fonction anonyme. Lis-la comme : « pour chaque n, vérifie si n >= 12 ». Le => se lit « va vers » ou « donne ». Tu verras les lambdas en détail dans un module avancé, mais LINQ les utilise partout.

LINQ avec des strings

C#List<string> prenoms = new List<string>
    { "Alice", "Bob", "Sharpy", "Charlie", "Anna" };

// Filtrer ceux qui commencent par 'A'
var avecA = prenoms.Where(p => p.StartsWith("A"));
Console.WriteLine($"Prénoms en A : {string.Join(", ", avecA)}");

// Trier par longueur
var parLongueur = prenoms.OrderBy(p => p.Length);
Console.WriteLine($"Par longueur : {string.Join(", ", parLongueur)}");

// Transformer en majuscules
var majuscules = prenoms.Select(p => p.ToUpper());
Console.WriteLine($"Majuscules : {string.Join(", ", majuscules)}");

// Premier élément ou valeur par défaut
string? premier = prenoms.FirstOrDefault(p => p.Length > 5);
Console.WriteLine($"Premier prénom > 5 lettres : {premier}");

N'oublie pas using System.Linq

Les méthodes LINQ (Where, Select, OrderBy, etc.) ne sont disponibles que si tu ajoutes using System.Linq; en haut de ton fichier. Sans ça, le compilateur ne les reconnaît pas et tu auras des erreurs. En .NET 6+, les using globaux les importent souvent par défaut.
