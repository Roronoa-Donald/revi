Module 06

Méthodes & Fonctions

Sharpy écrit le même code 10 fois dans son projet. Son mentor lui dit : « Arrête ! Mets ça dans une méthode. » Les méthodes sont des machines spécialisées qui exécutent une tâche précise à la demande.

Méthodes void — les machines qui font une action

Imagine une usine avec des machines spécialisées. Une machine pour peindre, une pour emballer, une pour étiqueter. Chaque machine fait une seule chose, et tu l'appelles quand tu en as besoin. En C#, une méthode est exactement ça : un bloc de code réutilisable avec un nom.

Une méthode void est une machine qui fait quelque chose mais ne renvoie rien. Elle exécute une action (afficher du texte, modifier une variable, envoyer un email) puis s'arrête. C'est comme une machine à café : tu appuies sur le bouton, le café coule, mais la machine ne te « donne » pas de valeur en retour.

C#// Déclarer une méthode void
static void DireBonjour()
{
    Console.WriteLine("Bonjour ! 👋");
    Console.WriteLine("Comment ça va ?");
}

// Méthode void avec un paramètre
static void DireBonjourA(string prenom)
{
    Console.WriteLine($"Bonjour {prenom} ! 👋");
}

// Méthode void avec plusieurs paramètres
static void AfficherProfil(string nom, int age, string ville)
{
    Console.WriteLine($"=== Profil ===");
    Console.WriteLine($"Nom    : {nom}");
    Console.WriteLine($"Âge    : {age} ans");
    Console.WriteLine($"Ville  : {ville}");
}

// Appeler les méthodes
DireBonjour();
DireBonjourA("Sharpy");
DireBonjourA("SharpBot");
AfficherProfil("Sharpy", 22, "Paris");
sortieBonjour ! 👋
Comment ça va ?
Bonjour Sharpy ! 👋
Bonjour SharpBot ! 👋
=== Profil ===
Nom    : Sharpy
Âge    : 22 ans
Ville  : Paris
Anatomie d'une méthode

Une méthode se compose de : 1) un modificateur d'accès (static, public, etc.), 2) un type de retour (void = rien, int = entier, etc.), 3) un nom en PascalCase, 4) des parenthèses avec les paramètres, 5) un corps entre accolades. C'est la recette de cuisine de ton code !

Pourquoi static ?

Pour l'instant, on met static devant toutes nos méthodes parce qu'on travaille dans la méthode Main (qui est elle-même static). Une méthode static appartient à la classe, pas à un objet. Tu comprendras la différence quand on abordera la POO (Module 12).

Méthodes avec retour — les machines qui produisent un résultat

Contrairement à void, une méthode avec retour est comme un distributeur automatique : tu mets des pièces (les paramètres), et il te donne un produit (la valeur de retour). Tu spécifies le type de retour au lieu de void, et tu utilises le mot-clé return pour renvoyer la valeur.

C#// Méthode qui retourne un int
static int Additionner(int a, int b)
{
    return a + b;
}

// Méthode qui retourne un double
static double CalculerMoyenne(int[] notes)
{
    int somme = 0;
    foreach (int note in notes)
        somme += note;
    return (double)somme / notes.Length;
}

// Méthode qui retourne un bool
static bool EstMajeur(int age)
{
    return age >= 18;
}

// Méthode qui retourne un string
static string ObtenirMention(double moyenne)
{
    if (moyenne >= 16) return "Très Bien 🌟";
    if (moyenne >= 14) return "Bien 👍";
    if (moyenne >= 12) return "Assez Bien 👌";
    if (moyenne >= 10) return "Passable 😅";
    return "Recalé 😢";
}

// Utilisation
int resultat = Additionner(15, 27);
Console.WriteLine($"15 + 27 = {resultat}");

int[] mesNotes = { 15, 12, 18, 14, 16 };
double moy = CalculerMoyenne(mesNotes);
Console.WriteLine($"Moyenne : {moy:F1}/20");
Console.WriteLine($"Mention : {ObtenirMention(moy)}");

Console.WriteLine($"Sharpy est majeure ? {EstMajeur(22)}");

// On peut aussi utiliser le retour directement
Console.WriteLine($"5 + 3 = {Additionner(5, 3)}");
sortie15 + 27 = 42
Moyenne : 15.0/20
Mention : Bien 👍
Sharpy est majeure ? True
5 + 3 = 8
Type de retour incohérent

Si ta méthode est déclarée int, tu dois renvoyer un int. Écrire return "texte"; dans une méthode int provoque une erreur CS0029: Cannot implicitly convert type 'string' to 'int'. Le compilateur vérifie que le type de retour correspond à chaque chemin possible — même les else doivent retourner une valeur.

Convention PascalCase

En C#, les noms de méthodes utilisent le PascalCase : chaque mot commence par une majuscule, sans underscore. CalculerMoyenne, EstMajeur, ObtenirMention. C'est différent de Java (calculerMoyenne) ou Python (calculer_moyenne). Respecte cette convention — c'est un standard Microsoft officiel.

Paramètres par défaut & surcharge

Parfois Sharpy veut que sa méthode soit flexible. Les paramètres par défaut permettent d'appeler une méthode sans fournir tous les arguments. Et la surcharge (overloading) permet d'avoir plusieurs versions de la même méthode avec des paramètres différents.

Paramètres par défaut

C#// Paramètre avec valeur par défaut
static void Saluer(string prenom, string salutation = "Bonjour")
{
    Console.WriteLine($"{salutation}, {prenom} !");
}

// Prix avec TVA (défaut : 20%)
static double CalculerPrixTTC(double prixHT, double tauxTVA = 0.20)
{
    return prixHT * (1 + tauxTVA);
}

// Appels
Saluer("Sharpy");                    // Utilise la valeur par défaut
Saluer("SharpBot", "Hey");           // Remplace la valeur par défaut

Console.WriteLine($"TTC 20% : {CalculerPrixTTC(100):F2}€");
Console.WriteLine($"TTC 5.5% : {CalculerPrixTTC(100, 0.055):F2}€");
sortieBonjour, Sharpy !
Hey, SharpBot !
TTC 20% : 120.00€
TTC 5.5% : 105.50€

Surcharge de méthodes (Overloading)

La surcharge c'est comme un couteau suisse : un seul nom, plusieurs outils. Tu crées plusieurs méthodes avec le même nom mais des paramètres différents (nombre ou types). Le compilateur choisit automatiquement la bonne version selon les arguments fournis :

C#// Trois surcharges de la méthode Calculer
static int Calculer(int a, int b)
{
    return a + b;
}

static double Calculer(double a, double b)
{
    return a + b;
}

static string Calculer(string a, string b)
{
    return a + " " + b;
}

// Le compilateur choisit la bonne surcharge
Console.WriteLine(Calculer(5, 3));             // int → 8
Console.WriteLine(Calculer(2.5, 3.7));        // double → 6.2
Console.WriteLine(Calculer("Hello", "World")); // string → "Hello World"
sortie8
6.2
Hello World
Console.WriteLine() est surchargée !

Tu utilises la surcharge tous les jours sans le savoir ! Console.WriteLine() peut recevoir un int, un string, un double, un bool… C'est parce que Microsoft a créé des dizaines de surcharges de cette méthode. Chaque type a sa propre version.

La surcharge ne dépend PAS du type de retour

Deux méthodes avec les mêmes paramètres mais des types de retour différents ne sont pas une surcharge valide. Le compilateur ne peut pas deviner quelle version appeler juste en regardant ce que tu fais du résultat. Il faut des paramètres différents (nombre, types, ou ordre).

Aperçu : static vs instance

Sharpy a remarqué qu'on met toujours static devant nos méthodes. Mais dans le vrai monde C#, il existe deux types de méthodes :

C#class Calculatrice
{
    // Méthode STATIC : appartient à la CLASSE
    // On l'appelle via le nom de la classe
    public static int Additionner(int a, int b)
    {
        return a + b;
    }

    // Propriété d'instance
    public string Marque { get; set; }

    // Méthode d'INSTANCE : appartient à un OBJET
    // On l'appelle via une instance de la classe
    public void AfficherMarque()
    {
        Console.WriteLine($"Calculatrice : {Marque}");
    }
}

// Appel static : pas besoin de créer un objet
int somme = Calculatrice.Additionner(5, 3);
Console.WriteLine($"Somme : {somme}");

// Appel instance : il faut créer un objet d'abord
Calculatrice calc = new Calculatrice();
calc.Marque = "SharpCalc 3000";
calc.AfficherMarque();
sortieSomme : 8
Calculatrice : SharpCalc 3000
Résumé — Quand utiliser quoi ?

void → la méthode fait une action sans renvoyer de valeur
return → la méthode calcule et renvoie un résultat
paramètres par défaut → rends les arguments optionnels
surcharge → même nom, paramètres différents
static → méthode utilitaire, pas besoin d'objet (pour l'instant, c'est ce qu'on utilise !)

Le principe DRY

DRY = Don't Repeat Yourself (ne te répète pas). Si tu copies-colles le même code plus de 2 fois, crée une méthode. Chaque méthode doit faire une seule chose et la faire bien. C'est l'un des principes fondamentaux du développement logiciel propre.
