Module 07

Manipulation de Strings

Sharpy doit nettoyer des noms d'utilisateurs, formater des messages et construire des rapports. Son mentor lui dit : « Les strings, c'est comme des blocs LEGO — tu peux les découper, les réarranger et les recoller à volonté ! »

Propriétés & méthodes de base

En C#, un string est un objet qui représente une séquence de caractères. Chaque string possède des propriétés (comme Length) et des méthodes (comme ToUpper()) intégrées. Imagine un string comme un collier de perles : chaque perle est un caractère, et tu peux compter les perles, les retourner, ou en retirer certaines.

C#string nom = "  Sharpy Le Dev  ";

// Propriété Length — nombre de caractères
Console.WriteLine($"Longueur : {nom.Length}");

// Transformation de casse
Console.WriteLine(nom.ToUpper());    // Tout en majuscules
Console.WriteLine(nom.ToLower());    // Tout en minuscules

// Suppression des espaces
Console.WriteLine(nom.Trim());      // Supprime espaces début/fin
Console.WriteLine(nom.TrimStart()); // Supprime espaces au début
Console.WriteLine(nom.TrimEnd());   // Supprime espaces à la fin

// Accéder à un caractère par index
char premiere = nom[2];  // 'S' (index 0 et 1 = espaces)
Console.WriteLine($"Caractère à l'index 2 : {premiere}");

Les index commencent à 0

Comme les tableaux, les caractères d'un string sont indexés à partir de 0. Le premier caractère est à l'index 0, le deuxième à l'index 1, etc. Si tu accèdes à un index hors limites (ex : nom[100]), tu obtiens une IndexOutOfRangeException.

Les strings sont immuables (immutable)

En C#, un string ne peut jamais être modifié une fois créé. Chaque opération (ToUpper(), Trim(), Replace()…) crée un nouveau string en mémoire. L'original reste intact. C'est comme photocopier un document puis modifier la copie — l'original ne change pas.

string s = "hello"; s.ToUpper(); → s vaut toujours "hello" !
Il faut écrire : s = s.ToUpper(); pour sauvegarder le résultat.

Substring, Replace, Contains — découper et chercher

Sharpy doit extraire des morceaux de texte, remplacer des mots et vérifier si une chaîne contient un mot-clé. Ce sont les outils de « chirurgie des strings » :

C#string phrase = "Sharpy apprend le C# avec SharpBot";

// Substring — extraire une portion
string debut = phrase.Substring(0, 6);    // "Sharpy"
string fin = phrase.Substring(25);         // "SharpBot"
Console.WriteLine($"Début : {debut}");
Console.WriteLine($"Fin   : {fin}");

// Replace — remplacer du texte
string modifie = phrase.Replace("C#", "C# 12");
Console.WriteLine(modifie);

// Contains — vérifier la présence d'un mot
bool contientBot = phrase.Contains("SharpBot");
bool contientJava = phrase.Contains("Java");
Console.WriteLine($"Contient SharpBot ? {contientBot}");
Console.WriteLine($"Contient Java ? {contientJava}");

// StartsWith / EndsWith
Console.WriteLine($"Commence par Sharpy ? {phrase.StartsWith("Sharpy")}");
Console.WriteLine($"Finit par Bot ? {phrase.EndsWith("Bot")}");

// IndexOf — trouver la position d'un sous-texte
int pos = phrase.IndexOf("C#");
Console.WriteLine($"Position de 'C#' : {pos}");

Substring — deux signatures

Substring(startIndex) retourne tout à partir de l'index donné. Substring(startIndex, length) retourne length caractères à partir de startIndex. Attention : si l'index dépasse la longueur du string, tu obtiens une ArgumentOutOfRangeException !

Contains est sensible à la casse

Par défaut, Contains("sharpbot") retourne false si le string contient "SharpBot". Pour une recherche insensible à la casse, utilise : phrase.Contains("sharpbot", StringComparison.OrdinalIgnoreCase). C'est un piège classique pour les débutants !

Split & Join — éclater et recoller

C'est ici que l'analogie des blocs LEGO prend tout son sens ! Split casse un string en morceaux (un tableau), et Join recolle les morceaux ensemble. C'est comme démonter un collier de perles puis le remonter dans un autre ordre.

C#// Split — découper une chaîne en tableau
string csv = "Sharpy,22,Paris,C#,Junior";
string[] morceaux = csv.Split(',');

Console.WriteLine($"Nombre de morceaux : {morceaux.Length}");
foreach (string m in morceaux)
    Console.WriteLine($"  → {m}");

// Split avec plusieurs séparateurs
string texte = "un;deux,trois|quatre";
string[] parties = texte.Split(new char[] { ';', ',', '|' });
foreach (string p in parties)
    Console.WriteLine($"  → {p}");

// Join — recoller un tableau en string
string[] fruits = { "pomme", "banane", "cerise", "kiwi" };
string listeVirgule = string.Join(", ", fruits);
string listeTiret  = string.Join(" - ", fruits);

Console.WriteLine(listeVirgule);
Console.WriteLine(listeTiret);

// Exemple pratique : inverser les mots d'une phrase
string phrase = "SharpBot aide Sharpy";
string[] mots = phrase.Split(' ');
Array.Reverse(mots);
string inverse = string.Join(" ", mots);
Console.WriteLine($"Inversé : {inverse}");

Split et les chaînes vides

Si tu fais "a,,b".Split(','), tu obtiens ["a", "", "b"] — une chaîne vide au milieu ! Pour ignorer les entrées vides, utilise : text.Split(',', StringSplitOptions.RemoveEmptyEntries). C'est particulièrement utile quand tu traites des fichiers CSV mal formatés.

Split attend un char, pas un string

Attention : Split(',') avec des guillemets simples (char) fonctionne, mais Split(",") avec des guillemets doubles (string) nécessite une surcharge différente. Si tu veux séparer par un string comme "::", utilise : text.Split("::", StringSplitOptions.None). C'est une source fréquente d'erreurs de compilation.

Interpolation & Verbatim strings

Sharpy en avait marre de concaténer les strings avec des + partout. Son mentor SharpBot lui a montré deux outils magiques : l'interpolation ($"") et les verbatim strings (@""). Depuis, son code est 10 fois plus lisible !

Interpolation de strings ($"")

C#string prenom = "Sharpy";
int age = 22;
double note = 17.5;

// ❌ L'ancienne méthode — concaténation avec +
string ancien = "Bonjour " + prenom + ", tu as " + age + " ans.";

// ✅ La méthode moderne — interpolation avec $
string moderne = $"Bonjour {prenom}, tu as {age} ans.";

// On peut mettre des EXPRESSIONS dans les {}
Console.WriteLine($"{prenom} aura {age + 1} ans l'an prochain.");
Console.WriteLine($"Note formatée : {note:F1}/20");
Console.WriteLine($"En majuscules : {prenom.ToUpper()}");
Console.WriteLine($"Date : {DateTime.Now:dd/MM/yyyy}");

// Alignement dans l'interpolation
Console.WriteLine($"|{"Nom",-15}|{"Note",8}|");
Console.WriteLine($"|{prenom,-15}|{note,8:F1}|");

Verbatim strings (@"")

C#// ❌ Sans verbatim — il faut échapper les backslashes
string chemin1 = "C:\\Users\\Sharpy\\Documents\\projet";

// ✅ Avec verbatim — pas besoin d'échapper
string chemin2 = @"C:\Users\Sharpy\Documents\projet";

// Verbatim pour du texte multi-lignes
string sql = @"
    SELECT nom, prenom, age
    FROM etudiants
    WHERE age >= 18
    ORDER BY nom ASC";

Console.WriteLine(chemin2);
Console.WriteLine(sql);

// Combiner $ et @ — interpolation + verbatim
string user = "Sharpy";
string path = $@"C:\Users\{user}\Desktop";
Console.WriteLine(path);

Bonnes pratiques d'interpolation

Toujours préférer $"" à la concaténation avec +. C'est plus lisible, plus performant, et tu peux insérer des expressions directement. Pour les chemins Windows, combine $@"" pour avoir à la fois l'interpolation et les backslashes sans échappement. Depuis C# 11, tu peux aussi utiliser les raw string literals avec """ pour du texte encore plus complexe.

Formatage dans l'interpolation

Tu peux formater les valeurs directement dans les {} : {prix:C} pour la monnaie (€), {note:F2} pour 2 décimales, {date:dd/MM/yyyy} pour les dates, {nombre:N0} pour les milliers avec séparateurs. C'est le même formatage que String.Format().

StringBuilder — concaténation performante

Sharpy construit un rapport de 1000 lignes en ajoutant du texte dans une boucle. Avec des string classiques, chaque += crée un nouveau string en mémoire — c'est comme recopier un livre entier à chaque fois que tu ajoutes une phrase ! La solution : StringBuilder, une classe spécialement conçue pour la concaténation massive.

C#using System.Text;  // Nécessaire pour StringBuilder

// ❌ MAUVAIS — concaténation avec string (lent pour les boucles)
string resultatLent = "";
for (int i = 0; i < 5; i++)
{
    resultatLent += $"Ligne {i}\n";  // Crée un NOUVEAU string à chaque tour
}

// ✅ BON — StringBuilder (rapide et efficace)
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 5; i++)
{
    sb.AppendLine($"Ligne {i}");  // Modifie le même buffer
}
string resultatRapide = sb.ToString();

Console.WriteLine(resultatRapide);

// Méthodes utiles de StringBuilder
StringBuilder builder = new StringBuilder("Bonjour");
builder.Append(" Sharpy");          // Ajouter à la fin
builder.AppendLine(" !");           // Ajouter + saut de ligne
builder.Insert(0, ">>> ");         // Insérer au début
builder.Replace("Sharpy", "SharpBot"); // Remplacer

Console.WriteLine(builder.ToString());
Console.WriteLine($"Longueur : {builder.Length}");

String vs StringBuilder — quand choisir quoi ?

string → quelques concaténations, texte fixe, interpolation simple. Parfait pour 95% des cas.
StringBuilder → concaténation dans une boucle, construction de rapports, génération de HTML/SQL dynamique. Dès que tu fais plus de ~10 concaténations dans une boucle, passe à StringBuilder. La différence de performance peut être x100 sur de gros volumes.

Résumé — la boîte à outils des strings

Length → nombre de caractères
ToUpper() / ToLower() → changer la casse
Trim() → supprimer les espaces
Substring() → extraire une portion
Replace() → remplacer du texte
Contains() / StartsWith() / EndsWith() → chercher
Split() → éclater en tableau
string.Join() → recoller un tableau
$"" → interpolation
@"" → verbatim string
StringBuilder → concaténation en boucle

Mnémotechnique — Les 5 outils string

🧠 « L.T.S.C.R — Les Textes Se Coupent et se Remplacent »

  • Length → nombre de caractères (propriété, pas de parenthèses)
  • Trim() → supprime les espaces aux extrémités
  • Substring() → extrait une portion du texte
  • Contains() → « est-ce que ce mot est dedans ? » → true/false
  • Replace() → remplace un morceau par un autre

$"..." = interpolation. Mets $ devant le string et tes variables dans {accolades} : $"Salut {nom} !"
