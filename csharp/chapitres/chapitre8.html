Module 08

Les Dictionnaires

Sharpy doit stocker des paires clé-valeur : noms → numéros de téléphone, produits → prix, mots → nombre d'occurrences. Son mentor SharpBot lui dit : « Un dictionnaire, c'est comme un vrai annuaire téléphonique — tu cherches par nom et tu obtiens le numéro instantanément ! »

Créer un Dictionary — l'annuaire de tes données

Un Dictionary<TKey, TValue> est une collection qui stocke des paires clé → valeur. Chaque clé est unique et permet d'accéder directement à sa valeur associée — sans parcourir toute la collection. C'est comme un vrai dictionnaire : tu ne lis pas tout le livre, tu vas directement au mot que tu cherches.

C#// Créer un dictionnaire vide
Dictionary<string, int> ages = new Dictionary<string, int>();

// Ajouter des éléments avec Add()
ages.Add("Sharpy", 22);
ages.Add("SharpBot", 1);
ages.Add("Alice", 25);

// Créer et initialiser en même temps
Dictionary<string, string> capitales = new Dictionary<string, string>
{
    { "France", "Paris" },
    { "Japon", "Tokyo" },
    { "Brésil", "Brasília" },
    { "Canada", "Ottawa" }
};

// Syntaxe alternative (C# 6+)
var notes = new Dictionary<string, double>
{
    ["Maths"] = 15.5,
    ["Français"] = 12.0,
    ["C#"] = 18.0
};

// Accéder à une valeur par sa clé
Console.WriteLine($"Âge de Sharpy : {ages["Sharpy"]}");
Console.WriteLine($"Capitale du Japon : {capitales["Japon"]}");
Console.WriteLine($"Note de C# : {notes["C#"]}");

// Modifier une valeur existante
ages["Sharpy"] = 23;  // Sharpy a eu un anniversaire !
Console.WriteLine($"Nouvel âge de Sharpy : {ages["Sharpy"]}");

// Nombre d'éléments
Console.WriteLine($"Nombre de capitales : {capitales.Count}");

Capitale du Japon : Tokyo

Nouvel âge de Sharpy : 23

Clés uniques obligatoires

Chaque clé doit être unique. Si tu fais ages.Add("Sharpy", 30) alors que "Sharpy" existe déjà, tu obtiens une ArgumentException. Pour modifier une valeur existante, utilise la syntaxe avec crochets : ages["Sharpy"] = 30; — ça ajoute la clé si elle n'existe pas, ou remplace la valeur si elle existe déjà.

KeyNotFoundException

Si tu accèdes à une clé qui n'existe pas — ages["Bob"] — tu obtiens une KeyNotFoundException. Ton programme plante ! Toujours vérifier l'existence d'une clé avant d'y accéder, avec ContainsKey() ou mieux encore TryGetValue(). C'est l'erreur n°1 des débutants avec les dictionnaires.

Vérifier, chercher et supprimer

Avant de chercher dans un annuaire, il faut s'assurer que le contact existe. En C#, on a deux approches : ContainsKey() pour vérifier d'abord, et TryGetValue() qui combine vérification et récupération en une seule opération.

C#var annuaire = new Dictionary<string, string>
{
    ["Sharpy"] = "06.12.34.56.78",
    ["SharpBot"] = "06.00.00.00.01",
    ["Alice"] = "06.98.76.54.32"
};

// Méthode 1 : ContainsKey (deux étapes)
if (annuaire.ContainsKey("Sharpy"))
{
    Console.WriteLine($"Sharpy : {annuaire["Sharpy"]}");
}

if (!annuaire.ContainsKey("Bob"))
{
    Console.WriteLine("Bob n'est pas dans l'annuaire.");
}

// Méthode 2 : TryGetValue (une seule étape — RECOMMANDÉ)
if (annuaire.TryGetValue("Alice", out string numero))
{
    Console.WriteLine($"Alice trouvée : {numero}");
}
else
{
    Console.WriteLine("Contact non trouvé.");
}

// Supprimer un élément
annuaire.Remove("SharpBot");
Console.WriteLine($"Après suppression : {annuaire.Count} contacts");

// Vérifier si une valeur existe (plus lent que ContainsKey)
bool aNumero = annuaire.ContainsValue("06.12.34.56.78");
Console.WriteLine($"Numéro trouvé ? {aNumero}");

// Tout effacer
// annuaire.Clear();  // Supprime TOUS les éléments
sortieSharpy : 06.12.34.56.78
Bob n'est pas dans l'annuaire.
Alice trouvée : 06.98.76.54.32
Après suppression : 2 contacts
Numéro trouvé ? True
TryGetValue vs ContainsKey — lequel choisir ?

TryGetValue() est toujours meilleur que ContainsKey() + accès par crochets. Pourquoi ? Parce que ContainsKey cherche la clé une première fois, puis l'accès par crochets la cherche une deuxième fois. TryGetValue fait tout en un seul passage. C'est plus rapide, plus propre, et plus idiomatique en C#.

Astuce — le pattern GetValueOrDefault

Depuis .NET Core, tu peux utiliser annuaire.GetValueOrDefault("Bob", "inconnu") qui retourne la valeur si la clé existe, ou la valeur par défaut sinon. Plus besoin de if/else pour un simple lookup avec fallback !

Parcourir un Dictionary avec foreach

Sharpy veut afficher tous les contacts de son annuaire. Pour parcourir un dictionnaire, on utilise foreach avec un KeyValuePair — c'est comme feuilleter chaque page de l'annuaire, une par une.

C#var inventaire = new Dictionary<string, int>
{
    ["Épée"] = 3,
    ["Bouclier"] = 1,
    ["Potion"] = 15,
    ["Flèche"] = 42,
    ["Gemme"] = 7
};

// Méthode 1 : foreach avec KeyValuePair
Console.WriteLine("=== Inventaire complet ===");
foreach (KeyValuePair<string, int> item in inventaire)
{
    Console.WriteLine($"  {item.Key} × {item.Value}");
}

// Méthode 2 : foreach avec var (plus concis)
Console.WriteLine("\n=== Version courte ===");
foreach (var item in inventaire)
{
    Console.WriteLine($"  {item.Key} : {item.Value}");
}

// Méthode 3 : destructuration (C# 7+)
Console.WriteLine("\n=== Destructuration ===");
foreach (var (objet, quantite) in inventaire)
{
    Console.WriteLine($"  {objet} → {quantite}");
}

// Parcourir uniquement les clés
Console.WriteLine("\n=== Clés seules ===");
foreach (string cle in inventaire.Keys)
{
    Console.WriteLine($"  {cle}");
}

// Parcourir uniquement les valeurs
int total = 0;
foreach (int qte in inventaire.Values)
{
    total += qte;
}
Console.WriteLine($"\nTotal d'objets : {total}");

Ne modifie pas un dictionnaire pendant le foreach !

Si tu essaies de faire inventaire.Remove("Potion") à l'intérieur d'un foreach sur inventaire, tu obtiens une InvalidOperationException: Collection was modified. Pour supprimer des éléments pendant l'itération, parcours d'abord inventaire.Keys.ToList() ou utilise LINQ avec Where pour filtrer.

Ordre d'itération

Un Dictionary ne garantit aucun ordre d'itération. Si tu as besoin d'un ordre trié par clé, utilise SortedDictionary<TKey, TValue>. Si tu as besoin de l'ordre d'insertion, utilise OrderedDictionary (de System.Collections.Specialized) ou un List<KeyValuePair>.

Exemple pratique — compteur de mots

SharpBot lance un défi à Sharpy : « Compte combien de fois chaque mot apparaît dans un texte. » C'est l'exemple classique d'utilisation d'un dictionnaire — et c'est une question fréquente en entretien d'embauche !

C#string texte = "le chat mange le poisson et le chat dort";

// Étape 1 : Découper en mots
string[] mots = texte.ToLower().Split(' ');

// Étape 2 : Compter avec un Dictionary
var compteur = new Dictionary<string, int>();

foreach (string mot in mots)
{
    if (compteur.ContainsKey(mot))
    {
        compteur[mot]++;            // Déjà vu → on incrémente
    }
    else
    {
        compteur[mot] = 1;           // Première occurrence
    }
}

// Version plus concise avec TryGetValue
var compteur2 = new Dictionary<string, int>();
foreach (string mot in mots)
{
    compteur2.TryGetValue(mot, out int count);
    compteur2[mot] = count + 1;
}

// Étape 3 : Afficher les résultats
Console.WriteLine("=== Fréquence des mots ===");
foreach (var (mot, freq) in compteur)
{
    string barre = new string('█', freq);
    Console.WriteLine($"  {mot,-10} {freq} {barre}");
}

Résumé — la boîte à outils des Dictionnaires

new Dictionary<K, V>() → créer
dict.Add(key, value) → ajouter (erreur si clé existe)
dict[key] = value → ajouter ou modifier
dict[key] → accéder (erreur si clé absente)
dict.TryGetValue(key, out val) → accéder en sécurité
dict.ContainsKey(key) → vérifier l'existence
dict.Remove(key) → supprimer
dict.Count → nombre d'éléments
dict.Keys / dict.Values → collections de clés/valeurs
foreach (var (k,v) in dict) → itérer
