Module 09

Entrées & Affichage

Sharpy veut que ses programmes interagissent avec l'utilisateur. Son mentor SharpBot lui dit : « La console, c'est comme un talkie-walkie — Write pour parler, ReadLine pour écouter. Maîtrise les deux et tes programmes prennent vie ! »

Console.Write & Console.WriteLine — parler à l'utilisateur

Ce sont les deux méthodes de base pour afficher du texte dans la console. La différence est simple : WriteLine ajoute un saut de ligne à la fin, Write reste sur la même ligne. C'est comme envoyer un message avec ou sans « Entrée » à la fin.

C#// WriteLine — chaque appel = une nouvelle ligne
Console.WriteLine("=== Profil de Sharpy ===");
Console.WriteLine("Nom    : Sharpy");
Console.WriteLine("Âge    : 22 ans");
Console.WriteLine();  // Ligne vide

// Write — reste sur la même ligne
Console.Write("Chargement");
Console.Write(".");
Console.Write(".");
Console.Write(".");
Console.WriteLine(" OK !");  // Finit la ligne

// Afficher différents types
Console.WriteLine(42);          // int
Console.WriteLine(3.14);        // double
Console.WriteLine(true);        // bool
Console.WriteLine('A');         // char

// Caractères d'échappement
Console.WriteLine("Ligne 1\nLigne 2");         // \n = saut de ligne
Console.WriteLine("Col1\tCol2\tCol3");          // \t = tabulation
Console.WriteLine("Il a dit \"Bonjour\"");       // \" = guillemet
Console.WriteLine("Chemin : C:\\Users\\Sharpy"); // \\ = backslash

Les caractères d'échappement essentiels

\n → saut de ligne (new line)
\t → tabulation (tab)
\\ → un seul backslash
\" → guillemet double
\0 → caractère nul
\r → retour chariot (carriage return)
Pour éviter les échappements, utilise les verbatim strings avec @"..." (vu au Module 07).

Console.ForegroundColor & Console.Clear()

Tu peux ajouter de la couleur à ta console ! Console.ForegroundColor = ConsoleColor.Green; change la couleur du texte. Console.BackgroundColor = ConsoleColor.DarkBlue; change le fond. Console.ResetColor(); remet les couleurs par défaut. Et Console.Clear(); efface tout l'écran. Parfait pour créer des menus colorés !

Console.ReadLine — écouter l'utilisateur

Console.ReadLine() attend que l'utilisateur tape quelque chose et appuie sur Entrée. Le résultat est toujours un string — même si l'utilisateur tape 42, tu reçois le texte "42", pas le nombre 42. Pour obtenir un nombre, il faut parser (convertir) la saisie.

C#// Lire un texte simple
Console.Write("Quel est ton prénom ? ");
string prenom = Console.ReadLine();
Console.WriteLine($"Bienvenue, {prenom} !");

// Lire un nombre entier — int.Parse()
Console.Write("Quel est ton âge ? ");
int age = int.Parse(Console.ReadLine());
Console.WriteLine($"Tu auras {age + 1} ans l'an prochain !");

// Lire un nombre décimal — double.Parse()
Console.Write("Ta note de C# ? ");
double note = double.Parse(Console.ReadLine());
Console.WriteLine($"Note : {note:F1}/20");

// Lire un booléen
Console.Write("Es-tu étudiant ? (true/false) ");
bool etudiant = bool.Parse(Console.ReadLine());
Console.WriteLine($"Étudiant : {etudiant}");

FormatException — le piège du parsing

Si l'utilisateur tape "abc" quand tu fais int.Parse(), BOOM : FormatException: Input string was not in a correct format. Ton programme plante immédiatement. C'est pourquoi on ne devrait jamais utiliser Parse() directement sur une saisie utilisateur sans validation. Utilise TryParse() à la place (voir section suivante) !

ReadLine retourne string? (nullable)

Depuis C# 8 avec les nullable reference types, Console.ReadLine() retourne string? (potentiellement null). En pratique, c'est null uniquement si l'entrée standard est redirigée et terminée. Pour le code interactif, tu peux ignorer le warning avec Console.ReadLine()! (opérateur null-forgiving) ou faire une vérification explicite.

TryParse — la conversion sécurisée

SharpBot avertit Sharpy : « Ne fais jamais confiance à l'utilisateur. » Si quelqu'un tape "pizza" au lieu d'un nombre, ton programme ne doit pas planter. TryParse() essaie de convertir et retourne true si ça marche, false sinon — sans lever d'exception.

C#// TryParse pour int — sécurisé
Console.Write("Entre un nombre entier : ");
string saisie = Console.ReadLine();

if (int.TryParse(saisie, out int nombre))
{
    Console.WriteLine($"Tu as entré : {nombre}");
    Console.WriteLine($"Le double : {nombre * 2}");
}
else
{
    Console.WriteLine($"'{saisie}' n'est pas un nombre valide !");
}

// Boucle de validation — redemander jusqu'à obtenir un bon nombre
int age;
do
{
    Console.Write("Ton âge (1-120) : ");
} while (!int.TryParse(Console.ReadLine(), out age) || age < 1 || age > 120);

Console.WriteLine($"Âge validé : {age} ans");

// TryParse pour double
Console.Write("Prix du produit : ");
if (double.TryParse(Console.ReadLine(), out double prix))
{
    double ttc = prix * 1.20;
    Console.WriteLine($"Prix TTC (20%) : {ttc:C}");
}
else
{
    Console.WriteLine("Prix invalide !");
}

Pattern de saisie robuste

Le pattern do/while + TryParse est le standard en C# pour lire un nombre de manière robuste : tu redemandes tant que la saisie est invalide. Tu peux combiner la vérification du format (TryParse) avec la vérification de la plage (|| age < 1) dans la même condition. Adopte ce pattern dès maintenant — tu l'utiliseras dans tous tes projets.

Attention aux séparateurs décimaux

En France, le séparateur décimal est la virgule (3,14), mais en anglais c'est le point (3.14). double.Parse() utilise la culture locale de ton système. Pour forcer le point, utilise : double.Parse(saisie, CultureInfo.InvariantCulture). C'est un bug classique quand tu déploies ton app sur un serveur en anglais !

Exemples pratiques — mini-programmes interactifs

Sharpy met en pratique tout ce qu'elle a appris en créant deux mini-programmes interactifs :

Exemple 1 : Calculatrice interactive

C#Console.WriteLine("╔═══════════════════════════╗");
Console.WriteLine("║   🧮 CALCULATRICE SHARPY  ║");
Console.WriteLine("╚═══════════════════════════╝");

// Lire le premier nombre
double a;
do { Console.Write("Premier nombre : "); }
while (!double.TryParse(Console.ReadLine(), out a));

// Lire l'opérateur
Console.Write("Opérateur (+, -, *, /) : ");
string op = Console.ReadLine();

// Lire le second nombre
double b;
do { Console.Write("Second nombre : "); }
while (!double.TryParse(Console.ReadLine(), out b));

// Calculer et afficher le résultat
double resultat = 0;
bool valide = true;

if (op == "+") resultat = a + b;
else if (op == "-") resultat = a - b;
else if (op == "*") resultat = a * b;
else if (op == "/" && b != 0) resultat = a / b;
else if (op == "/" && b == 0)
{
    Console.WriteLine("❌ Division par zéro impossible !");
    valide = false;
}
else
{
    Console.WriteLine($"❌ Opérateur '{op}' non reconnu.");
    valide = false;
}

if (valide)
    Console.WriteLine($"\n✅ {a} {op} {b} = {resultat}");

Exemple 2 : Système de menu

C#bool continuer = true;

while (continuer)
{
    Console.WriteLine("\n╔═══════════════════╗");
    Console.WriteLine("║   📋 MENU SHARPY  ║");
    Console.WriteLine("╠═══════════════════╣");
    Console.WriteLine("║ 1. Dire bonjour   ║");
    Console.WriteLine("║ 2. Calculer l'âge ║");
    Console.WriteLine("║ 3. Quitter        ║");
    Console.WriteLine("╚═══════════════════╝");
    Console.Write("Ton choix : ");

    string choix = Console.ReadLine();

    if (choix == "1")
    {
        Console.Write("Ton prénom ? ");
        string nom = Console.ReadLine();
        Console.WriteLine($"👋 Salut {nom} ! Bienvenue dans le programme de Sharpy.");
    }
    else if (choix == "2")
    {
        Console.Write("Année de naissance ? ");
        if (int.TryParse(Console.ReadLine(), out int annee))
            Console.WriteLine($"Tu as environ {DateTime.Now.Year - annee} ans.");
        else
            Console.WriteLine("Année invalide !");
    }
    else if (choix == "3")
    {
        Console.WriteLine("👋 À bientôt !");
        continuer = false;
    }
    else
    {
        Console.WriteLine("❌ Choix invalide, réessaie.");
    }
}
Résumé — Console I/O

Console.WriteLine() → afficher + saut de ligne
Console.Write() → afficher sans saut de ligne
Console.ReadLine() → lire une ligne (retourne un string)
int.Parse() → convertir en int (plante si invalide)
int.TryParse() → convertir en int (retourne false si invalide)
\n \t \\ \" → caractères d'échappement
Console.ForegroundColor → couleur du texte
Console.Clear() → effacer l'écran

Console.ReadKey() — lire une seule touche

Console.ReadKey() attend qu'une seule touche soit pressée (pas besoin d'appuyer sur Entrée). Elle retourne un ConsoleKeyInfo avec la touche, le caractère et les modificateurs (Shift, Ctrl, Alt). Très utile pour les menus où l'utilisateur choisit avec une lettre, ou pour le classique « Appuie sur une touche pour continuer… ».
