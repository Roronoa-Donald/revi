Chapitre 3

Dockerfile et construction d'images

FROM, RUN, COPY, CMD, ENTRYPOINT, cache et multi-stage

Dockerfile: la recette

Un Dockerfile est un fichier texte contenant une suite d'instructions pour construire une image. La commande docker build lit cette recette et crée l'image.

FROM ubuntu
RUN apt-get update
RUN apt-get -y install cowsay
CMD ["/usr/games/cowsay", "bonjour"]

Instructions essentielles

Instruction Rôle examen
FROM Définit l'image de base.
RUN Exécute une commande pendant le build.
COPY Copie depuis le contexte vers l'image.
EXPOSE Documente le port écouté par le conteneur.
CMD Commande par défaut, remplaçable au docker run.
ENTRYPOINT Commande principale du conteneur.

CMD vs ENTRYPOINT

CMD donne une commande ou des paramètres par défaut. ENTRYPOINT fixe la commande principale. Ensemble, ENTRYPOINT peut définir le programme et CMD ses arguments par défaut.

ENTRYPOINT ["/usr/games/cowsay"]
CMD ["hello world"]

# docker run mycowsay salut
# Ici, "salut" remplace la partie CMD.

Contexte, cache et bonnes pratiques

Le build context est le dossier envoyé au daemon Docker. Il faut éviter d'envoyer trop de fichiers inutiles.

  • Créer un dossier propre pour le Dockerfile.
  • Utiliser .dockerignore pour exclure fichiers lourds ou secrets.
  • Mettre les étapes coûteuses avant les fichiers qui changent souvent.
  • Utiliser le multi-stage pour produire une image finale plus petite.
Focus examen : Question probable: pourquoi ne pas utiliser / comme contexte de build ? Parce que tout le contenu serait envoyé au daemon.
