Manipuler les conteneurs

Après docker run, il faut savoir observer, entrer, arrêter, supprimer et diagnostiquer un conteneur.

Commande Rôle
docker ps Liste les conteneurs en cours.
docker exec -it ID bash Ouvre une commande dans un conteneur lancé.
docker logs ID Affiche les logs.
docker stop ID Arrête proprement.
docker rm ID Supprime un conteneur arrêté.
docker container prune Supprime les conteneurs arrêtés.

Options de lancement

Option Sens
--name Donner un nom au conteneur.
-it Mode interactif avec pseudo-terminal.
-d Mode détaché, en arrière-plan.
-p 8000:80 Port 8000 hôte vers port 80 conteneur.
-P Publie automatiquement les ports exposés.

Réseau Docker

Chaque conteneur a son espace réseau. Docker fournit plusieurs drivers: bridge, host, overlay et none.

  • Bridge: driver par défaut, adapté à une seule machine.
  • Bridge utilisateur: recommandé pour isoler les groupes de conteneurs et obtenir la résolution de noms.
  • Host: utilise directement le réseau de l'hôte, moins isolé côté ports.
  • Overlay: interconnecte des conteneurs sur plusieurs machines avec un orchestrateur.

Stockage des données

La couche writable d'un conteneur disparaît à sa suppression. Les données importantes doivent être sorties du conteneur.

Solution Usage
Volume Stockage géré par Docker, pratique pour persistance et partage.
Bind mount Chemin précis de l'hôte monté dans le conteneur.
tmpfs Stockage en mémoire pour données non persistantes. 
$ docker volume create my-vol
$ docker run -d --mount source=my-vol,target=/app debian
$ docker run --mount type=bind,source="$(pwd)"/code,target=/app debian
