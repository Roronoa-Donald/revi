Chapitre 5

Docker Compose et synthèse

YAML, services, dépendances, volumes et commandes

Pourquoi Compose ?

Configurer une application multi-services avec plusieurs docker run devient vite complexe: ports, volumes, réseaux, dépendances et variables.

Docker Compose décrit tout cela dans un fichier YAML. C'est de l'Infrastructure as Code.

Structure d'un compose

services:
  web:
    build: .
    ports:
      - "5000:5000"
    volumes:
      - .:/code
    depends_on:
      - redis
  redis:
    image: redis
volumes:
  appdata:

À comprendre sur l'exemple

  • web et redis sont deux services.
  • Compose crée un réseau bridge propre à la composition.
  • web peut contacter redis avec le nom redis.
  • depends_on donne un ordre de démarrage, pas une garantie que le service est prêt.
  • build construit depuis un Dockerfile; image récupère ou utilise une image existante.

Commandes Compose

Commande Rôle
docker compose up Démarre la composition.
docker compose up -d Démarre en arrière-plan.
docker compose ps Liste les services lancés.
docker compose kill Arrête brutalement.
docker compose rm Supprime les conteneurs arrêtés.
docker compose down Arrête et supprime ce qui a été créé.
