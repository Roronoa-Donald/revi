Chapitre 1

Notions generales & Definitions

Comprendre les bases : droit, informatique, donnees personnelles et droits fondamentaux.

Le droit, en bref

Le droit est un ensemble de regles qui organisent la vie en societe. Il protege les personnes, fixe des obligations et sanctionne les abus.

Sur Internet comme dans la rue, il existe des regles : ce qui est interdit dans la vie reelle l'est aussi en ligne.

Qu'est-ce que le droit informatique ?

Le droit informatique regroupe les regles qui encadrent :

  • les donnees personnelles,
  • les logiciels et le droit d'auteur,
  • les reseaux et la cybersecurite,
  • les activites en ligne (e-commerce, reseaux sociaux).
Objectif : proteger la vie privee, garantir la securite et encadrer l'economie numerique.

Donnees personnelles : definition

Une donnee personnelle est toute information permettant d'identifier une personne, directement ou indirectement.

  • Nom, prenom, numero de telephone
  • Adresse IP, identifiant, photo
  • Localisation, numero de carte d'identite
Les donnees sensibles (sante, religion, opinions, orientation) sont protegees encore plus strictement.

Les acteurs cles

  • Responsable de traitement : decide pourquoi et comment collecter.
  • Sous-traitant : traite pour le compte du responsable.
  • Personne concernee : l'utilisateur dont les donnees sont traitees.
Exemple : une ecole est responsable, un hebergeur cloud est sous-traitant.

Les droits des personnes

  • Droit d'acces : obtenir la copie des donnees, leurs usages, destinataires et durees.
  • Droit de rectification : corriger des donnees inexactes ou incompletes.
  • Droit a l'effacement (suppression) : demander la suppression quand la finalite est atteinte, consentement retire ou traitement illicite.
  • Droit d'opposition : refuser un traitement dans certains cas (prospection, interet legitime).
  • Droit a la portabilite : recevoir ses donnees dans un format reutilisable et les transmettre.
  • Droit a la limitation : geler temporairement un traitement pendant verification.
Une organisation doit pouvoir repondre en delai raisonnable a ces demandes.

Notions cles du QCM

  • Traitement : toute operation sur des donnees (collecte, stockage, modification, partage).
  • Finalite : objectif precis et legitime annonce a l'avance.
  • Base legale : consentement, contrat, obligation legale, interet legitime.
  • Consentement valable : libre, specifique, eclaire, univoque.
  • Anonymisation : identification impossible ; Pseudonymisation : identite remplacee mais recoupement possible.
  • Transparence : informer qui traite, pourquoi, combien de temps.
  • Cookies : traceurs pouvant identifier indirectement une personne.
  • Droit d'auteur : protege les logiciels et creations originales.
  • Identification indirecte : recoupement IP, cookies, contexte.
Astuce : relie chaque notion a un exemple concret (IP, cookie, email pro).

Le cycle de vie d'une donnee

  1. Collecter : pourquoi ? sur quelle base legale ?
  2. Stocker : ou ? combien de temps ?
  3. Utiliser : finalite claire, acces limite.
  4. Partager : uniquement si necessaire.
  5. Supprimer : quand la finalite est atteinte.
Principe central : minimiser les donnees collectees.

Exemple simple

Un site e-commerce collecte l'email pour envoyer une facture. Il ne doit pas l'utiliser pour la publicite sans consentement explicite.

Sans consentement, l'envoi de publicite peut etre illegal.
