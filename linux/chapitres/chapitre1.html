Module 01

Commandes de Base

Le premier jour de Sami au terminal. Navigation, affichage, aide — les fondations de tout sysadmin.

Le terminal, ton outil numéro 1

Imagine que Sami arrive le premier jour de son stage. Son chef lui dit : "Oublie la souris, ici on fait tout au terminal." Le terminal (aussi appelé shell ou console) est l'interface en ligne de commande où tu tapes des instructions que Linux exécute immédiatement.

Sous Debian 13, le shell par défaut est Bash (Bourne Again SHell). Quand tu ouvres un terminal, tu vois un prompt qui ressemble à ça :

sami@debian:~$

Ce prompt te dit : l'utilisateur sami, sur la machine debian, dans le dossier ~ (ton répertoire personnel). Le $ signifie que tu es un utilisateur normal. Si tu vois #, tu es root (administrateur).

Définition — Shell

Le shell est un programme qui interprète tes commandes et les transmet au noyau Linux (kernel). Bash est le shell le plus courant, mais il en existe d'autres (zsh, fish, dash).

Attention

Linux est sensible à la casse (case-sensitive). Fichier.txt et fichier.txt sont deux fichiers différents. LS ne marchera pas, il faut taper ls.

Naviguer dans l'arborescence

Linux organise tout dans une arborescence (un arbre inversé). La racine est / et tout part de là. Voici les commandes essentielles :

pwd — Où suis-je ?

Print Working Directory — affiche le chemin absolu du dossier actuel.

bashpwd
/home/sami

ls — Qu'est-ce qu'il y a ici ?

List — liste le contenu du dossier courant.

bashls                 # Liste simple
ls -l              # Format long (permissions, taille, date)
ls -la             # Format long + fichiers cachés (commencent par .)
ls -lh             # Format long + tailles lisibles (Ko, Mo)
ls -lt             # Trié par date de modification
ls /etc             # Lister un dossier spécifique
Astuce pro

Crée un alias pour ne plus jamais taper ls -la : alias ll='ls -la'. Tu peux l'ajouter dans ton fichier ~/.bashrc pour qu'il soit permanent.

cd — Se déplacer

Change Directory — change le dossier courant.

bashcd /var/log        # Aller dans /var/log (chemin absolu)
cd Documents       # Entrer dans le sous-dossier Documents (relatif)
cd ..              # Remonter d'un niveau (dossier parent)
cd ../..           # Remonter de 2 niveaux
cd                  # Retourner dans ~ (home)
cd ~               # Même chose : retour au home
cd -               # Retourner au dossier précédent
Chemin absolu vs relatif

Absolu : commence par / — chemin complet depuis la racine (ex: /home/sami/Documents).
Relatif : part du dossier actuel (ex: Documents/projet). . = dossier actuel, .. = dossier parent.

Afficher du contenu

cat — Afficher un fichier

Concatenate — affiche le contenu d'un fichier en entier dans le terminal.

bashcat /etc/hostname        # Affiche le nom de la machine
cat -n fichier.txt       # Avec numéros de ligne

less et more — Lire page par page

Pour les fichiers longs, cat défile tout d'un coup. Utilise less pour naviguer :

bashless /var/log/syslog     # Navigue avec ↑↓, q pour quitter
more /var/log/syslog     # Similaire mais plus ancien

head et tail — Début et fin

bashhead fichier.txt          # Les 10 premières lignes
head -n 5 fichier.txt     # Les 5 premières lignes
tail fichier.txt          # Les 10 dernières lignes
tail -f /var/log/syslog  # Suivi en temps réel (follow)
Astuce pro — tail -f

tail -f est l'un des outils les plus utilisés en admin sys. Il te permet de surveiller un fichier de log en temps réel. Ctrl+C pour arrêter.

echo — Afficher du texte

bashecho "Bonjour Sami"             # Affiche du texte
echo $USER                      # Affiche le nom d'utilisateur
echo $HOME                      # Affiche le répertoire home
echo $PATH                      # Affiche le PATH (chemins des exécutables)

Obtenir de l'aide

man — Le manuel

La commande la plus importante de Linux. Quand tu ne sais pas comment fonctionne une commande, tape man nom_commande.

bashman ls              # Manuel de ls (navigue avec ↑↓, q pour quitter)
man -k "copy file" # Chercher un mot-clé dans tous les manuels

--help — L'aide rapide

bashls --help          # Aide rapide et concise
chmod --help       # Fonctionne avec la plupart des commandes

Autres commandes utiles

bashclear              # Efface l'écran (ou Ctrl+L)
history            # Affiche l'historique des commandes
!42                # Relance la commande n°42 de l'historique
!!                 # Relance la dernière commande
whoami             # Affiche ton nom d'utilisateur
hostname           # Affiche le nom de la machine
date               # Affiche la date et l'heure
uptime             # Depuis combien de temps la machine tourne
uname -a           # Infos système (noyau, architecture...)
Ne JAMAIS faire ça

Ne tape jamais rm -rf / ou rm -rf * sans savoir exactement ce que tu fais. Ces commandes suppriment des fichiers de manière irréversible. Sami a failli le faire le premier jour — heureusement, son chef était là !

Le pipe | — Un avant-goût

Le pipe (|) envoie la sortie d'une commande vers l'entrée d'une autre. Exemple : history | grep "ls" — cherche toutes les commandes contenant "ls" dans ton historique. On approfondit ça au module 4 !
