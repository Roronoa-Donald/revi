Module 02

Manipulation de Fichiers

Sami apprend à créer, copier, déplacer et supprimer — les gestes quotidiens d'un admin.

Créer des fichiers et dossiers

touch — Créer un fichier vide

Si le fichier n'existe pas, touch le crée. S'il existe, il met à jour sa date de modification.

bashtouch rapport.txt                # Crée un fichier vide
touch fic1.txt fic2.txt fic3.txt   # Crée 3 fichiers d'un coup
touch /tmp/test.log               # Crée dans un chemin spécifique

mkdir — Créer un dossier

bashmkdir projets                    # Crée le dossier "projets"
mkdir -p projets/web/css          # Crée toute l'arborescence d'un coup
mkdir -v dossier1 dossier2       # Crée 2 dossiers avec confirmation verbose
Le flag -p (parents)

Sans -p, si le dossier parent n'existe pas, mkdir échoue. Avec -p, il crée tous les dossiers intermédiaires. C'est le flag que Sami utilise le plus !

Copier et déplacer

cp — Copier

bashcp source.txt destination.txt      # Copie un fichier
cp fichier.txt /tmp/                # Copie dans /tmp (garde le même nom)
cp -r dossier/ /backup/             # Copie récursive d'un dossier
cp -i a.txt b.txt                   # Mode interactif (demande confirmation si écrasement)
cp -v *.conf /backup/               # Copie tous les .conf avec verbose
Piège classique

Sans -r, cp ne copie PAS les dossiers. Tu auras l'erreur : cp: omitting directory 'dossier/'. Pense toujours à -r pour les dossiers !

mv — Déplacer / Renommer

mv fait 2 choses : déplacer ET renommer. Tout dépend de la destination.

bashmv ancien.txt nouveau.txt          # Renomme le fichier
mv fichier.txt /tmp/                # Déplace dans /tmp
mv dossier/ /home/sami/archives/    # Déplace un dossier entier
mv -i *.log /backup/               # Mode interactif
Astuce

mv n'a pas besoin de -r pour les dossiers. Il déplace tout en un seul mouvement, c'est instantané sur le même système de fichiers.

Supprimer

rm — Supprimer des fichiers

bashrm fichier.txt                   # Supprime un fichier
rm -i fichier.txt                # Demande confirmation
rm -r dossier/                   # Supprime un dossier et son contenu
rm -rf dossier/                  # Supprime sans demander (DANGEREUX)
rm *.tmp                         # Supprime tous les fichiers .tmp

rmdir — Supprimer un dossier vide

bashrmdir dossier_vide/              # Ne fonctionne QUE si le dossier est vide
Attention CRITIQUE

Sous Linux, il n'y a pas de corbeille en ligne de commande. rm supprime définitivement. Sami a appris ça à ses dépens quand il a supprimé un fichier de config... Utilise rm -i en cas de doute.

Archiver et compresser

tar — Archiver

bash# Créer une archive tar.gz
tar -czf backup.tar.gz dossier/

# Extraire une archive
tar -xzf backup.tar.gz

# Lister le contenu sans extraire
tar -tzf backup.tar.gz

# Extraire dans un dossier spécifique
tar -xzf backup.tar.gz -C /tmp/restore/
Retenir les flags de tar

c = create, x = extract, t = list, z = gzip, f = file. Moyen mnémotechnique : créer/xtraire, zip, fichier.

zip / unzip

Plus commun dans le monde Windows, mais parfois nécessaire. Installer si manquant :

bashsudo apt install zip unzip          # Installation si nécessaire

zip -r archive.zip dossier/          # Créer un zip
unzip archive.zip                    # Extraire un zip
unzip -l archive.zip               # Lister le contenu

ln — Liens symboliques

Un lien symbolique est comme un raccourci Windows : il pointe vers un autre fichier.

bashln -s /etc/nginx/nginx.conf ~/nginx-config
# Crée un raccourci ~/nginx-config → /etc/nginx/nginx.conf

ls -l ~/nginx-config
lrwxrwxrwx 1 sami sami 24 jan 15 10:00 nginx-config -> /etc/nginx/nginx.conf
Lien symbolique vs dur

ln -s = lien symbolique (raccourci, peut casser si la cible est supprimée).
ln (sans -s) = lien dur (copie le pointeur inode, ne casse jamais tant qu'un lien existe). En pratique, tu utiliseras presque toujours -s.

Mémo Technique — Manipulation de Fichiers

🧠 C.D.M.R — « Ce Déménageur Malin Range »

  • Copier → cp (dupliquer)
  • Déplacer → mv (aussi pour renommer !)
  • Mkdir → mkdir (créer un dossier)
  • Retirer → rm (supprimer — IRRÉVERSIBLE !)

⚠️ rm -rf / = détruire tout le système. Réfléchir avant chaque rm !
