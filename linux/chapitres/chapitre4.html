Module 04

Recherche de Fichiers

Sami cherche un fichier de config perdu dans /etc. Les outils de recherche vont lui sauver la vie.

find — Recherche avancée de fichiers

find parcourt l'arborescence en temps réel pour trouver des fichiers selon de nombreux critères.

Par nom

bashfind /home -name "rapport.txt"        # Recherche exacte
find /etc -name "*.conf"               # Tous les .conf dans /etc
find . -iname "readme*"                # Insensible à la casse

Par type

bashfind /var -type f                      # Fichiers uniquement
find /var -type d                      # Dossiers uniquement
find /usr -type l                      # Liens symboliques

Par taille

bashfind /var/log -size +100M              # Fichiers > 100 Mo
find /tmp -size -1k                    # Fichiers < 1 Ko
find /home -empty                      # Fichiers et dossiers vides

Par date

bashfind /var/log -mtime -7               # Modifiés dans les 7 derniers jours
find /tmp -atime +30                   # Non accédés depuis 30+ jours
find . -newer référence.txt              # Plus récents que référence.txt

Actions sur les résultats

bash# Supprimer les fichiers trouvés
find /tmp -name "*.tmp" -delete

# Exécuter une commande sur chaque résultat
find /var/log -name "*.log" -exec ls -lh {} \;

# Copier tous les .conf trouvés dans un dossier
find /etc -name "*.conf" -exec cp {} /backup/configs/ \;
{} et \; dans -exec

{} est remplacé par le fichier trouvé. \; termine la commande. Tu peux aussi utiliser + au lieu de \; pour passer tous les fichiers en une seule fois (plus rapide).

grep — Recherche dans le contenu

grep cherche un texte (ou une regex) à l'intérieur des fichiers. C'est l'outil le plus puissant du terminal.

bashgrep "error" /var/log/syslog         # Cherche "error" dans syslog
grep -i "error" fichier.log           # Insensible à la casse
grep -r "TODO" /home/sami/projets/   # Recherche récursive dans un dossier
grep -n "root" /etc/passwd           # Affiche les numéros de ligne
grep -c "error" *.log                # Compte les occurrences
grep -v "#" /etc/fstab               # Lignes qui NE contiennent PAS "#"
grep -l "password" /etc/*            # Liste les fichiers qui contiennent le mot

grep + pipes : la combinaison magique

bashcat /etc/passwd | grep "sami"       # Cherche "sami" dans passwd
ps aux | grep "apache"               # Cherche les processus apache
history | grep "chmod"               # Retrouve tes commandes chmod
dmesg | grep -i "usb"               # Cherche les messages USB du kernel
Astuce pro — grep -rn

La combinaison grep -rn "mot" dossier/ (recursive + numéros de ligne) est LA commande la plus utilisée par les développeurs pour rechercher du code. Sami l'utilise 50 fois par jour.

Autres outils de recherche

locate — Recherche ultra-rapide

locate utilise une base de données pré-indexée. Beaucoup plus rapide que find, mais les résultats peuvent ne pas être à jour.

bashsudo apt install mlocate             # Installer si nécessaire
sudo updatedb                          # Met à jour la base de données
locate "nginx.conf"                    # Recherche instantanée
locate -i "readme"                    # Insensible à la casse

which et whereis — Trouver des commandes

bashwhich python3          # Chemin de l'exécutable python3
/usr/bin/python3
whereis bash            # Binaire + sources + man pages
bash: /usr/bin/bash /usr/share/man/man1/bash.1.gz

wc — Compter

bashwc fichier.txt          # Lignes, mots, octets
wc -l fichier.txt       # Nombre de lignes uniquement
ls /etc | wc -l        # Combien de fichiers dans /etc ?

sort et uniq — Trier et dédupliquer

bashsort noms.txt                        # Tri alphabétique
sort -n nombres.txt                  # Tri numérique
sort -r fichier.txt                  # Tri inversé
sort fichier.txt | uniq              # Supprime les doublons consécutifs
sort fichier.txt | uniq -c           # Compte les occurrences de chaque ligne
uniq sans sort = piège

uniq ne supprime que les doublons consécutifs. Si "chat" apparaît aux lignes 1, 5 et 8, seuls les doublons adjacents sont retirés. Toujours faire sort | uniq pour être sûr !

Les Pipes et Redirections

Le pipe |

Le pipe envoie la sortie (stdout) d'une commande comme entrée de la suivante. C'est l'ADN de Linux !

bashcat /var/log/syslog | grep "error" | wc -l
# Combien de lignes contiennent "error" dans syslog ?

ps aux | grep "apache" | grep -v "grep"
# Liste les processus apache (en excluant la commande grep elle-même)

du -sh /var/* | sort -hr | head -5
# Top 5 des plus gros dossiers dans /var

Redirections

bashecho "Hello" > fichier.txt          # Écrit dans fichier (ÉCRASE le contenu)
echo "World" >> fichier.txt         # Ajoute à la fin (APPEND)
ls /inexistant 2> erreurs.log        # Redirige les erreurs (stderr)
commande &> tout.log                # Redirige stdout ET stderr
commande > /dev/null 2>&1         # Envoie tout dans le néant (silence total)
> vs >>

> écrase le fichier. >> ajoute à la fin. Si tu utilises > par erreur, tu perds l'ancien contenu ! Sami a perdu un fichier de log comme ça — depuis il utilise toujours >> sauf quand il veut vraiment écraser.

Mémo Technique — Recherche

🧠 F.G.W — « Find cherche les noms, Grep cherche les mots, Which cherche les commandes »

  • Find → cherche par nom de fichier dans les dossiers
  • Grep → cherche par contenu à l'intérieur des fichiers
  • Which → cherche une commande dans le PATH

Combo magique : find / -name "*.log" | xargs grep "error" → trouver tous les logs contenant "error".
