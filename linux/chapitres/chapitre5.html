Module 05

Éditeurs Vi/Vim & Nano

Sami doit modifier un fichier de config sur un serveur sans interface graphique. Il n'a que le terminal.

Nano — L'éditeur simple

Nano est l'éditeur le plus simple de Linux. Si tu débutes, c'est celui que tu utiliseras le plus. Il est installé par défaut sur Debian 13.

Ouvrir et éditer

bashnano fichier.txt           # Ouvrir (crée le fichier s'il n'existe pas)
sudo nano /etc/hostname    # Éditer un fichier système (besoin de sudo)
nano -l script.sh          # Ouvrir avec numéros de ligne

Raccourcis clavier essentiels

Ctrl+O    Sauvegarder (puis Entrée pour confirmer le nom)
Ctrl+X    Quitter (demande de sauvegarder si modifié)
Ctrl+K    Couper la ligne entière
Ctrl+U    Coller la ligne coupée
Ctrl+W    Rechercher un mot
Ctrl+\    Rechercher et remplacer
Ctrl+G    Afficher l'aide
Ctrl+C    Afficher la position du curseur (ligne, colonne)
Alt+U     Annuler (Undo)
Alt+E     Refaire (Redo)
Les raccourcis sont affichés en bas

Nano affiche les raccourcis dans la barre du bas. Le ^ signifie Ctrl. Exemple : ^X Exit = Ctrl+X pour quitter. Pas besoin de tout retenir !

Vi/Vim — L'éditeur puissant

Vim est l'éditeur des vrais sysadmins. Il est présent sur TOUS les systèmes Unix/Linux. Il est déroutant au début, mais extrêmement puissant une fois maîtrisé.

bashsudo apt install vim      # Installer vim (vi basique est par défaut)
vim fichier.txt              # Ouvrir un fichier
sudo vim /etc/fstab         # Éditer un fichier système
Les 3 modes de Vim

1. Mode Normal (par défaut à l'ouverture) — navigation et commandes
2. Mode Insertion — tu tapes du texte (entrée avec i, sortie avec Esc)
3. Mode Commande — commandes spéciales (entrée avec :)

Survivre dans Vim — Le minimum vital

── Mode Normal (par défaut) ──
i         Passer en mode insertion (avant le curseur)
a         Passer en mode insertion (après le curseur)
o         Nouvelle ligne en dessous + insertion
O         Nouvelle ligne au-dessus + insertion
Esc       Revenir en mode Normal

── Sauvegarder et quitter ──
:w        Sauvegarder
:q        Quitter
:wq       Sauvegarder et quitter
:q!       Quitter SANS sauvegarder (forcer)
:wq!      Forcer sauvegarde et quitter

── Navigation ──
h j k l   ← ↓ ↑ → (ou utilise les flèches)
gg        Aller au début du fichier
G         Aller à la fin du fichier
:42       Aller à la ligne 42

── Édition rapide ──
dd        Supprimer (couper) la ligne entière
yy        Copier la ligne entière
p         Coller après le curseur
u         Annuler (Undo)
Ctrl+R    Refaire (Redo)
x         Supprimer le caractère sous le curseur

── Recherche ──
/mot      Chercher "mot" (n = suivant, N = précédent)
:%s/ancien/nouveau/g  Remplacer partout dans le fichier
Le piège n°1 de Vim

Tu ouvres Vim, tu tapes du texte... et rien ne se passe comme prévu ! C'est parce que tu es en mode Normal. Tape i d'abord pour passer en mode Insertion. Sami a bloqué 10 minutes sur ça son premier jour !

Comment quitter Vim

La blague classique : "Comment quitter Vim ?" — Appuie sur Esc puis tape :q! et Entrée. Si ça ne marche pas : Esc Esc :q!. Ce meme a généré des millions de recherches Google !

Le conseil de Sami

Utilise Nano pour les petites éditions rapides (modifier une ligne de config). Apprends les bases de Vim pour les cas où Nano n'est pas disponible (serveur minimal, conteneur Docker). En entretien, savoir utiliser Vim montre que tu es un vrai sysadmin !

Changer l'éditeur par défaut

bash# Voir l'éditeur par défaut actuel
echo $EDITOR

# Changer pour nano
export EDITOR=nano

# Ou utiliser la commande Debian dédiée
sudo update-alternatives --config editor

# Rendre permanent : ajouter dans ~/.bashrc
echo 'export EDITOR=nano' >> ~/.bashrc

Gestion de paquets — APT & dpkg

Sur Debian/Ubuntu, les logiciels sont distribués sous forme de paquets (.deb). Le gestionnaire APT (Advanced Package Tool) permet de les installer, mettre à jour et supprimer automatiquement, en gérant les dépendances.

apt — Le gestionnaire principal

bash# Mettre à jour la liste des paquets disponibles
sudo apt update

# Mettre à jour tous les paquets installés
sudo apt upgrade

# Mettre à jour + gérer les nouvelles dépendances
sudo apt full-upgrade

# Installer un paquet
sudo apt install htop

# Installer plusieurs paquets d'un coup
sudo apt install vim curl wget git

# Supprimer un paquet (garde les fichiers de config)
sudo apt remove htop

# Supprimer un paquet ET ses fichiers de config
sudo apt purge htop

# Supprimer les paquets orphelins (dépendances inutilisées)
sudo apt autoremove

# Rechercher un paquet dans les dépôts
apt search serveur web

# Afficher les infos d'un paquet
apt show nginx

# Lister les paquets installés
apt list --installed

# Lister les paquets avec mises à jour disponibles
apt list --upgradable
update ≠ upgrade

apt update met à jour la liste des paquets (comme vérifier le catalogue). apt upgrade installe les nouvelles versions. Toujours faire update avant upgrade !

dpkg — Gestion bas niveau

dpkg est l'outil bas niveau qui gère directement les fichiers .deb. APT l'utilise en interne, mais parfois tu dois l'utiliser directement.

bash# Installer un paquet .deb téléchargé manuellement
sudo dpkg -i paquet.deb

# Si des dépendances manquent après dpkg -i
sudo apt install -f

# Lister tous les paquets installés
dpkg -l

# Vérifier si un paquet est installé
dpkg -l | grep nginx

# Voir les fichiers installés par un paquet
dpkg -L nginx

# Trouver à quel paquet appartient un fichier
dpkg -S /usr/bin/vim
Scénario de Sami

Son chef lui demande d'installer un serveur web. Sami tape :
sudo apt update → met à jour le catalogue
sudo apt install nginx → installe Nginx + toutes ses dépendances
dpkg -l | grep nginx → vérifie que c'est bien installé
Et voilà, le serveur web tourne !

Les dépôts — /etc/apt/sources.list

bash# Voir les dépôts configurés
cat /etc/apt/sources.list

# Les dépôts additionnels sont dans :
ls /etc/apt/sources.list.d/

# Après avoir modifié les sources, toujours :
sudo apt update
