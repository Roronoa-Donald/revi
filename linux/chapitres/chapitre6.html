Module 06

Utilisateurs & Groupes

Sami doit créer un compte pour un nouveau collègue et lui donner les bons accès. Bienvenue dans l'administration multi-utilisateur !

Les fichiers clés

Linux stocke toutes les infos des utilisateurs et groupes dans 4 fichiers texte essentiels :

Fichiers système/etc/passwd    # Infos utilisateurs (nom, UID, GID, home, shell)
/etc/shadow    # Mots de passe chiffrés (accès root uniquement)
/etc/group     # Infos groupes (nom, GID, membres)
/etc/gshadow   # Mots de passe des groupes (rarement utilisé)

Décoder /etc/passwd

bashcat /etc/passwd | grep sami
sami:x:1001:1001:Sami Stagiaire:/home/sami:/bin/bash
#  │   │  │    │       │            │          │
#  │   │  │    │       │            │          └─ Shell de connexion
#  │   │  │    │       │            └─ Répertoire personnel
#  │   │  │    │       └─ Commentaire (nom complet)
#  │   │  │    └─ GID (Group ID principal)
#  │   │  └─ UID (User ID)
#  │   └─ x = mot de passe dans /etc/shadow
#  └─ Nom d'utilisateur
UID et GID

UID 0 = root (superutilisateur)
UID 1-999 = comptes système (services comme www-data, mysql...)
UID 1000+ = comptes utilisateurs normaux
Chaque utilisateur a un groupe principal (même GID) et peut appartenir à des groupes secondaires.

Gérer les utilisateurs

Créer un utilisateur

bash# Créer un utilisateur avec répertoire home
sudo useradd -m alice

# Créer avec shell bash + commentaire + home
sudo useradd -m -s /bin/bash -c "Alice Dupont" alice

# Créer et ajouter au groupe sudo directement
sudo useradd -m -s /bin/bash -G sudo alice

# Définir un mot de passe
sudo passwd alice
-m est indispensable

Sans -m, le répertoire /home/alice ne sera PAS créé. L'utilisateur existera mais n'aura pas de dossier personnel. Sur Debian, adduser alice (interactif) est une alternative plus simple qui crée tout automatiquement.

Modifier un utilisateur

bash# Ajouter au groupe sudo (droits admin)
sudo usermod -aG sudo alice

# Ajouter à plusieurs groupes secondaires
sudo usermod -aG www-data,docker alice

# Changer le shell
sudo usermod -s /bin/zsh alice

# Changer le répertoire home
sudo usermod -d /home/nouveau_home -m alice

# Verrouiller un compte (désactiver la connexion)
sudo usermod -L alice

# Déverrouiller un compte
sudo usermod -U alice

# Renommer un utilisateur
sudo usermod -l alice_new alice
Attention au -G sans -a

usermod -G sudo alice (sans -a) REMPLACE tous les groupes secondaires par "sudo" uniquement ! Utilise toujours -aG (append + Group) pour ajouter sans supprimer les groupes existants.

Supprimer un utilisateur

bash# Supprimer l'utilisateur (garde le home)
sudo userdel alice

# Supprimer l'utilisateur ET son répertoire home
sudo userdel -r alice

Gérer les groupes

bash# Créer un groupe
sudo groupadd developpeurs

# Créer un groupe avec un GID spécifique
sudo groupadd -g 2000 webmasters

# Supprimer un groupe
sudo groupdel developpeurs

# Renommer un groupe
sudo groupmod -n devs developpeurs

# Voir les groupes d'un utilisateur
groups alice
alice : alice sudo www-data

# Voir son propre ID et ses groupes
id
uid=1001(sami) gid=1001(sami) groupes=1001(sami),27(sudo)

# Lister les membres d'un groupe
getent group sudo
sudo:x:27:sami,alice

Su & Sudo — Élévation de privilèges

su — Changer d'utilisateur

bash# Devenir root (demande le mot de passe root)
su -

# Devenir alice (demande le mot de passe d'alice)
su - alice

# Exécuter une seule commande en tant qu'alice
su - alice -c "whoami"
alice

# Revenir à l'utilisateur précédent
exit

sudo — Exécuter en tant que root

bash# Exécuter une commande en root (mot de passe de l'utilisateur courant)
sudo apt update

# Ouvrir un shell root
sudo -i

# Relancer la dernière commande avec sudo
sudo !!

# Voir les droits sudo de l'utilisateur actuel
sudo -l

Configurer sudo — Le fichier sudoers

bash# TOUJOURS utiliser visudo pour éditer sudoers
sudo visudo

# Syntaxe d'une ligne sudoers :
# utilisateur  machine=(en_tant_que)  commandes
alice   ALL=(ALL:ALL)  ALL           # Alice peut tout faire
bob     ALL=(ALL)  /usr/bin/apt      # Bob peut seulement utiliser apt
%webdev ALL=(ALL)  /usr/bin/systemctl restart apache2  # Le groupe webdev peut redémarrer Apache
Toujours visudo, jamais nano

N'édite JAMAIS /etc/sudoers directement avec nano ou vim ! visudo vérifie la syntaxe avant de sauvegarder. Une erreur dans sudoers peut t'empêcher d'utiliser sudo et te bloquer hors du système.

Commandes utiles

bash# Changer son propre mot de passe
passwd

# Changer le mot de passe d'un autre utilisateur (root)
sudo passwd alice

# Forcer le changement de mot de passe à la prochaine connexion
sudo passwd -e alice

# Voir qui est connecté
who
w         # Plus détaillé (activité, charge système)

# Dernières connexions
last       # Historique des connexions réussies
lastlog    # Dernière connexion de chaque utilisateur

# Lister tous les utilisateurs du système
cut -d: -f1 /etc/passwd

# Lister seulement les utilisateurs « humains » (UID ≥ 1000)
awk -F: '$3 >= 1000 && $3 < 65534 {print $1}' /etc/passwd
Scénario complet de Sami

Un nouveau développeur arrive. Sami doit :
sudo useradd -m -s /bin/bash -c "Bob Dev" -G sudo,www-data bob
sudo passwd bob
sudo passwd -e bob (force le changement au 1er login)
Vérifie : id bob → uid=1002(bob) gid=1002(bob) groupes=1002(bob),27(sudo),33(www-data)
