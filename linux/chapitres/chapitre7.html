Module 07

Réseau Statique & SSH

Sami doit configurer l'adresse IP d'un serveur et s'y connecter à distance. On entre dans le vrai monde de l'administration système !

Diagnostiquer le réseau

Avant de configurer, il faut savoir lire l'état actuel du réseau :

bash# Voir les interfaces réseau et leurs IP
ip addr                     # ou ip a (forme courte)

# Voir la table de routage
ip route                    # ou ip r

# Voir les DNS configurés
cat /etc/resolv.conf

# Tester la connectivité
ping -c 4 8.8.8.8          # Ping Google DNS (4 paquets)
ping -c 4 google.fr         # Teste aussi la résolution DNS

# Voir les ports ouverts / connexions actives
ss -tulnp                   # Ports TCP/UDP en écoute

# Voir le nom d'hôte
hostname
hostnamectl
Interfaces courantes

lo = loopback (127.0.0.1, interface locale)
eth0 ou ens33 = carte réseau filaire (le nom varie selon le matériel)
wlan0 = carte Wi-Fi
Utilise ip addr pour identifier le nom exact de ton interface.

Configuration IP statique (Debian)

Sur Debian 13, la configuration réseau se fait dans le fichier /etc/network/interfaces :

bash# Éditer le fichier de configuration réseau
sudo nano /etc/network/interfaces

Contenu pour une IP statique

/etc/network/interfaces# Loopback
auto lo
iface lo inet loopback

# Interface réseau principale (adapter le nom !)
auto ens33
iface ens33 inet static
    address   192.168.1.100
    netmask   255.255.255.0
    gateway   192.168.1.1
    dns-nameservers 8.8.8.8 8.8.4.4

Contenu pour DHCP (automatique)

/etc/network/interfacesauto ens33
iface ens33 inet dhcp

Appliquer les changements

bash# Redémarrer le service réseau
sudo systemctl restart networking

# OU redémarrer une interface spécifique
sudo ifdown ens33 && sudo ifup ens33

# Vérifier que la config est appliquée
ip addr show ens33
Attention en SSH !

Si tu es connecté en SSH et que tu changes l'adresse IP, tu seras déconnecté ! Assure-toi d'avoir un accès console (écran + clavier) au serveur avant de modifier la config réseau à distance.

Configuration DNS

bash# Voir les serveurs DNS actuels
cat /etc/resolv.conf

# Fichier hosts local (résolution manuelle)
sudo nano /etc/hosts

Le fichier /etc/hosts

/etc/hosts127.0.0.1       localhost
127.0.1.1       debian-server

# Ajouts personnalisés
192.168.1.200   serveur-web
192.168.1.201   serveur-bdd
/etc/hosts vs DNS

Le fichier /etc/hosts est consulté avant les serveurs DNS. C'est pratique pour définir des noms locaux sans configurer un serveur DNS complet. Exemple : tu peux accéder à serveur-web au lieu de taper 192.168.1.200.

Changer le hostname

bash# Voir le hostname actuel
hostnamectl

# Changer le hostname (persistant)
sudo hostnamectl set-hostname serveur-debian

# Mettre à jour /etc/hosts en conséquence
sudo nano /etc/hosts

SSH — Connexion à distance

Installer le serveur SSH

bash# Installer le serveur OpenSSH
sudo apt install openssh-server

# Vérifier que le service est actif
sudo systemctl status ssh

# Démarrer / arrêter / redémarrer
sudo systemctl start ssh
sudo systemctl stop ssh
sudo systemctl restart ssh

# Activer au démarrage
sudo systemctl enable ssh

Se connecter en SSH

bash# Connexion basique (depuis un autre PC)
ssh sami@192.168.1.100

# Avec un port différent
ssh -p 2222 sami@192.168.1.100

# Exécuter une commande à distance (sans session interactive)
ssh sami@192.168.1.100 "uptime"

# Copier des fichiers via SCP
scp fichier.txt sami@192.168.1.100:/home/sami/
scp -r dossier/ sami@192.168.1.100:/home/sami/    # Dossier récursif

# Télécharger un fichier depuis le serveur
scp sami@192.168.1.100:/home/sami/log.txt ./

Authentification par clé SSH (sans mot de passe)

bash# 1. Générer une paire de clés (sur ta machine locale)
ssh-keygen -t ed25519 -C "sami@monpc"
# Entrée pour le chemin par défaut, puis un passphrase (optionnel)

# 2. Copier la clé publique sur le serveur
ssh-copy-id sami@192.168.1.100

# 3. Se connecter sans mot de passe !
ssh sami@192.168.1.100
# → Connexion directe 🎉
Clés SSH = Sécurité + Confort

Les clés SSH sont plus sécurisées que les mots de passe et permettent l'automatisation (scripts, déploiements). La clé privée (~/.ssh/id_ed25519) reste sur ta machine. La clé publique (~/.ssh/id_ed25519.pub) va sur le serveur (~/.ssh/authorized_keys).

Sécuriser SSH

Le fichier de configuration du serveur SSH :

bashsudo nano /etc/ssh/sshd_config

Modifications recommandées

/etc/ssh/sshd_config# Changer le port par défaut (22 → 2222)
Port 2222

# Interdire la connexion root
PermitRootLogin no

# N'autoriser que certains utilisateurs
AllowUsers sami alice

# Désactiver l'authentification par mot de passe
# (seulement après avoir configuré les clés SSH !)
PasswordAuthentication no

# Limiter les tentatives de connexion
MaxAuthTries 3
bash# Appliquer les changements
sudo systemctl restart ssh
Ne te verrouille pas !

Avant de désactiver PasswordAuthentication, vérifie que ta clé SSH fonctionne ! Ouvre une deuxième session SSH pour tester avant de fermer la première. Si les clés ne marchent pas et que les mots de passe sont désactivés, tu seras bloqué !

Services avec systemctl

Sami doit gérer des services (SSH, Apache, MySQL...) avec systemctl :

bash# Voir l'état d'un service
sudo systemctl status ssh

# Démarrer / Arrêter / Redémarrer
sudo systemctl start ssh
sudo systemctl stop ssh
sudo systemctl restart ssh

# Recharger la config (sans couper le service)
sudo systemctl reload ssh

# Activer / Désactiver au démarrage
sudo systemctl enable ssh     # Démarre auto au boot
sudo systemctl disable ssh    # Ne démarre plus au boot

# Lister tous les services actifs
systemctl list-units --type=service --state=running
