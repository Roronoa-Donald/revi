Module 08

Introduction au Bash Scripting

Sami en a marre de taper les mêmes commandes tous les matins. Il va automatiser tout ça avec des scripts Bash !

Premier script

Créer et exécuter

bash# 1. Créer le fichier script
nano mon_script.sh

# 2. Ajouter le contenu (voir ci-dessous)

# 3. Rendre exécutable
chmod +x mon_script.sh

# 4. Exécuter
./mon_script.sh

# OU sans chmod :
bash mon_script.sh

Structure d'un script

mon_script.sh#!/bin/bash
# Mon premier script Bash
# Auteur : Sami | Date : 2025

echo "Bonjour, bienvenue sur $(hostname) !"
echo "Nous sommes le $(date +%d/%m/%Y)"
echo "Il est $(date +%H:%M)"
echo "Tu es connecté en tant que : $USER"
echo "Uptime : $(uptime -p)"
Le shebang #!/bin/bash

La première ligne #!/bin/bash (shebang) indique au système quel interpréteur utiliser. Sans elle, le script pourrait être interprété par un autre shell. C'est obligatoire dans tout script Bash.

Variables

variables.sh#!/bin/bash

# Déclarer une variable (PAS d'espace autour du =)
nom="Sami"
age=22
dossier="/home/sami/projets"

# Utiliser une variable avec $
echo "Bonjour $nom, tu as $age ans"
echo "Tes projets sont dans : $dossier"

# Stocker le résultat d'une commande
date_actuelle=$(date +%d/%m/%Y)
nb_users=$(who | wc -l)
echo "Date : $date_actuelle — $nb_users utilisateur(s) connecté(s)"

# Variables spéciales
echo "Nom du script : $0"
echo "Argument 1 : $1"
echo "Argument 2 : $2"
echo "Tous les arguments : $@"
echo "Nombre d'arguments : $#"
echo "Code retour dernière commande : $?"

Lire une saisie utilisateur

saisie.sh#!/bin/bash

echo -n "Quel est ton nom ? "
read nom
echo "Salut $nom !"

# Avec un prompt intégré
read -p "Ton âge : " age

# Saisie masquée (mot de passe)
read -sp "Mot de passe : " mdp

Piège classique

nom = "Sami" avec des espaces autour du = provoque une erreur ! Bash interprète nom comme une commande. Écris toujours nom="Sami" sans espace.

Conditions (if/else)

conditions.sh#!/bin/bash

# Structure de base
if [[ $USER == "root" ]]; then
    echo "⚠️ Tu es root ! Fais attention."
else
    echo "Tu es $USER, utilisateur normal."
fi

# if / elif / else
read -p "Entrez un nombre : " n

if [[ $n -gt 100 ]]; then
    echo "Plus grand que 100"
elif [[ $n -eq 100 ]]; then
    echo "Exactement 100"
else
    echo "Plus petit que 100"
fi

Opérateurs de comparaison

── Nombres ──
-eq    Égal           (equal)
-ne    Différent      (not equal)
-gt    Plus grand     (greater than)
-lt    Plus petit     (less than)
-ge    Plus grand ou égal (greater or equal)
-le    Plus petit ou égal (less or equal)

── Chaînes ──
==     Égale
!=     Différente
-z     Chaîne vide (zero)
-n     Chaîne non vide (not zero)

── Fichiers ──
-f     Le fichier existe et est un fichier
-d     Le dossier existe
-e     Existe (fichier ou dossier)
-r     Est lisible (readable)
-w     Est modifiable (writable)
-x     Est exécutable

── Logiques ──
&&     ET logique
||     OU logique
!      NON logique

Exemples pratiques

check_file.sh#!/bin/bash

if [[ -f $fichier ]]; then

Boucles

Boucle for

boucle_for.sh#!/bin/bash

# Parcourir une liste
for fruit in pomme banane orange; do
    echo "J'aime les ${fruit}s"
done

# Parcourir des nombres
for i in {1..5}; do
    echo "Iteration $i"
done

# Parcourir les fichiers .log
for fichier in /var/log/*.log; do
    echo "Log trouvé : $fichier"
done

# Style C (compteur classique)
for (( i=0; i<5; i++ )); do
    echo "Compteur : $i"
done

Boucle while

boucle_while.sh#!/bin/bash

# Compteur simple
compteur=1
while [[ $compteur -le 5 ]]; do
    echo "Tour n°$compteur"
    (( compteur++ ))
done

# Lire un fichier ligne par ligne
while IFS= read -r ligne; do
    echo "Ligne : $ligne"
done < /etc/hostname

# Menu interactif
while true; do
    echo "1) Voir l'heure  2) Voir le disque  3) Quitter"
    read -p "Choix : " choix
    case $choix in
        1) date ;;
        2) df -h ;;
        3) echo "Au revoir !"; break ;;
        *) echo "Choix invalide" ;;
    esac
done

Fonctions

fonctions.sh#!/bin/bash

# Définir une fonction
saluer() {
    echo "Bonjour $1 ! Tu as $2 ans."
}

# Appeler la fonction avec des arguments
saluer "Sami" 22
saluer "Alice" 25

# Fonction avec valeur de retour
fichier_existe() {
    if [[ -f "$1" ]]; then
        return 0   # succès
    else
        return 1   # échec
    fi
}

if fichier_existe "/etc/passwd"; then
    echo "Le fichier existe"
fi

# Fonction utile : log avec timestamp
log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1"
}

log "Script démarré"
log "Mise à jour en cours..."

Scripts pratiques

Script de sauvegarde

backup.sh#!/bin/bash
# Script de sauvegarde simple

SOURCE="/home/sami/projets"
DEST="/home/sami/backups"
DATE=$(date +%Y%m%d_%H%M%S)
ARCHIVE="backup_${DATE}.tar.gz"

# Créer le dossier de destination s'il n'existe pas
mkdir -p "$DEST"

# Créer l'archive
tar -czf "${DEST}/${ARCHIVE}" "$SOURCE"

if [[ $? -eq 0 ]]; then
    echo "✅ Sauvegarde réussie : ${DEST}/${ARCHIVE}"
else
    echo "❌ Erreur lors de la sauvegarde"
    exit 1
fi

# Supprimer les sauvegardes de plus de 7 jours
find "$DEST" -name "backup_*.tar.gz" -mtime +7 -delete
echo "🧹 Anciennes sauvegardes nettoyées"

Script d'info système

sysinfo.sh#!/bin/bash
# Affiche un résumé du système

echo "═══════════════════════════════════"
echo " 🖥️  INFORMATIONS SYSTÈME"
echo "═══════════════════════════════════"
echo "Hostname    : $(hostname)"
echo "OS          : $(lsb_release -ds 2>/dev/null || cat /etc/os-release | grep PRETTY | cut -d= -f2)"
echo "Kernel      : $(uname -r)"
echo "Uptime      : $(uptime -p)"
echo "Utilisateur : $USER"
echo "IP          : $(hostname -I | awk '{print $1}')"
echo "RAM         : $(free -h | awk '/Mem/{print $3 "/" $2}')"
echo "Disque /    : $(df -h / | awk 'NR==2{print $3 "/" $2 " (" $5 ")"}')"
echo "Connectés   : $(who | wc -l) utilisateur(s)"
echo "═══════════════════════════════════"
Automatiser avec cron

Tu peux automatiser tes scripts avec crontab :
crontab -e puis ajoute une ligne :
0 2 * * * /home/sami/backup.sh → Exécute le backup tous les jours à 2h du matin
*/5 * * * * /home/sami/check.sh → Toutes les 5 minutes
Format : minute heure jour mois jour_semaine commande

Déboguer un script

bash# Exécuter en mode debug (affiche chaque commande)
bash -x mon_script.sh

# Activer le debug dans le script
set -x    # Active le mode debug
set +x    # Désactive le mode debug

# Arrêter le script à la première erreur
set -e

# Bonnes pratiques en début de script
set -euo pipefail
# -e = exit on error
# -u = error on undefined variable
# -o pipefail = catch errors in pipes
Codes de retour

Chaque commande retourne un code : $?
0 = succès, 1+ = erreur
exit 0 = le script se termine avec succès
exit 1 = le script se termine avec une erreur
