Module 1 — Commandes de baseFacile
Exploration du système

Sans utiliser d'interface graphique :

  1. Affiche le répertoire courant
  2. Navigue vers /etc et liste son contenu en détail
  3. Affiche les 5 premières lignes du fichier /etc/hostname
  4. Retourne à ton répertoire personnel en une commande
  5. Affiche la date, le nom d'utilisateur et le hostname
pwd
cd /etc && ls -la
head -5 /etc/hostname
cd ~
date && echo $USER && hostname

Module 2 — Manipulation de fichiersFacile
Arborescence de projet

Crée la structure suivante en une seule ligne de commande, puis vérifie avec ls -R :

projet/
├── src/
│   ├── main.py
│   └── utils.py
├── docs/
└── README.md
mkdir -p projet/{src,docs} && touch projet/src/{main.py,utils.py} projet/README.md
ls -R projet/

Astuce : Les accolades {} permettent de créer plusieurs éléments en une seule commande. C'est le brace expansion de Bash.

Module 3 — PermissionsMoyen
Sécuriser un script
  1. Crée un fichier deploy.sh
  2. Donne les permissions : propriétaire peut tout faire, groupe peut lire et exécuter, autres ne peuvent rien faire
  3. Vérifie les permissions avec ls -l
  4. Quel est le code octal correspondant ?
touch deploy.sh
chmod 750 deploy.sh
ls -l deploy.sh
-rwxr-x--- 1 sami sami 0 ... deploy.sh
# Réponse : 750 (rwx=7, r-x=5, ---=0)

Module 4 — RechercheMoyen
Audit des gros fichiers
  1. Trouve tous les fichiers de plus de 50 Mo dans /var
  2. Cherche le mot "error" dans /var/log/syslog (insensible à la casse)
  3. Compte combien de lignes contiennent le mot "error"
  4. Liste les 10 plus gros fichiers dans /var/log
sudo find /var -type f -size +50M 2>/dev/null
grep -i "error" /var/log/syslog
grep -ic "error" /var/log/syslog
sudo du -ah /var/log | sort -rh | head -10

Module 5 — ÉditeursFacile
Édition de fichier config
  1. Ouvre le fichier /etc/hostname avec Nano (en sudo)
  2. Note le nom actuel, modifie-le en "linux-test"
  3. Sauvegarde et quitte
  4. Vérifie la modification avec cat
  5. Remet le nom original

Bonus : Fais la même chose avec Vim.

# Avec Nano :
sudo nano /etc/hostname
# → Modifier → Ctrl+O → Entrée → Ctrl+X
cat /etc/hostname

# Avec Vim :
sudo vim /etc/hostname
# → i (mode insertion) → modifier → Esc → :wq

Module 6 — UtilisateursMoyen
Gestion d'une équipe

Tu dois intégrer 2 nouveaux membres dans l'équipe :

  1. Crée le groupe "webteam"
  2. Crée l'utilisateur "alice" avec home, shell bash et groupe webteam
  3. Crée l'utilisateur "bob" avec les mêmes paramètres
  4. Définis un mot de passe pour chacun
  5. Ajoute alice au groupe sudo
  6. Vérifie les groupes des deux utilisateurs
sudo groupadd webteam
sudo useradd -m -s /bin/bash -G webteam alice
sudo useradd -m -s /bin/bash -G webteam bob
sudo passwd alice
sudo passwd bob
sudo usermod -aG sudo alice
groups alice   # alice : alice webteam sudo
groups bob     # bob : bob webteam

Module 7 — RéseauMoyen
Configuration IP statique

Configure ton interface réseau en IP statique :

  • IP : 192.168.1.50
  • Masque : 255.255.255.0
  • Passerelle : 192.168.1.1
  • DNS : 8.8.8.8 et 1.1.1.1
  1. Identifie le nom de ton interface réseau
  2. Écris la configuration dans le bon fichier
  3. Applique les changements
  4. Vérifie avec ip addr et ping
ip addr   # → Repérer le nom (ex: ens33)

sudo nano /etc/network/interfaces
# Ajouter :
auto ens33
iface ens33 inet static
    address   192.168.1.50
    netmask   255.255.255.0
    gateway   192.168.1.1
    dns-nameservers 8.8.8.8 1.1.1.1

sudo systemctl restart networking
ip addr show ens33
ping -c 4 8.8.8.8

Module 7 — SSHDifficile
Sécuriser SSH
  1. Installe le serveur SSH
  2. Génère une paire de clés ed25519
  3. Change le port SSH de 22 à 2222
  4. Interdit la connexion root
  5. Redémarre le service et vérifie
sudo apt install openssh-server
ssh-keygen -t ed25519

sudo nano /etc/ssh/sshd_config
# Modifier :
Port 2222
PermitRootLogin no

sudo systemctl restart ssh
sudo ss -tulnp | grep 2222
# Pour se connecter : ssh -p 2222 user@ip

Module 8 — BashMoyen
Script de bienvenue

Écris un script welcome.sh qui :

  1. Affiche un message de bienvenue avec le nom de l'utilisateur
  2. Affiche la date et l'heure actuelles
  3. Affiche l'espace disque restant sur /
  4. Si l'utilisateur est root, affiche un avertissement
#!/bin/bash

echo "═══════════════════════════"
echo "  Bienvenue $USER !"
echo "═══════════════════════════"
echo "Date : $(date '+%d/%m/%Y %H:%M')"
echo "Disque : $(df -h / | awk 'NR==2{print $4}') disponibles"

if [[ $USER == "root" ]]; then
    echo "⚠️  ATTENTION : Tu es connecté en ROOT !"
fi

Module 8 — BashDifficile
Script de sauvegarde automatique

Crée un script backup.sh qui prend en argument un dossier source et :

  1. Vérifie que le dossier source existe (sinon, erreur)
  2. Crée un dossier ~/backups s'il n'existe pas
  3. Crée une archive tar.gz avec la date dans le nom
  4. Affiche un message de succès avec la taille de l'archive
  5. Supprime les archives de plus de 7 jours
#!/bin/bash
set -e

SOURCE="$1"
BACKUP_DIR="$HOME/backups"
DATE=$(date +%Y%m%d_%H%M%S)

# Vérifier l'argument
if [[ -z "$SOURCE" ]]; then
    echo "❌ Usage : $0 <dossier_source>"
    exit 1
fi

if [[ ! -d "$SOURCE" ]]; then
    echo "❌ Le dossier '$SOURCE' n'existe pas"
    exit 1
fi

# Créer le dossier de backup
mkdir -p "$BACKUP_DIR"

# Créer l'archive
ARCHIVE="${BACKUP_DIR}/backup_${DATE}.tar.gz"
tar -czf "$ARCHIVE" "$SOURCE"

# Afficher le résultat
TAILLE=$(du -h "$ARCHIVE" | cut -f1)
echo "✅ Sauvegarde créée : $ARCHIVE ($TAILLE)"

# Nettoyage des vieilles archives
find "$BACKUP_DIR" -name "backup_*.tar.gz" -mtime +7 -delete
echo "🧹 Archives > 7 jours supprimées"

TransversalDifficile
Mise en production d'un serveur web

Scénario complet : tu configures un serveur Debian pour héberger un site web.

  1. Configure une IP statique (192.168.1.200/24, gw 192.168.1.1)
  2. Crée un utilisateur "webmaster" dans le groupe www-data
  3. Installe Apache (apache2)
  4. Crée le dossier /var/www/monsite avec les bons droits (propriétaire webmaster:www-data, 755)
  5. Crée un fichier index.html basique
  6. Sécurise SSH (port 2222, pas de root login)
# 1. IP statique
sudo nano /etc/network/interfaces
# auto ens33 / iface ens33 inet static / address 192.168.1.200 ...
sudo systemctl restart networking

# 2. Utilisateur
sudo useradd -m -s /bin/bash -G www-data webmaster
sudo passwd webmaster

# 3. Apache
sudo apt install apache2
sudo systemctl enable apache2

# 4. Dossier site
sudo mkdir -p /var/www/monsite
sudo chown -R webmaster:www-data /var/www/monsite
sudo chmod -R 755 /var/www/monsite

# 5. index.html
echo "<h1>Mon site</h1>" | sudo tee /var/www/monsite/index.html

# 6. SSH
sudo nano /etc/ssh/sshd_config
# Port 2222 + PermitRootLogin no
sudo systemctl restart ssh

Module 8 — BashDifficile
Script de monitoring système

Crée un script monitor.sh qui :

  1. Affiche un résumé système (hostname, IP, uptime, RAM, disque)
  2. Vérifie si le service SSH est actif
  3. Si le disque est utilisé à plus de 80%, affiche une alerte
  4. Sauvegarde le rapport dans /var/log/monitor.log avec timestamp
#!/bin/bash

LOG="/var/log/monitor.log"
DATE=$(date '+%Y-%m-%d %H:%M:%S')

log() { echo "[$DATE] $1" | sudo tee -a "$LOG"; }

log "══ Rapport système ══"
log "Hostname : $(hostname)"
log "IP       : $(hostname -I | awk '{print $1}')"
log "Uptime   : $(uptime -p)"
log "RAM      : $(free -h | awk '/Mem/{print $3"/"$2}')"
log "Disque / : $(df -h / | awk 'NR==2{print $3"/"$2" ("$5")"}')"

# Vérifier SSH
if systemctl is-active --quiet ssh; then
    log "SSH : ✅ Actif"
else
    log "SSH : ❌ Inactif !"
fi

# Alerte disque
USAGE=$(df / | awk 'NR==2{print $5}' | tr -d '%')
if [[ $USAGE -gt 80 ]]; then
    log "⚠️ ALERTE : Disque utilisé à ${USAGE}% !"
fi

log "══ Fin du rapport ══"
