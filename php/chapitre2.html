L'Héritage

L'héritage est un principe fondamental de la POO qui permet à une classe d'hériter des propriétés et méthodes d'une autre classe. Cela favorise la réutilisation du code.

Principe de base

Une classe peut être une extension ou une spécialisation d'une autre classe. La classe dérivée (classe fille) hérite de toutes les méthodes et attributs de la classe de base (classe mère), sauf ceux définis comme private.

En PHP, l'héritage se fait avec le mot clé extends. PHP ne supporte que l'héritage simple (une seule classe mère).

PHP 
class Voiture { 
    protected $marque;
    
    public function affiche() {
        echo "La voiture est de marque " . $this->marque;
    }
}

class VoitureDeLuxe extends Voiture {
    // Cette classe hérite de $marque et affiche()
    public function setMarque($marque) {
        $this->marque = $marque;
    }
}

$luxe = new VoitureDeLuxe();
$luxe->setMarque("Ferrari");
$luxe->affiche(); // Affiche: La voiture est de marque Ferrari

L'instruction parent

Le mot clé parent:: permet d'accéder aux méthodes ou propriétés de la classe mère, notamment dans le cas d'une redéfinition de méthode.

PHP 
class A {
    function example() {
        echo "Je suis A::example().\n";
    }
}

class B extends A {
    function example() {
        echo "Je suis B::example().\n";
        parent::example(); // Appel de la méthode de la classe mère
    }
}

$b = new B();
$b->example();
// Affiche :
// Je suis B::example().
// Je suis A::example().

Le Polymorphisme

Le polymorphisme permet à des objets de différentes classes d'être traités de manière uniforme. En PHP, cela se traduit souvent par la redéfinition de méthodes dans les classes filles.

PHP 
class Forme {
    public function calculerSurface() {
        return 0;
    }
}

class Cercle extends Forme {
    private $rayon;
    
    public function __construct($r) {
        $this->rayon = $r;
    }
    
    // Redéfinition
    public function calculerSurface() {
        return 3.14159 * pow($this->rayon, 2);
    }
}

class Rectangle extends Forme {
    private $largeur;
    private $hauteur;
    
    public function __construct($l, $h) {
        $this->largeur = $l;
        $this->hauteur = $h;
    }
    
    // Redéfinition
    public function calculerSurface() {
        return $this->largeur * $this->hauteur;
    }
}

$formes = [new Cercle(5), new Rectangle(3, 2)];
foreach($formes as $forme) {
    echo "Surface : " . $forme->calculerSurface() . "\n";
}

Classes et Méthodes Abstraites

Une classe abstraite ne peut pas être instanciée. Elle sert de modèle pour ses classes filles. Si une classe contient au moins une méthode abstraite, elle doit être déclarée abstraite.

Une méthode abstraite déclare une signature (nom, paramètres) mais ne fournit pas d'implémentation. Les classes filles DOIVENT définir ces méthodes.

abstract class Utilisateur {
    protected $nom;
    
    // Méthode abstraite : pas de corps, juste la déclaration
    abstract public function calculerDroitAccee();
    
    public function getNom() {
        return $this->nom;
    }
}

class Admin extends Utilisateur {
    // Obligation d'implémenter la méthode abstraite
    public function calculerDroitAccee() {
        return "Accès total";
    }
}

Interfaces

Une interface est un "contrat" qui spécifie quelles méthodes une classe doit implémenter. Contrairement aux classes abstraites, une interface ne peut contenir que des signatures de méthodes (et des constantes), mais aucune implémentation.

Toutes les méthodes d'une interface doivent être publiques. Une classe peut implémenter plusieurs interfaces. 
interface GestionStock {
    public function ajouterProduit($produit);
    public function retirerProduit($id);
}

interface Exportable {
    public function toCSV();
}

// Implémentation multiple
class Magasin implements GestionStock, Exportable {
    public function ajouterProduit($produit) {
        // Code...
    }
    
    public function retirerProduit($id) {
        // Code...
    }
    
    public function toCSV() {
        // Code...
    }
}
