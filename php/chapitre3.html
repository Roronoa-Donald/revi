Inclusion de fichiers

Pour éviter de dupliquer du code, PHP permet d'inclure des fichiers dans d'autres. C'est essentiel pour organiser son code (une classe par fichier).

include vs require

  • include : Si le fichier n'est pas trouvé, génère un avertissement (E_WARNING) mais le script continue.
  • require : Si le fichier n'est pas trouvé, génère une erreur fatale (E_ERROR) et le script s'arrête.

Les variantes include_once et require_once s'assurent que le fichier n'est inclus qu'une seule fois, évitant les erreurs de redéfinition.

Autoloading

Plutôt que d'écrire des dizaines de require en début de fichier, l'autoloading permet de charger automatiquement les classes quand elles sont utilisées.

PHP 
// Ancienne méthode (dépréciée en PHP 7.2+, supprimée en 8.0)
function __autoload($class_name) {
    require 'classes/' . $class_name . '.php';
}

// Nouvelle méthode recommandée : spl_autoload_register
spl_autoload_register(function ($class_name) {
    echo "Tentative de chargement de la classe : " . $class_name . "\n";
    // include 'classes/' . $class_name . '.php';
});

$obj = new MaClasse(); // Le fichier classes/MaClasse.php est inclus automatiquement

Namespaces (Espaces de noms)

Les Namespaces permettent de résoudre les conflits de noms entre classes (ex: deux classes User dans deux bibliothèques différentes) et d'organiser le code.

PHP 
namespace MonProjet\Model;

class User {
    public function direBonjour() {
        echo "Bonjour depuis le Namespace MonProjet\\Model !";
    }
}

// Simulation de l'utilisation
$u = new User();
$u->direBonjour();

Utilisation avec use

Pour utiliser une classe située dans un namespace, on l'importe avec use, ou on utilise son chemin complet (Fully Qualified Name).

use MonProjet\Model\User;

$u = new User(); // Utilise MonProjet\Model\User

// Ou avec un alias
use MonProjet\Model\User as Utilisateur;
$u = new Utilisateur();

Les Traits

PHP ne supporte pas l'héritage multiple. Les Traits sont un mécanisme de réutilisation de code qui permet de composer des classes avec des ensembles de méthodes, contournant cette limitation.

Un Trait ressemble à une classe mais ne peut pas être instancié.

trait Loggable {
    public function log($msg) {
        echo "Log: $msg";
    }
}

class User {
    use Loggable; // Injection des méthodes du trait
}

class Product {
    use Loggable;
}

$u = new User();
$u->log("Création utilisateur"); // Affiche "Log: Création utilisateur"

Exercices Pratiques

Exercice 1 : Autoload

Créez une structure de dossier avec classes/Client.php et index.php. Dans index.php, mettez en place un autoloader avec spl_autoload_register pour charger la classe Client automatiquement lors de son instanciation.

Exercice 2 : Namespace

Placez la classe Client dans le namespace App\Entity. Essayez de l'instancier dans index.php en utilisant le mot clé use.
