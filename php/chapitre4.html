1. Principe des Exceptions

Les exceptions sont le mécanisme standard de la POO pour gérer les erreurs. L'objectif est de pouvoir déceler une erreur, interrompre le flux normal, et traiter l'erreur proprement sans arrêter brutalement le script (comme le ferait une Error Fatale).

En PHP, les exceptions sont représentées par la classe Exception.

2. Le bloc Try ... Catch

Pour gérer une exception potentielle, on place le code "à risque" dans un bloc try. Si une erreur survient, le code s'arrête et saute directement au bloc catch.

PHP 
try {
    // Code qui peut échouer
    echo "Début du traitement...\n";
    $division = 10 / 0; // Erreur !
    echo "Ceci ne sera pas affiché.";
} catch (Exception $e) {
    // Code exécuté en cas d'erreur
    echo "Une erreur est survenue : Division par zéro impossible.";
} finally {
    echo "\nFin du bloc.";
}

3. Lancer une exception (Throw)

Vous pouvez déclencher manuellement une exception avec le mot clé throw. C'est utile pour valider des données.

PHP 
function verifierAge($age) {
    if ($age < 18) {
        throw new Exception("Vous devez être majeur.");
    }
    return "Accès autorisé.";
}

try {
    echo verifierAge(15);
} catch (Exception $e) {
    echo "Erreur : " . $e->getMessage(); 
}

4. Exceptions Personnalisées

On peut créer ses propres types d'erreurs en héritant de la classe Exception. Cela permet de différencier les types d'erreurs dans plusieurs blocs catch.

PHP 
class MonException extends Exception {
    public function afficherErreur() {
        return "ERREUR [MON_EXCEPTION] : " . $this->getMessage();
    }
}

try {
    throw new MonException("Oups !");
} catch (MonException $e) {
    echo $e->afficherErreur();
} catch (Exception $e) {
    echo "Erreur générique.";
}
