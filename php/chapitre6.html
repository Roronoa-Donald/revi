1. Architecture MVC

Le pattern MVC (Modèle - Vue - Contrôleur) est l'architecture la plus répandue pour les applications web.

  • Modèle : Gère les données et la logique métier (accès BDD, validations).
  • Vue : Gère l'affichage (HTML, templates). Ne contient aucune logique métier.
  • Contrôleur : Fait le lien — reçoit la requête, appelle le modèle, passe les données à la vue.

Flux d'une requête MVC

1. Navigateur → /articles?id=5
2. Routeur    → ArticleController::show(5)
3. Controller → Article::find(5)        (appelle le Modèle)
4. Modèle     → SELECT * FROM articles WHERE id = 5
5. Controller → render('article.php', $data) (passe à la Vue)
6. Vue        → HTML final envoyé au navigateur

Structure de dossiers

// Structure type d'un projet MVC
projet/
├── public/           // Point d'entrée (index.php)
│   └── index.php
├── src/
│   ├── Controller/   // Contrôleurs
│   │   └── ArticleController.php
│   ├── Model/        // Modèles
│   │   └── Article.php
│   └── View/         // Vues (templates)
│       ├── layout.php
│       └── article/
│           ├── list.php
│           └── show.php
├── config/
│   └── database.php
└── vendor/           // Dépendances Composer

Mnémotechnique MVC

🎯 « M = Données, V = Écran, C = Chef d'orchestre »

  • Le Modèle est le cuisinier (prépare les données)
  • La Vue est le serveur (présente le plat au client)
  • Le Contrôleur est le chef (décide quel plat préparer)

2. Le Routeur

Le routeur est le point d'entrée de l'application (souvent public/index.php). Il analyse l'URL et appelle le bon contrôleur.

PHP 
// Routeur simple avec tableau de routes
$routes = [
    '/'         => ['HomeController', 'index'],
    '/articles' => ['ArticleController', 'list'],
    '/contact'  => ['ContactController', 'show'],
];

$uri = parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH);

if (isset($routes[$uri])) {
    [$controller, $method] = $routes[$uri];
    echo "Route : $uri → {$controller}::{$method}()\n";
    // $ctrl = new $controller();
    // $ctrl->$method();
} else {
    http_response_code(404);
    echo "Page non trouvée !";
}

Exemple de Contrôleur

class ArticleController {
    private $model;
    
    public function __construct() {
        $this->model = new Article(); // Le modèle
    }
    
    public function list() {
        $articles = $this->model->getAll(); // Appel au modèle
        require 'src/View/article/list.php'; // Rendu de la vue
    }
    
    public function show(int $id) {
        $article = $this->model->find($id);
        if (!$article) {
            http_response_code(404);
            require 'src/View/errors/404.php';
            return;
        }
        require 'src/View/article/show.php';
    }
}

Exemple de Modèle

class Article {
    private PDO $db;
    
    public function __construct() {
        $this->db = Database::getInstance(); // Singleton
    }
    
    public function getAll(): array {
        $stmt = $this->db->query('SELECT * FROM articles ORDER BY created_at DESC');
        return $stmt->fetchAll(PDO::FETCH_ASSOC);
    }
    
    public function find(int $id): ?array {
        $stmt = $this->db->prepare('SELECT * FROM articles WHERE id = ?');
        $stmt->execute([$id]);
        return $stmt->fetch(PDO::FETCH_ASSOC) ?: null;
    }
}

3. Le pattern Singleton (BDD)

Le Singleton garantit qu'une seule connexion à la base de données existe dans toute l'application :

class Database {
    private static ?PDO $instance = null;
    
    // Constructeur privé → empêche new Database()
    private function __construct() {}
    
    public static function getInstance(): PDO {
        if (self::$instance === null) {
            self::$instance = new PDO(
                'mysql:host=localhost;dbname=test;charset=utf8mb4',
                'root', '',
                [PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION]
            );
        }
        return self::$instance;
    }
}

Erreurs courantes MVC

  • Mettre du SQL dans la Vue → toujours passer par le Modèle !
  • Mettre du HTML dans le Contrôleur → c'est le rôle de la Vue !
  • Contrôleur trop « gras » → déplacez la logique métier dans le Modèle ou un Service
