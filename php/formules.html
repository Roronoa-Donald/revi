Classes & Objets

Classe de base

class User {
    public string $nom;
    private int $age;
    
    public function __construct(string $nom, int $age) {
        $this->nom = $nom;
        $this->age = $age;
    }
    
    public function getAge(): int {
        return $this->age;
    }
}

$u = new User("Alice", 25);

Visibilité

publicAccessible partout
protectedClasse + enfants
privateClasse uniquement

🎯 Règle : toujours private par défaut, protected si héritage prévu.

Méthodes Magiques

  • __construct() — Initialisation
  • __destruct() — Nettoyage
  • __toString() — Conversion en string
  • __get/$__set() — Accesseurs magiques
  • __call() — Méthode inexistante
  • __clone() — Clonage

Static & Constantes

class Config {
    const VERSION = '2.0';
    private static int $count = 0;
    
    public static function getCount(): int {
        return self::$count;
    }
}

echo Config::VERSION;    // 2.0
echo Config::getCount(); // 0

⚠️ self:: = classe courante, static:: = late binding

Héritage & Polymorphisme

Héritage

class Admin extends User {
    private array $permissions;
    
    public function __construct(string $nom, int $age) {
        parent::__construct($nom, $age);
        $this->permissions = ['all'];
    }
}

✅ extends = héritage simple (1 seul parent)
✅ parent:: = appeler la méthode parente

Traits

trait Loggable {
    public function log(string $msg): void {
        echo "[LOG] $msg";
    }
}

class Service {
    use Loggable;
}

Traits = réutilisation horizontale de code (pas d'héritage multiple en PHP).

Erreurs & Base de Données

Exceptions

try {
    // code risqué
    throw new \InvalidArgumentException("Oups");
} catch (InvalidArgumentException $e) {
    echo $e->getMessage();
} catch (Exception $e) {
    // fallback général
} finally {
    // toujours exécuté
}

💡 Du plus spécifique au plus général dans les catch.

PDO — Connexion

$pdo = new PDO(
    'mysql:host=localhost;dbname=test;charset=utf8mb4',
    'root', '',
    [PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
     PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC]
);

⚠️ Toujours ERRMODE_EXCEPTION + charset=utf8mb4

Requêtes Préparées

// Marqueurs positionnels
$stmt = $pdo->prepare('SELECT * FROM users WHERE id = ?');
$stmt->execute([5]);

// Marqueurs nommés
$stmt = $pdo->prepare('INSERT INTO users (nom, email) 
    VALUES (:nom, :email)');
$stmt->execute([':nom' => 'Alice', ':email' => 'a@b.com']);

🔒 JAMAIS de concaténation → injection SQL !

CRUD rapide

  • Create : INSERT INTO ... VALUES (...)
  • Read : SELECT * FROM ... WHERE ...
  • Update : UPDATE ... SET ... WHERE ...
  • Delete : DELETE FROM ... WHERE ...

$pdo->lastInsertId() → récupérer l'ID auto-incrémenté

fetch() = 1 ligne | fetchAll() = toutes

Architecture & Patterns

MVC

  • Modèle → données & logique métier (SQL)
  • Vue → affichage HTML (pas de SQL !)
  • Contrôleur → reçoit requête, orchestre M et V

🎯 Règle : pas de SQL dans la Vue, pas de HTML dans le Contrôleur.

Singleton

class DB {
    private static ?PDO $instance = null;
    private function __construct() {}
    
    public static function get(): PDO {
        if (self::$instance === null)
            self::$instance = new PDO(...);
        return self::$instance;
    }
}

1 seule connexion BDD dans toute l'app.

PHP 8 — Nouveautés clés

  • match — switch amélioré avec retour
  • ?-> — null-safe operator
  • Named arguments : foo(y: 2, x: 1)
  • Union types : int|string
  • Enum : enum Color { case Red; }
  • Constructor promotion : public function __construct(private string $nom)

Pièges d'examen

  • ❌ new AbstractClass() → Fatal Error
  • ❌ Interface avec propriétés → Interdit
  • ❌ Oublier parent::__construct() dans l'enfant
  • ❌ catch(Exception $e) {} → bug silencieux
  • ❌ $bdd->query("... '$var' ...") → injection SQL
  • ✅ $this->nom (pas $this.nom)
