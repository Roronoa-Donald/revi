TP 3 : Encapsulation — Protéger les données

L'encapsulation, c'est cacher et protéger les attributs d'une classe. C'est le concept le plus testé à l'examen du prof !

🔒 Analogie : Un distributeur automatique : tu ne peux pas mettre la main dans la machine pour prendre un café. Tu passes par les boutons (getters/setters). Le mécanisme interne est caché (private).
1

Le problème avec public

❌ Avec public — dangereux
$livre = new Livre("PHP", "Jean", 300);
$livre->nbPages = -50;  // Accepté !
$livre->nbPages = "abc"; // Accepté !
// Aucun contrôle, n'importe quelle valeur passe
✅ Avec private + setter — sécurisé
$livre = new Livre("PHP", "Jean", 300);
$livre->setNbPages(-50);  // Refusé !
$livre->setNbPages(400);  // Accepté ✓
// Le setter vérifie la valeur avant de l'accepter

Micro-exercice 3.1 — Identifier le danger

Regarde ce code. Qu'est-ce qui ne va pas ?

class CompteBancaire {
    public $solde;
    public function __construct($solde) { $this->solde = $solde; }
}
$compte = new CompteBancaire(1000);
$compte->solde = -999999; // ???

Correction

Le problème : $solde est public, donc n'importe qui peut le modifier directement sans contrôle. On pourrait mettre un solde négatif, ce qui est incohérent pour un compte bancaire.

Solution : rendre $solde private et fournir un setter avec validation.

2

Mnémonique : public = Tout le monde, protected = Famille (héritage), private = Moi seul

Micro-exercice 3.2 — Passer en private

Reprends la classe Livre du TP2 et change public $nbPages en private $nbPages. Essaie d'accéder à $livre->nbPages depuis l'extérieur. Que se passe-t-il ?

Correction

<?php
class Livre {
    public $titre;
    public $auteur;
    private $nbPages;   // ← Changé en private

    public function __construct($titre, $auteur, $nbPages) {
        $this->titre = $titre;
        $this->auteur = $auteur;
        $this->nbPages = $nbPages;   // OK : on est DANS la classe
    }
}

$livre = new Livre("PHP", "Jean", 300);
echo $livre->titre;     // ✅ OK (public)
echo $livre->nbPages;   // ❌ ERREUR ! Cannot access private property
?>
💡 C'est exactement ça l'encapsulation :
L'attribut private est inaccessible depuis l'extérieur de la classe. On ne peut y accéder que via des méthodes publiques (getters/setters) qui contrôlent l'accès.
3

Getters et Setters

Getter = méthode pour lire un attribut private → getNbPages()
Setter = méthode pour modifier un attribut private avec validation → setNbPages($pages)

Classe Livre avec encapsulation complète
<?php
class Livre {
    private $titre;      // Tous les attributs sont private
    private $auteur;
    private $nbPages;

    public function __construct($titre, $auteur, $nbPages) {
        $this->titre = $titre;
        $this->auteur = $auteur;
        $this->setNbPages($nbPages);  // Utilise le setter même dans le constructeur !
    }

    // --- GETTERS (lecture seule) ---
    public function getTitre()   { return $this->titre; }
    public function getAuteur()  { return $this->auteur; }
    public function getNbPages() { return $this->nbPages; }

    // --- SETTER avec VALIDATION ---
    public function setNbPages($pages) {
        if (!is_int($pages) || $pages <= 0) {
            echo "Erreur : le nombre de pages doit être un entier positif !\n";
            return;  // On sort sans modifier
        }
        $this->nbPages = $pages;  // Modification acceptée
    }

    public function afficherInfos() {
        echo "Titre : {$this->titre}, Auteur : {$this->auteur}, Pages : {$this->nbPages}\n";
    }
}

$livre = new Livre("PHP POO", "Jean", 300);
echo $livre->getTitre();      // ✅ "PHP POO"
echo $livre->getNbPages();    // ✅ 300

$livre->setNbPages(450);      // ✅ Accepté
$livre->setNbPages(-10);      // ❌ "Erreur : le nombre de pages..."
$livre->setNbPages("abc");    // ❌ "Erreur : le nombre de pages..."

echo $livre->getNbPages();    // 450 (la valeur invalide n'a pas été enregistrée)
?>

Micro-exercice 3.3 — Encapsuler CompteBancaire

Reprends la classe CompteBancaire. Rends $solde private. Crée :
- un getter getSolde()
- un setter setSolde($montant) qui refuse les valeurs négatives
- une méthode deposer($montant) qui ajoute au solde (si montant > 0)
- une méthode retirer($montant) qui retire du solde (si montant > 0 ET solde suffisant)

Correction

<?php
class CompteBancaire {
    private $solde;   // private = accessible uniquement dans la classe

    public function __construct($solde) {
        $this->setSolde($solde);  // On passe par le setter pour valider
    }

    // Getter : permet de LIRE le solde
    public function getSolde() {
        return $this->solde;
    }

    // Setter : permet de MODIFIER avec validation
    public function setSolde($montant) {
        if ($montant < 0) {
            echo "Erreur : le solde ne peut pas être négatif.\n";
            return;
        }
        $this->solde = $montant;
    }

    // Méthode métier : déposer de l'argent
    public function deposer($montant) {
        if ($montant <= 0) {
            echo "Erreur : le montant doit être positif.\n";
            return;
        }
        $this->solde += $montant;  // On modifie directement (on est dans la classe)
        echo "Dépôt de $montant€. Nouveau solde : {$this->solde}€\n";
    }

    // Méthode métier : retirer de l'argent
    public function retirer($montant) {
        if ($montant <= 0) {
            echo "Erreur : le montant doit être positif.\n";
            return;
        }
        if ($montant > $this->solde) {
            echo "Erreur : solde insuffisant ({$this->solde}€).\n";
            return;
        }
        $this->solde -= $montant;
        echo "Retrait de $montant€. Nouveau solde : {$this->solde}€\n";
    }
}

$compte = new CompteBancaire(1000);
echo $compte->getSolde();     // 1000
$compte->deposer(500);        // Dépôt de 500€. Nouveau solde : 1500€
$compte->retirer(200);        // Retrait de 200€. Nouveau solde : 1300€
$compte->retirer(5000);       // Erreur : solde insuffisant (1300€).
$compte->deposer(-100);       // Erreur : le montant doit être positif.
?>
💡 Schéma d'encapsulation :
Extérieur → setter (vérifie) → attribut private
Attribut private → getter (lit) → Extérieur
Les méthodes deposer() et retirer() sont des méthodes métier : elles font des opérations qui ont du sens dans le contexte du problème, tout en protégeant les données.

Exercice de Synthèse — Encapsulation du Livre 5 pts

Contexte : On veut sécuriser la gestion du nombre de pages d'un livre. (Examen 2024/2025 – Exercice 2)

  1. (2 pts) Modifiez la classe Livre pour que l'attribut nbPages soit privé.
  2. (2 pts) Ajoutez une méthode setNbPages($pages) qui permet de modifier nbPages uniquement si la valeur est un entier strictement positif.
  3. (1 pt) Testez cette méthode avec des valeurs valides et invalides.

Correction détaillée — Épreuve réelle !

<?php
// ===========================================================
// QUESTION 1 (2 pts) : Attribut privé
// ===========================================================
class Livre {
    public $titre;
    public $auteur;
    private $nbPages;     // ← private au lieu de public

    public function __construct($titre, $auteur, $nbPages) {
        $this->titre = $titre;
        $this->auteur = $auteur;
        // On utilise le setter pour valider dès la construction
        $this->setNbPages($nbPages);
    }

    // Getter pour lire nbPages (pas demandé mais bonne pratique)
    public function getNbPages() {
        return $this->nbPages;
    }

    // ===========================================================
    // QUESTION 2 (2 pts) : Setter avec validation
    // ===========================================================
    public function setNbPages($pages) {
        // Condition : entier ET strictement positif
        if (is_int($pages) && $pages > 0) {
            $this->nbPages = $pages;
        } else {
            echo "Erreur : nbPages doit être un entier strictement positif.\n";
        }
    }
    // Points clés :
    // - is_int() vérifie que c'est bien un entier (pas "abc", pas 3.5)
    // - > 0 vérifie que c'est strictement positif (pas 0, pas -5)

    public function afficherInfos() {
        echo "Titre : {$this->titre}, Auteur : {$this->auteur}, ";
        echo "Pages : {$this->nbPages}\n";
    }
}

// ===========================================================
// QUESTION 3 (1 pt) : Tests valides et invalides
// ===========================================================
$livre = new Livre("PHP POO", "Jean Dupont", 300);
$livre->afficherInfos();
// Titre : PHP POO, Auteur : Jean Dupont, Pages : 300

// Test valide
$livre->setNbPages(450);
echo "Après modification : " . $livre->getNbPages() . " pages\n"; // 450

// Tests invalides
$livre->setNbPages(-10);    // Erreur : nbPages doit être un entier strictement positif.
$livre->setNbPages(0);      // Erreur : nbPages doit être un entier strictement positif.
$livre->setNbPages("abc");  // Erreur : nbPages doit être un entier strictement positif.
$livre->setNbPages(3.5);    // Erreur : nbPages doit être un entier strictement positif.

// Vérifier que la valeur n'a pas changé
echo "Toujours : " . $livre->getNbPages() . " pages\n"; // 450
?>
💡 Barème du prof :
✅ Q1 : private au lieu de public (1 pt) + constructeur correct (1 pt)
✅ Q2 : is_int() (1 pt) + > 0 + affectation conditionnelle (1 pt)
✅ Q3 : Au moins 1 test valide + 1 test invalide (1 pt)
Piège fréquent : écrire >= 0 au lieu de > 0 (le prof dit "strictement positif" = pas zéro !)
