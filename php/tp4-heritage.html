Héritage avec extends

Classe parent → Classe enfant
<?php
// CLASSE PARENT (ou classe "mère")
class Livre {
    private $titre;
    private $auteur;
    private $nbPages;

    public function __construct($titre, $auteur, $nbPages) {
        $this->titre = $titre;
        $this->auteur = $auteur;
        $this->nbPages = $nbPages;
    }

    public function getTitre()   { return $this->titre; }
    public function getAuteur()  { return $this->auteur; }
    public function getNbPages() { return $this->nbPages; }

    public function afficherInfos() {
        echo "Titre : {$this->titre}, Auteur : {$this->auteur}, ";
        echo "Pages : {$this->nbPages}\n";
    }
}

// CLASSE ENFANT : hérite de TOUT ce que Livre a de public/protected
class LivreElectronique extends Livre {
    private $tailleFichier;  // Nouvel attribut spécifique

    // Le constructeur appelle celui du parent avec parent::
    public function __construct($titre, $auteur, $nbPages, $tailleFichier) {
        parent::__construct($titre, $auteur, $nbPages);  // ← CRUCIAL
        $this->tailleFichier = $tailleFichier;
    }

    public function getTailleFichier() {
        return $this->tailleFichier;
    }
}

// Utilisation
$ebook = new LivreElectronique("PHP POO", "Jean", 300, 5.2);
$ebook->afficherInfos();    // ✅ Héritée du parent !
// Titre : PHP POO, Auteur : Jean, Pages : 300

echo $ebook->getTailleFichier(); // 5.2 (propre à l'enfant)
?>
⚠️ Piège #1 : Si tu oublies parent::__construct(...), les attributs du parent ne seront PAS initialisés !
⚠️ Piège #2 : L'enfant n'a PAS accès aux attributs private du parent. Il doit passer par les getters. Pour donner accès aux enfants, utilise protected.

Micro-exercice 4.1 — Créer LivreElectronique

En partant de la classe Livre ci-dessus, crée une classe LivreElectronique qui hérite de Livre et ajoute un attribut $tailleFichier (en Mo). N'oublie pas le constructeur avec parent::__construct().

Correction

<?php
class LivreElectronique extends Livre {
    private $tailleFichier;  // en Mo

    public function __construct($titre, $auteur, $nbPages, $tailleFichier) {
        // 1. Appeler le constructeur du parent pour initialiser titre, auteur, nbPages
        parent::__construct($titre, $auteur, $nbPages);
        // 2. Initialiser notre propre attribut
        $this->tailleFichier = $tailleFichier;
    }

    public function getTailleFichier() {
        return $this->tailleFichier;
    }
}

$ebook = new LivreElectronique("PHP Avancé", "Marie", 250, 3.8);
echo $ebook->getTitre();          // "PHP Avancé" (hérité)
echo $ebook->getTailleFichier();  // 3.8 (propre)
?>
💡 Schéma mental :
LivreElectronique = tout ce que Livre sait faire + ses propres ajouts.
Le mot-clé extends = "étendre" la classe parent avec de nouvelles fonctionnalités.

Redéfinition de méthode (surcharge)

Redéfinir = réécrire une méthode du parent dans l'enfant pour changer son comportement. C'est ça le polymorphisme : le même nom de méthode, un comportement différent.

Redéfinir afficherInfos() dans LivreElectronique
<?php
class LivreElectronique extends Livre {
    private $tailleFichier;

    public function __construct($titre, $auteur, $nbPages, $tailleFichier) {
        parent::__construct($titre, $auteur, $nbPages);
        $this->tailleFichier = $tailleFichier;
    }

    // REDÉFINITION : même nom que dans le parent, mais comportement différent
    public function afficherInfos() {
        // 1. Appeler la version du parent
        parent::afficherInfos();
        // 2. Ajouter nos propres informations
        echo "Taille du fichier : {$this->tailleFichier} Mo\n";
    }

    public function getTailleFichier() {
        return $this->tailleFichier;
    }
}

$livre  = new Livre("PHP Bases", "Jean", 300);
$ebook  = new LivreElectronique("PHP Avancé", "Marie", 250, 3.8);

$livre->afficherInfos();
// Titre : PHP Bases, Auteur : Jean, Pages : 300

$ebook->afficherInfos();
// Titre : PHP Avancé, Auteur : Marie, Pages : 250
// Taille du fichier : 3.8 Mo    ← AJOUTÉ par la redéfinition
?>

Micro-exercice 4.2 — Redéfinir afficherInfos()

Redéfinis la méthode afficherInfos() dans LivreElectronique pour qu'elle affiche aussi la taille du fichier. Utilise parent::afficherInfos() pour ne pas réécrire le code du parent.

Correction

<?php
// Dans la classe LivreElectronique :
public function afficherInfos() {
    parent::afficherInfos();  // Réutilise le code du parent (DRY!)
    echo "Taille : {$this->tailleFichier} Mo\n";
}

// Test
$ebook = new LivreElectronique("C# Guide", "Paul", 400, 7.5);
$ebook->afficherInfos();
// Titre : C# Guide, Auteur : Paul, Pages : 400
// Taille : 7.5 Mo
?>
💡 parent:: vs $this-> :
• parent:: = appeler une méthode/constructeur de la classe parent
• $this-> = accéder à un attribut/méthode de l'objet courant
Le prof vérifie que tu utilises parent::afficherInfos() plutôt que de dupliquer le code.

Classes et méthodes abstraites

Une classe abstraite est un "moule incomplet" : on ne peut pas l'instancier directement. Elle force les enfants à implémenter certaines méthodes.

Classe abstraite Forme
<?php
// abstract = on ne peut PAS faire new Forme()
abstract class Forme {
    protected $couleur;

    public function __construct($couleur) {
        $this->couleur = $couleur;
    }

    // Méthode abstraite = PAS de corps, les enfants DOIVENT la définir
    abstract public function calculerSurface();

    // Méthode normale (héritée telle quelle)
    public function getCouleur() {
        return $this->couleur;
    }
}

class Cercle extends Forme {
    private $rayon;

    public function __construct($couleur, $rayon) {
        parent::__construct($couleur);
        $this->rayon = $rayon;
    }

    // OBLIGÉ d'implémenter calculerSurface()
    public function calculerSurface() {
        return M_PI * $this->rayon ** 2;  // π × r²
    }
}

class Rectangle extends Forme {
    private $largeur;
    private $hauteur;

    public function __construct($couleur, $largeur, $hauteur) {
        parent::__construct($couleur);
        $this->largeur = $largeur;
        $this->hauteur = $hauteur;
    }

    public function calculerSurface() {
        return $this->largeur * $this->hauteur;
    }
}

// $f = new Forme("rouge");  // ❌ ERREUR ! Classe abstraite
$c = new Cercle("bleu", 5);
$r = new Rectangle("vert", 4, 6);

echo $c->calculerSurface();  // 78.54 (π × 25)
echo $r->calculerSurface();  // 24 (4 × 6)
?>

Micro-exercice 4.3 — Classe abstraite Animal

Crée une classe abstraite Animal avec un attribut $nom et une méthode abstraite crier(). Crée Chien (crier = "Wouf!") et Chat (crier = "Miaou!") qui en héritent.

Correction

<?php
abstract class Animal {
    protected $nom;   // protected : accessible par les enfants

    public function __construct($nom) {
        $this->nom = $nom;
    }

    // Méthode abstraite : chaque animal crie différemment
    abstract public function crier();

    public function sePresenter() {
        echo "Je suis {$this->nom} et je fais : ";
        $this->crier();  // Polymorphisme : chaque animal a sa propre version
    }
}

class Chien extends Animal {
    public function crier() {
        echo "Wouf !\n";
    }
}

class Chat extends Animal {
    public function crier() {
        echo "Miaou !\n";
    }
}

$rex = new Chien("Rex");
$felix = new Chat("Félix");

$rex->sePresenter();   // "Je suis Rex et je fais : Wouf !"
$felix->sePresenter(); // "Je suis Félix et je fais : Miaou !"
?>
💡 Polymorphisme en action :
sePresenter() appelle $this->crier(). Selon que l'objet est un Chien ou un Chat, c'est la bonne version de crier() qui s'exécute. C'est ça le polymorphisme : même appel, comportement différent.

Interfaces

Une interface = un contrat. "Si tu implémentes cette interface, tu DOIS avoir ces méthodes."
Différence avec abstract : une classe peut implémenter plusieurs interfaces, mais ne peut hériter que d'une seule classe.

Interface et implements
<?php
// Interface = contrat (que des signatures de méthodes, pas de corps)
interface Affichable {
    public function afficher();  // Pas de corps ! Juste la signature
}

interface Exportable {
    public function exporter($format);
}

// Une classe peut implémenter PLUSIEURS interfaces
class Rapport implements Affichable, Exportable {
    private $contenu;

    public function __construct($contenu) {
        $this->contenu = $contenu;
    }

    // OBLIGÉ d'implémenter afficher() (contrat Affichable)
    public function afficher() {
        echo $this->contenu;
    }

    // OBLIGÉ d'implémenter exporter() (contrat Exportable)
    public function exporter($format) {
        return "[$format] $this->contenu";
    }
}

$r = new Rapport("Résultats du semestre");
$r->afficher();               // "Résultats du semestre"
echo $r->exporter("PDF");     // "[PDF] Résultats du semestre"
?>

Mot-cléextendsimplements
Héritage multiple ?❌ 1 seule✅ Plusieurs
Attributs ?✅ Oui❌ Non
Corps de méthode ?✅ Oui (mixte)❌ Que des signatures
Logique"JE SUIS un type""JE SAIS FAIRE"

Micro-exercice 4.4 — Interface FormeGeometrique

Crée une interface FormeGeometrique avec les méthodes calculerSurface() et calculerPerimetre(). Crée une classe Carre qui implémente cette interface (côté en paramètre).

Correction

<?php
interface FormeGeometrique {
    public function calculerSurface();
    public function calculerPerimetre();
}

class Carre implements FormeGeometrique {
    private $cote;

    public function __construct($cote) {
        $this->cote = $cote;
    }

    public function calculerSurface() {
        return $this->cote ** 2;  // côté²
    }

    public function calculerPerimetre() {
        return 4 * $this->cote;   // 4 × côté
    }
}

$c = new Carre(5);
echo "Surface : " . $c->calculerSurface();    // 25
echo "Périmètre : " . $c->calculerPerimetre(); // 20
?>

Exercice de Synthèse — Héritage LivreElectronique 5 pts

Contexte : On veut étendre notre système avec des livres électroniques. (Examen 2024/2025 – Exercice 3)

  1. (2 pts) Créez une classe LivreElectronique qui hérite de Livre et ajoute un nouvel attribut tailleFichier (en Mo).
  2. (2 pts) Redéfinissez la méthode afficherInfos() pour inclure la taille du fichier dans l'affichage.
  3. (1 pt) Instanciez un objet LivreElectronique et testez l'affichage.

Correction détaillée — Épreuve réelle !

<?php
// Classe parent (du TP2/TP3)
class Livre {
    private $titre;
    private $auteur;
    private $nbPages;

    public function __construct($titre, $auteur, $nbPages) {
        $this->titre = $titre;
        $this->auteur = $auteur;
        $this->nbPages = $nbPages;
    }

    public function getTitre() { return $this->titre; }
    public function getAuteur() { return $this->auteur; }
    public function getNbPages() { return $this->nbPages; }

    public function afficherInfos() {
        echo "Titre : {$this->titre}, ";
        echo "Auteur : {$this->auteur}, ";
        echo "Nombre de pages : {$this->nbPages}\n";
    }
}

// ===========================================================
// QUESTION 1 (2 pts) : Héritage + nouveau attribut
// ===========================================================
class LivreElectronique extends Livre {
    private $tailleFichier;  // en Mo — spécifique aux ebooks

    public function __construct($titre, $auteur, $nbPages, $tailleFichier) {
        // parent::__construct() = appeler le constructeur de Livre
        parent::__construct($titre, $auteur, $nbPages);
        $this->tailleFichier = $tailleFichier;
    }
    // Barème : extends (0.5pt), parent::__construct (0.5pt),
    // nouveau attribut (0.5pt), constructeur complet (0.5pt)

    public function getTailleFichier() {
        return $this->tailleFichier;
    }

    // ===========================================================
    // QUESTION 2 (2 pts) : Redéfinition de afficherInfos()
    // ===========================================================
    public function afficherInfos() {
        // Appeler la version du parent (ne pas réécrire le code !)
        parent::afficherInfos();
        // Ajouter l'info spécifique
        echo "Taille du fichier : {$this->tailleFichier} Mo\n";
    }
    // Barème : parent::afficherInfos() (1pt),
    // affichage tailleFichier correct (1pt)
}

// ===========================================================
// QUESTION 3 (1 pt) : Instanciation et test
// ===========================================================
$ebook = new LivreElectronique(

// Titre : Apprendre PHP, Auteur : Jean Dupont, Nombre de pages : 280
