TP 5 : Try, Catch, Throw & Exceptions

L'exercice 4 de l'examen porte sur la gestion d'erreurs. On apprend à gérer les erreurs proprement au lieu de crasher.

🎯 Mnémonique : try = ESSAIE ce code | catch = ATTRAPE l'erreur si ça rate | finally = TOUJOURS exécuté (qu'il y ait erreur ou pas) | throw = LANCE une erreur volontairement

1

try / catch — Attraper les erreurs

Syntaxe de base
<?php
// SANS try/catch : le programme CRASHE
// echo 10 / 0;  // Warning: Division by zero

// AVEC try/catch : on gère l'erreur proprement
try {
    // Le code qui POURRAIT causer une erreur
    $resultat = 10 / 0;
    echo "Résultat : $resultat\n";
} catch (DivisionByZeroError $e) {
    // Ce bloc s'exécute SI une erreur survient dans le try
    echo "Erreur attrapée : " . $e->getMessage() . "\n";
}

echo "Le programme continue normalement !\n";
// Sans try/catch, on n'arriverait jamais ici
?>

Micro-exercice 5.1 — Premier try/catch

Écris une fonction diviser($a, $b) qui divise a par b. Utilise un try/catch pour attraper l'erreur si b vaut 0.

Correction

<?php
function diviser($a, $b) {
    try {
        if ($b == 0) {
            // On LANCE une exception manuellement
            throw new Exception("Division par zéro impossible !");
        }
        return $a / $b;
    } catch (Exception $e) {
        // $e->getMessage() contient notre message d'erreur
        echo "Erreur : " . $e->getMessage() . "\n";
        return null;  // On retourne null en cas d'erreur
    }
}

echo diviser(10, 2);  // 5
echo diviser(10, 0);  // "Erreur : Division par zéro impossible !"
echo diviser(15, 3);  // 5 (le programme continue !)
?>
💡 Pourquoi c'est mieux que "if" ?
Avec if ($b == 0) { echo "erreur"; }, tu gères UN cas. Avec try/catch, tu gères tous les cas d'erreur possibles d'un bloc de code. Et surtout, le programme ne crashe jamais.
2

throw — Lancer une exception

throw permet de déclencher volontairement une exception. Utile pour valider des données et signaler les problèmes.

throw + finally
<?php
function inscrireEtudiant($nom, $age) {
    try {
        // Validation : on LANCE des exceptions si les données sont invalides
        if (empty($nom)) {
            throw new Exception("Le nom ne peut pas être vide !");
        }
        if ($age < 16 || $age > 65) {
            throw new Exception("L'âge doit être entre 16 et 65 ans !");
        }

        echo "Étudiant $nom ($age ans) inscrit avec succès.\n";
        return true;

    } catch (Exception $e) {
        echo "Inscription refusée : " . $e->getMessage() . "\n";
        return false;

    } finally {
        // TOUJOURS exécuté, même après un return !
        echo "--- Fin de la procédure d'inscription ---\n";
    }
}

inscrireEtudiant("Alice", 21);
// Étudiant Alice (21 ans) inscrit avec succès.
// --- Fin de la procédure d'inscription ---

inscrireEtudiant("", 21);
// Inscription refusée : Le nom ne peut pas être vide !
// --- Fin de la procédure d'inscription ---

inscrireEtudiant("Bob", 12);
// Inscription refusée : L'âge doit être entre 16 et 65 ans !
// --- Fin de la procédure d'inscription ---
?>
⚠️ Piège d'examen sur finally : Le bloc finally s'exécute TOUJOURS, même s'il y a un return dans le try ou le catch. C'est pour ça qu'on l'utilise pour le "nettoyage" (fermer un fichier, une connexion...).

Micro-exercice 5.2 — Valider un retrait bancaire

Écris une fonction retirer($compte, $montant) qui lance une Exception si le montant est négatif ou si le solde est insuffisant. Utilise try/catch/finally.

Correction

<?php
function retirer($solde, $montant) {
    try {
        if ($montant <= 0) {
            throw new Exception("Le montant doit être positif !");
        }
        if ($montant > $solde) {
            throw new Exception("Solde insuffisant ! (Solde: {$solde}€)");
        }

        $nouveauSolde = $solde - $montant;
        echo "Retrait de {$montant}€ effectué.\n";
        return $nouveauSolde;

    } catch (Exception $e) {
        echo "Erreur : " . $e->getMessage() . "\n";
        return $solde;  // On retourne le solde inchangé

    } finally {
        echo "Transaction terminée.\n";
    }
}

$solde = retirer(1000, 200);   // Retrait de 200€ effectué. Transaction terminée.
echo "Solde : {$solde}€\n";   // 800€

$solde = retirer(800, 5000);   // Erreur : Solde insuffisant ! Transaction terminée.
echo "Solde : {$solde}€\n";   // 800€ (inchangé)
?>

Exceptions personnalisées

On peut créer ses propres types d'exceptions en héritant de Exception. Ça permet de distinguer les différents types d'erreurs dans le catch.

Exceptions personnalisées + multi-catch
<?php
// Exceptions personnalisées héritant de Exception
class SoldeInsuffisantException extends Exception {}
class MontantInvalideException extends Exception {}

class CompteBancaire {
    private $solde;

    public function __construct($solde) {
        $this->solde = $solde;
    }

    public function retirer($montant) {
        if ($montant <= 0) {
            throw new MontantInvalideException(
                "Montant invalide : $montant"
            );
        }
        if ($montant > $this->solde) {
            throw new SoldeInsuffisantException(
                "Solde insuffisant : {$this->solde}€ < {$montant}€"
            );
        }
        $this->solde -= $montant;
        return $this->solde;
    }

    public function getSolde() { return $this->solde; }
}

// Utilisation avec MULTI-CATCH
$compte = new CompteBancaire(500);

try {
    $compte->retirer(200);
    echo "Retrait OK. Solde : " . $compte->getSolde() . "€\n";

    $compte->retirer(1000);  // Va lancer SoldeInsuffisantException

} catch (MontantInvalideException $e) {
    echo "⚠️ Montant invalide : " . $e->getMessage() . "\n";

} catch (SoldeInsuffisantException $e) {
    echo "💰 Solde insuffisant : " . $e->getMessage() . "\n";

} catch (Exception $e) {
    // Catch générique : attrape TOUTES les autres exceptions
    echo "Erreur inattendue : " . $e->getMessage() . "\n";
}
?>
⚠️ Règle du multi-catch : Toujours du plus spécifique au plus général !
✅ catch (SoldeInsuffisantException) → catch (Exception)
❌ catch (Exception) → catch (SoldeInsuffisantException) — jamais atteint !

Micro-exercice 5.3 — Exception NoteInvalideException

Crée une exception personnalisée NoteInvalideException. Dans la classe Etudiant, modifie ajouterNote($note) pour lancer cette exception si la note n'est pas entre 0 et 20. Attrape-la avec un try/catch.

Correction

<?php
// 1. Exception personnalisée
class NoteInvalideException extends Exception {}

class Etudiant {
    private $nom;
    private $notes;

    public function __construct($nom) {
        $this->nom = $nom;
        $this->notes = [];
    }

    // Lance NoteInvalideException si note hors bornes
    public function ajouterNote($note) {
        if (!is_numeric($note) || $note < 0 || $note > 20) {
            throw new NoteInvalideException(
                "Note invalide : $note (doit être entre 0 et 20)"
            );
        }
        $this->notes[] = $note;
    }

    public function getMoyenne() {
        if (empty($this->notes)) return 0;
        return array_sum($this->notes) / count($this->notes);
    }
}

// 2. Utilisation avec try/catch
$etu = new Etudiant("Alice");

try {
    $etu->ajouterNote(15);   // ✅ OK
    $etu->ajouterNote(12);   // ✅ OK
    $etu->ajouterNote(25);   // ❌ Lance NoteInvalideException !
} catch (NoteInvalideException $e) {
    echo "Erreur : " . $e->getMessage() . "\n";
    // "Erreur : Note invalide : 25 (doit être entre 0 et 20)"
}

echo "Moyenne : " . $etu->getMoyenne();  // 13.5 (seules 15 et 12 enregistrées)
?>
💡 Pourquoi des exceptions personnalisées ?
Elles permettent de distinguer les types d'erreurs. Au lieu d'un message générique, tu sais exactement ce qui a échoué. À l'examen, le prof demande souvent de créer une exception personnalisée et de l'utiliser avec throw/catch.

Exercice de Synthèse — Système bancaire robuste 5 pts

Contexte : On souhaite sécuriser un système de compte bancaire avec une gestion d'erreurs complète.

  1. (1 pt) Créez deux exceptions personnalisées : SoldeInsuffisantException et MontantInvalideException.
  2. (2 pts) Créez une classe CompteBancaire avec un attribut privé $solde, un getter, et des méthodes deposer($montant) et retirer($montant) qui lancent les exceptions appropriées.
  3. (2 pts) Écrivez un programme de test qui crée un compte, effectue plusieurs opérations et attrape chaque type d'exception séparément.

Correction détaillée

<?php
// ===========================================================
// QUESTION 1 (1 pt) : Exceptions personnalisées
// ===========================================================
class SoldeInsuffisantException extends Exception {}
class MontantInvalideException extends Exception {}
// C'est tout ! Elles héritent de getMessage(), getCode()...

// ===========================================================
// QUESTION 2 (2 pts) : Classe CompteBancaire
// ===========================================================
class CompteBancaire {
    private $solde;

    public function __construct($soldeInitial) {
        if ($soldeInitial < 0) {
            throw new MontantInvalideException("Le solde initial ne peut pas être négatif.");
        }
        $this->solde = $soldeInitial;
    }

    public function getSolde() {
        return $this->solde;
    }

    public function deposer($montant) {
        if ($montant <= 0) {
            throw new MontantInvalideException("Le dépôt doit être positif (reçu: $montant).");
        }
        $this->solde += $montant;
        echo "Dépôt de {$montant}€. Solde : {$this->solde}€\n";
    }

    public function retirer($montant) {
        if ($montant <= 0) {
            throw new MontantInvalideException("Le retrait doit être positif (reçu: $montant).");
        }
        if ($montant > $this->solde) {
            throw new SoldeInsuffisantException(
                "Solde insuffisant : {$this->solde}€ disponible, {$montant}€ demandé."
            );
        }
        $this->solde -= $montant;
        echo "Retrait de {$montant}€. Solde : {$this->solde}€\n";
    }
}

// ===========================================================
// QUESTION 3 (2 pts) : Programme de test
// ===========================================================
try {
    $compte = new CompteBancaire(1000);
    echo "Compte créé. Solde : " . $compte->getSolde() . "€\n";

    $compte->deposer(500);    // ✅ Dépôt OK
    $compte->retirer(200);    // ✅ Retrait OK
    $compte->retirer(5000);   // ❌ Lance SoldeInsuffisantException

} catch (MontantInvalideException $e) {
    echo "❌ Montant invalide : " . $e->getMessage() . "\n";

} catch (SoldeInsuffisantException $e) {
    echo "💰 Solde insuffisant : " . $e->getMessage() . "\n";

} catch (Exception $e) {
    echo "⚠️ Erreur inattendue : " . $e->getMessage() . "\n";

} finally {
    echo "--- Opérations bancaires terminées ---\n";
}

// Test montant invalide
try {
    $compte->deposer(-100);
} catch (MontantInvalideException $e) {
    echo "❌ " . $e->getMessage() . "\n";
}
?>
💡 Récap exceptions (ce que le prof note) :
✅ class MonException extends Exception {} — créer des exceptions custom
✅ throw new MonException("message") — lancer au bon moment
✅ try { } catch (Specifique $e) { } catch (Exception $e) { } — du spécifique au général
✅ $e->getMessage() — récupérer le message d'erreur
✅ finally { } — nettoyage (toujours exécuté)
C'est l'exercice 4 de l'examen — exceptions obligatoires !
