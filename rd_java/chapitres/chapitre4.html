Module 4

Structures conditionnelles

if, else, else if, switch et l'opérateur ternaire : faites prendre des décisions à votre programme.

Structure if / else

Un if, c'est comme un carrefour : si la condition est vraie, on tourne à gauche (bloc if), sinon on tourne à droite (bloc else).

int age = 20;

if (age >= 18) {
    System.out.println("Vous êtes majeur");
} else {
    System.out.println("Vous êtes mineur");
}

Chaînage avec else if

int note = 14;

if (note >= 16) {
    System.out.println("Très bien");
} else if (note >= 12) {
    System.out.println("Bien");
} else if (note >= 10) {
    System.out.println("Passable");
} else {
    System.out.println("Insuffisant");
}

L'opérateur ternaire

C'est un raccourci pour un if/else simple :

// Syntaxe : condition ? valeurSiVrai : valeurSiFaux

String statut = (age >= 18) ? "Majeur" : "Mineur";
int max = (a > b) ? a : b;

Le switch

Idéal lorsque vous comparez une variable à plusieurs valeurs fixes :

int jour = 3;

switch (jour) {
    case 1:
        System.out.println("Lundi");
        break;
    case 2:
        System.out.println("Mardi");
        break;
    case 3:
        System.out.println("Mercredi");
        break;
    default:
        System.out.println("Autre jour");
}

Switch avec String (Java 7+)

String couleur = "rouge";

switch (couleur) {
    case "rouge":  System.out.println("Danger");  break;
    case "vert":   System.out.println("OK");      break;
    case "orange": System.out.println("Attention"); break;
    default:       System.out.println("Inconnu");
}
⚠️ N'oubliez pas le break ! Sans lui, Java exécute tous les cases suivants (fall-through).

Comparer des String

String a = "Java";
String b = "Java";

// ❌ NE PAS utiliser == pour les String
if (a == b) { ... }  // Compare les références mémoire

// ✅ Utiliser .equals()
if (a.equals(b)) {
    System.out.println("Identiques !");
}

// Ignorer la casse
if (a.equalsIgnoreCase("JAVA")) { ... }
🚨 Règle d'or : == compare les références (adresses mémoire). .equals() compare les valeurs. Pour les String, utilisez toujours .equals().
