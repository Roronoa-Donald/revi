Module 7

Fonctions & Procédures

Créez des méthodes statiques pour structurer et réutiliser votre code.

Pourquoi utiliser des méthodes ?

Une méthode est comme une recette de cuisine : vous la définissez une fois (les ingrédients = paramètres, le plat final = retour), puis vous l'utilisez autant de fois que nécessaire.

  • Réutilisation : écrire le code une seule fois
  • Lisibilité : noms explicites = code compréhensible
  • Maintenance : modifier à un seul endroit
  • Décomposition : diviser un problème en sous-problèmes

Fonction vs Procédure

ConceptType de retourExemple
Fonctionint, double, String…Calcule et retourne un résultat
ProcédurevoidExécute une action, ne retourne rien
 
// Fonction : retourne un résultat
public static int somme(int a, int b) {
    return a + b;
}

// Procédure : ne retourne rien (void)
public static void direBonjour(String nom) {
    System.out.println("Bonjour " + nom + " !");
}

// Appels dans le main
public static void main(String[] args) {
    int r = somme(3, 5);      // r = 8
    direBonjour("Alice");       // Bonjour Alice !
}

Anatomie d'une méthode

//  modificateur  type_retour  nom        (paramètres)
    public static  double      moyenne   (int a, int b) {
        return (a + b) / 2.0;
    }
ÉlémentDescription
publicAccessible partout
staticAppartient à la classe (pas besoin d'objet)
Type de retourint, double, void…
NomcamelCase, verbe d'action
ParamètresEntrées de la méthode
returnRenvoie le résultat (absent si void)

La surcharge (overloading)

Plusieurs méthodes avec le même nom mais des paramètres différents :

public static int additionner(int a, int b) {
    return a + b;
}

public static int additionner(int a, int b, int c) {
    return a + b + c;
}

public static double additionner(double a, double b) {
    return a + b;
}

// Java choisit la bonne version automatiquement
additionner(2, 3);         // → int version (2 params)
additionner(1, 2, 3);      // → int version (3 params)
additionner(1.5, 2.5);    // → double version
💡 La surcharge se distingue par le nombre ou le type des paramètres. Le type de retour seul ne suffit pas.

Exemples pratiques

// Vérifier si un nombre est pair
public static boolean estPair(int n) {
    return n % 2 == 0;
}

// Trouver le maximum d'un tableau
public static int max(int[] tab) {
    int m = tab[0];
    for (int v : tab) {
        if (v > m) m = v;
    }
    return m;
}

// Afficher un tableau
public static void afficher(int[] tab) {
    for (int v : tab) {
        System.out.print(v + " ");
    }
    System.out.println();
}
