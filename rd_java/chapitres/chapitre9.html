Module 9

Attributs, Méthodes & Encapsulation

private, public, getters/setters, static, toString et les bonnes pratiques de la POO.

Qu'est-ce que l'encapsulation ?

L'encapsulation, c'est comme un distributeur automatique : vous appuyez sur les boutons (méthodes publiques) pour obtenir un produit, mais vous ne touchez pas au mécanisme interne (attributs privés). Le distributeur contrôle ce que vous pouvez faire.

L'encapsulation consiste à :

  • Cacher les attributs (les rendre private)
  • Contrôler l'accès via des méthodes publiques (getters/setters)
  • Protéger les données contre les modifications invalides

Getters et Setters

public class Personne {
    private String nom;
    private int age;

    // Constructeur
    public Personne(String nom, int age) {
        this.nom = nom;
        setAge(age);  // Validation via le setter
    }

    // Getter — lire la valeur
    public String getNom() {
        return this.nom;
    }

    // Setter — modifier avec validation
    public void setAge(int age) {
        if (age >= 0 && age <= 150) {
            this.age = age;
        } else {
            System.out.println("Âge invalide !");
        }
    }

    public int getAge() {
        return this.age;
    }
}
✅ Avantage : Le setter setAge() vérifie que l'âge est valide. Sans encapsulation, on pourrait écrire p.age = -5; sans aucun contrôle !

Le mot-clé static

Un membre static appartient à la classe, pas à une instance. Il est partagé entre tous les objets.

public class Personne {
    private String nom;
    private static int compteur = 0;  // Partagé

    public Personne(String nom) {
        this.nom = nom;
        compteur++;  // Chaque nouvelle personne incrémente
    }

    public static int getCompteur() {
        return compteur;
    }
}

// Utilisation
Personne p1 = new Personne("Alice");
Personne p2 = new Personne("Bob");
System.out.println(Personne.getCompteur());  // 2

toString() et @Override

public class Personne {
    private String nom;
    private int age;

    public Personne(String nom, int age) {
        this.nom = nom;
        this.age = age;
    }

    @Override
    public String toString() {
        return "Personne{" + "nom='" + nom + "', age=" + age + "}";
    }
}

// Sans toString() : Personne@1a2b3c (adresse mémoire)
// Avec toString() : Personne{nom='Alice', age=20}
Personne p = new Personne("Alice", 20);
System.out.println(p);  // Appelle automatiquement toString()
💡 @Override indique qu'on redéfinit une méthode héritée de la classe Object. Ce n'est pas obligatoire, mais c'est une bonne pratique (le compilateur vérifie).

Exemple complet

public class CompteBancaire {
    private String titulaire;
    private double solde;
    private static int nbComptes = 0;

    public CompteBancaire(String titulaire, double soldeInitial) {
        this.titulaire = titulaire;
        this.solde = soldeInitial;
        nbComptes++;
    }

    public void deposer(double montant) {
        if (montant > 0) this.solde += montant;
    }

    public void retirer(double montant) {
        if (montant > 0 && montant <= this.solde) {
            this.solde -= montant;
        } else {
            System.out.println("Solde insuffisant !");
        }
    }

    public double getSolde() { return solde; }
    public static int getNbComptes() { return nbComptes; }

    @Override
    public String toString() {
        return titulaire + " : " + solde + " €";
    }
}
🧠 Mémo — « P.G.S = Propriété Gardée Soigneusement »
• Private → attributs toujours en private
• Getter → getNom() pour lire
• Setter → setNom() pour modifier (avec validation !)
static = partagé entre TOUS les objets. Comme un compteur de classe.
toString() est appelé automatiquement par println(objet). Toujours le redéfinir pour du debug lisible.
