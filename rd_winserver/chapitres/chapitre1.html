Module 1

Fondamentaux Windows Server

Blaise arrive en stage. Son responsable lui tend un ISO de Windows Server 2022 : « Installe-le, configure-le, et on en reparle. » Premier jour, premières commandes.

Qu'est-ce que Windows Server ?

Windows Server est un système d'exploitation conçu pour gérer des infrastructures professionnelles. Contrairement à Windows 10/11 que tu utilises pour surfer et jouer, Windows Server est le « cerveau » qui distribue les ressources dans un réseau d'entreprise.

Imagine un hôtel 5 étoiles. Windows 10/11, c'est la chambre — confortable, personnelle. Windows Server, c'est la réception — elle gère les clés, les réservations, le room service, la sécurité. Sans réception, l'hôtel est un chaos. Sans serveur, le réseau est un chaos.

Desktop Experience vs Server Core

Lors de l'installation, tu as deux choix :

  • Desktop Experience (GUI) — Interface graphique complète, familière, idéale pour débuter
  • Server Core — Pas d'interface graphique, uniquement la ligne de commande. Plus léger, plus sécurisé (moins de surface d'attaque)
💡 Pour débuter, choisis Desktop Experience. Tu pourras toujours migrer vers Core plus tard. En production, Core est souvent préféré car il consomme moins de ressources et réduit les risques de sécurité.

Post-installation : les premiers réflexes

  1. Renommer le serveur — SRV-DC01 est plus parlant que WIN-A12B3C4D
  2. Configurer une IP statique — Un serveur ne doit JAMAIS être en DHCP
  3. Appliquer les mises à jour — Première chose à faire, toujours
  4. Activer le Bureau à distance — Pour l'administrer à distance
  5. Configurer le pare-feu — Ouvrir uniquement les ports nécessaires
⚠️ Règle d'or : Un serveur = une IP statique. Imagine un hôpital dont le numéro de téléphone change tous les jours — personne ne pourrait appeler les urgences. C'est pareil pour un serveur.

Server Manager — Le Tableau de Bord

Server Manager est l'outil central de Windows Server. Il s'ouvre automatiquement au démarrage et te donne une vue d'ensemble de ton serveur.

Que fait Server Manager ?

  • 📊 Dashboard — Vue globale : alertes, événements, état des rôles
  • 🔧 Ajouter des rôles — Installer AD, DNS, DHCP, Hyper-V, etc.
  • 🖥️ Gérer plusieurs serveurs — Un seul tableau de bord pour tout
  • 📋 Événements & services — Voir ce qui tourne, ce qui plante
Server Manager, c'est le tableau de bord d'un avion. Le pilote ne pilote pas « à l'aveugle » : il consulte ses instruments. Toi non plus, tu ne gères pas un serveur à l'aveugle.

Rôles et Fonctionnalités

Un Windows Server tout neuf ne fait rien de spécial. Il devient utile quand tu lui donnes des rôles.

Fonctionnalités = les outils complémentaires

Les fonctionnalités sont des options supplémentaires qui enrichissent un rôle ou le serveur lui-même :

  • .NET Framework — Pour les applications qui en ont besoin
  • Windows Server Backup — Outil de sauvegarde intégré
  • Failover Clustering — Haute disponibilité
  • BitLocker — Chiffrement des disques
  • Telnet Client — Tests réseau basiques

Installer un rôle

Via Server Manager :

Server Manager → Manage → Add Roles and Features
→ Suivre l'assistant → Sélectionner le rôle → Install

Via PowerShell (plus rapide) :

Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools
Install-WindowsFeature -Name DNS
Install-WindowsFeature -Name DHCP
Get-WindowsFeature   # Lister tous les rôles/fonctionnalités

PowerShell — Premiers Pas

PowerShell est le langage de commande de Windows Server. Si tu viens du monde Linux, c'est l'équivalent de Bash — mais en version « objet ».

PowerShell, c'est comme passer une commande par texto au lieu de se déplacer au guichet. Plus rapide, plus précis, et tu gardes un historique de tout ce que tu as demandé.

Commandes essentielles

# Informations système
Get-ComputerInfo                    # Toutes les infos du serveur
hostname                             # Nom du serveur
Get-Date                             # Date et heure

# Réseau
Get-NetIPAddress                     # Adresses IP
Get-NetAdapter                       # Cartes réseau

# Services
Get-Service                          # Tous les services
Get-Service -Name WinRM             # Un service spécifique
Restart-Service -Name Spooler       # Redémarrer un service

# Gestion du serveur
Rename-Computer -NewName SRV-DC01 -Restart
Get-WindowsFeature                   # Rôles installés et disponibles
Get-EventLog -LogName System -Newest 10

La logique Verbe-Nom

PowerShell suit toujours le schéma Verbe-Nom :

Configurer une IP Statique

Première mission de Blaise : donner une adresse fixe à son serveur.

Via l'interface graphique

Control Panel → Network and Sharing Center
→ Change adapter settings → Clic droit sur la carte → Properties
→ Internet Protocol Version 4 (TCP/IPv4) → Properties
→ Renseigner : IP, Masque, Passerelle, DNS

Via PowerShell (recommandé)

# Identifier la carte réseau
Get-NetAdapter

# Configurer l'IP statique
New-NetIPAddress -InterfaceAlias "Ethernet" `
    -IPAddress 192.168.1.10 `
    -PrefixLength 24 `
    -DefaultGateway 192.168.1.1

# Configurer le DNS
Set-DnsClientServerAddress -InterfaceAlias "Ethernet" `
    -ServerAddresses 192.168.1.10,8.8.8.8

# Vérifier
Get-NetIPConfiguration
💡 Pourquoi le DNS pointe sur lui-même ? Quand le serveur sera contrôleur de domaine, il hébergera son propre service DNS. Il doit donc se référencer lui-même en DNS primaire.

Bureau à Distance (RDP)

En entreprise, les serveurs sont dans une salle serveur. Tu ne vas pas te déplacer avec clavier et écran à chaque intervention. Tu utilises le Remote Desktop Protocol (RDP).

Activer RDP

# Via PowerShell
Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' `
    -Name "fDenyTSConnections" -Value 0

# Ouvrir le port dans le pare-feu
Enable-NetFirewallRule -DisplayGroup "Remote Desktop"

# Se connecter depuis un poste client
mstsc /v:192.168.1.10
⚠️ Sécurité RDP : Ne jamais exposer RDP directement sur Internet. Utilise un VPN ou un bastion pour protéger l'accès.
