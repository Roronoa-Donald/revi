Module 3

GPO & Sécurité

Blaise découvre le pouvoir des GPO : « Tu veux interdire l'USB sur 200 postes ? Une seule GPO et c'est fait. » C'est la télécommande magique de l'admin.

Qu'est-ce qu'une GPO ?

Une Group Policy Object (GPO) est un ensemble de règles que tu appliques à des utilisateurs ou des ordinateurs via Active Directory. Plutôt que de configurer chaque poste un par un, tu définis une règle une seule fois et elle s'applique à tous.

Les GPO, c'est comme les lois d'un pays. Le gouvernement (l'admin) publie une loi (GPO) qui s'applique à tous les citoyens (utilisateurs/ordinateurs) d'une zone (OU). Pas besoin d'aller voir chaque citoyen individuellement — la loi s'applique automatiquement.

Les deux types de paramètres

CatégorieS'applique àExemples
Computer ConfigurationLa machine (au démarrage)Pare-feu, BitLocker, services, scripts de démarrage
User ConfigurationL'utilisateur (à la connexion)Fond d'écran, lecteurs réseau, menu Démarrer, restrictions

Héritage et Ordre d'Application

Les GPO suivent un ordre strict d'application appelé LSDOU :

OrdreNiveauDescription
1LocalGPO locale de la machine (gpedit.msc)
2SiteGPO liée au site AD (rarement utilisé)
3DomaineGPO liée au domaine (ex : politique de mots de passe)
4OUGPO liée à l'OU (la plus spécifique gagne)
💡 Règle : La dernière GPO appliquée gagne en cas de conflit. Comme les OU sont appliquées en dernier, une GPO d'OU peut écraser une GPO de domaine.

Bloquer l'héritage & Forcer l'application

  • Block Inheritance — Sur une OU, empêche les GPO parentes de s'appliquer
  • Enforced (No Override) — Force une GPO à s'appliquer même si l'héritage est bloqué
Block Inheritance = un quartier qui déclare son indépendance et refuse les lois de la ville. Enforced = une loi constitutionnelle que même les quartiers indépendants doivent respecter.

Créer et Lier une GPO

Via la console GPMC

Server Manager → Tools → Group Policy Management
→ Domaine → Clic droit sur l'OU → Create a GPO in this domain, and Link it here...
→ Nommer la GPO → Clic droit → Edit

Via PowerShell

# Créer une GPO
New-GPO -Name "Interdire USB"

# Lier la GPO à une OU
New-GPLink -Name "Interdire USB" `
    -Target "OU=Ordinateurs,DC=entreprise,DC=local"

# Forcer la mise à jour GPO sur un poste (sans attendre)
Invoke-GPUpdate -Computer "PC-COMPTA01" -Force

# Depuis le poste client (vérifier les GPO appliquées)
gpresult /r         # Résumé texte
gpresult /h rapport.html  # Rapport HTML

GPO Courantes — Exemples Pratiques

🔒 Politique de mot de passe

Computer Config → Policies → Windows Settings
→ Security Settings → Account Policies → Password Policy

Minimum password length     : 12 caractères

Maximum password age         : 90 jours

🖥️ Fond d'écran imposé

User Config → Policies → Administrative Templates
→ Desktop → Desktop Wallpaper

Computer Config → Policies → Administrative Templates
→ System → Removable Storage Access

User Config → Preferences → Windows Settings → Drive Maps
→ Clic droit → New → Mapped Drive

→ Logon → Ajouter le script (.bat ou .ps1)

Filtrage des GPO

Parfois, tu veux qu'une GPO s'applique seulement à certains utilisateurs/ordinateurs dans une OU.

Security Filtering

Par défaut, une GPO s'applique à « Authenticated Users ». Tu peux restreindre :

GPMC → Sélectionner la GPO → Onglet Scope
→ Security Filtering → Supprimer Authenticated Users
→ Ajouter le groupe spécifique (ex : GRP_Direction)

WMI Filtering

Appliquer une GPO selon des critères techniques (OS, RAM, modèle…) :

# Exemple : GPO uniquement pour Windows 11
SELECT * FROM Win32_OperatingSystem WHERE Version LIKE "10.0.22%"
💡 Le WMI Filtering est puissant mais peut ralentir l'application des GPO. Utilise-le avec parcimonie.

Audit & Sécurité

Activer l'audit des connexions

Computer Config → Policies → Windows Settings
→ Security Settings → Advanced Audit Policy Configuration
→ Logon/Logoff

Audit Logon            : Success, Failure
Audit Account Lockout  : Success, Failure

Verrouillage de compte

Computer Config → Security Settings → Account Lockout Policy

Account lockout threshold    : 5 tentatives
Account lockout duration     : 30 minutes
Reset lockout counter after  : 15 minutes

Commandes de diagnostic GPO

gpresult /r                    # GPO appliquées (résumé)
gpresult /h C:\gpo-report.html  # Rapport HTML détaillé
gpupdate /force                 # Forcer l'application immédiate
rsop.msc                        # Resultant Set of Policy (console)

# PowerShell : rapport étendu
Get-GPResultantSetOfPolicy -ReportType Html `
    -Path C:\rsop-report.html

Résumé du Module

  • GPO = règles centralisées pour utilisateurs/ordinateurs
  • Deux catégories : Computer Configuration et User Configuration
  • Ordre d'application : LSDOU (Local, Site, Domaine, OU)
  • Block Inheritance et Enforced pour contrôler l'héritage
  • Security Filtering = restreindre à un groupe spécifique
  • WMI Filtering = restreindre selon des critères techniques
  • gpresult /r pour diagnostiquer, gpupdate /force pour appliquer
  • Toujours tester sur une OU de test avant déploiement global
