Module 4

DNS & DHCP

« Comment mon PC sait-il que google.com = 142.250.179.78 ? » Blaise découvre les deux services réseau qui font tourner le monde silencieusement.

DNS — Le Traducteur Universel

Le Domain Name System (DNS) traduit les noms de domaine en adresses IP. Sans DNS, tu devrais mémoriser des adresses IP pour chaque site et chaque serveur.

DNS, c'est l'annuaire téléphonique d'Internet. Tu cherches « Papa » (nom) → l'annuaire te donne 06 12 34 56 78 (IP). Tu n'as pas besoin de mémoriser le numéro, juste le nom.

Pourquoi DNS est vital pour AD

Active Directory ne peut pas fonctionner sans DNS. Chaque fois qu'un PC joint le domaine ou qu'un utilisateur se connecte, le système interroge le DNS pour localiser le contrôleur de domaine.

🚨 Si le DNS tombe : Plus d'authentification AD, plus de résolution de noms, plus d'accès aux partages réseau. Le DNS est le talon d'Achille du réseau. Protège-le !

Les Zones DNS

Zone de recherche directe (Forward Lookup Zone)

Traduit un nom → IP. C'est la zone la plus utilisée.

srv-dc01.entreprise.local  →  192.168.1.10

Zone de recherche inversée (Reverse Lookup Zone)

Traduit une IP → nom. Utile pour le diagnostic.

192.168.1.10  →  srv-dc01.entreprise.local

Commandes DNS PowerShell

# Installer le rôle DNS
Install-WindowsFeature -Name DNS -IncludeManagementTools

# Créer une zone directe
Add-DnsServerPrimaryZone -Name "entreprise.local" -ReplicationScope "Forest"

# Ajouter un enregistrement A
Add-DnsServerResourceRecordA -ZoneName "entreprise.local" `
    -Name "srv-files" -IPv4Address 192.168.1.20

# Ajouter un CNAME
Add-DnsServerResourceRecordCName -ZoneName "entreprise.local" `
    -Name "intranet" -HostNameAlias "srv-web01.entreprise.local"

# Diagnostiquer
nslookup srv-dc01.entreprise.local
Resolve-DnsName -Name "srv-dc01" -Type A
Get-DnsServerResourceRecord -ZoneName "entreprise.local"

DHCP — L'Attributeur d'Adresses

Le Dynamic Host Configuration Protocol (DHCP) distribue automatiquement les adresses IP aux appareils qui se connectent au réseau.

DHCP, c'est le réceptionniste d'un hôtel. Quand un client arrive (un PC se connecte), le réceptionniste lui attribue une chambre (IP) avec une durée de séjour (bail). Quand le client part, la chambre redevient disponible.

Ce que DHCP distribue

  • Adresse IP — depuis une plage configurée (étendue/scope)
  • Masque de sous-réseau — pour délimiter le réseau
  • Passerelle par défaut — pour sortir du réseau local
  • Serveur DNS — pour la résolution de noms
  • Suffixe DNS — entreprise.local
  • Durée du bail — combien de temps l'IP est réservée

Le processus DORA

ÉtapeMessageQui → QuiDescription
DDiscoverClient → Broadcast« Y'a un serveur DHCP ici ? »
OOfferServeur → Client« Oui ! Voici une IP disponible »
RRequestClient → Serveur« OK, je prends cette IP »
AAcknowledgeServeur → Client« C'est noté ! Elle est à toi »

Configurer DHCP

Installer et configurer via PowerShell

# Installer le rôle DHCP
Install-WindowsFeature -Name DHCP -IncludeManagementTools

# Autoriser le serveur DHCP dans AD
Add-DhcpServerInDC -DnsName "srv-dc01.entreprise.local"

# Créer une étendue (scope)
Add-DhcpServerv4Scope -Name "LAN Principal" `
    -StartRange 192.168.1.100 `
    -EndRange 192.168.1.200 `
    -SubnetMask 255.255.255.0 `
    -LeaseDuration 8.00:00:00   # 8 jours

# Options du scope : passerelle
Set-DhcpServerv4OptionValue -ScopeId 192.168.1.0 `
    -Router 192.168.1.1

# Options du scope : DNS
Set-DhcpServerv4OptionValue -ScopeId 192.168.1.0 `
    -DnsServer 192.168.1.10 `
    -DnsDomain "entreprise.local"

# Réservation (IP fixe pour une machine spécifique)
Add-DhcpServerv4Reservation -ScopeId 192.168.1.0 `
    -IPAddress 192.168.1.50 `
    -ClientId "AA-BB-CC-DD-EE-FF" `
    -Description "Imprimante Compta"

Exclusions

Les adresses à ne jamais distribuer (serveurs, routeurs…) :

Add-DhcpServerv4ExclusionRange -ScopeId 192.168.1.0 `
    -StartRange 192.168.1.1 -EndRange 192.168.1.20
💡 Bonne pratique : Réserve toujours une plage pour les IP statiques (serveurs, imprimantes). Ex : .1–.20 = statique, .100–.200 = DHCP.

Diagnostic DNS & DHCP

# ── DNS ──
nslookup entreprise.local             # Résolution simple
nslookup -type=MX entreprise.local   # Enregistrement MX
Resolve-DnsName "srv-dc01"              # PowerShell natif
Clear-DnsClientCache                    # Vider le cache DNS local
ipconfig /flushdns                     # Alternative classique
dcdiag /test:dns                       # Test DNS pour AD

# ── DHCP ──
Get-DhcpServerv4Scope                   # Voir les étendues
Get-DhcpServerv4Lease -ScopeId 192.168.1.0  # Bails actifs
ipconfig /release                     # Libérer l'IP (client)
ipconfig /renew                       # Renouveler l'IP (client)

Résumé du Module

  • DNS traduit les noms en IP — indispensable pour AD
  • Zones : directe (nom→IP), inversée (IP→nom), intégrée AD recommandée
  • Enregistrements clés : A, AAAA, CNAME, MX, NS, SRV, PTR
  • DHCP distribue automatiquement les IP via le processus DORA
  • Configurer : étendue (scope), options, exclusions, réservations
  • Toujours autoriser DHCP dans AD pour éviter les serveurs DHCP non autorisés
  • Diagnostic : nslookup, Resolve-DnsName, ipconfig /flushdns
