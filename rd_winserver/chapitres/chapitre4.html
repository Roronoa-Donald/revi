Virtualisation & VMware

Hyperviseurs, ESXi, vCenter, réseaux virtuels (NAT/Bridge/Host-Only), images disque et ressources CPU.

Hyperviseur Type 1 vs Type 2

Un hyperviseur est un logiciel qui permet de créer et gérer des machines virtuelles (VM).

TypeDescriptionExemples
Type 1 (bare-metal) Permet la virtualisation directement sur le matériel physique, sans OS intermédiaire. VMware ESXi
Type 2 (hébergé) S'installe sur un OS existant (Windows, Linux, macOS). VMware Workstation, Oracle VirtualBox

ESXi, vCenter et vSphere Client

Dans un environnement VMware professionnel :

ComposantRôle
VMware ESXiHyperviseur de type 1 installé directement sur le serveur physique. Héberge et exécute les VM.
vCenterServeur de gestion centralisée permettant d'administrer plusieurs hôtes ESXi, créer des clusters, et configurer la HA.
vSphere ClientInterface graphique (web) permettant d'administrer l'infrastructure : créer des VM, allouer des ressources, surveiller les performances.
Exemple (examen) : Max ouvre vSphere Client, choisit un hôte ESXi, alloue 2 vCPU, 8 Go de RAM et 100 Go de stockage pour créer une VM Ubuntu Server 22.04.

Gestion des Réseaux Virtuels

Les machines virtuelles peuvent être connectées au réseau de différentes manières. Il existe 3 modes réseau principaux :

1. NAT (Network Address Translation)

  • La VM utilise l'adresse IP de l'hôte pour accéder à Internet.
  • L'hôte fait office de passerelle.
  • Avantages : accès Internet sécurisé, pas besoin de configurer le réseau.
  • Limites : la VM n'est pas directement accessible depuis l'extérieur.

2. Bridge (Pont)

  • La VM se comporte comme une machine physique sur le même réseau que l'hôte.
  • Elle obtient une adresse IP depuis le serveur DHCP du réseau physique.
  • Avantages : la VM peut communiquer librement avec toutes les machines du réseau.
  • Inconvénients : peut poser des problèmes de sécurité et de configuration.

3. Host-Only

  • Réseau privé entre l'hôte et les VM.
  • Pas d'accès Internet par défaut.
  • Utilisé pour des environnements isolés (test, dev).
  • Possibilité d'ajouter une seconde carte NAT pour l'accès Internet.

Images Disque (ISO, VMDK, VHD)

Dans la virtualisation, l'image disque est un fichier qui représente l'intégralité du contenu d'un disque dur ou d'un lecteur optique (comme un CD/DVD).

FormatDescription
ISO Image d'un disque optique (CD, DVD). Utilisé pour installer un système d'exploitation dans une VM. Contient une copie exacte secteur par secteur d'un disque optique.
VMDK
(Virtual Machine Disk)		 Format utilisé principalement par VMware. Contient toutes les données du disque virtuel de la VM. Peut être utilisé en mode thick (réservé d'avance) ou thin (s'agrandit selon l'usage).
VHD / VHDX
(Virtual Hard Disk)		 Format développé par Microsoft. Principalement utilisé avec Hyper-V. Représente un disque dur complet pour une VM. VHDX = version améliorée avec meilleure tolérance aux erreurs.

vCPU, CPU Ready Time et Surallocation

vCPU vs CPU physique

Un vCPU (virtual CPU) est un processeur virtuel alloué à une VM. Il correspond à un cœur physique (ou un thread) du processeur de l'hôte. Une VM peut avoir plusieurs vCPU (ex : 2 vCPU pour une VM).

CPU Ready Time

Le CPU Ready Time est l'unité de mesure de la consommation CPU dans un environnement VMware. Il indique le temps qu'une VM attend pour obtenir du temps CPU sur le processeur physique.

Surallocation de vCPU

La surallocation (overcommit) se produit quand le nombre total de vCPU alloués aux VM dépasse le nombre de cœurs physiques disponibles.

⚠️ Conséquence principale : L'allongement du CPU Ready Time — les VM doivent attendre plus longtemps pour accéder au processeur physique, ce qui dégrade les performances.

Thin Provisioning

Le thin provisioning signifie que l'espace disque est alloué dynamiquement selon les besoins. Au lieu de réserver tout l'espace disque immédiatement (thick provisioning), la VM n'utilise que l'espace réellement nécessaire et s'agrandit au fil du temps.

Résumé Express

  • Hyperviseur type 1 : directement sur le matériel (ESXi) / Type 2 : sur un OS (VirtualBox, Workstation)
  • ESXi : hyperviseur / vCenter : gestion centralisée / vSphere Client : interface web
  • NAT : Internet via l'hôte, VM non accessible depuis le LAN
  • Bridge : VM = machine physique sur le réseau, accessible partout
  • Host-Only : réseau privé, pas d'Internet par défaut
  • ISO : image CD/DVD / VMDK : disque VMware / VHD/VHDX : disque Microsoft Hyper-V
  • VMDK thick : espace réservé d'avance / thin : espace alloué dynamiquement
  • vCPU : processeur virtuel / CPU Ready Time : temps d'attente CPU
  • Surallocation vCPU : plus de vCPU que de cœurs physiques → allongement CPU Ready Time
