Snapshots, Clones & Templates

Instantanés, clones complet/lié, templates de VM, vMotion et Haute Disponibilité.

Snapshot (Instantané)

Un snapshot est une capture de l'état d'une machine virtuelle à un moment donné. Il enregistre :

  • Le contenu de la mémoire
  • L'état du processeur
  • L'état des disques virtuels

Utilité

  • Faire des sauvegardes rapides avant une mise à jour ou un changement critique.
  • Permet de revenir en arrière si un problème survient.
⚠️ Important :
  • Les snapshots consomment de l'espace disque supplémentaire.
  • Ils ne doivent pas être conservés longtemps en production.

Snapshot vs Sauvegarde

CritèreSnapshotSauvegarde
Durée de conservationTemporaire (court terme)Long terme
UtilitéRetour rapide avant/après changementRestauration complète en cas de perte
Espace disqueConsomme de plus en plus avec le tempsEspace prévu et planifié
Usage en productionNe pas conserver longtempsConçue pour le long terme

Clones de VM

Un clone est une copie complète d'une machine virtuelle existante. Il existe deux types :

Clone Complet

  • Fait une copie indépendante de la VM (fichiers de disque, configuration).
  • Fonctionne même si la VM d'origine est supprimée.
✅ Avantage : totalement autonome et indépendant de la VM source.

Clone Lié (Linked Clone)

  • La nouvelle VM partage une partie des fichiers de disque avec l'originale.
  • Moins gourmand en espace disque mais dépendant de la VM source.
⚠️ Attention : Si la VM source est supprimée, le clone lié ne fonctionnera plus.

Comparaison

CritèreClone CompletClone Lié
Indépendance✅ Totalement indépendant❌ Dépend de la VM source
Espace disquePlus volumineuxMoins gourmand
Suppression de la source✅ Continue de fonctionner❌ Ne fonctionne plus
Utilité : Le clonage permet de déployer rapidement plusieurs machines basées sur un modèle existant (ex : laboratoires de test, fermes de serveurs).

Templates de VM

Un template est une machine virtuelle configurée avec :

  • Un système d'exploitation installé
  • Les outils de base (mises à jour, logiciels)
  • Des paramètres prédéfinis (réseau, sécurité)

Le template est figé (souvent en lecture seule) pour être utilisé comme modèle lors de la création de nouvelles VM.

Utilité des Templates

  • Standardiser les déploiements de VM.
  • Gagner du temps : plus besoin de réinstaller et configurer chaque VM.
  • Garantir que toutes les VM créées sont cohérentes et conformes aux politiques internes.
En résumé : Un template permet de cloner rapidement une VM préconfigurée pour standardiser les déploiements.

vMotion

vMotion permet de migrer une VM en cours d'exécution d'un hôte physique vers un autre, sans interruption de service.

Exemple (examen) : La VM de Max devient lente car plusieurs VM surconsomment les ressources sur le même hôte. Il utilise vMotion pour déplacer la VM vers un autre hôte moins sollicité. Résultat : les performances sont de retour.

Haute Disponibilité (HA) et Sauvegarde Veeam

Cluster HA

La fonctionnalité essentielle pour la haute disponibilité des VM est le cluster de virtualisation. Grâce à la configuration d'un cluster HA :

  • Les VM redémarrent automatiquement sur un autre hôte disponible du cluster en cas de panne.
  • Cela minimise l'impact des pannes matérielles.
Exemple (examen) : L'entreprise de Max subit une panne électrique majeure. Grâce au cluster HA, les VM redémarrent automatiquement sur un autre hôte disponible.

Sauvegarde avec Veeam

Veeam est un outil de sauvegarde automatique pour les VM. Il permet de configurer des sauvegardes automatiques quotidiennes et de restaurer rapidement une VM en cas de suppression accidentelle.

Exemple (examen) : Un collègue de Max supprime accidentellement une VM critique. Grâce à la sauvegarde Veeam configurée, Max restaure la VM en moins de 30 minutes.

Bonnes Pratiques d'Administration

  • Documenter chaque action effectuée sur l'infrastructure
  • Surveiller régulièrement les performances avec vCenter
  • Programmer des snapshots avant chaque mise à jour majeure
  • Configurer des sauvegardes automatiques (Veeam)
  • Configurer la HA pour le redémarrage automatique des VM

Résumé Express

  • Snapshot : capture mémoire + processeur + disques virtuels, temporaire, pas pour le long terme
  • Snapshot ≠ sauvegarde : snapshot = court terme / sauvegarde = long terme
  • Clone complet : copie indépendante, fonctionne même si la source est supprimée
  • Clone lié : partage les fichiers disque avec la source, moins d'espace mais dépendant
  • Template : VM préconfigurée figée, modèle pour déploiements standardisés
  • vMotion : migration de VM en cours d'exécution sans interruption
  • Cluster HA : redémarrage automatique des VM sur un autre hôte en cas de panne
  • Veeam : sauvegarde automatique et restauration rapide des VM
