Module 5

Sauvegarde & Restauration

Un ransomware chiffre les données de l'entreprise un vendredi soir. Le responsable se tourne vers Blaise : « Dis-moi qu'on a des sauvegardes récentes. » Ce module t'apprend à ne jamais être celui qui répond « non ».

Pourquoi sauvegarder ?

Les sauvegardes protègent contre 5 types de désastres :

MenaceExemple concretImpact sans sauvegarde
🧑 Erreur humaineUn admin supprime une OU entièreTous les comptes perdus
💻 Panne matérielleDisque dur en panneDonnées irrécupérables
🦠 CyberattaqueRansomware chiffre les fichiersRançon ou perte totale
⚡ Panne de courantCoupure longue sans onduleurCorruption système
🌊 Catastrophe naturelleInondation de la salle serveurDestruction physique
🚨 Réalité : 93% des entreprises qui perdent leurs données pendant plus de 10 jours font faillite dans l'année. La sauvegarde n'est pas optionnelle — c'est une assurance vie.

Les 3 Types de Sauvegarde

1. Sauvegarde Complète (Full Backup)

Copie l'intégralité des données à chaque exécution. Simple, fiable, mais volumineuse et lente.

C'est comme photocopier tout le dossier à chaque fois. Tu es sûr de tout avoir, mais c'est beaucoup de papier et de temps.
✅ Avantages❌ Inconvénients
Restauration simple (1 seul jeu)Lente à exécuter
Toutes les données en un pointOccupe beaucoup d'espace

2. Sauvegarde Incrémentielle (Incremental)

Après une complète initiale, ne sauvegarde que les données modifiées depuis la dernière sauvegarde (complète OU incrémentielle).

Tu as photocopié le dossier lundi. Mardi, tu ne photocopies que les nouvelles pages ajoutées depuis lundi. Mercredi, que celles ajoutées depuis mardi. Rapide, mais pour restaurer mercredi, tu as besoin de la complète + lundi + mardi + mercredi.
✅ Avantages❌ Inconvénients
Très rapide et légèreRestauration complexe (chaîne complète nécessaire)
Peu d'espace disqueSi un maillon manque, restauration partielle

3. Sauvegarde Différentielle (Differential)

Après une complète, sauvegarde tout ce qui a changé depuis la dernière sauvegarde COMPLÈTE. Grossit progressivement.

Tu photocopies toujours par rapport au dossier de lundi. Mardi = pages changées depuis lundi. Mercredi = pages changées depuis lundi aussi (donc mardi est inclus). Pour restaurer, tu n'as besoin que de la complète + la dernière différentielle.
✅ Avantages❌ Inconvénients
Restauration simple (complète + 1 diff)Grossit jour après jour
Compromis vitesse/simplicitéPlus lente qu'incrémentielle

Comparaison visuelle

# Semaine type avec une Full le dimanche

Complète       : Dim[████████] Lun[████████] Mar[████████] ...
                   → Espace : ÉNORME  |  Restauration : FACILE

Incrémentielle : Dim[████████] Lun[█] Mar[██] Mer[█] Jeu[███] ...
                   → Espace : MINIMAL  |  Restauration : COMPLEXE

Différentielle : Dim[████████] Lun[█] Mar[███] Mer[█████] Jeu[███████]
                   → Espace : MOYEN    |  Restauration : SIMPLE

La Stratégie 3-2-1

La stratégie 3-2-1 est le standard de l'industrie pour protéger les données :

RègleSignificationExemple
3 copiesAu moins 3 copies de tes donnéesOriginal + 2 sauvegardes
2 supportsSur 2 types de supports différentsDisque local + NAS / Cloud
1 hors-siteAu moins 1 copie géographiquement séparéeDatacenter distant ou cloud
Imagine tes photos de famille. Tu les as sur ton téléphone (1), tu les copies sur un disque dur à la maison (2), et tu les envoies sur Google Drive (3, hors-site). Si ta maison brûle, tu as encore le cloud. Si Google disparaît, tu as le disque dur.

RPO et RTO

Deux métriques essentielles pour planifier tes sauvegardes :

MétriqueQuestionExemple
RPO (Recovery Point Objective)Combien de données peut-on se permettre de perdre ?RPO = 4h → sauvegarde toutes les 4h
RTO (Recovery Time Objective)En combien de temps doit-on restaurer le service ?RTO = 2h → restauration max en 2h
⚠️ Plus le RPO/RTO est bas, plus la solution est coûteuse. Un RPO de 0 (aucune perte) nécessite de la réplication en temps réel ! C'est un arbitrage business.

Windows Server Backup

L'outil intégré de sauvegarde de Windows Server.

Installation

Install-WindowsFeature -Name Windows-Server-Backup

Sauvegarde complète via PowerShell

# Créer une politique de sauvegarde
$policy = New-WBPolicy

# Ajouter tous les volumes
$volumes = Get-WBVolume -AllVolumes
Add-WBVolume -Policy $policy -Volume $volumes

# Inclure l'état système (AD, registre, boot)
Add-WBSystemState -Policy $policy

# Cible : disque dédié
$target = New-WBBackupTarget -VolumePath "E:"
Add-WBBackupTarget -Policy $policy -Target $target

# Planifier : tous les jours à 22h
Set-WBSchedule -Policy $policy -Schedule 22:00

# Appliquer la politique
Set-WBPolicy -Policy $policy

# Lancer une sauvegarde manuelle immédiate
Start-WBBackup -Policy (Get-WBPolicy)

Restaurer

# Voir les sauvegardes disponibles
Get-WBBackupSet

# Restaurer l'état système (mode DSRM requis pour AD)
wbadmin start systemstaterecovery -version:01/15/2025-22:00

PCA et PRA

Deux concepts complémentaires pour la résilience de l'entreprise :

PCA (Plan de Continuité d'Activité)PRA (Plan de Reprise d'Activité)
ObjectifMaintenir le service PENDANT l'incidentRestaurer le service APRÈS l'incident
QuandAvant et pendant la criseAprès la crise
ApprocheRedondance, basculeSauvegarde, restauration
Exemple2 DC, clustering, NLBRestore depuis backup E:
CoûtÉlevé (infrastructure double)Modéré (stockage + procédures)
✅ Règle d'or : Teste régulièrement tes restaurations. Une sauvegarde qui n'a jamais été testée n'est PAS une sauvegarde — c'est un espoir. Et l'espoir n'est pas une stratégie.

Résumé du Module

  • 3 types : Complète (tout) > Différentielle (depuis full) > Incrémentielle (depuis dernière)
  • Stratégie 3-2-1 : 3 copies, 2 supports, 1 hors-site
  • RPO = données perdues max, RTO = temps de restauration max
  • Windows Server Backup : outil intégré, planifiable
  • PCA = continuité pendant, PRA = reprise après
  • TOUJOURS tester la restauration régulièrement
