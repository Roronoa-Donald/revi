Module 7

Maintenance Windows Server

Blaise pensait qu'une fois le serveur installé, c'était fini. Erreur de débutant. Un serveur, c'est comme une voiture : sans entretien régulier, elle tombe en panne au pire moment. Ce module t'apprend à garder tes serveurs en pleine forme.

Les 3 Types de Maintenance

1. Maintenance Préventive

Actions planifiées pour éviter les pannes avant qu'elles ne surviennent.

C'est la révision annuelle de ta voiture. Tu changes l'huile, vérifies les freins et les pneus. Tu n'attends pas que le moteur casse pour réagir.
  • Mises à jour du système et des rôles
  • Monitoring des ressources (CPU, RAM, disques, réseau)
  • Vérification des sauvegardes → test de restauration
  • Audit des comptes et des permissions
  • Nettoyage des fichiers temporaires et des logs
  • Vérification de la santé matérielle (températures, SMART)

2. Maintenance Corrective

Intervention après une panne pour restaurer le fonctionnement normal.

Ta voiture est en panne sur le bord de la route. Tu appelles le dépanneur. C'est stressant, coûteux, et ça aurait pu être évité avec de la préventive.

Exemples typiques :

  • Service AD DS qui ne démarre plus
  • Réplication DNS cassée
  • Volume disque plein à 100%
  • Performance dégradée (CPU à 95% en permanence)
  • Serveur DHCP qui ne distribue plus d'adresses

3. Maintenance Évolutive

Mise à niveau pour améliorer les capacités ou s'adapter aux nouveaux besoins.

Tu ajoutes un GPS et une caméra de recul à ta voiture. Elle fonctionnait déjà, mais maintenant elle est mieux adaptée aux besoins actuels.
  • Migration de Windows Server 2016 vers 2022
  • Ajout de RAM ou de stockage
  • Déploiement de nouveaux rôles
  • Passage à Hyper-V pour la virtualisation

Les Outils de Surveillance

Event Viewer (Observateur d'événements)

Le journal de bord de ton serveur. Chaque erreur, avertissement et information y est consigné.

# Lancer l'Observateur d'événements
eventvwr.msc

# Les 50 dernières erreurs système
Get-EventLog -LogName System -EntryType Error -Newest 50

# Erreurs sur les dernières 24h
Get-WinEvent -FilterHashtable @{
    LogName = 'System'
    Level = 2  # 2=Error, 3=Warning, 4=Info
    StartTime = (Get-Date).AddHours(-24)
}

# Erreurs d'authentification (sécurité)
Get-WinEvent -FilterHashtable @{
    LogName = 'Security'
    Id = 4625  # Échec de connexion
} -MaxEvents 20
💡 IDs critiques à surveiller : 4625 (échec auth), 1074 (shutdown/reboot), 7036 (service arrêté), 41 (crash système), 1001 (BSOD bugcheck).

Performance Monitor (perfmon)

Monitoring en temps réel des compteurs de performance :

# Lancer Performance Monitor
perfmon.msc

# Vérifier l'utilisation CPU via PowerShell
Get-Counter '\Processor(_Total)\% Processor Time'

# Mémoire disponible
Get-Counter '\Memory\Available MBytes'

# Longueur de la file d'attente disque
Get-Counter '\PhysicalDisk(_Total)\Current Disk Queue Length'

# Monitoring continu (toutes les 5 secondes, 12 fois)
Get-Counter -Counter @(
    '\Processor(_Total)\% Processor Time',
    '\Memory\Available MBytes',
    '\PhysicalDisk(_Total)\% Disk Time'
) -SampleInterval 5 -MaxSamples 12

WSUS (Windows Server Update Services)

WSUS permet de centraliser et contrôler les mises à jour Windows dans ton réseau.

Sans WSUS, chaque PC télécharge ses mises à jour directement sur Internet (= 200 employés qui téléchargent la même mise à jour de 500 MB = 100 GB de bande passante gaspillés). Avec WSUS, le serveur télécharge une seule fois, et distribue en interne. C'est un entrepôt central de pièces détachées.

Installation et configuration

# Installer WSUS
Install-WindowsFeature -Name UpdateServices -IncludeManagementTools

# Post-installation : configurer le dossier de stockage
cd "C:\Program Files\Update Services\Tools"
.\wsusutil.exe postinstall CONTENT_DIR=D:\WSUS

# Configurer les clients via GPO :
# Computer Configuration > Administrative Templates
# > Windows Components > Windows Update
# → Specify intranet Microsoft update service location
# → http://SRV-WSUS:8530

Vérifications de Santé Quotidiennes

# === SCRIPT DE SANTÉ QUOTIDIEN ===

# 1. Espace disque
Get-WmiObject Win32_LogicalDisk -Filter "DriveType=3" |
    Select-Object DeviceID,
        @{N='TailleGB';E={[math]::Round($_.Size/1GB,1)}},
        @{N='LibreGB';E={[math]::Round($_.FreeSpace/1GB,1)}},
        @{N='%Libre';E={[math]::Round($_.FreeSpace/$_.Size*100,1)}}

# 2. Services critiques
$services = "DNS","NTDS","W32Time","DHCPServer","Spooler"
$services | ForEach-Object {
    $svc = Get-Service $_ -ErrorAction SilentlyContinue
    [PSCustomObject]@{
        Nom = $_
        Statut = if($svc){$svc.Status}else{"Non installé"}
    }
}

# 3. Uptime du serveur
(Get-Date) - (Get-CimInstance Win32_OperatingSystem).LastBootUpTime

# 4. Dernière sauvegarde réussie
Get-WBSummary

# 5. Réplication AD
repadmin /replsummary
dcdiag /s:$env:COMPUTERNAME

Planning de Maintenance

FréquenceTâches
QuotidienVérifier Event Viewer, vérifier sauvegardes, vérifier espace disque, surveiller performances
HebdomadaireAppliquer les mises à jour testées, vérifier réplication AD, nettoyer fichiers temporaires
MensuelTester une restauration de sauvegarde, auditer les comptes/groupes AD, analyser les logs de sécurité, rapport WSUS

✅ Conseil de Blaise : Crée un checklist et coche chaque vérification. Un admin qui ne vérifie pas, c'est un gardien de but qui ferme les yeux pendant le match.

Dépannage : Méthodologie

Face à un problème, suis cette méthode en 6 étapes :

ÉtapeActionOutils
1Identifier le symptômeSignalements utilisateurs, monitoring
2Établir la chronologieEvent Viewer (quand ça a commencé ?)
3Chercher les changements récentsQui a modifié quoi ? MAJ ? GPO ?
4Isoler la causeTests, logs, commandes diag
5Appliquer la correctionFix, rollback, restart
6DocumenterBase de connaissances interne

Commandes de diagnostic essentielles

# Réseau
Test-NetConnection -ComputerName "SRV-DC01" -Port 389
Test-NetConnection "8.8.8.8" -TraceRoute

# DNS
Resolve-DnsName "srv-dc01.lab.local"
nslookup srv-dc01.lab.local

# Active Directory
dcdiag /v
repadmin /showrepl

# Services
Get-Service | Where-Object {$_.Status -eq 'Stopped' -and $_.StartType -eq 'Automatic'}

# Processus les plus gourmands
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10 Name,CPU,WS

Résumé du Module

  • Préventive = éviter les pannes, Corrective = réparer, Évolutive = améliorer
  • Event Viewer : le journal de bord (IDs critiques : 4625, 1074, 41)
  • Performance Monitor : CPU < 85%, RAM libre > 10%, Disk Queue < 2
  • WSUS : centralise les mises à jour, teste avant de déployer
  • Planning : quotidien → hebdo → mensuel → trimestriel → annuel
  • Dépannage en 6 étapes : symptôme → chrono → changements → isoler → corriger → documenter
