Exercices Pratiques

12 TP guidés pour mettre en pratique chaque module. Chaque exercice inclut le contexte, les étapes et la correction complète.

TP1 — Installation et Post-Installation

Contexte : Vous venez de recevoir un serveur neuf. Préparez-le pour rejoindre l'infrastructure.

Objectifs

  • Renommer le serveur en SRV-DC01
  • Configurer l'IP statique : 192.168.1.10/24, GW 192.168.1.1, DNS 192.168.1.10
  • Activer le Bureau à distance
  • Vérifier la configuration
📝 Voir la correction 
# 1. Renommer
Rename-Computer -NewName "SRV-DC01" -Restart

# 2. IP statique
New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 192.168.1.10 -PrefixLength 24 -DefaultGateway 192.168.1.1
Set-DnsClientServerAddress -InterfaceAlias "Ethernet" -ServerAddresses 192.168.1.10

# 3. Activer RDP
Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' -Name "fDenyTSConnections" -Value 0
Enable-NetFirewallRule -DisplayGroup "Remote Desktop"

# 4. Vérifier
Get-ComputerInfo | Select-Object CsName
Get-NetIPAddress -InterfaceAlias "Ethernet"

TP2 — Créer une forêt Active Directory

Contexte : Promouvoir SRV-DC01 en premier contrôleur de domaine de la forêt lab.local.

Objectifs

  • Installer le rôle AD DS
  • Promouvoir en DC avec le domaine lab.local
  • Vérifier le domaine et les rôles FSMO
📝 Voir la correction 
Install-WindowsFeature AD-Domain-Services -IncludeManagementTools
Install-ADDSForest -DomainName "lab.local" `
    -DomainNetbiosName "LAB" `
    -ForestMode "WinThreshold" `
    -DomainMode "WinThreshold" `
    -InstallDns `
    -SafeModeAdministratorPassword (ConvertTo-SecureString "Recup@2025!" -AsPlainText -Force)

# Après redémarrage :
Get-ADDomain
Get-ADForest
netdom query fsmo

TP3 — Structure AD : OU, Utilisateurs, Groupes

Contexte : Créer la structure organisationnelle pour les services IT, RH et Direction.

📝 Voir la correction 
# Créer les OUs
New-ADOrganizationalUnit -Name "Entreprise" -Path "DC=lab,DC=local"
"IT","RH","Direction" | ForEach-Object {
    New-ADOrganizationalUnit -Name $_ -Path "OU=Entreprise,DC=lab,DC=local"
}

# Créer des utilisateurs
$pass = ConvertTo-SecureString "User@2025!" -AsPlainText -Force
New-ADUser -Name "Alice Martin" -SamAccountName "amartin" -Path "OU=IT,OU=Entreprise,DC=lab,DC=local" -AccountPassword $pass -Enabled $true
New-ADUser -Name "Bob Durand" -SamAccountName "bdurand" -Path "OU=RH,OU=Entreprise,DC=lab,DC=local" -AccountPassword $pass -Enabled $true

# Créer des groupes
New-ADGroup -Name "GRP_IT" -GroupScope Global -Path "OU=IT,OU=Entreprise,DC=lab,DC=local"
Add-ADGroupMember -Identity "GRP_IT" -Members "amartin"

TP4 — Créer et appliquer une GPO de sécurité

Contexte : Imposer une politique de mots de passe forts sur tout le domaine.

📝 Voir la correction 
# Créer et lier la GPO
New-GPO -Name "Politique MDP Fort"
New-GPLink -Name "Politique MDP Fort" -Target "DC=lab,DC=local"

# Configurer via GPMC :
# Computer Config > Policies > Windows Settings
# > Security Settings > Account Policies > Password Policy
#   - Minimum password length : 12
#   - Password must meet complexity : Enabled
#   - Maximum password age : 90 days
#   - Enforce password history : 24

# Forcer l'application
gpupdate /force
gpresult /r

TP5 — Configurer DNS et DHCP

Contexte : Installer et configurer DNS et DHCP pour le réseau 192.168.1.0/24.

📝 Voir la correction 
# DNS (déjà installé avec AD DS normalement)
Add-DnsServerPrimaryZone -NetworkId "192.168.1.0/24" -ReplicationScope "Forest"
Add-DnsServerResourceRecordA -ZoneName "lab.local" -Name "srv-web" -IPv4Address 192.168.1.20

# DHCP
Install-WindowsFeature DHCP -IncludeManagementTools
Add-DhcpServerInDC -DnsName "srv-dc01.lab.local" -IPAddress 192.168.1.10
Add-DhcpServerv4Scope -Name "LAN" -StartRange 192.168.1.51 -EndRange 192.168.1.200 -SubnetMask 255.255.255.0 -LeaseDuration 08:00:00
Set-DhcpServerv4OptionValue -Router 192.168.1.1 -DnsServer 192.168.1.10 -DnsDomain "lab.local"
Add-DhcpServerv4ExclusionRange -ScopeId 192.168.1.0 -StartRange 192.168.1.1 -EndRange 192.168.1.50

# Vérifier
nslookup srv-web.lab.local
Get-DhcpServerv4Scope

TP6 — Mettre en place une sauvegarde planifiée

Contexte : Configurer une sauvegarde complète quotidienne de l'état système sur le volume E:.

📝 Voir la correction 
Install-WindowsFeature Windows-Server-Backup

$policy = New-WBPolicy
Add-WBSystemState -Policy $policy
$target = New-WBBackupTarget -VolumePath "E:"
Add-WBBackupTarget -Policy $policy -Target $target
Set-WBSchedule -Policy $policy -Schedule 22:00
Set-WBPolicy -Policy $policy

# Test immédiat
Start-WBBackup -Policy (Get-WBPolicy)
Get-WBSummary

TP7 — Créer un Failover Cluster

Contexte : Configurer un cluster à 2 nœuds pour la haute disponibilité.

📝 Voir la correction 
# Sur chaque nœud
Install-WindowsFeature Failover-Clustering -IncludeManagementTools

# Valider (OBLIGATOIRE)
Test-Cluster -Node "SRV-NODE1","SRV-NODE2"

# Créer le cluster
New-Cluster -Name "CLUSTER-PROD" `
    -Node "SRV-NODE1","SRV-NODE2" `
    -StaticAddress 192.168.1.200

# Configurer le quorum (File Share Witness)
Set-ClusterQuorum -NodeAndFileShareMajority "\\SRV-FS\ClusterWitness"

# Vérifier
Get-Cluster
Get-ClusterNode
Get-ClusterQuorum

TP8 — Script de santé quotidien

Contexte : Créer un script PowerShell de vérification de santé à exécuter chaque matin.

📝 Voir la correction 
# HealthCheck.ps1
Write-Host "=== SANTÉ SERVEUR ===" -ForegroundColor Cyan

# Uptime
$boot = (Get-CimInstance Win32_OperatingSystem).LastBootUpTime
Write-Host "Uptime : $((Get-Date) - $boot)"

# Espace disque
Get-WmiObject Win32_LogicalDisk -Filter "DriveType=3" |
    Format-Table DeviceID,
        @{N='Libre(GB)';E={[math]::Round($_.FreeSpace/1GB,1)}},
        @{N='%Libre';E={[math]::Round($_.FreeSpace/$_.Size*100,1)}}

# Services critiques arrêtés
Get-Service DNS,NTDS,W32Time,DHCPServer -ErrorAction SilentlyContinue |
    Where-Object Status -ne "Running" |
    Format-Table Name,Status

# Erreurs système (24h)
$errors = (Get-EventLog System -EntryType Error -After (Get-Date).AddHours(-24) -ErrorAction SilentlyContinue).Count
Write-Host "Erreurs système (24h) : $errors"

# Dernière sauvegarde
Get-WBSummary -ErrorAction SilentlyContinue

TP10 — Créer une VM Hyper-V

Contexte : Installer Hyper-V et créer une VM Windows Server Génération 2.

📝 Voir la correction 
Install-WindowsFeature Hyper-V -IncludeManagementTools -Restart

# Créer un switch externe
New-VMSwitch -Name "Production" -NetAdapterName "Ethernet" -AllowManagementOS $true

# Créer la VM
New-VM -Name "SRV-WEB01" -Generation 2 `
    -MemoryStartupBytes 4GB `
    -NewVHDPath "D:\VMs\SRV-WEB01\disk.vhdx" `
    -NewVHDSizeBytes 60GB `
    -SwitchName "Production"

Set-VMProcessor -VMName "SRV-WEB01" -Count 2
Set-VMMemory -VMName "SRV-WEB01" -DynamicMemoryEnabled $true -MinimumBytes 1GB -MaximumBytes 8GB
Set-VM -Name "SRV-WEB01" -CheckpointType Production
Add-VMDvdDrive -VMName "SRV-WEB01" -Path "D:\ISOs\WinServer2022.iso"

Start-VM "SRV-WEB01"

TP11 — Gérer les Snapshots

Contexte : Créer, restaurer et supprimer un snapshot avant une mise à jour risquée.

📝 Voir la correction 
# 1. Créer le snapshot AVANT la MAJ
Checkpoint-VM -Name "SRV-WEB01" -SnapshotName "Pre-MAJ-IIS"

# 2. Vérifier qu'il existe
Get-VMSnapshot -VMName "SRV-WEB01"

# 3. Faire la mise à jour...
# (si problème, restaurer) :
Restore-VMSnapshot -VMName "SRV-WEB01" -Name "Pre-MAJ-IIS" -Confirm:$false

# 4. Si tout va bien, SUPPRIMER le snapshot (<72h !)
Remove-VMSnapshot -VMName "SRV-WEB01" -Name "Pre-MAJ-IIS"

TP12 — Exporter et Importer une VM

Contexte : Exporter une VM pour la transférer vers un autre hôte Hyper-V.

📝 Voir la correction 
# Exporter (même si la VM est allumée)
Export-VM -Name "SRV-WEB01" -Path "E:\Exports"

# Copier le dossier E:\Exports\SRV-WEB01 vers l'autre hôte

# Importer sur le nouvel hôte
Import-VM -Path "E:\Exports\SRV-WEB01\Virtual Machines\*.vmcx" `
    -Copy `
    -GenerateNewId

# Vérifier
Get-VM
