Chapitre 4

State, evenements et formulaires

Faire reagir l interface sans modifier le DOM a la main.

useState

Le state permet de memoriser une valeur qui evolue: compteur, texte, filtre, panier.

useState retourne un tableau: la valeur actuelle et la fonction qui la modifie.

const [compte, setCompte] = useState(0);
setCompte(compte + 1);
Memo rapide : Valeur + setter. Tu lis la valeur, tu modifies avec le setter.

Mise a jour fonctionnelle

Quand la nouvelle valeur depend de l ancienne, utilise la forme fonctionnelle. Elle evite les surprises si plusieurs mises a jour arrivent vite.

C est particulierement utile pour les compteurs, les tableaux et les toggles.

setCompte((prev) => prev + 1);
setOn((prev) => !prev);
Piege : Ne fais jamais compte = compte + 1. Modifier directement une variable state ne declenche pas le rendu.

Formulaire controle

Un input controle a sa value liee au state et son onChange met a jour ce state.

Le composant devient la source de verite: ce qui est affiche dans le champ vient de React.

const [email, setEmail] = useState("");
<input value={email} onChange={(e) => setEmail(e.target.value)} />
Tip : Formulaire controle = value + onChange. Si un manque, le comportement devient flou.

Atelier guide

Complete un compteur simple. Le correcteur cherche les elements essentiels.

Correction guidee :
import { useState } from "react";

function Compteur() {
  const [compte, setCompte] = useState(0);
  return (
    <div>
      <p>Compteur : {compte}</p>
      <button onClick={() => setCompte((prev) => prev + 1)}>+1</button>
      <button onClick={() => setCompte(0)}>Reset</button>
    </div>
  );
}

La valeur compte est affichee. Les boutons utilisent le setter, donc React sait qu il doit re-rendre le composant.
