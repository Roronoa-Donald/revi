Chapitre 5

Listes et donnees

Afficher des tableaux avec map, key et filtrage.

map

map transforme un tableau en un autre tableau. En React, on l utilise pour transformer des donnees en elements JSX.

Chaque element genere doit avoir une key stable.

{produits.map((produit) => (
  <li key={produit.id}>{produit.nom}</li>
))}
Piege : Evite key={index} si la liste peut changer. Une suppression ou un tri peut produire des bugs visuels.

Ajouter et supprimer

Avec un tableau dans le state, on cree un nouveau tableau au lieu de modifier l ancien.

spread ajoute proprement. filter retire proprement.

setAnimaux((prev) => [...prev, nouvelAnimal]);
setAnimaux((prev) => prev.filter((a) => a.id !== id));
Memo rapide : Ajouter = [...prev, item]. Supprimer = filter.

Deriver plutot que dupliquer

Si une liste filtree peut etre calculee a partir du state existant, ne la stocke pas dans un deuxieme state.

Moins de state signifie moins de bugs de synchronisation.

const visibles = animaux.filter((animal) => animal.nom.includes(recherche));
Tip : State minimal: garde la source, calcule le reste.

Questions de fin de chapitre

Reponds avant de lire la correction. Elle se debloque apres une bonne reponse, ou apres deux erreurs.

1. Quelle méthode JavaScript utilise-t-on le plus fréquemment pour transformer un tableau de données en éléments JSX ?
Correction : map retourne un nouveau tableau d'éléments JSX à partir du tableau d'origine, parfait pour le rendu de listes.
2. Pourquoi chaque élément retourné au sein d'une boucle map doit-il posséder une prop 'key' unique ?
Correction : La key stable permet à React de réordonner les éléments physiques du DOM plutôt que de tout reconstruire, optimisant les performances.
3. Quelle valeur doit-on privilégier pour configurer la prop 'key' d'un élément de liste ?
Correction : L'identifiant unique de la base de données est le meilleur choix car il est stable d'un rendu à l'autre.
4. Quel problème peut survenir si on utilise l'index du tableau ('key={index}') dans une liste dynamique ?
Correction : Si un élément en tête est supprimé, l'ancien index 1 devient l'index 0. React peut alors associer de fausses données de DOM à l'élément restant.
5. Pourquoi est-il déconseillé d'utiliser 'Math.random()' pour générer des keys lors du rendu ?
Correction : Une clé doit être stable dans le temps. Si elle change à chaque calcul, React considère que tous les éléments sont neufs.
