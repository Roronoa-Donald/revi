Chapitre 6

useEffect et appels API

Charger des donnees apres le rendu.

Effet de bord

useEffect execute du code apres le rendu: appel API, abonnement, timer, titre du document.

Le tableau de dependances controle quand l effet se relance.

DependancesEffet
[]Une fois au montage
[id]A chaque changement de id
Aucun tableauApres chaque rendu, risque de boucle
Memo rapide : useEffect = apres rendu + dependances.

Fetch

Un appel API a au moins trois etats utiles: chargement, donnees, erreur.

Le support recommande de gerer explicitement loading et error pour eviter une page vide.

useEffect(() => {
  fetch("https://jsonplaceholder.typicode.com/users")
    .then((res) => res.json())
    .then(setUsers)
    .catch(setError);
}, []);
Piege : Ne mets pas setState dans un effet qui depend de ce meme state sans condition. C est la boucle infinie classique.

AbortController

Si un composant disparait avant la fin d une requete, on peut annuler la requete.

C est une bonne pratique pour les pages qui changent vite ou les recherches dynamiques.

const controller = new AbortController();
fetch(url, { signal: controller.signal });
return () => controller.abort();
Tip : Nettoyage = return dans useEffect.

Questions de fin de chapitre

Reponds avant de lire la correction. Elle se debloque apres une bonne reponse, ou apres deux erreurs.

1. Qu'est-ce qu'un effet de bord (side effect) en React ?
Correction : Les effets de bord connectent React avec le monde extérieur ou des APIs du navigateur après l'affichage du composant.
2. Quel Hook React permet de gérer les effets de bord ?
Correction : useEffect planifie l'exécution d'une fonction de rappel après que le rendu a été validé et injecté dans le DOM.
3. Que se passe-t-il si on omet totalement le tableau de dépendances de 'useEffect(callback)' (aucun tableau passé) ?
Correction : Sans tableau, React relance l'effet à chaque cycle. Si l'effet modifie un state, cela relance un rendu, créant une boucle infinie.
4. Comment indique-t-on à 'useEffect' de ne s'exécuter qu'une seule fois, au montage du composant ?
Correction : Le tableau vide indique à React que l'effet ne dépend d'aucune variable réactive, il ne se lancera donc qu'au chargement.
5. À quel moment s'exécute la fonction de nettoyage (cleanup) retournée par 'useEffect' ?
Correction : Le cleanup permet de nettoyer les écouteurs d'événements, d'annuler les requêtes en cours ou de stopper les timers.
