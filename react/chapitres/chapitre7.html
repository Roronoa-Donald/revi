Chapitre 7

Routing

Construire une mini application multi-pages.

React Router DOM

React ne gere pas les routes tout seul. On ajoute react-router-dom pour faire une SPA avec plusieurs vues.

Link navigue sans rechargement complet. Routes associe un chemin a un composant.

npm i react-router-dom
Memo rapide : BrowserRouter autour, Link pour naviguer, Routes/Route pour declarer.

Route dynamique

Une route comme /utilisateur/:id capture un parametre.

useParams permet de lire cet id et de charger la bonne ressource.

const { id } = useParams();
fetch(`https://jsonplaceholder.typicode.com/users/${id}`);
Piege : Si la route peut etre inconnue, ajoute une route * pour afficher une page 404 propre.

Deploiement

npm run build cree le dossier dist. Sur Netlify, Vercel ou GitHub Pages, il faut parfois une regle de redirection pour les routes cote client.

Le test local du build evite les surprises apres publication.

npm run build
npx serve -s dist
Tip : Dev OK ne veut pas dire build OK. Toujours tester le build.

Questions de fin de chapitre

Reponds avant de lire la correction. Elle se debloque apres une bonne reponse, ou apres deux erreurs.

1. Quelle bibliothèque standard utilise-t-on pour gérer la navigation et les routes dans une application React ?
Correction : react-router-dom est le standard pour gérer la navigation côté client dans les Single Page Applications React pour le web.
2. Quel composant doit envelopper l'intégralité de l'application pour activer le routage ?
Correction : BrowserRouter utilise l'API History d'HTML5 pour synchroniser l'URL du navigateur avec l'application.
3. Comment déclare-t-on une association entre le chemin '/contact' et le composant '<Contact />' ?
Correction : Dans React Router v6, on utilise le composant Route avec les props path et element.
4. Quel composant remplace la balise HTML classique '<a href="...">' pour naviguer sans recharger la page en React ?
Correction : Le composant Link intercepte le clic pour mettre à jour l'URL et l'affichage localement sans faire de requête de page au serveur.
5. Comment déclare-t-on une route dynamique capturant un paramètre nommé 'id' (ex: pour afficher un profil) ?
Correction : Le symbole deux-points (:) indique à React Router qu'il s'agit d'un paramètre dynamique de chemin.
