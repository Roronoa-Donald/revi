Chapitre 8

Projet Todo List

Assembler props, state, listes, filtres et localStorage.

Fonctionnalites attendues

Le projet Todo List reprend l ensemble du cours: ajout, suppression, completion, filtrage, compteur et persistance locale.

L objectif n est pas de faire joli d abord; l objectif est de structurer les donnees et les interactions.

  • Une tache a id, texte, completed.
  • Le filtre change l affichage sans supprimer les donnees.
  • localStorage sauvegarde et restaure les taches.
  • Le compteur derive des taches actives.

Persistance locale

Au montage, on lit localStorage. A chaque modification des taches, on sauvegarde.

JSON.stringify transforme le tableau en texte. JSON.parse relit le texte.

useEffect(() => {
  localStorage.setItem("tasks", JSON.stringify(tasks));
}, [tasks]);
Memo rapide : localStorage ne stocke que du texte: stringify pour entrer, parse pour sortir.

Plan de code

Decoupe possible: TodoForm pour ajouter, TodoItem pour afficher une tache, TodoFilters pour changer le filtre, App pour coordonner.

Cette decoupe rend le projet lisible et proche d un vrai code professionnel.

Piege : Ne mets pas la logique de filtrage dans chaque TodoItem. Le parent decide la liste visible.

Questions de fin de chapitre

Reponds avant de lire la correction. Elle se debloque apres une bonne reponse, ou apres deux erreurs.

1. Dans le projet de Todo List, pourquoi sépare-t-on la logique de filtrage des composants enfants comme 'TodoItem' ?
Correction : La centralisation de la logique dans le composant parent coordonnateur simplifie le flux de données et évite les désynchronisations.
2. Comment sauvegarde-t-on le tableau des tâches dans le navigateur pour qu'il survive au rechargement ?
Correction : localStorage stocke des données textuelles de manière persistante sur le poste du client sans date d'expiration.
3. Quelle méthode permet de lire et restaurer les tâches sauvegardées dans le localStorage au démarrage de l'application ?
Correction : On récupère le texte avec getItem, puis JSON.parse le retransforme en tableau exploitable par React.
4. Que doit-on prévoir si 'localStorage.getItem()' ne trouve aucune tâche sauvegardée (première visite) ?
Correction : Il faut toujours prévoir une valeur de repli (fallback) pour éviter de passer une valeur nulle à useState.
5. Où place-t-on idéalement le code de sauvegarde automatique du localStorage dans la Todo List ?
Correction : useEffect s'exécutera automatiquement après chaque modification des tâches, assurant une sauvegarde transparente.
