Chapitre 8

Projet Todo List

Assembler props, state, listes, filtres et localStorage.

Fonctionnalites attendues

Le projet Todo List reprend l ensemble du cours: ajout, suppression, completion, filtrage, compteur et persistance locale.

L objectif n est pas de faire joli d abord; l objectif est de structurer les donnees et les interactions.

  • Une tache a id, texte, completed.
  • Le filtre change l affichage sans supprimer les donnees.
  • localStorage sauvegarde et restaure les taches.
  • Le compteur derive des taches actives.

Persistance locale

Au montage, on lit localStorage. A chaque modification des taches, on sauvegarde.

JSON.stringify transforme le tableau en texte. JSON.parse relit le texte.

useEffect(() => {
  localStorage.setItem("tasks", JSON.stringify(tasks));
}, [tasks]);
Memo rapide : localStorage ne stocke que du texte: stringify pour entrer, parse pour sortir.

Plan de code

Decoupe possible: TodoForm pour ajouter, TodoItem pour afficher une tache, TodoFilters pour changer le filtre, App pour coordonner.

Cette decoupe rend le projet lisible et proche d un vrai code professionnel.

Piege : Ne mets pas la logique de filtrage dans chaque TodoItem. Le parent decide la liste visible.
