Exercices progressifs

Applique les notions avec des corrections qui se debloquent apres reussite ou deux echecs.

Exercice 1 - Carte de presentation

Cree un composant CartePresentation qui affiche ton nom, ton role et trois technologies.

Correction :
function CartePresentation() {
  return (
    <div className="carte">
      <h1>Clarisse</h1>
      <p>Developpeuse React en devenir</p>
      <ul><li>HTML</li><li>CSS</li><li>React</li></ul>
    </div>
  );
}

Le composant est une fonction et retourne un seul bloc JSX.

Exercice 2 - Produit avec props

Cree un composant Produit avec nom, prix, disponible et categorie optionnelle.

Correction :
function Produit({ nom, prix, disponible, categorie }) {
  return <article><h2>{nom}</h2><p>{prix} EUR</p><p>{disponible ? "En stock" : "Rupture"}</p>{categorie && <small>{categorie}</small>}</article>;
}

Les props rendent le composant reutilisable sans dupliquer le code.

Exercice 3 - Compteur

Implemente un compteur avec +, -, reset et pas dynamique.

Correction :
const [compte, setCompte] = useState(0);
const [pas, setPas] = useState(1);
<button onClick={() => setCompte(c => c + pas)}>+{pas}</button>
<input type="number" value={pas} onChange={e => setPas(+e.target.value)} />

Le pas est lui aussi un state, car il change avec l input.

Exercice 4 - Apercu texte

Cree un formulaire controle qui affiche la previsualisation en temps reel.

Correction :
const [texte, setTexte] = useState("");
<input value={texte} onChange={e => setTexte(e.target.value)} />
<textarea value={texte} onChange={e => setTexte(e.target.value)} />
<p>{texte}</p>

value + onChange rendent le champ controle par React.

Exercice 5 - Liste animaux

Affiche une liste depuis un tableau et ajoute/supprime sans utiliser index comme key.

Correction :
{animaux.map(a => <li key={a.id}>{a.nom}<button onClick={() => setAnimaux(prev => prev.filter(x => x.id !== a.id))}>Supprimer</button></li>)}

La key stable permet a React de suivre chaque item.

Exercice 6 - Liste utilisateurs API

Charge les utilisateurs jsonplaceholder avec loading et error.

Correction :
useEffect(() => {
  setLoading(true);
  fetch(url).then(r => r.json()).then(setUsers).catch(setError).finally(() => setLoading(false));
}, []);

Un appel API propre gere chargement, succes et erreur.

Exercice 7 - Routing simple

Cree Accueil, Services, Contact et une route 404.

Correction :
<Routes>
  <Route path="/" element={<Accueil />} />
  <Route path="/services" element={<Services />} />
  <Route path="/contact" element={<Contact />} />
  <Route path="*" element={<NotFound />} />
</Routes>

La route * capture les chemins inconnus.

Exercice 8 - Profil dynamique

Utilise useParams pour charger /utilisateur/:id.

Correction :
const { id } = useParams();
useEffect(() => { fetch(`/users/${id}`).then(r => r.json()).then(setUser); }, [id]);

id est une dependance: si l URL change, l effet doit relancer la requete.

Exercice 9 - Build production

Explique les commandes pour generer et tester dist.

Correction :
npm run build
npx serve -s dist

Le build verifie que l application peut etre livree hors serveur de developpement.

Exercice 10 - Todo List

Decris la structure state du projet Todo.

Correction :
const task = { id: Date.now(), text: "Reviser React", completed: false };
localStorage.setItem("tasks", JSON.stringify(tasks));

Un bon modele de donnees rend le reste du projet simple.
