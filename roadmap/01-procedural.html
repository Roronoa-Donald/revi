Roadmap PHP vs C# — Procédural TP PHP

Programmation Procédurale

Variables, types, tableaux, boucles, fonctions — La syntaxe de base, côte à côte.

Variables & Types

PHP est dynamiquement typé (pas besoin de déclarer le type). C# est statiquement typé (type obligatoire).

PHP
// Pas de déclaration de type
$nom = "Alice";        // string
$age = 22;             // int
$moyenne = 14.5;       // float
$estInscrit = true;    // bool

// Concaténation avec .
echo $nom . " a " . $age . " ans";

// Type juggling (conversion auto)
$resultat = "10" + 5;  // 15 (int!)
C#
// Type obligatoire à la déclaration
string nom = "Alice";     // string
int age = 22;              // int
double moyenne = 14.5;     // double
bool estInscrit = true;    // bool

// Concaténation avec + ou $""
Console.WriteLine($"{nom} a {age} ans");

// Pas de conversion auto string→int
// int resultat = "10" + 5; ❌ Erreur!
int resultat = int.Parse("10") + 5; // ✅
⚡ Piège examen : En PHP, "10" + 5 donne 15 (conversion automatique). En C#, c'est une erreur de compilation. C'est LA différence fondamentale entre typage dynamique et statique.

Tableaux

PHP a des tableaux indexés ET associatifs dans le même type. C# sépare les arrays, List et Dictionary.

PHP
// Tableau indexé
$notes = [15, 12, 18, 9];
echo $notes[0]; // 15

// Tableau associatif
$etudiant = [
    "nom" => "Alice",
    "age" => 22,
    "moyenne" => 14.5
];
echo $etudiant["nom"]; // Alice

// Ajouter un élément
$notes[] = 16; // ajoute à la fin

// Fonctions utiles
count($notes);       // 5
array_sum($notes);   // 70
sort($notes);        // tri
in_array(18, $notes); // true
C#
// Array (taille fixe)
int[] notes = {15, 12, 18, 9};
Console.WriteLine(notes[0]); // 15

// Dictionary (= associatif PHP)
Dictionary<string, object> etudiant = new() {
    {"nom", "Alice"},
    {"age", 22},
    {"moyenne", 14.5}
};
Console.WriteLine(etudiant["nom"]);

// List (taille dynamique)
List<int> notesList = new() {15, 12, 18, 9};
notesList.Add(16); // ajoute à la fin

// Équivalents
notesList.Count;          // 5
notesList.Sum();          // 70 (avec using System.Linq)
notesList.Sort();
notesList.Contains(18);   // true
💡 Truc à retenir : En PHP, $tab[] ajoute un élément. En C#, les arrays ont une taille fixe → utilise List<T> pour l'équivalent.

Boucles & Conditions

if/else, for, while sont quasi identiques. La grande différence : foreach.

PHP
// if/else — identique
if ($note >= 10) {
    echo "Admis";
} elseif ($note >= 8) {
    echo "Rattrapage";
} else {
    echo "Recalé";
}

// for — identique
for ($i = 0; $i < 5; $i++) {
    echo $i;
}

// foreach — syntaxe propre à PHP
$notes = [15, 12, 18];
foreach ($notes as $note) {
    echo $note;
}

// foreach associatif
foreach ($etudiant as $cle => $valeur) {
    echo "$cle : $valeur";
}
C#
// if/else — identique
if (note >= 10) {
    Console.WriteLine("Admis");
} else if (note >= 8) {    // else if (2 mots)
    Console.WriteLine("Rattrapage");
} else {
    Console.WriteLine("Recalé");
}

// for — identique
for (int i = 0; i < 5; i++) {
    Console.Write(i);
}

// foreach — avec in au lieu de as
int[] notes = {15, 12, 18};
foreach (int note in notes) {
    Console.Write(note);
}

// foreach Dictionary
foreach (var kvp in etudiant) {
    Console.WriteLine($"{kvp.Key} : {kvp.Value}");
}
⚡ Différences clés :
• PHP: elseif (1 mot) — C#: else if (2 mots)
• PHP: foreach ($arr as $val) — C#: foreach (Type val in arr)
• PHP: $cle => $valeur — C#: kvp.Key / kvp.Value

Fonctions

PHP : function + pas de type retour obligatoire. C# : type retour obligatoire + static dans une classe.

PHP
// Fonction simple
function saluer($nom) {
    return "Bonjour $nom !";
}
echo saluer("Alice");

// Avec valeur par défaut
function calculerMoyenne($notes, $bonus = 0) {
    $somme = array_sum($notes);
    $nb = count($notes);
    return ($somme / $nb) + $bonus;
}

echo calculerMoyenne([15, 12, 18]);
echo calculerMoyenne([15, 12], 1); // avec bonus

// Avec type hints (PHP 7+)
function isReussi(float $moy): bool {
    return $moy >= 10;
}
C#
// Fonction (= méthode static)
static string Saluer(string nom) {
    return $"Bonjour {nom} !";
}
Console.WriteLine(Saluer("Alice"));

// Avec valeur par défaut
static double CalculerMoyenne(int[] notes,
                               double bonus = 0) {
    double somme = notes.Sum();
    int nb = notes.Length;
    return (somme / nb) + bonus;
}

Console.WriteLine(CalculerMoyenne(new[]{15,12,18}));
Console.WriteLine(CalculerMoyenne(new[]{15,12}, 1));

// Type retour OBLIGATOIRE
static bool IsReussi(double moy) {
    return moy >= 10;
}
🔑 Règle C# : Toute fonction doit être dans une class. Pour du "procédural", on utilise static. En PHP, les fonctions existent hors des classes.

Exercice — Bulletin de notes (PHP puis C#)

Écris le même programme dans les deux langages :

Consigne :
1. Crée un tableau associatif/Dictionary avec 3 matières et leurs notes
2. Écris une fonction calculerMoyenne qui renvoie la moyenne
3. Affiche chaque matière et sa note avec une boucle
4. Affiche la moyenne et "Admis" si ≥ 10, "Recalé" sinon
PHP
<?php
$bulletin = [
    "Maths" => 14,
    "PHP" => 16,
    "Réseaux" => 11
];

function calculerMoyenne($notes) {
    return array_sum($notes) / count($notes);
}

echo "=== Bulletin ===\n";
foreach ($bulletin as $matiere => $note) {
    echo "$matiere : $note/20\n";
}

$moy = calculerMoyenne($bulletin);
echo "\nMoyenne : " . round($moy, 2) . "/20\n";
echo $moy >= 10 ? "✅ Admis" : "❌ Recalé";
C#
using System;
using System.Linq;
using System.Collections.Generic;

Dictionary<string, int> bulletin = new() {
    {"Maths", 14},
    {"C#", 16},
    {"Réseaux", 11}
};

static double CalculerMoyenne(
    Dictionary<string, int> notes) {
    return notes.Values.Average();
}

Console.WriteLine("=== Bulletin ===");
foreach (var kvp in bulletin) {
    Console.WriteLine($"{kvp.Key} : {kvp.Value}/20");
}

double moy = CalculerMoyenne(bulletin);
Console.WriteLine($"\nMoyenne : {moy:F2}/20");
Console.WriteLine(moy >= 10 ? "✅ Admis" : "❌ Recalé");
Pourquoi c'est important : Ce pattern (tableau → boucle → calcul → condition) est la base de TOUT exercice procédural, que ce soit en PHP ou C#. La logique est identique, seule la syntaxe change.
