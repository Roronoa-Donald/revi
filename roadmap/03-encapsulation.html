Roadmap PHP vs C# — Encapsulation TP PHP

Encapsulation

Protéger les données avec private, getters/setters, validation — Exercice 2 de l'examen.

Les 3 niveaux de visibilité

Identiques en PHP et C#. La seule différence : PHP a un $, C# a un type.

Mot-clé Même classe Classe enfant Extérieur Quand l'utiliser
public ✅ ✅ ✅ Méthodes que l'extérieur doit appeler
private ✅ ❌ ❌ Attributs (données à protéger)
protected ✅ ✅ ❌ Attributs partagés avec enfants
⚡ Règle d'or (identique PHP & C#) : Attributs = private. Accès = via getters/setters public. C'est ça l'encapsulation.

Getters & Setters

C'est ici que PHP et C# diffèrent le plus. PHP utilise des méthodes classiques, C# utilise des propriétés.

PHP — Méthodes get/set
class Livre {
    private $titre;
    private $prix;

    public function __construct($titre, $prix) {
        $this->titre = $titre;
        $this->setPrix($prix); // validation!
    }

    // GETTER
    public function getTitre() {
        return $this->titre;
    }

    // GETTER
    public function getPrix() {
        return $this->prix;
    }

    // SETTER avec validation
    public function setPrix($prix) {
        if ($prix < 0) {
            throw new Exception("Prix négatif !");
        }
        $this->prix = $prix;
    }
}

$l = new Livre("PHP", 25);
echo $l->getTitre();    // PHP
// $l->titre;           // ❌ ERREUR (private)
$l->setPrix(30);        // ✅ via setter
// $l->setPrix(-5);     // ❌ Exception !
C# — Propriétés { get; set; }
class Livre {
    private string titre;
    private double prix;

    public Livre(string titre, double prix) {
        this.titre = titre;
        Prix = prix; // passe par le set → validation
    }

    // PROPRIÉTÉ en lecture seule
    public string Titre {
        get { return titre; }
    }

    // PROPRIÉTÉ avec validation dans set
    public double Prix {
        get { return prix; }
        set {
            if (value < 0)
                throw new Exception("Prix négatif !");
            prix = value;  // "value" = valeur assignée
        }
    }
}

Livre l = new Livre("C#", 25);
Console.WriteLine(l.Titre);  // C#
// l.titre → ❌ private
l.Prix = 30;                 // ✅ passe par le set
// l.Prix = -5;              // ❌ Exception !
🔑 La clé C# : Le mot value dans un set représente automatiquement la valeur qu'on essaie d'assigner. C'est comme le paramètre $prix de setPrix($prix) en PHP, mais implicite.
💡 Raccourci C# : Si pas de validation, utilise les propriétés auto :
public string Titre { get; set; } — C# génère automatiquement le champ privé.

Exemple complet — Classe encapsulée

Le schéma complet tel qu'il tomberait à l'examen :

PHP
class Livre {
    private $titre;
    private $auteur;
    private $prix;

    public function __construct($titre, $auteur, $prix) {
        $this->titre = $titre;
        $this->auteur = $auteur;
        $this->setPrix($prix);
    }

    public function getTitre() { return $this->titre; }
    public function getAuteur() { return $this->auteur; }
    public function getPrix() { return $this->prix; }

    public function setPrix($prix) {
        if ($prix < 0) {
            throw new Exception("Prix invalide");
        }
        $this->prix = $prix;
    }

    public function afficher() {
        return "{$this->titre} par {$this->auteur}"
             . " — {$this->prix} €";
    }
}
C#
class Livre {
    public string Titre { get; private set; }
    public string Auteur { get; private set; }

    private double prix;
    public double Prix {
        get => prix;
        set {
            if (value < 0)
                throw new Exception("Prix invalide");
            prix = value;
        }
    }

    public Livre(string titre, string auteur, double prix) {
        Titre = titre;
        Auteur = auteur;
        Prix = prix; // validation via set
    }

    public string Afficher() {
        return $"{Titre} par {Auteur} — {Prix} €";
    }
}
⚡ Comparaison rapide :
Lecture seulePHP: pas de set | C#: { get; private set; }
ValidationPHP: dans setPrix() | C#: dans set { }
AccèsPHP: $l->getPrix() | C#: l.Prix (comme un champ!)

Exercice type examen — Encapsulation Livre (PHP + C#)

Consigne (identique examen 2024/2025, exercice 2) :
1. Reprends la classe Livre avec attributs private
2. Ajoute getters pour titre et auteur
3. Ajoute getter + setter pour prix avec validation (prix ≥ 0)
4. Modifie le constructeur pour utiliser le setter
5. Teste avec un prix valide puis un prix négatif
PHP
<?php
class Livre {
    private $titre;
    private $auteur;
    private $prix;

    public function __construct($t, $a, $p) {
        $this->titre = $t;
        $this->auteur = $a;
        $this->setPrix($p);
    }

    public function getTitre() { return $this->titre; }
    public function getAuteur() { return $this->auteur; }
    public function getPrix() { return $this->prix; }

    public function setPrix($p) {
        if ($p < 0) {
            throw new Exception("Prix négatif interdit");
        }
        $this->prix = $p;
    }
}

// Test
$l = new Livre("PHP POO", "Alice", 25);
echo $l->getTitre() . " — " . $l->getPrix() . "€\n";

$l->setPrix(30);
echo "Nouveau prix : " . $l->getPrix() . "€\n";

try {
    $l->setPrix(-10); // ❌
} catch (Exception $e) {
    echo "Erreur : " . $e->getMessage();
}
C#
class Livre {
    public string Titre { get; private set; }
    public string Auteur { get; private set; }

    private double prix;
    public double Prix {
        get => prix;
        set {
            if (value < 0)
                throw new Exception("Prix négatif interdit");
            prix = value;
        }
    }

    public Livre(string t, string a, double p) {
        Titre = t;
        Auteur = a;
        Prix = p; // passe par set → validation
    }
}

// Test
Livre l = new Livre("C# POO", "Alice", 25);
Console.WriteLine($"{l.Titre} — {l.Prix}€");

l.Prix = 30;
Console.WriteLine($"Nouveau prix : {l.Prix}€");

try {
    l.Prix = -10; // ❌
} catch (Exception e) {
    Console.WriteLine($"Erreur : {e.Message}");
}
Pourquoi c'est important : L'encapsulation représente 5 points à l'examen. Le prof vérifie 3 choses : (1) attributs private, (2) getters/setters public, (3) validation dans le setter. Si tu fais ces 3 choses → 5/5.
