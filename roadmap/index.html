Cours C# Portail

C# → PHP

Tu connais C#, il te manque juste la syntaxe PHP. Lis les différences, retiens les mémos, fonce aux exercices.

80% de la logique est identique. Seule la syntaxe change. Cette page te montre uniquement ce qui diffère, avec un mémo-technique à chaque fois pour le retenir en 5 secondes. Après lecture, tu peux attaquer les TPs directement.

1

Variables & Bases

TP1 — Procédural
Déclarer une variable
string nom = "Alice"; int age = 20;
→
$nom = "Alice"; $age = 20;
$ = Dollar = Data. En PHP toute variable commence par $. Pas besoin de déclarer le type.
Tableau
string[] fruits = {"pomme", "banane"}; fruits.Length
→
$fruits = ["pomme", "banane"]; count($fruits)
Crochets [ ] et count(). PHP utilise [ ] au lieu de { } pour les tableaux, et count() au lieu de .Length.
Fonction
static int Additionner(int a, int b) { return a + b; }
→
function additionner($a, $b) { return $a + $b; }
function au lieu de static + type. Pas de type de retour, pas de static, juste le mot function. Et les paramètres ont un $.
Affichage
Console.WriteLine("Bonjour"); Console.WriteLine($"Age: {age}");
→
echo "Bonjour"; echo "Age: $age";
echo remplace Console.WriteLine. L'interpolation est directe dans les guillemets doubles : "$var" suffit (pas besoin de $ devant les accolades).
Concaténation
"Hello " + nom
→
"Hello " . $nom
Le point . remplace le + pour concaténer. Retiens : en PHP le + est réservé aux maths.
2

Classes & Objets

TP2 — POO
Constructeur
class Livre { public Livre(string titre) { this.titre = titre; } }
→
class Livre { public function __construct($titre) { $this->titre = $titre; } }
__construct() = le constructeur magique. 2 underscores + construct. Accès aux attributs : $this-> avec la flèche au lieu du point.
Accès aux membres
livre.GetTitre() livre.titre
→
$livre->getTitre() $livre->titre
La flèche -> remplace le point. Retiens : en PHP le . c'est la concaténation, donc on utilise -> pour les objets.
Instanciation
Livre l = new Livre("PHP");
→
$l = new Livre("PHP");
Pas de type devant la variable. new est identique ! Juste $ devant le nom et pas de type.
3

Encapsulation

TP3 — Encapsulation
Propriété / Getter-Setter
private string titre; public string Titre { get { return titre; } set { titre = value; } }
→
private $titre; public function getTitre() { return $this->titre; } public function setTitre($t) { $this->titre = $t; }
Pas de { get; set; } en PHP. Tu dois écrire les méthodes getTitre() et setTitre() toi-même. C'est la différence la plus visible à l'examen.
Validation dans le setter
set { if (value < 0) throw new ArgumentException(); prix = value; }
→
public function setPrix($p) { if ($p < 0) throw new Exception("Prix négatif"); $this->prix = $p; }
Le paramètre remplace value. En C# le setter reçoit value implicitement. En PHP, c'est toi qui nommes le paramètre ($p).
4

Héritage & Polymorphisme

TP4 — Héritage
Hériter d'une classe
class Roman : Livre { public Roman(string t) : base(t) { } }
→
class Roman extends Livre { public function __construct($t) { parent::__construct($t); } }
extends au lieu de : et parent:: au lieu de base(). Retiens : PHP est plus "verbeux" → extends (7 lettres) et parent:: (avec ::).
Surcharge de méthode (override)
// Classe parente public virtual string Afficher() { ... } // Enfant public override string Afficher() { ... }
→
// Classe parente public function afficher() { ... } // Enfant — juste réécrire ! public function afficher() { ... }
Pas de virtual/override en PHP ! PHP surcharge automatiquement : tu réécris la méthode, c'est tout. C'est plus simple qu'en C#.
Classe abstraite & Interface
abstract class Forme { } interface IDessinable { }
→
abstract class Forme { } interface Dessinable { }
Identique sauf le I. Les interfaces en PHP n'ont pas le préfixe I. abstract et interface sont les mêmes mots-clés.
Implémenter une interface
class Cercle : Forme, IDessinable
→
class Cercle extends Forme implements Dessinable
implements pour les interfaces. En C# tout est avec :. En PHP : extends pour la classe, implements pour les interfaces.
5

Exceptions

TP5 — Exceptions
Try / Catch
try { // ... } catch (Exception ex) { Console.WriteLine(ex.Message); }
→
try { // ... } catch (Exception $ex) { echo $ex->getMessage(); }
$ devant la variable + getMessage() avec flèche. La structure try/catch est identique ! Seul l'accès au message change : $ex->getMessage() vs ex.Message.
Exception personnalisée
class PrixNegatifException : Exception { public PrixNegatifException(string msg) : base(msg) { } }
→
class PrixNegatifException extends Exception { public function __construct($msg) { parent::__construct($msg); } }
extends + parent:: — la même règle que l'héritage. C'est logique : une exception personnalisée, c'est juste de l'héritage.
