50 Exercices

🥋 Entraînement Intensif

La maîtrise vient par la pratique. Voici 50 scénarios progressifs pour tester vos connaissances, du simple SELECT aux fenêtrages avancés. Utilisez les indices si vous bloquez !

Progression 0/50 (0%)
01 - DDL

Architecte de Base de Données

Contexte : Vous lancez une startup EdTech et devez stocker les profils étudiants.
Objectif : Créez une table Etudiants.
Détails : Colonnes id (INT, Clé Primaire), nom (VARCHAR 100), et moyenne (DECIMAL 4,2).

CREATE TABLE Etudiants (
  id INT PRIMARY KEY,
  nom VARCHAR(100),
  moyenne DECIMAL(4,2)
);
02 - DDL

Évolution du Schéma

Contexte : L'équipe marketing souhaite lancer une newsletter mais n'a pas les adresses des clients.
Objectif : Modifiez la table Clients existante pour ajouter une colonne.
Action : Ajoutez email (VARCHAR 255).

ALTER TABLE Clients
ADD email VARCHAR(255);
03 - DDL

Sécurité des Données

Contexte : Un audit de sécurité révèle des doublons d'emails problématiques pour l'authentification.
Objectif : Imposez une contrainte d'unicité sur la colonne email.
Commande : Ajoutez une contrainte UNIQUE sur la table Users.

ALTER TABLE Users
ADD CONSTRAINT uq_email UNIQUE (email);
04 - DDL

Nettoyage RGPD

Contexte : Pour respecter la législation, nous devons supprimer les données obsolètes.
Objectif : Supprimez définitivement une table d'archivage.
Cible : Table Old_Logs.

DROP TABLE Old_Logs;
05 - DDL

Standardisation

Contexte : Le CTO impose une nomenclature francophone pour tout le projet.
Objectif : Renommez la colonne fullname en nom_complet.
Table : Employes.

-- SQL Server
EXEC sp_rename 'Employes.fullname', 'nom_complet', 'COLUMN';

-- PostgreSQL / MySQL
ALTER TABLE Employes RENAME COLUMN fullname TO nom_complet;
06 - DML

Gestion de Stock

Contexte : Une nouvelle référence produit vient d'arriver à l'entrepôt.
Objectif : Insérez ce nouveau produit dans la base.
Données : 'T-Shirt', Prix: 19.99, Stock: 100.

INSERT INTO Produits (nom, prix, stock)
VALUES ('T-Shirt', 19.99, 100);
07 - DML

Activation en Masse

Contexte : Après vérification manuelle, tous les comptes créés cette année sont validés.
Objectif : Mettez à jour leur statut à 'Actif'.
Cible : Users créés en 2025.

UPDATE Users
SET statut = 'Actif'
WHERE YEAR(created_at) = 2025;
08 - DML

Maintenance Serveur

Contexte : Le disque dur est plein. L'admin système demande de purger les logs inutiles.
Objectif : Supprimez les alertes peu importantes.
Condition : Niveau 'Low' ou 'Info'.

DELETE FROM Alertes
WHERE niveau IN ('Low', 'Info');
09 - DML

Archivage de Sécurité

Contexte : Avant de purger les vieux messages, nous devons en garder une copie.
Objectif : Copiez les messages d'avant 2020 dans la table Archive.
Méthode : INSERT INTO ... SELECT ...

INSERT INTO Archive (msg, date)
SELECT msg, date FROM Messages WHERE date < '2020-01-01';
10 - DML

Nettoyage de Données

Contexte : L'interface affiche "null" pour les téléphones manquants, ce qui n'est pas professionnel.
Objectif : Remplacez les valeurs NULL par le texte 'Non renseigné'.
Table : Clients.

UPDATE Clients
SET telephone = 'Non renseigné'
WHERE telephone IS NULL;
11 - SELECT

Moteur de Recherche

Contexte : L'utilisateur tape "Mar" dans la barre de recherche du CRM.
Objectif : Trouvez tous les clients dont le nom commence par 'M'.
Outil : Clause LIKE avec wildcard.

SELECT * FROM Clients
WHERE nom LIKE 'M%';
12 - SELECT

Catalogue Papier

Contexte : Pour l'impression du catalogue, les produits doivent être groupés par rayon, puis mis en avant par prix.
Objectif : Triez par Catégorie (A-Z), puis par Prix (Décroissant).
Clause : ORDER BY ... ASC, ... DESC.

SELECT * FROM Produits
ORDER BY categorie ASC, prix DESC;
13 - SELECT

Pagination Web

Contexte : Le site affiche 10 résultats par page. L'utilisateur clique sur "Page 2".
Objectif : Récupérez les enregistrements 11 à 20.
Méthode : LIMIT / OFFSET.

-- Standard SQL / MySQL / PG
SELECT * FROM Table
ORDER BY id
LIMIT 10 OFFSET 10;
14 - SELECT

Affichage Prix TTC

Contexte : La base stocke du Hors Taxe, mais le client doit voir le TTC (TVA 20%).
Objectif : Calculez une colonne virtuelle 'Prix TTC'.
Formule : prix * 1.2 avec Alias.

SELECT nom, prix * 1.2 AS prix_ttc
FROM Produits;
15 - SELECT

Cartographie

Contexte : L'équipe logistique veut une liste des villes desservies pour optimiser les tournées.
Objectif : Listez chaque ville une seule fois, sans doublon.
Mot-clé : DISTINCT.

SELECT DISTINCT ville
FROM Clients;
16 - CALCULS

Rapport Financier

Contexte : Le directeur financier demande le chiffre d'affaires du mois de Mars 2024.
Objectif : Calculez la somme des montants des ventes.
Filtre : Mois = 3, Année = 2024.

SELECT SUM(montant)
FROM Ventes
WHERE MONTH(date) = 3 AND YEAR(date) = 2024;
17 - CALCULS

Inventaire par Rayon

Contexte : Le responsable magasin veut savoir combien de références existent dans chaque rayon.
Objectif : Comptez les produits groupés par catégorie.
Fonctions : COUNT et GROUP BY.

SELECT categorie, COUNT(*)
FROM Produits
GROUP BY categorie;
18 - CALCULS

Analyse Salariale

Contexte : Les RH veulent identifier les départements les plus coûteux.
Objectif : Affichez les départements dont le salaire moyen dépasse 50,000.
Indice : Utilisez HAVING après le GROUP BY.

SELECT dept, AVG(salaire)
FROM Employes
GROUP BY dept
HAVING AVG(salaire) > 50000;
19 - CALCULS

Audit Temporel

Contexte : Pour calculer la "Lifetime Value", nous avons besoin de l'amplitude d'activité.
Objectif : Trouvez la date de la toute première et de la toute dernière commande.
Fonctions : MIN() et MAX().

SELECT MIN(date_cmd), MAX(date_cmd)
FROM Commandes;
20 - CALCULS

Base Clients Active

Contexte : Le marketing veut savoir combien de personnes *différentes* ont acheté quelque chose.
Objectif : Comptez les `client_id` uniques dans la table Commandes.
Syntaxe : COUNT(DISTINCT col).

SELECT COUNT(DISTINCT client_id)
FROM Commandes;
21 - LOGIC

Produits Premium

Contexte : Le marketing veut mettre en avant les produits "Haut de gamme".
Objectif : Sélectionnez les produits dont le prix est supérieur à la moyenne globale.
Outil : Sous-requête dans le WHERE.

SELECT * FROM Produits
WHERE prix > (
  SELECT AVG(prix) FROM Produits
);
22 - LOGIC

Ciblage Précis

Contexte : Promotion flash sur les articles bleus ou rouges bon marché.
Objectif : Produits (Rouge OU Bleu) ET (Prix <= 50).
Attention : Aux parenthèses !

SELECT * FROM Produits
WHERE (couleur = 'Rouge' OR couleur = 'Bleu')
AND prix <= 50;
23 - LOGIC

Étiquetage Automatique

Contexte : L'affichage client doit montrer une catégorie de prix simplifiée.
Objectif : Créez une colonne 'Etiquette' : 'Cher' si > 100, sinon 'Abordable'.
Syntaxe : CASE WHEN ... THEN ... END.

SELECT nom,
CASE
  WHEN prix > 100 THEN 'Cher'
  ELSE 'Abordable'
END as etiquette
FROM Produits;
24 - LOGIC

Programme VIP

Contexte : Nous avons une table séparée pour les VIPs. Qui sont-ils dans la table principale ?
Objectif : Sélectionnez les Clients dont l'ID est dans la table VIP.
Méthode : WHERE id IN (...).

SELECT * FROM Clients
WHERE id IN (SELECT client_id FROM VIP);
25 - LOGIC

Localisation Date

Contexte : L'interface utilisateur française attend un format jour-mois-année.
Objectif : Formatez la date de commande en 'JJ-MM-AAAA'.
Note : La fonction dépend du SGBD (FORMAT, DATE_FORMAT...).

-- SQL Server
SELECT FORMAT(date, 'dd-MM-yyyy')

-- MySQL
SELECT DATE_FORMAT(date, '%d-%m-%Y')
26 - JOIN

Facturation

Contexte : Pour générer une facture, nous avons besoin du nom du client sur chaque commande.
Objectif : Jointure entre Commandes et Clients.
Type : INNER JOIN (Seulement si correspondance).

SELECT C.nom, O.date_cmd
FROM Clients C
INNER JOIN Commandes O ON C.id = O.client_id;
27 - JOIN

Prospection Commerciale

Contexte : Nous voulons relancer les clients qui n'ont jamais rien acheté.
Objectif : Listez TOUS les clients, et voyez NULL s'ils n'ont pas de commande.
Type : LEFT JOIN.

SELECT C.nom, O.id as commande_id
FROM Clients C
LEFT JOIN Commandes O ON C.id = O.client_id;
28 - JOIN

Détail Complet

Contexte : Le service client veut savoir "Qui a acheté Quoi et Quand".
Objectif : Joignez Clients -> Commandes -> Details -> Produits.
Défi : Triple Jointure.

SELECT C.nom, O.date, P.nom
FROM Clients C
JOIN Commandes O ON C.id = O.client_id
JOIN Details_Commande D ON O.id = D.cmd_id
JOIN Produits P ON D.prod_id = P.id;
29 - JOIN

Hiérarchie RH

Contexte : Générer l'organigramme.
Objectif : Pour chaque employé, affichez le nom de son manager.
Technique : Self Join (La table se joint à elle-même).

SELECT E.nom as Employe, M.nom as Manager
FROM Employes E
LEFT JOIN Employes M ON E.manager_id = M.id;
30 - JOIN

Réconciliation

Contexte : Comparaison de deux sources de données A et B.
Objectif : Affichez tout de A et tout de B, qu'il y ait correspondance ou non.
Type : FULL OUTER JOIN.

SELECT * FROM TableA A
FULL OUTER JOIN TableB B ON A.id = B.a_id;
31 - PROC

Encapsulation

Contexte : Pour des raisons de sécurité, les développeurs front-end n'ont pas accès direct aux tables.
Objectif : Créez une procédure stockée qui renvoie la liste des clients.
Avantage : Abstraction du schéma.

CREATE PROCEDURE GetClients
AS
SELECT * FROM Clients;
GO;
32 - PROC

Recherche Backend

Contexte : Le moteur de recherche du site envoie le nom de la ville.
Objectif : Créez une procédure avec un paramètre d'entrée @ville.
Syntaxe : CREATE PROCEDURE ... (@p type).

CREATE PROCEDURE GetClientByCity (@ville VARCHAR(50))
AS
SELECT * FROM Clients WHERE ville = @ville;
33 - FUNC

Logique Métier Partagée

Contexte : Le calcul de la TVA change souvent. Il doit être centralisé.
Objectif : Créez une fonction scalaire qui prend un montant et retourne le montant * 1.20.
Retour : Un décimal.

CREATE FUNCTION CalculTVA (@prix DECIMAL)
RETURNS DECIMAL
AS BEGIN
RETURN @prix * 1.20
END;
34 - PROC

KPI Dashboard

Contexte : Le tableau de bord a besoin du nombre total de clients pour un widget.
Objectif : Utilisez un paramètre de sortie OUTPUT.
Action : Renvoyez le COUNT(*) dans @total.

CREATE PROCEDURE CountClients (@total INT OUTPUT)
AS
SELECT @total = COUNT(*) FROM Clients;
35 - FUNC

Utilisation de Fonction

Contexte : Génération du catalogue avec prix TTC.
Objectif : Appelez votre fonction `CalculTVA` à l'intérieur d'un SELECT.
Note : N'oubliez pas le schéma (dbo.).

SELECT nom, dbo.CalculTVA(prix) as PrixTTC
FROM Produits;
36 - TRANS

Virement Bancaire

Contexte : Débit du compte A, Crédit du compte B. Si l'un échoue, tout doit échouer.
Objectif : Enveloppez deux INSERTs/UPDATEs dans une transaction.
Instruction : BEGIN ... COMMIT.

BEGIN TRANSACTION;
INSERT INTO A VALUES (1);
INSERT INTO B VALUES (2);
COMMIT;
37 - TRANS

Filet de Sécurité

Contexte : Un script d'import massif est risqué.
Objectif : Si une erreur survient, annulez toutes les modifications en cours.
Mécanisme : TRY / CATCH avec ROLLBACK.

BEGIN TRY
  BEGIN TRANSACTION
    -- SQL Dangereux
  COMMIT
END TRY
BEGIN CATCH
  ROLLBACK
END CATCH
38 - TRIG

Traçabilité (Audit)

Contexte : Conformité légale. Nous devons savoir quand un utilisateur est créé.
Objectif : Créez un Trigger qui insère une ligne dans `Logs` après insertion dans `Users`.
Type : AFTER INSERT.

CREATE TRIGGER trg_UserAudit
ON Users AFTER INSERT
AS
INSERT INTO Logs (msg) VALUES ('Nouvel user créé');
39 - TRIG

Verrouillage de Table

Contexte : La table `Critical_Data` ne doit JAMAIS perdre de lignes.
Objectif : Interceptez l'ordre de suppression et ne faites rien (ou affichez une erreur).
Type : INSTEAD OF DELETE.

CREATE TRIGGER trg_NoDelete
ON Critical_Data INSTEAD OF DELETE
AS
PRINT 'Suppression interdite !';
40 - TRIG

Historisation des Prix

Contexte : Analyse de l'inflation. Avant qu'un prix change, sauvegardez l'ancien.
Objectif : Trigger sur UPDATE qui lit la pseudotable DELETED.
Note : DELETED contient l'état avant modif.

CREATE TRIGGER trg_History
ON Produits AFTER UPDATE
AS
INSERT INTO History (old_price)
SELECT prix FROM DELETED;
41 - VIEW

Vue Simplifiée

Contexte : Donnez un accès aux stagiaires sans exposer les salaires.
Objectif : Créez une vue V_Employes_Public sans la colonne salaire.
Code : CREATE VIEW...

CREATE VIEW V_Employes_Public AS
SELECT id, nom, poste FROM Employes;
42 - INDEX

Optimisation de Recherche

Contexte : La recherche par email est lente sur 1M d'utilisateurs.
Objectif : Créez un index sur la colonne email de la table Users.
But : Accélérer les SELECT WHERE email = ...

CREATE INDEX idx_user_email
ON Users (email);
43 - CTE

Requête Lisiable (CTE)

Contexte : Une sous-requête complexe rend le code illisible.
Objectif : Refactorisez avec WITH (Common Table Expression).
Exemple : Définissez 'BigOrders' puis sélectionnez dedans.

WITH BigOrders AS (
  SELECT * FROM Commandes WHERE total > 1000
)
SELECT * FROM BigOrders;
44 - RANK

Classement (Window Func)

Contexte : Donnez un rang à chaque vendeur selon ses ventes.
Objectif : Utilisez RANK() OVER (ORDER BY ventes DESC).
Résultat : 1, 2, 3...

SELECT nom, ventes,
RANK() OVER (ORDER BY ventes DESC) as classement
FROM Vendeurs;
45 - SUM

Cumul Progressif

Contexte : Afficher l'évolution du CA jour après jour (Running Total).
Objectif : SUM(montant) OVER (ORDER BY date).
Concept : Fenêtrage cumulatif.

SELECT date, montant,
SUM(montant) OVER (ORDER BY date) as cumul
FROM Ventes;
46 - JSON

Extraction JSON

Contexte : Les configs utilisateurs sont stockées en JSON brutes.
Objectif : Extraire la clé 'theme' de la colonne preferences.
PostgreSQL : col->>'key'.

-- PostgreSQL
SELECT preferences->>'theme' as theme
FROM Users;
47 - CONCAT

Concaténation Avancée

Contexte : Générer un nom complet lisible pour l'export CSV.
Objectif : "NOM Prénom" (Nom en majuscule).
Fonctions : UPPER() et CONCAT() (ou ||).

SELECT CONCAT(UPPER(nom), ' ', prenom)
FROM Employes;
48 - UPSERT

Merge / Upsert

Contexte : Synchro nocturne. Si le user existe, update, sinon insert.
Objectif : Syntaxe "ON CONFLICT" (PG) ou "ON DUPLICATE KEY" (MySQL).
Cas : Sauvegarde de profil.

-- PostgreSQL
INSERT INTO Users (id, nom) VALUES (1, 'Bob')
ON CONFLICT (id) DO UPDATE SET nom = 'Bob';
49 - EXIST

Check Existence

Contexte : Vérifier si une catégorie est vide avant de la supprimer.
Objectif : Renvoyez Vrai si la sous-requête renvoie des lignes.
Opérateur : EXISTS (...) est souvent plus rapide que COUNT > 0.

SELECT * FROM Categories C
WHERE NOT EXISTS (
  SELECT 1 FROM Produits P WHERE P.cat_id = C.id
);
50 - OPTIM

Plan d'Exécution

Contexte : Une requête est lente. Pourquoi ?
Objectif : Affichez comment le moteur SQL exécute la requête (Index Scan vs Seq Scan).
Commande : EXPLAIN ANALYZE (PG) ou EXPLAIN (MySQL).

EXPLAIN SELECT * FROM Users WHERE email = 'x';
