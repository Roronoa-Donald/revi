1. La Langue des Données

Bonjour et bienvenue ! Si vous êtes ici, c'est que vous voulez apprendre à parler à vos données.

Imaginez que vous êtes dans une immense bibliothèque (votre serveur). Vous ne pouvez pas juste crier "Donne-moi le livre rouge !". La bibliothèque a des règles strictes. Vous devez parler sa langue : le SQL (Structured Query Language).

Ne vous inquiétez pas, c'est une langue avec très peu de mots (moins de 30 mots importants) et une grammaire très logique. Aujourd'hui, nous n'allons pas lire des livres (SELECT), nous allons construire les étagères (DDL).

2. De quoi est faite une Base ?

Pour bien comprendre, oublions le code une seconde. Pensez à un fichier Excel. Une base de données SQL, c'est comme un fichier Excel super-puissant, mais avec des règles beaucoup plus strictes pour éviter le chaos.

La Base (Database)

C'est votre fichier .xlsx entier. C'est le conteneur principal. Une entreprise a souvent une Base 'RH', une Base 'Ventes', etc.

La Table

C'est un onglet dans Excel. Une grille dédiée à un sujet précis : Clients, Commandes, Produits.
Règle d'Or : Une table = Un sujet.

La Colonne

C'est l'en-tête de vos colonnes Excel (A, B, C...). Elle définit QUOI on stocke : Nom, Prénom, Prix, Date.

La grande différence avec Excel ?

Dans Excel, vous pouvez écrire "Jean" dans la case A1 et "12/05/2023" dans la case A2. C'est le bazar.
En SQL, c'est INTERDIT. Si une colonne est définie pour contenir des Noms, elle ne contiendra JAMAIS de dates. C'est ce qu'on appelle le Typage Fort.

3. L'Anatomie d'une Commande

Nous allons apprendre notre première phrase magique : CREATE TABLE. L'objectif est de dire à l'ordinateur : "Fabrique-moi un nouveau tiroir appelé 'Stagiaires' et prépare-toi à recevoir des ID et des Noms".

CREATE TABLE stagiaires (
    id INT,
    nom VARCHAR(50)
)

Décryptage pas à pas :

  1. CREATE TABLE : C'est l'ordre. On déclare "Je veux créer une structure de table".
  2. stagiaires : C'est le nom que VOUS choisissez. Ça pourrait être "Clients" ou "Pizza".
  3. ( ... ) : Les parenthèses contiennent la définition des colonnes. Tout ce qui est à l'intérieur définit le contenu de l'étagère.
  4. id, nom : Ce sont les noms des colonnes (les en-têtes).
  5. INT, VARCHAR : Ce sont les TYPES DE DONNÉES. Très important !

Zoom sur : Les Types de Données

Pourquoi tant de types différents ?

Vous vous demandez peut-être : "Pourquoi ne pas tout mettre en texte ?".
Imaginez que vous déménagez. Vous avez des cartons de différentes tailles :

  • Le type INT (Entier) C'est un tout petit carton très solide, fait pour contenir juste un nombre (1, 2, 500). Il prend très peu de place dans le camion (mémoire). Très rapide à manipuler.
  • Le type VARCHAR (Texte Variable) C'est un carton extensible. Si vous écrivez "Jo", il rétrécit. Si vous écrivez "Jean-Sébastien", il s'agrandit (jusqu'à une limite). C'est flexible, mais un peu plus complexe à gérer pour l'ordinateur.

Le Duel : CHAR vs VARCHAR

CHAR(10)
Taille Fixe
"Hello"

Stocke "Hello" + 5 espaces vides.
Gaspillage d'espace !

VARCHAR(10)
Taille Optimisée
"Hello"

Stocke uniquement "Hello".
Économie d'espace !

1. Création Simple

Solution

Créez une table 'Clients' avec id (INT) et email (VARCHAR).

CREATE TABLE Clients (id INT, email VARCHAR(100));

2. Types de données

Solution

Créez 'Produits' avec un prix exact (DECIMAL) et une description textuelle longue (TEXT ou VARCHAR MAX).

CREATE TABLE Produits (prix DECIMAL(10,2), description VARCHAR(MAX));

3. Suppression

Solution

Supprimez définitivement la table 'OldLogs'.

DROP TABLE OldLogs;

4. Modification

Solution

Ajoutez une colonne 'age' (INT) à la table 'Users'.

ALTER TABLE Users ADD age INT;

5. Réflexion

Solution

Pourquoi ne pas stocker un numéro de téléphone en INT ?

Parce que le 0 initial disparaîtrait (0612... devient 612...) et on ne fait pas de calculs mathématiques avec. Utilisez VARCHAR.
