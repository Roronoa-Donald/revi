1. La Vie des Données

Créer des tables (DDL), c'est comme construire une maison. Maintenant, il faut y vivre ! C'est le rôle du DML (Data Manipulation Language).

Vos données ne sont pas des statues de pierre. Elles naissent (INSERT), elles changent (UPDATE) et elles meurent (DELETE).

2. La Naissance (INSERT)

Poster une lettre

Pour ajouter une ligne, imaginez que vous remplissez un formulaire papier et que vous le glissez dans une boîte.

INSERT INTO Clients (nom, ville)
VALUES ('Alice', 'Paris');
Attention à l'ordre !
Nom -> Alice
Ville -> Paris

3. La Modification & Suppression

C'est ici que le SQL devient puissant... et dangereux.

UPDATE (La Gomme)

Permet de corriger une valeur existante.

UPDATE table SET prix = 20
WHERE id = 5

Sans le WHERE, vous changez TOUS les prix du magasin à 20€ !

DELETE (La Broyeuse)

Supprime définitivement une ou plusieurs lignes.

DELETE FROM table
WHERE date < '2020'

Sans le WHERE, la table est vide.

Zoom : TRUNCATE vs DELETE

Comment vider une table efficacement ?

Imaginez que vous devez vider une armoire d'archives géante (1 million de dossiers).

DELETE FROM Table

Méthode Douce

Vous prenez chaque dossier un par un, vous vérifiez s'il doit être jeté (WHERE), et vous le passez au broyeur.

  • Lent (Ligne par ligne)
  • Consomme beaucoup de ressources
  • Mais annulable (Loggué)

TRUNCATE TABLE

Méthode Brutale

Vous mettez le feu à l'armoire entière. Vous ne regardez pas ce qu'il y a dedans.

  • Immédiat (Désallocation de page)
  • Extrêmement performant
  • Impossible d'utiliser WHERE

Zone d'Entraînement

1. Insertion Simple

Ajoutez un utilisateur 'Alice' avec l'email 'alice@test.com' dans la table Users (colonnes name, email).

INSERT INTO Users (name, email)
VALUES ('Alice', 'alice@test.com');

2. Correction d'Email

L'utilisateur d'ID 5 a changé d'email pour 'new@test.com'. Mettez à jour sa fiche.

UPDATE Users
SET email = 'new@test.com'
WHERE id = 5;

3. Suppression Ciblée

Supprimez toutes les commandes (Orders) datant d'avant 2020 (colonne created_at).

DELETE FROM Orders
WHERE created_at < '2020-01-01';

4. Augmentation Générale

Le produit 'Laptop' voit son prix augmenter de 50€ pour tous les modèles en stock.

UPDATE Products
SET price = price + 50
WHERE name = 'Laptop';

5. Vide-Grenier (TRUNCATE)

Videz entièrement la table 'Logs' de manière performante (sans scanner ligne par ligne).

TRUNCATE TABLE Logs;
