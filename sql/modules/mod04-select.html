1. L'Art de la Recherche

Stocker des données, c'est facile. Retrouver exactement l'information dont vous avez besoin parmi des millions de lignes, c'est un art.

Le SELECT est votre lampe de poche dans l'obscurité de la base de données.

2. Le Supermarché (SELECT & FROM)

Faire ses courses

Imaginez votre base de données comme un supermarché géant.

  • FROM C'est choisir le rayon (La Table).
  • SELECT C'est choisir les produits (Les Colonnes).

SELECT * revient à acheter tout le rayon ! C'est lourd à porter (Performance).

3. Organiser le Chaos

Le Classement (ORDER BY)

Par défaut, SQL vous rend les données en vrac. Mettez de l'ordre !

ORDER BY prix DESC
-- Du plus cher au moins cher

L'Unique (DISTINCT)

Si vous demandez "Quels pays ?", vous ne voulez pas voir "France" 1000 fois.

SELECT DISTINCT pays
FROM Clients

Zoom : L'Ordre Secret

Ce que vous écrivez vs Ce que SQL lit

SQL est un langage étrange. Vous écrivez la fin (SELECT) au début !

1
FROM
Je prends la table
2
WHERE
Je filtre les lignes
3
SELECT
Je garde les colonnes
4
ORDER BY
Je trie le résultat

Zone d'Entraînement

1. Sélection de Colonnes

Ne sélectionnez que le nom et le prix des produits.

SELECT name, price FROM Products;

2. Tri Temporel

Listez les commandes de la plus récente à la plus ancienne.

SELECT * FROM Orders ORDER BY created_at DESC;

3. Anti-Doublons

Quels sont les différents pays de nos clients ? (Sans répétition)

SELECT DISTINCT country FROM Customers;

4. Podium

Les 3 produits les plus chers.

SELECT TOP 3 * FROM Products ORDER BY price DESC;
-- ou LIMIT 3 selon SGBD

5. Tri Multi-Critères

Trier par Pays (A-Z), puis par salaire (Haut-Bas) pour chaque pays.

SELECT * FROM Employees
ORDER BY country ASC, salary DESC;
