3. Le Panier de Tri (GROUP BY)

Calculer le total global, c'est bien. Mais si on veut le total par catégorie ? Imaginez que vous triez votre chariot avant de passer en caisse : "Légumes" d'un côté, "Tech" de l'autre.

SELECT category, SUM(price)
FROM Products
GROUP BY category;

Zoom : WHERE vs HAVING

Le piège de l'entretien d'embauche

Les deux servent à filtrer, mais ils ne vivent pas dans le même monde temporel.

Le Vigile de l'Entrée

WHERE

Filtre les lignes individuelles.

  • S'applique AVANT de faire les groupes.
  • Ex: "Je ne veux que les ventes de 2023".
  • Interdit d'utiliser SUM() ici.
Le Vigile de la Sortie

HAVING

Filtre les résultats groupés.

  • S'applique APRÈS les calculs.
  • Ex: "Montrez-moi les totaux > 1000€".
  • C'est ici qu'on filtre sur SUM().

Zone d'Entraînement

1. Somme Totale

Calculez la valeur totale du stock (somme de la colonne price).

SELECT SUM(price) FROM Products;

2. Répartition Géographique

Comptez le nombre d'utilisateurs par pays.

SELECT country, COUNT(*) FROM Users
GROUP BY country;

3. Salaire Moyen

Quel est le salaire moyen des employés du département 'HR' ?

SELECT AVG(salary) FROM Employees
WHERE dept = 'HR';

4. Groupes Importants (HAVING)

Listez les catégories qui ont plus de 50 produits.

SELECT category, COUNT(*)
FROM Products
GROUP BY category
HAVING COUNT(*) > 50;

5. Bornes de Prix

Affichez le prix minimum et maximum pour chaque catégorie.

SELECT category, MIN(price), MAX(price)
FROM Products
GROUP BY category;
