1. Le Temps et le Vide

Deux ennemis invisibles guettent le développeur SQL : le temps et le vide.

Le temps semble simple — une date, une heure — mais il cache des pièges redoutables : fuseaux horaires, formats régionaux, calculs entre jours fériés... Et puis il y a NULL : l'absence totale de valeur. Ce n'est pas 0, ce n'est pas '' (chaîne vide). C'est l'inconnu.

"Dans SQL, NULL est comme un trou noir : tout ce qui le touche disparaît." — NULL + 5 = NULL, NULL = NULL renverra FALSE. Étrange, non ?

2. Les Types Temporels

SQL Server offre plusieurs types pour stocker le temps :

DATE

Date seule (sans heure).
'2025-01-15'

TIME

Heure seule (sans date).
'14:30:00'

DATETIME

Date + heure. Précision 3.33ms.
'2025-01-15 14:30:00'

DATETIME2

Plus précis (100ns). Recommandé.
'2025-01-15 14:30:00.1234567'

Conseil : Utilisez DATETIME2 pour les nouveaux projets. DATETIME est conservé pour la compatibilité mais est moins précis et a une plage plus limitée.

3. Fonctions de Dates

Fonction Description Exemple
GETDATE() Date et heure actuelles du serveur SELECT GETDATE()
SYSDATETIME() Comme GETDATE() mais plus précis SELECT SYSDATETIME()
DATEADD(unit, n, date) Ajoute n unités à une date DATEADD(month, 3, '2025-01-01') → 2025-04-01
DATEDIFF(unit, d1, d2) Différence entre deux dates DATEDIFF(day, '2025-01-01', '2025-03-15') → 73
DATEPART(unit, date) Extrait une partie (année, mois...) DATEPART(year, GETDATE()) → 2025
FORMAT(date, fmt) Formater l'affichage FORMAT(GETDATE(), 'dd/MM/yyyy') → 15/01/2025
YEAR() / MONTH() / DAY() Raccourcis pour extraire YEAR('2025-06-15') → 2025

4. NULL — L'Ennemi Invisible

Règle d'or : NULL n'est PAS une valeur. C'est l'absence de valeur. Toute opération avec NULL donne NULL.
NULL + 5 = NULL

Propagation du NULL

NULL = NULL → FALSE

On ne peut pas comparer

IS NULL → TRUE

Le seul test valide

Les outils anti-NULL

ISNULL(expr, remplacement)

Remplace NULL par une valeur. Spécifique SQL Server.

SELECT ISNULL(telephone, 'Non renseigné')
FROM Clients;
COALESCE(a, b, c, ...)

Retourne le premier non-NULL. Standard SQL.

SELECT COALESCE(mobile, fixe, 'Aucun')
FROM Contacts;
NULLIF(a, b)

Retourne NULL si a = b. Utile pour éviter div/0.

SELECT total / NULLIF(quantite, 0)
FROM Ventes;
IS NULL / IS NOT NULL

Les seuls opérateurs pour tester NULL dans WHERE.

SELECT * FROM Clients
WHERE email IS NOT NULL;
