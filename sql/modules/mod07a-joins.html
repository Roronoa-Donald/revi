Module 07: Les Jointures

XP: 0

1. Pourquoi les Jointures ?

En bases de données relationnelles, les données sont réparties dans plusieurs tables pour éviter la redondance. Une table Clients, une table Commandes, une table Produits...

Mais comment afficher "le nom du client avec ses commandes" si ces infos sont dans des tables séparées ? C'est exactement le rôle des JOIN : relier les tables entre elles.

🔗 Sans les JOIN, SQL ne serait qu'un tableur glorifié. Avec les JOIN, c'est un moteur de requêtes relationnelles surpuissant.

3. INNER JOIN — L'Intersection

Le INNER JOIN retourne uniquement les lignes qui ont une correspondance dans les deux tables. C'est la jointure la plus courante et la plus utilisée.

Clients
Commandes
INNER JOIN
correspondances

Syntaxe

SELECT C.Nom, C.Ville, O.CmdID, O.Montant
FROM Clients C
INNER JOIN Commandes O ON C.ClientID = O.ClientID;
Résultat (3 lignes)
NomVilleCmdIDMontant
AliceParis101250 €
AliceParis103120 €
BobLyon102430 €

Charlie et Diana ont disparu (pas de commandes). La commande 104 (ClientID=5) a aussi disparu (pas de client correspondant). INNER JOIN = uniquement les correspondances.

4. LEFT JOIN — Tout à Gauche

Le LEFT JOIN (ou LEFT OUTER JOIN) retourne TOUTES les lignes de la table de gauche, même si elles n'ont pas de correspondance dans la table de droite. Les colonnes manquantes seront remplies de NULL.

Clients
(TOUT)
Commandes
LEFT JOIN
gauche = prioritaire

Syntaxe

SELECT C.Nom, C.Ville, O.CmdID, O.Montant
FROM Clients C
LEFT JOIN Commandes O ON C.ClientID = O.ClientID;
Résultat (5 lignes)
NomVilleCmdIDMontant
AliceParis101250 €
AliceParis103120 €
BobLyon102430 €
CharlieMarseilleNULLNULL
DianaBordeauxNULLNULL

Cas d'usage typique : Trouver les clients sans commande : 

SELECT C.Nom
FROM Clients C
LEFT JOIN Commandes O ON C.ClientID = O.ClientID
WHERE O.CmdID IS NULL;

→ Retourne : Charlie, Diana (ceux sans commande)

5. RIGHT JOIN — Tout à Droite

Le RIGHT JOIN est le miroir du LEFT JOIN : il retourne TOUTES les lignes de la table de droite, avec NULL si aucune correspondance dans la table de gauche. 

SELECT C.Nom, O.CmdID, O.Montant
FROM Clients C
RIGHT JOIN Commandes O ON C.ClientID = O.ClientID;
Résultat (4 lignes)
NomCmdIDMontant
Alice101250 €
Bob102430 €
Alice103120 €
NULL10480 €

La commande 104 (ClientID=5) apparaît avec NULL comme Nom car aucun client n'a l'ID 5. En pratique, on utilise rarement RIGHT JOIN — un LEFT JOIN en inversant les tables fait la même chose et est plus lisible.

6. FULL OUTER JOIN — L'Union Totale

Le FULL OUTER JOIN combine LEFT et RIGHT : il retourne toutes les lignes des deux tables, avec NULL là où il n'y a pas de correspondance.

(TOUT)
Commandes
(TOUT)
FULL OUTER
tout le monde 
SELECT C.Nom, C.Ville, O.CmdID, O.Montant
FROM Clients C
FULL OUTER JOIN Commandes O ON C.ClientID = O.ClientID;
Résultat (6 lignes)
NomVilleCmdIDMontant
AliceParis101250 €
AliceParis103120 €
BobLyon102430 €
CharlieMarseilleNULLNULL
DianaBordeauxNULLNULL
NULLNULL10480 €

Cas d'usage : Détecter les orphelins des deux côtés — clients sans commande ET commandes sans client valide.

7. CROSS JOIN — Le Produit Cartésien

Le CROSS JOIN combine chaque ligne de la première table avec chaque ligne de la deuxième. Pas de clause ON ! Si table A a 4 lignes et table B a 4 lignes : le résultat a 4 × 4 = 16 lignes. 

SELECT C.Nom, P.NomProduit
FROM Clients C
CROSS JOIN Produits P;

Danger ! 1 000 clients × 10 000 produits = 10 millions de lignes. N'utilisez CROSS JOIN que pour de petites tables ou des cas très spécifiques (génération de combinaisons, calendriers, etc.).

Cas utile : Générer toutes les combinaisons possibles taille/couleur : 

SELECT T.Taille, Co.Couleur
FROM Tailles T
CROSS JOIN Couleurs Co;
-- S-Rouge, S-Bleu, M-Rouge, M-Bleu, L-Rouge, L-Bleu...

8. Self-JOIN — La Table contre Elle-même

Un Self-JOIN est une jointure d'une table avec elle-même. C'est utile quand une table contient des relations hiérarchiques (employé → manager, catégorie → sous-catégorie).

Table Employes
EmpIDNomManagerID
1PDGNULL
2Alice1
3Bob1
4Charlie2

Trouver le manager de chaque employé

SELECT 
    E.Nom AS Employe,
    M.Nom AS Manager
FROM Employes E
LEFT JOIN Employes M ON E.ManagerID = M.EmpID;
Résultat
EmployeManager
PDGNULL
AlicePDG
BobPDG
CharlieAlice

Astuce : Les alias sont obligatoires en Self-JOIN pour distinguer les deux « copies » de la même table (E pour employé, M pour manager).

9. Jointures Multi-Tables

En pratique, on joint souvent 3 tables ou plus. Il suffit d'enchaîner les JOIN :

Clients + Commandes + Produits

SELECT 
    C.Nom AS Client,
    O.CmdID,
    P.NomProduit,
    LC.Quantite,
    P.Prix * LC.Quantite AS SousTotal
FROM Clients C
INNER JOIN Commandes O ON C.ClientID = O.ClientID
INNER JOIN LignesCommande LC ON O.CmdID = LC.CmdID
INNER JOIN Produits P ON LC.ProduitID = P.ProduitID
ORDER BY C.Nom, O.CmdID;

Ordre de lecture SQL : FROM → JOIN → WHERE → GROUP BY → HAVING → SELECT → ORDER BY. Les jointures sont résolues avant le filtrage WHERE.

JOIN avec agrégats

-- Total des commandes par client
SELECT 
    C.Nom,
    COUNT(O.CmdID) AS NbCommandes,
    SUM(O.Montant) AS TotalDepense
FROM Clients C
LEFT JOIN Commandes O ON C.ClientID = O.ClientID
GROUP BY C.Nom
ORDER BY TotalDepense DESC;

11. Pièges et Erreurs Courantes

Oublier la clause ON

Sans ON, un INNER JOIN devient un CROSS JOIN — explosion de lignes !

-- MAUVAIS : produit cartésien involontaire
SELECT * FROM Clients JOIN Commandes; -- Manque le ON !

Colonnes ambiguës

Si deux tables ont une colonne de même nom, le SGBD ne sait pas laquelle choisir.

-- ERREUR : ambiguous column 'ClientID'
SELECT ClientID FROM Clients JOIN Commandes ON ...

-- CORRECT : utiliser l'alias
SELECT C.ClientID FROM Clients C JOIN Commandes O ON ...

WHERE qui annule le LEFT JOIN

Filtrer sur une colonne de la table de droite dans le WHERE transforme un LEFT JOIN en INNER JOIN.

-- MAUVAIS : le WHERE élimine les NULL → LEFT JOIN = INNER JOIN
SELECT C.Nom, O.Montant
FROM Clients C LEFT JOIN Commandes O ON C.ClientID = O.ClientID
WHERE O.Montant > 100;

-- CORRECT : condition dans le ON
SELECT C.Nom, O.Montant
FROM Clients C LEFT JOIN Commandes O 
  ON C.ClientID = O.ClientID AND O.Montant > 100;

🧠 MÉMO TECHNIQUE — Les Jointures SQL

« I.L.R.F.C — Inner, Left, Right, Full, Cross »

  • 🔹 INNER JOIN → uniquement les correspondances (intersection ∩)
  • 🔹 LEFT JOIN → tout à gauche + correspondances
  • 🔹 RIGHT JOIN → tout à droite + correspondances
  • 🔹 FULL JOIN → tout des deux côtés
  • 🔹 CROSS JOIN → produit cartésien (chaque ligne × chaque ligne)

💡 90% du temps, tu utiliseras INNER ou LEFT.
