1. Le Cerveau de SQL

Jusqu'ici, SQL exécutait vos ordres sans réfléchir. Mais grâce à T-SQL (Transact-SQL), SQL Server devient un vrai langage de programmation : il peut décider, calculer, et s'adapter.

C'est comme passer du GPS basique (« Tourne à droite ») au GPS intelligent (« S'il y a un embouteillage, prends la déviation, sinon reste sur l'autoroute »).

T-SQL = SQL standard + variables + conditions + boucles + fonctions. C'est l'extension Microsoft de SQL.

2. Les Variables

En T-SQL, les variables commencent toujours par @. Elles stockent des valeurs temporaires.

Déclaration

DECLARE @nom VARCHAR(50) = 'Alice'
DECLARE @age INT
SET @age = 25

Via SELECT

DECLARE @total INT
SELECT @total = COUNT(*)
FROM Clients
Attention : Les variables n'existent que pendant le batch (lot de commandes). Si vous relancez la requête, la variable est recréée. Pas de persistance !

3. IF / ELSE — La Condition

La structure de décision la plus fondamentale :

DECLARE @age INT = 20

IF (@age >= 18)
BEGIN
    PRINT 'Majeur — accès autorisé'
END
ELSE
BEGIN
    PRINT 'Mineur — accès refusé'
END
Rappel : BEGIN...END est obligatoire si le bloc contient plus d'une instruction. Pour une seule ligne, c'est optionnel mais recommandé pour la lisibilité.

4. CASE WHEN — Le Switch SQL

CASE WHEN est l'équivalent d'un switch/case. Utilisable directement dans un SELECT !

CASE Recherché (avec conditions)
SELECT nom,
  CASE
    WHEN prix < 10 THEN 'Pas cher'
    WHEN prix < 100 THEN 'Moyen'
    ELSE 'Cher'
  END AS categorie
FROM Produits;
CASE Simple (avec valeur)
SELECT nom,
  CASE role
    WHEN 'Admin' THEN 'Administrateur'
    WHEN 'User' THEN 'Utilisateur'
    ELSE 'Inconnu'
  END AS role_complet
FROM Users;
Différence clé : IF/ELSE est pour le flux de contrôle (exécuter des blocs). CASE WHEN est une expression qui retourne une valeur (utilisable dans SELECT, WHERE, ORDER BY).
