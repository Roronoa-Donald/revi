1. Le Cerveau de SQL

Jusqu'ici, SQL exécutait vos ordres sans réfléchir. Mais grâce à T-SQL (Transact-SQL), SQL Server devient un vrai langage de programmation : il peut décider, calculer, et s'adapter.

C'est comme passer du GPS basique (« Tourne à droite ») au GPS intelligent (« S'il y a un embouteillage, prends la déviation, sinon reste sur l'autoroute »).

T-SQL = SQL standard + variables + conditions + boucles + fonctions. C'est l'extension Microsoft de SQL.

2. Les Variables

En T-SQL, les variables commencent toujours par @. Elles stockent des valeurs temporaires.

Déclaration

DECLARE @nom VARCHAR(50) = 'Alice'
DECLARE @age INT
SET @age = 25

Via SELECT

DECLARE @total INT
SELECT @total = COUNT(*)
FROM Clients
Attention : Les variables n'existent que pendant le batch (lot de commandes). Si vous relancez la requête, la variable est recréée. Pas de persistance !

3. IF / ELSE — La Condition

La structure de décision la plus fondamentale :

DECLARE @age INT = 20

IF (@age >= 18)
BEGIN
    PRINT 'Majeur — accès autorisé'
END
ELSE
BEGIN
    PRINT 'Mineur — accès refusé'
END
Rappel : BEGIN...END est obligatoire si le bloc contient plus d'une instruction. Pour une seule ligne, c'est optionnel mais recommandé pour la lisibilité.

4. CASE WHEN — Le Switch SQL

CASE WHEN est l'équivalent d'un switch/case. Utilisable directement dans un SELECT !

CASE Recherché (avec conditions)
SELECT nom,
  CASE
    WHEN prix < 10 THEN 'Pas cher'
    WHEN prix < 100 THEN 'Moyen'
    ELSE 'Cher'
  END AS categorie
FROM Produits;
CASE Simple (avec valeur)
SELECT nom,
  CASE role
    WHEN 'Admin' THEN 'Administrateur'
    WHEN 'User' THEN 'Utilisateur'
    ELSE 'Inconnu'
  END AS role_complet
FROM Users;
Différence clé : IF/ELSE est pour le flux de contrôle (exécuter des blocs). CASE WHEN est une expression qui retourne une valeur (utilisable dans SELECT, WHERE, ORDER BY).

7. Introduction aux Sous-requêtes

Une sous-requête (subquery) est un SELECT imbriqué dans un autre SELECT, WHERE, FROM ou HAVING. C'est un outil puissant pour écrire des requêtes complexes.

7.1 Sous-requête avec WHERE ... IN

Filtrer les lignes selon une liste dynamique calculée par une autre requête.

-- Clients ayant au moins une commande > 500 €
SELECT Nom, Email
FROM Clients
WHERE ClientID IN (
    SELECT ClientID
    FROM Commandes
    WHERE Montant > 500
);

7.2 Sous-requête corrélée

La sous-requête fait référence à l'alias de la requête externe. Elle est réévaluée pour chaque ligne.

-- Produits dont le prix est supérieur à la moyenne de leur catégorie
SELECT P.NomProduit, P.Prix, P.CategorieID
FROM Produits P
WHERE P.Prix > (
    SELECT AVG(P2.Prix)
    FROM Produits P2
    WHERE P2.CategorieID = P.CategorieID
);

7.3 EXISTS / NOT EXISTS

EXISTS teste si la sous-requête retourne au moins une ligne. Plus performant que IN sur de grands jeux de données.

-- Clients avec au moins une commande (EXISTS)
SELECT C.Nom
FROM Clients C
WHERE EXISTS (
    SELECT 1 FROM Commandes O
    WHERE O.ClientID = C.ClientID
);

-- Clients sans AUCUNE commande (NOT EXISTS)
SELECT C.Nom
FROM Clients C
WHERE NOT EXISTS (
    SELECT 1 FROM Commandes O
    WHERE O.ClientID = C.ClientID
);

7.4 Sous-requête dans FROM (table dérivée)

On peut utiliser un SELECT comme une « table temporaire » dans le FROM.

-- Classement des catégories par chiffre d'affaires
SELECT Classement.Categorie, Classement.CA
FROM (
    SELECT 
        C.NomCategorie AS Categorie,
        SUM(P.Prix * LC.Quantite) AS CA
    FROM Categories C
    INNER JOIN Produits P ON C.CategorieID = P.CategorieID
    INNER JOIN LignesCommande LC ON P.ProduitID = LC.ProduitID
    GROUP BY C.NomCategorie
) AS Classement
WHERE Classement.CA > 10000
ORDER BY Classement.CA DESC;

Conseil : Préférez EXISTS à IN pour de meilleures performances. Les CTE (WITH ... AS) sont souvent plus lisibles que les tables dérivées dans le FROM.
