Module 09: Boucles

XP: 0

1. La Machine à Répétition

Imaginez un robot qui doit numéroter 10 000 dossiers. Lui dire « Numérotez le dossier 1... Numérotez le dossier 2... » serait absurde. On lui dit plutôt : « Tant qu'il reste des dossiers, numérotez le suivant. »

C'est le principe de la boucle WHILE en T-SQL. Mais attention — un robot mal programmé pourrait tourner en boucle à l'infini. D'où l'importance de BREAK et CONTINUE.

Règle d'or en SQL : Privilégiez les opérations ensemblistes (UPDATE, INSERT...SELECT) aux boucles. SQL est conçu pour traiter des ensembles de lignes, pas des lignes une par une. Les boucles sont un dernier recours.

2. WHILE — Tant que...

La seule boucle native de T-SQL. Syntaxe :

DECLARE @i INT = 1

WHILE @i <= 5
BEGIN
    PRINT 'Itération ' + CAST(@i AS VARCHAR)
    SET @i = @i + 1
END
Boucle infinie : Si vous oubliez SET @i = @i + 1, la condition reste toujours vraie et la boucle ne s'arrête jamais. SQL Server finira par timeout ou vous devrez annuler manuellement la requête.

3. BREAK & CONTINUE

BREAK

Sort immédiatement de la boucle, peu importe la condition.

WHILE 1=1 -- infinie
BEGIN
  SET @i = @i + 1
  IF @i > 10 BREAK
END

CONTINUE

Saute le reste du bloc et retourne au test de condition.

WHILE @i <= 10
BEGIN
  SET @i = @i + 1
  IF @i % 2 = 0 CONTINUE
  PRINT @i -- impairs
END
Astuce : WHILE 1=1 + BREAK est un pattern courant quand la condition de sortie est complexe.

4. CURSOR — Le Dernier Recours

Un curseur parcourt un résultat ligne par ligne. C'est l'anti-pattern de SQL, mais parfois nécessaire.

-- 1. Déclarer le curseur
DECLARE cur CURSOR FOR
    SELECT nom FROM Clients

-- 2. Ouvrir
OPEN cur

-- 3. Lire ligne par ligne
DECLARE @nom VARCHAR(50)
FETCH NEXT FROM cur INTO @nom
WHILE @@FETCH_STATUS = 0
BEGIN
    PRINT @nom
    FETCH NEXT FROM cur INTO @nom
END

-- 4. Fermer & libérer
CLOSE cur
DEALLOCATE cur
Performance : Les curseurs sont 10x à 100x plus lents qu'un simple UPDATE ou INSERT...SELECT. Toujours chercher une solution ensembliste d'abord !
Quand utiliser un curseur
  • • Envoi d'emails par client
  • • Appel API externe par ligne
  • • Migration de données complexe
Quand NE PAS utiliser
  • • Mise à jour de colonnes
  • • Calculs/agrégations
  • • Insertions en masse

5. L'Alternative Ensembliste

Avant de boucler, demandez-vous : « Est-ce faisable en un seul SELECT/UPDATE ? »

❌ Avec CURSOR (lent)
FETCH NEXT...
WHILE @@FETCH_STATUS = 0
  UPDATE Clients
  SET remise = 10
  WHERE id = @id
  FETCH NEXT...
✅ Ensembliste (rapide)
UPDATE Clients
SET remise = 10
WHERE actif = 1

Une seule commande, toutes les lignes traitées d'un coup !
