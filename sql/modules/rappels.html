1. Qu'est-ce qu'une Base de Données ?

Une base de données est un système organisé pour stocker, gérer et retrouver des informations. Pensez à un classeur géant : chaque tiroir est une table, chaque fiche est une ligne, et les champs sur la fiche sont les colonnes.

Définition formelle : Une base de données relationnelle organise les données en tables (relations) composées de lignes (tuples) et de colonnes (attributs), reliées entre elles par des clés.

Ce qu'une BDD fait bien

  • • Stocker des millions de lignes efficacement
  • • Retrouver des données en millisecondes
  • • Garantir la cohérence (pas de données contradictoires)
  • • Permettre un accès simultané par plusieurs utilisateurs

Ce qu'Excel ne fait PAS

  • • Pas de gestion de droits d'accès granulaire
  • • Pas de relations entre feuilles vérifiées
  • • Performances catastrophiques au-delà de ~100k lignes
  • • Aucune garantie d'intégrité des données

2. Le Modèle Relationnel

Inventé par Edgar F. Codd en 1970, le modèle relationnel est la base de tous les SGBDR (Systèmes de Gestion de Bases de Données Relationnelles) : SQL Server, MySQL, PostgreSQL, Oracle...

Table (Relation)

Un ensemble structuré de données sur un sujet unique. Ex : Clients, Produits.

Colonne (Attribut)

Un champ typé qui décrit une propriété. Ex : nom VARCHAR(100).

Ligne (Tuple)

Un enregistrement unique. Chaque ligne = une entité distincte.

Exemple : Table Clients

id (PK) nom email ville
1 Alice Martin alice@mail.com Paris
2 Bob Dupont bob@mail.com Lyon
3 Clara Noel clara@mail.com Paris

3. Clés et Relations

Les tables ne vivent pas isolées. Elles sont reliées entre elles par un système de clés, comme des liens hypertexte entre pages web.

Clé Primaire (PK)

L'identifiant unique de chaque ligne. Comme un numéro de sécurité sociale.

  • • Toujours unique (pas de doublons)
  • • Jamais NULL
  • • Souvent un INT auto-incrémenté
  • • Une seule par table

Clé Étrangère (FK)

Un lien vers la clé primaire d'une autre table. C'est le pont relationnel.

  • • Référence la PK d'une autre table
  • • Peut contenir des NULL (relation optionnelle)
  • • Empêche les données orphelines
  • • Plusieurs FK possibles par table
Analogie : La PK est comme votre numéro de badge à l'entreprise. La FK, c'est quand un formulaire demande « Numéro de badge du manager » — il référence le badge (PK) de quelqu'un d'autre.

En SQL Server, chaque colonne a un type strict. Pas de mélange possible (contrairement à Excel).

Erreur courante : Utiliser VARCHAR pour stocker des nombres (prix, quantités). Le tri sera alphabétique ("9" > "10") au lieu de numérique. Toujours utiliser le bon type !

5. SQL : Un Langage Déclaratif

Contrairement à Python ou Java (langages impératifs : vous dites comment faire), SQL est déclaratif : vous dites ce que vous voulez, et le moteur se débrouille.

Python (impératif)

résultat = []
for client in clients:
    if client.ville == "Paris":
        résultat.append(client.nom)
résultat.sort()

SQL (déclaratif)

SELECT nom
FROM Clients
WHERE ville = 'Paris'
ORDER BY nom;
Avantage : SQL est plus court, plus lisible, et le moteur optimise automatiquement l'exécution. Vous n'avez pas à penser aux boucles ou aux index — le SGBD le fait pour vous.

8. Vocabulaire Essentiel

Les termes que vous rencontrerez tout au long du cours :

SGBDR

Système de Gestion de Bases de Données Relationnelles

DDL

Data Definition Language — Créer/modifier la structure

DML

Data Manipulation Language — Insérer/modifier/supprimer des données

DQL

Data Query Language — Interroger (SELECT)

TCL

Transaction Control Language — COMMIT, ROLLBACK

T-SQL

Transact-SQL — Extension SQL de Microsoft (variables, boucles...)

NULL

L'absence de valeur. Ce n'est PAS zéro ni une chaîne vide.

Requête (Query)

Une instruction SQL envoyée au serveur pour obtenir un résultat
