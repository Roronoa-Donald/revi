Module 4

Diagramme d'Activités

Modélisez les flux de travail et processus : actions, transitions, décisions, synchronisation et couloirs d'activités.

Qu'est-ce qu'un diagramme d'activités ?

Un diagramme d'activités est comme un organigramme (flowchart) amélioré : il décrit le déroulement d'un processus étape par étape, avec des choix, des actions parallèles et des responsabilités.

Un diagramme d'activités permet de :

  • Représenter graphiquement le comportement d'une méthode
  • Décrire le déroulement d'un cas d'utilisation
  • Modéliser un processus métier
💡 Rappel : Le flot d'exécution est modélisé par des nœuds reliés par des arcs (transitions). Les transitions sont automatiques : la fin d'une activité déclenche immédiatement la suivante.

Symboles du diagramme d'activités

SymboleNomDescription
●Nœud initialDébut du processus (point noir plein)
▢Action / ActivitéRectangle arrondi contenant la description de l'action
◇Décision / FusionLosange avec conditions sur les branches sortantes
━━Barre de synchronisationFork (parallélisation) ou Join (synchronisation)
→TransitionFlèche indiquant le flux directionnel
⊗Fin de fluxFin d'un flux spécifique (pas de tous les flux)
⊕Nœud finalFin de TOUS les flux (cercle avec point)

Décisions et Conditions

Le nœud de décision (losange ◇) permet de créer des branchements conditionnels :

  • Il possède au moins 2 branches sortantes
  • Chaque branche porte une condition entre crochets : [condition]
  • Les conditions doivent être mutuellement exclusives et complètes
Exemple de décision :

         ◇ ─── [montant > solde] ──→ Refuser retrait
         |
         └── [montant ≤ solde] ──→ Effectuer retrait
⚠️ Attention : Chaque nœud de décision doit avoir une condition [sinon] (else) pour couvrir tous les cas possibles.

Synchronisation (Fork / Join)

Les barres de synchronisation permettent de gérer les activités parallèles :

Fork (bifurcation)

Sépare un flux en plusieurs flux parallèles. Toutes les activités qui partent de la barre s'exécutent en même temps.

Join (jonction)

Réunit plusieurs flux parallèles en un seul. On ne franchit la barre qu'après la réalisation de TOUTES les activités en entrée.

Exemple de fork/join :

    Recevoir commande
           |
       ━━━━━━━━━  ← Fork
       |        |
  Préparer   Facturer     ← Actions parallèles
  colis      client
       |        |
       ━━━━━━━━━  ← Join (attend les 2)
           |
    Expédier commande
✅ À retenir : Fork = 1 entrée, N sorties (paralléliser). Join = N entrées, 1 sortie (synchroniser).

Couloirs d'activités (Swimlanes)

Les couloirs d'activités (swimlanes) permettent d'organiser les actions selon les responsables :

  • Chaque couloir représente un acteur, un rôle ou un service
  • Les activités sont placées dans le couloir du responsable de l'action
  • Les transitions peuvent traverser les couloirs
Exemple avec swimlanes :

| Client        | Système       | Banque        |
|               |               |               |
| Insérer carte |               |               |
|       ────────→ Vérifier      |               |
|               | carte         |               |
|               |       ────────→ Autoriser     |
|               |               | transaction   |
|               | ←────────────── OK            |
| ←──────────── Distribuer     |               |
| Retirer       | billets       |               |
| billets       |               |               |
💡 Avantage : Les swimlanes montrent clairement QUI fait QUOI dans le processus, facilitant l'identification des responsabilités.

TD — Fiche de réparation

Créez un diagramme d'activité pour le cas d'utilisation « Créer une fiche de réparation » :

  • Le chef d'atelier saisit les critères de recherche de voitures
  • Le système fournit la liste des voitures correspondantes
  • Si la voiture existe, le chef la sélectionne → le système fournit les infos
  • Si la voiture est sous garantie, saisir la date de demande de réparation
  • Si la voiture n'existe pas, saisir les informations du nouveau véhicule
  • Dans tous les cas : saisir les dates de réception et restitution
  • Si le dommage est payé par l'assurance, sélectionner l'assurance adéquate
  • Le système enregistre la fiche de réparation
Mnémotechnique — Séquence = Conversation

🧠 « L.M.R.A — Ligne de vie, Message, Retour, Activation »

  • Ligne de vie → la barre verticale pointillée sous chaque objet (son « existence »)
  • Message → flèche pleine → = synchrone (attend la réponse), pointillée → = retour
  • Retour → flèche pointillée ← (la réponse au message)
  • Activation → le rectangle fin sur la ligne de vie (l'objet « travaille »)

Fragments : alt = if/else, loop = boucle, opt = optionnel. « Le diagramme de séquence est un scénario de film : on lit du haut vers le bas. »
